# サムネイルデザインガイド

> チャンネルのサムネイル設計における **原則** と **方向性** のリファレンス

---

## チャンネル概要

- **ジャンル**: 技術解説動画（圧縮アルゴリズム、ファイル形式、通信プロトコル等）
- **方針**: 3Blue1Brown的。直感→仕組み→洞察の流れで技術の本質を解き明かす
- **視聴者**: 20〜40代、エンジニア・技術好奇心が強い層

---

## 守るべき原則

### 1. サムネを見ただけで動画タイトルが想起できること

最も重要な原則。サムネイルを見た瞬間に「この動画は何について語っているか」が分かること。
キャッチコピー＋ビジュアルの**両方**でテーマを伝える。

### 2. テキストが主役

サムネイルはイラスト作品ではなく **テキストが主役の広告媒体**。
キャッチコピーは画面の中で最も視認性が高く、最も大きく配置する。ビジュアルはテキストを引き立てる役割。

### 3. ビジュアルは1つの主題に絞る

複雑なシーンや多数の要素を詰め込まない。**1つの強い主題**（1つの図、1つのフロー、1つのアイコン構図）に絞り、残りはテキストと背景で構成する。

### 4. タイトルだけから作る

サムネイルのデザインは **動画タイトルだけから想像できる内容** に限定。台本の具体的内容（研究者名、実験データ、数値、結論）は使わない。

### 5. 盛らない

タイトルから読み取れる事実をそのまま力強く出す。書いていないことを盛るのは禁止。

**NG例**: 「全部ウソ」「衝撃の真実」「ヤバすぎる」「まだ信じてるの？」

### 6. 高コントラスト

スマホの小さい画面で一瞬で認識できる。テキストと背景のコントラストは常に高く。

### 7. テキストの文字数制限

| 要素 | 文字数 | 備考 |
|---|---|---|
| メインキャッチ | **8〜15文字** | 最も大きく、最も目立つ |
| サブテキスト | **5〜15文字** | 無い方が良い。補足 |

## 画風

すべてのサムネイルは **テクニカルUI / フラットデザイン** を基調とする。

### 基本スタイル

- **ダーク背景**（#0d1117〜#1a1a2e系のダークブルー/ダークグレー）
- **カラフルなフラットアイコン・カード**を使った構図
- 各要素に**ネオンカラーのアクセント**（シアン、マゼンタ、グリーン、オレンジ等）
- **細い線・角丸ボーダー**で区切られたUI風レイアウト
- テキストは**等幅フォント風・テック感のあるサンセリフ**
- ファイル拡張子・コード断片・技術記号をラベルとして配置
- **グロー効果は控えめ**（ネオンサインではなく、UIのアクセントカラー程度）
- アイコンは**大きく、立体感のあるフラット**（OS標準アイコン〜VSCodeアイコンテーマ的な質感）

### 参考イメージ

- VSCodeのアイコンテーマ / ファイルマネージャーのグリッド表示
- ダッシュボードUI / 管理画面
- macOS / Windows のフラットアイコン（大きめ・立体感あり）
- 技術ブログのOGP画像

### 禁止

- 写実・フォトリアル
- AI風の厚塗り・重いレンダリング
- カートゥーン調（アメコミ的・ディズニー的）
- アニメキャラクター・擬人化キャラ
- AIっぽいキラキラ（パーティクル、光の粒子、過度なグロー）
- 複雑な風景・室内シーン
- 書き込みすぎ（ディテール過多で主題がぼやける）

## レイアウト構成の原則

### 情報量は「3要素」で完結させる

サムネイルに詰め込む要素は最大3つ。

| 要素 | 役割 | サイズ感 |
|---|---|---|
| **テキスト（メインキャッチ）** | 最重要。一番大きく配置 | 画面の30〜40% |
| **メインビジュアル** | テーマを体現するUI要素・アイコン・図 | 画面の中心に据える |
| **アクセント要素** | ラベル・バッジ・ステータス表示・矢印 | 小〜中 |

> ✅ テキスト＋ビジュアル1つ＋アクセント1つ = これで完成
> ❌ 要素を4つ以上詰め込む = NG

### カラーは「ダーク背景 + アクセント2〜3色」

| 役割 | 色 | 例 |
|---|---|---|
| **背景** | ダーク1色 | #0d1117, #1a1a2e, #111827 |
| **プライマリアクセント** | 鮮やか1色 | シアン (#06b6d4), グリーン (#10b981) |
| **セカンダリアクセント** | 鮮やか1色 | マゼンタ (#ec4899), オレンジ (#f59e0b) |
| **強調・否定** | 赤系 | #ef4444（×印、エラー表示等） |

> ダーク背景 × ネオンカラーの高コントラストが基本。白背景は使わない。

### アクセント要素のアイデア集

方向性に応じて以下から選ぶ（毎回同じにしない）：

- **ステータスバッジ**: 「✓ LOSSLESS」「✗ PATENT」「⚠ DEPRECATED」等のラベル
- **矢印・フロー**: A → B の変換を示す矢印
- **バーチャート・メーター**: サイズ比較、性能比較
- **×印 / ✓印**: 否定・肯定のシンプルなマーク
- **ボーダーカラー**: カードの枠線色で状態を表現（緑＝OK、赤＝NG）
- **ファイル拡張子ラベル**: .jpg .png .gif 等のテック感あるラベル

## デザインの方向性

以下は **「構図・レイアウト」** の選択肢。すべてダーク背景 × フラットUI を前提とする。

### 🌟 セントラル・展開図（おすすめ）

中央に誰もが知る象徴的なアイコン（ロゴやメインテーマ）を**かなり大きなサイズ**で堂々と配置し、その周囲の空間に**関連するコンポーネント、UIパーツ、コード断片などをバランス良く散りばめる**構図。
ごちゃごちゃと密集させるのではなく、**シンプルでスタイリッシュな要素が中央の主役を囲むように配置されている状態**がベスト。技術のエコシステムや構成要素をワクワクする形で見せる。

**レイアウトのルール**:
- 中央アイコンは**明確に大きく**、画面の主役として中心に配置する
- 周囲の要素は**5〜8個程度**に絞り、背景空間に美しく散りばめる（情報過多にしない）
- 各要素自体はフラットでシンプルなデザインを保つ。複雑な描写を入れすぎない
- スカスカにならないよう適度に配置しつつ、息苦しくなるほど敷き詰めない「ちょうど良い密度」を狙う

**使いどころ**: 特定の技術が「どう構成されているか」「何を支えているか」を俯瞰するテーマ
**具体的な表現**:
- 中央にVSCodeの大きなアイコン、周囲にHTMLアイコン・歯車・ターミナル窓・フォルダ等が等間隔に浮遊する
- 中央に大きめのペンギン（Tux）、周囲にスマホ・サーバーラック・ロケットなどのフラットアイコンが適度に散らばる
- 中央に輝く大きな鍵穴、周囲にシールド・データベース・ハッシュ文字列などが数個配置される

### 🖥️ グリッドカード

ダッシュボード・ファイルブラウザ・開発ツール風のUI画面。カラフルなカード・アイコンが**グリッドで並び**、技術テーマの多様性や一覧性を一目で伝える。

**使いどころ**: 複数の技術・形式・概念を俯瞰するテーマ
**具体的な表現**:
- ファイル形式のアイコンがグリッドで並ぶ（各カードに異なる色とシンボル）
- ダッシュボード：グラフ・メーター・ステータスバーが並ぶ管理画面
- ファイルブラウザ：フォルダとファイルが整然と並ぶ

### 🔀 フローチャート / パイプライン

技術プロセスが**矢印で繋がるフロー図**。入力→変換→出力の流れを一枚の図で見せる。

**使いどころ**: 技術の処理手順・変換プロセスを説明するテーマ
**具体的な表現**:
- A → B → C のステップカードが矢印で接続
- 分岐フロー：1つの入力から2つの出力へ分岐（可逆 vs 非可逆など）
- パイプライン：圧縮・変換・符号化の各ステップをカードで表現

### 🎯 ミニマルアイコン

**2〜3個のシンプルなアイコンと矢印だけ**で概念を伝える。余白を最大限に活かした最もシンプルな構図。「説明しなくても見ればわかる」を極めたスタイル。

**使いどころ**: 変換・対比・プロセスをワンアクションで伝えたいテーマ
**具体的な表現**:
- ファイル → ゴミ箱：「捨てる/削除」の概念
- 大きい箱 → 小さい箱：「圧縮」の概念
- 元画像 → 劣化画像：「非可逆」の概念
- 鍵 → 解錠：「暗号化/復号」の概念
- 1つ → 複数に分裂：「分割/変換」の概念
- A ≠ B：「互換性なし/違い」の概念

### ⚔️ 比較 / VS

2つの技術・形式を**左右に並べて対比**する構図。スペック比較カードやバーチャートで優劣や違いを視覚化する。

**使いどころ**: 2つの技術の違い・対決・トレードオフを扱うテーマ
**具体的な表現**:
- 左右2カラムに形式名＋スペックカード
- バーチャートが向かい合う比較UI
- 左に✓右に✗（または逆）で得意・苦手を対比

### 🔬 メカニズム断面

技術の**内部構造・仕組み**を「分解図」「断面図」で示す。ネオンカラーで描かれた回路図・ブロック図がビジュアルの主役。

**使いどころ**: 技術の内部動作・アルゴリズムの構造を扱うテーマ
**具体的な表現**:
- 8×8ブロックのグリッド（JPEG DCT）
- レイヤー分離図（RGBチャンネル分離等）
- 回路図風のデータフロー

---

## やらないこと

- 写実・フォトリアル
- AI風の厚塗り・重いレンダリング
- カートゥーン調（アメコミ・ディズニー的）
- アニメキャラクター・擬人化キャラ
- AIっぽいキラキラ（パーティクル、光の粒子、グロー）
- 台本の具体的内容（研究者名、データ、数値、結論）
- 盛った表現
- 白背景（ダーク背景に統一）
- 複雑なシーン描写（要素詰め込みすぎ）
- テキストよりビジュアルが目立つ構図
