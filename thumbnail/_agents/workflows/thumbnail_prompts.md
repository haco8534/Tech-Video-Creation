# 🎨 サムネイルプロンプト一括生成ワークフロー

## 概要

動画タイトルを入力として、YouTube動画サムネイル用の画像生成プロンプトをYAML形式で生成する。
出力は Chrome拡張「Gemini Image Auto Generator」にそのままコピペして使える。

### インプット
- **動画タイトル**（1本 or 複数本）

### アウトプット
- **YAML形式のデザインスペック**を `---` 区切りで出力するコードブロック
- そのままChrome拡張にコピペ → `---` 区切りで自動的に複数プロンプトとして認識される

---

## デザインリファレンス（必須参照）

プロンプト生成時は、必ず以下のガイドを参照すること：

**📄 `d:\myfolder\動画生成\技術解説\thumnail\channel_thumbnail_guide.md`**

このガイドに記載されている以下の項目を厳守する：
- 守るべき原則（テキスト主役、盛らない、ダーク背景等）
- 6つのデザイン方向性（セントラル展開図 / グリッドカード / フローチャート / ミニマルアイコン / 比較VS / メカニズム断面）。グリッドカードはカラフルなカード多数配置のスタイル
- やらないことリスト

---

## 実行手順

### Step 1: ガイドの読み込み

`channel_thumbnail_guide.md` を読み込んで、原則と方向性を把握する。

### Step 2: タイトルの受け取り

ユーザーが指定した動画タイトルを受け取る。

```
# 入力例
- 「JPEGは「人間の目を騙す」天才的な仕組み」
- 「なぜ画像形式はこんなにたくさんあるのか」
```

### Step 3: デザイン案の生成

各タイトルにつき **3〜6パターン** のデザイン案を生成する。

#### 生成ルール

1. ガイドの **6つの方向性** から、テーマに合うものを **異なる方向性** で選ぶ（同じ方向性を2回使わない）
2. **メインキャッチは8〜15文字**（サブテキストは**絶対に含めない**こと）
3. メインキャッチは**タイトルとそのまま同じにしない**（別角度の表現にする）
4. **表示するテキストやビジュアルはタイトルのみから考える**。動画の中身やスクリプトを読み取ったり参照したりしないこと。
5. **すべてダーク背景 × リッチで丸みを帯びた立体感（ソフト3D、クレイモーフィズム風）**で統一する。平面的すぎるフラットや、ギラギラした安っぽいAI特有の質感・露骨なグラデーションは排除し、上品で滑らかな質感にすること。
6. アイコンやビジュアル要素は**大きく**出す。例えば主役のアイコンなどに視点を1点に絞るか、もしくは多くのバリエーション（サービスアイコンなど）を美しく整列配置すること。
7. フォント指定には必ず「**ZenMaruGothic風の丸く柔らかい太字フォント**」を含めること。
8. アニメイラスト・キャラクターは使わないこと。
9. **グリッドカード方向性**を使う場合：カードは6〜12枚をグリッド配置し、各カードは**ネオンカラーのボーダー＋暗めの同系色背景**で、カードごとに色を変える。各カードに**アイコン1つ+ラベル1つ**。`cards` フィールドで各カードの内容と色を具体的に列挙すること。

### Step 4: ファイルとして出力

生成したデザイン案を **`d:\myfolder\動画生成\技術解説\thumnail\output\prompts.txt`** に一括保存する。

#### ファイル仕様

- ファイル名: `prompts.txt`（固定）
- 全タイトル・全デザイン案を **1つのファイル** に `---` 区切りでまとめる
- 新規生成時は既存ファイルを上書き。追記する場合はファイル末尾に `---` で区切って追加
- そのままChrome拡張にコピペ可能な形式を維持

### Step 5: チャットにもコピペ可能なYAMLブロックとして出力

すべてのデザイン案を **1つのコードブロック** に、**`---` 区切り** でまとめてチャットにも出力する。
この出力をそのまま Chrome拡張「Gemini Image Auto Generator」のテキスト入力欄にコピペすれば、自動的に `---` で分割されて複数プロンプトとしてキューに登録される。

#### 出力スキーマ（1ブロック = 1プロンプト）

```yaml
title: "動画タイトル"
direction: "方向性名（6つから選択: セントラル展開図 / グリッドカード / フローチャート / ミニマルアイコン / 比較VS / メカニズム断面）"
mood: "狙いたい雰囲気・印象（1〜2文）"
main_catch: "メインキャッチコピー（8〜15文字）"
cards: "（グリッドカード方向性の場合のみ）グリッドに並べるカードの一覧。各カードの内容（アイコン＋ラベル）と色を具体的に列挙する"
visual_concept: "ビジュアルの核となるアイデア。視点を1点に大きく絞るか、多数のバリエーションを美しく並べる"
style: "ダーク背景 × リッチで丸みを帯びた立体感。上品なグラデーション、ZenMaruGothic風の丸く柔らかい太字フォントなどの補足"
```

## ⚠️ 重要な注意事項

1. **入力はタイトルだけ**。台本全文の読み込みは不要
2. **出力は `---` 区切りのYAMLブロック**。ファイル保存 + チャット出力の両方
3. **各デザイン案で異なる方向性を使う**（同じ方向性の繰り返しはNG）
4. **すべてダーク背景 × テクニカルUI系で統一**。アニメキャラ・イラスト系は使わない
5. **メインキャッチは8〜15文字**（これだけは厳守）
6. **タイトルから読み取れない誇張は禁止**
7. **グリッドカード方向性**の場合：カードは6〜12枚、カードごとに色を変え、`cards`フィールドで列挙する
8. **コードブロック内に余計な装飾を入れない**（YAML + `---` 区切りのみ）
9. **ファイルは `thumnail\output\prompts.txt` に一括保存**する（全タイトル1ファイル）

## 複数タイトルの一括処理

### 入力方法

```
# 方法1: タイトルを列挙
- 「JPEGは…」
- 「なぜ画像形式は…」

# 方法2: 「全部」
台本作成にあるすべてのタイトル

# 方法3: 「サムネ未作成のもの全部」
完成品フォルダにあるが、prompts.txt に title が含まれないタイトルを自動検出
```

### 処理フロー

1. 指定されたすべてのタイトルを受け取る
2. `prompts.txt` に既に含まれるタイトルはスキップ
3. 各タイトルについて Step 3 を実行
4. **全デザイン案を `thumnail\output\prompts.txt` に一括保存**（Step 4）
5. **すべてのデザイン案をチャットにも `---` 区切りでまとめて出力**（Step 5）
