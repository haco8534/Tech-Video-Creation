## 📌 台本構成への提言

### 核心メッセージ
「ファイル削除 ≠ データ消去。削除とは参照を外すこと。物理データは上書きされるまで残る。」

### 推奨構成

1. **導入（Block 1-2）**: 身近な体験から入る。「Deleteキーを押した瞬間、何が起きるか知ってる？」→ 図書館の蔵書目録アナロジーで基本概念構築
2. **本論① ファイルシステムの仕組み（Block 3）**: 「目録カードを捨てるだけ」の80%の理解を構築。ゴミ箱→メタデータ削除→物理データ残存の3段構造
3. **本論② HDD vs SSD の衝撃（Block 4）**: 直感破壊。「HDDならデータは残る」→「SSDだと本当に消える」。TRIMとerase-before-write制約の本質的理解
4. **本論③ デジタルフォレンジックと完全消去（Block 5）**: パターン統合。「データが残る」世界（フォレンジック）と「完全に消したい」ニーズ（DoD/Gutmann/Secure Erase）の対比
5. **終わりに（Block 6）**: 冒頭の問いの回収。「Deleteキーとは何をするキーなのか？」→「参照を外す指示を出すキー」

### 3Blue1Brown的ポイント
- **核アナロジー**: 図書館の蔵書目録システム（全体を貫くアナロジー）
- **思考実験**: 「もしOSが毎回データを上書きしていたら、PCはどうなる？」
- **直感の構築→破壊→再構築**: HDD「データ残る！」→ SSD「でもSSDは本当に消す」→「なぜSSDだけ消すのか＝NANDの物理制約」
- **パターン統合**: FAT/NTFS/ext4はやり方が違うが「メタデータ更新＋領域マーク」という構造は同じ
