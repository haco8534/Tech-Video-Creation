# 「テストを書かないコードはゴミ」ってまじ？

「テストを書かないコードはゴミ」——プログラマなら一度は聞いたことがある過激な主張。その原典はMichael Feathersの名著『レガシーコード改善ガイド』にありました。

この動画では、テストの重要性を証明する衝撃的な実例（Knight Capital事件：45分で550億円消滅、CrowdStrike事件：世界850万台同時クラッシュ、Therac-25事件：ソフトウェアバグによる死亡事故）から、テスト至上主義の落とし穴（DHH「TDD is Dead」、Googleのカバレッジ研究）、AI時代のハリボテテスト問題まで、ソフトウェアテストの光と影を徹底検証します。

テストは「動くことの証明」ではなく「安全に変更する権利」を手に入れるためのもの。量より質、教条主義より実用主義——本当に価値のあるテストとは何かを一緒に考えましょう。
