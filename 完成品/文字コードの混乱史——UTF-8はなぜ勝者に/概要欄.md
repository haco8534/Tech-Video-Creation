# 文字コードの混乱史——UTF-8はなぜ勝者になったのか

あなたが今読んでいるこの文字は、コンピュータにとっては「ただの数字」です。
「65番 → A」「66番 → B」——この変換ルールを「文字コード」と呼びます。

しかし、この変換ルールが世界中で何十種類も作られたことから、40年にわたる混乱が始まりました。

この動画では、ASCII → コードページ → Unicode → UTF-8 という文字コードの進化の歴史を辿りながら、
なぜUTF-8が世界のWebページの98.9%を制覇するに至ったのか、その技術的・歴史的理由を解説します。

■ タイムスタンプ
0:00 文字化けの正体——コンピュータは文字を知らない
1:33 ASCIIの128文字——なぜ7ビットだったのか
3:57 コードページ地獄——各国バラバラの拡張
6:43 Unicodeの野望と挫折——16ビットの罠
9:50 UTF-8の天才的設計——ダイナーで生まれた奇跡
14:19 まとめ・後方互換性という最強の武器


■ 動画のポイント
✅ ASCIIの7ビット制限がすべての混乱の始まりだった
✅ Unicode（文字の辞書）と UTF-8（その書き方）は別物
✅ UTF-8はダイナーのプレースマットで設計された
✅ 後方互換性こそが技術の勝敗を決める最大の武器
