# リサーチ: プログラミング言語はなぜ700種類以上ある

## 核心メッセージ
プログラミング言語が700種類以上存在するのは「誰も整理しなかった」からではなく、人間の問題解決のアプローチが本質的に多様だからである。言語は「道具」であり、ハンマー1本で家を建てられないように、1つの言語で全ての問題を最適に解決することは原理的に不可能。

## 1. 数字のファクト

### プログラミング言語の総数
- Wikipedia掲載: 約700言語（notable languages）
- 歴史上存在した言語: 8,000〜9,000以上
- 実際に商用利用されている: 50〜100言語程度

## 2. なぜ言語が増え続けるのか：5つの力学

### 力学1: 問題領域の多様性（ドメイン特化）
- 科学計算 → FORTRAN, Julia
- Web → JavaScript, PHP
- データ分析 → Python, R
- システム開発 → C, Rust
- モバイル → Swift, Kotlin
- ゲーム → C++, C#（Unity）
- 1つの言語で全領域をカバーするのは「万能包丁」の幻想

### 力学2: プログラミングパラダイムの進化
- 手続き型（FORTRAN, C）: 手順を上から順に書く
- オブジェクト指向（Java, C++）: データと処理を「物」としてまとめる
- 関数型（Haskell, Erlang）: 数学的な関数の合成で表現
- 論理型（Prolog）: 事実とルールから推論
- 新しい考え方が生まれるたびに、それを体現する言語が必要になる

### 力学3: 既存言語への不満 → 改良型の誕生
- C → C++（オブジェクト指向が欲しい）
- C++ → Go（複雑すぎる、コンパイルが遅い / Googleの不満）
- Objective-C → Swift（モダンな構文が欲しい / Apple）
- Java → Kotlin（冗長性を減らしたい）
- JavaScript → TypeScript（型安全性が欲しい）

### 力学4: 新しい技術領域の出現
- 1960年代: メインフレーム → COBOL
- 1990年代: World Wide Web → JavaScript, PHP
- 2000年代: スマートフォン → Swift, Kotlin
- 2010年代: AI/ML → Pythonの台頭 + Julia
- 2020年代: ブロックチェーン → Solidity

### 力学5: 人間の思想・哲学の違い
- Python: 「1つのことをやる方法は1つだけであるべき」
- Perl: 「やり方は複数あっていい（TMTOWTDI）」
- Ruby: 「プログラマを幸せにする言語」（まつもとゆきひろ）
- Go: 「シンプルさは信頼性」（GoogleのRob Pike）
- Rust: 「安全性とパフォーマンスは両立できる」

## 3. 面白いエピソード

### FORTRAN: 機械との「白兵戦」からの解放
- John Backusが「プログラミングは機械との白兵戦だ」と表現
- 高水準言語は不可能だと信じられていた → 最初の最適化コンパイラで証明
- IBM が核研究所にパンチカードの大箱を送りつけた（マニュアルなし）

### Go言語: 「コンパイル待ちの怒り」から生まれた
- GoogleのRob Pike, Ken Thompson, Robert Griesemerらが
  C++のコンパイル待ちの45分間にGoの設計を始めた
- 「C++は複雑すぎる、Javaは冗長すぎる、Pythonは遅すぎる」

### JavaScript: 10日で作られた言語が世界を支配
- Brendan Eichが10日で作ったプロトタイプ
- 当初はLiveScript、Javaの人気に便乗してJavaScriptに改名
- typeof null === "object" はバグだが修正不可能（互換性のため）

### Ruby: 日本発、「プログラマの幸福」を追求
- まつもとゆきひろが「プログラマが楽しいと思える言語」を目指して個人開発
- 個人プロジェクトから世界的言語になった稀有な例

### 難解プログラミング言語（Esoteric Languages）
- Brainfuck: たった8文字の命令でチューリング完全
- Piet: 色の変化で命令を表現（ソースコードがモンドリアン風絵画）
- Whitespace: スペース・タブ・改行のみで記述（目に見えないコード）
- ArnoldC: シュワルツェネッガーの映画のセリフが命令

## 4. 「1つに統一できない」理論的な理由

### トレードオフの不可能三角形
- 実行速度（C, Rust）
- 開発速度（Python, Ruby）  
- 安全性（Haskell, Rust）
- 3つを同時に完璧にすることは原理的に困難

### コンピューティングの本質的な多様性
- 組み込みシステム（数KBメモリ → C）
- Webブラウザ（リアルタイムUI → JavaScript）
- 量子コンピュータ → Qiskit(Python), Q#
- 用途が多様=道具が多様は必然

## 5. 現在のトレンドと未来

### マルチパラダイム化
- 現代の言語はパラダイムを混ぜている
- Rust: 手続き型 + 関数型
- Kotlin: オブジェクト指向 + 関数型
- Python: マルチパラダイム

### AI時代の言語
- LLM時代にプログラミング言語は不要になる？
- → むしろ「何を」「どう伝えるか」の抽象化が高まるだけ
- プロンプト言語やDSLが新たなレイヤーとして登場する可能性
