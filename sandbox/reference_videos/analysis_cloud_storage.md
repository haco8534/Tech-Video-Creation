# 脱法クラウドストレージ — 台本分析

YouTube / Discord / マインクラフトなど本来ストレージではないサービスに、データを動画・画像・ブロックとして埋め込んで「無限ストレージ」化する技術を、霊夢・魔理沙の掛け合いで解説する 16 分動画。

## 構造

### 序論：例示の先制射撃で「？」を畳みかける

冒頭 30 秒は「説明」を一切せず、結論だけを 3 連続で叩きつける構成になっている。

> [00:00:00] これはマインクラフトのワールド。このブロックの塊には動画データが格納されている。容量は34MBほどだ。
> [00:00:06] え？
> [00:00:07] これは画像内に格納された音声データ。容量は5.93MBだ。
> [00:00:11] え？
> [00:00:11] これは動画に格納されたマイクラのセーブデータ。容量は1.63GBだ。
> [00:00:16] ちなみにこの方法でYouTubeやDiscord、ニコ動などに復元可能な動画を投稿すれば、無限に近いデータを保存できる。
> [00:00:22] は？

「結論3連 → ジャンルの命名」というパターン。視聴者の頭に「？」を作ったところで [00:00:28] 「『脱法クラウドストレージ』と称されるぶっ飛んだデータ変換技術を紹介していくぜ！」と固有名詞を投げ込み、その「？」を一語で回収させる。煽りや疑問形ではなく、**現象の異常さそのもの**でフックを作っている。

### ボディ：失敗 → 成功の階段、サービスごとの章立て

全体は「サービス名でセクションを切る」素直な構成だが、各セクション内部は **「思いつき → やってみる → 失敗 → 改良案」** の螺旋になっている。

- YouTube 章：mp4 直書き換え案 → [00:02:56] 「残念なことにこの手法は上手く行かなかった」 → 視覚情報案
- 視覚情報案：理論上 78GB 格納可能 → [00:05:08] 「YouTubeに投稿される動画は全てシステム側で再エンコードされる」で再びブレーキ → 480p 低画質で回避
- Discord 章：そのまま動画案 → [00:06:32] 「Discordには『1ファイル25MBまで』という誓約がある」で失敗 → 画像分割案 → 仕様変更でリンク切れ問題 → Dev Detour の txt 化案
- マインクラフト章：[00:11:39] 「じゃあクラウドストレージじゃないじゃん！」で本来の目的を一旦否定 → 情報隠匿という別用途へリフレーミング

「次の手法」が常に「前の手法の致命的欠陥」から駆動される。**前章の欠点を次章のフックにする**形でテンポを作っている。

### 結論：オチをずらして広げる

最終章は対象の総まとめではなく、**応用例を投げっぱなしで提示してから DNA ストレージに飛ぶ**。

> [00:14:56] 例えばオンライン将棋で2プレイヤーを両方操作できる状態にして、8つの歩に3ビットずつ割り当てる。
> [00:15:17] Cities: Skylinesで商業地区を0と、住居地区を1とし、横にそれらを並べてバイナリを表現する。
> [00:15:34] 現在、DNAストレージは実用化一歩手前で〜

「YouTube/Discord/マイクラ」の三本柱で締めず、**より飛んだ仮想例（将棋・シティーズ）→ 現実の研究（DNA）** に開いて、視聴者が自分でも応用を思いつける余白を残してから [00:16:04] 「人間そのものをデータストレージとして活用できるかもな」で一周回って馬鹿話に着地する。締めは挨拶 [00:16:15] 「じゃ、さらだばー！」のみで余韻を引きずらない。

## 掛け合い

### ロール分担：魔理沙＝解説、霊夢＝視聴者代弁

魔理沙が一貫して情報を出し、霊夢は「素朴な驚き」「素朴な疑問」「素朴なツッコミ」の 3 種だけを担当する。霊夢の発言は短く、平均 1〜2 文。

霊夢の発言は機能別に分類できる：

- **驚き相槌**：[00:00:06] 「え？」 [00:00:11] 「え？」 [00:00:22] 「は？」 [00:04:01] 「テレビの砂嵐っぽいね。」
- **理解確認の言い換え**：[00:01:25] 「え？ちょっとまって。例えば動画とか音楽とかゲームをダウンロードして、パソコンの容量がパンパンになったとするじゃん。〜YouTubeにアップロードすれば、無限にデータを保存できるって事！？」
- **次の話題を引き出す質問**：[00:05:26] 「そんでどうなんの？」 [00:11:48] 「これ、データってどうなるの？」
- **ツッコミ**：[00:05:33] 「ダメじゃん！」 [00:10:30] 「いや何で？」 [00:11:39] 「じゃあクラウドストレージじゃないじゃん！」

### 「言い換え返し」が会話のリズムを作る

魔理沙が抽象的・専門的に語った直後、霊夢が**具体例で言い換え直す**パターンが頻出する。

> [00:01:21] （魔理沙）なんと、YouTubeをファイル保管庫として利用することができるようになるのだ。
> [00:01:25] （霊夢）え？ちょっとまって。例えば動画とか音楽とかゲームとかをダウンロードして、パソコンの容量がパンパンになったとするじゃん。〜

霊夢が要約 → 魔理沙が「その通り」or「その通り…と言いたいところだが」で受け、次の制約条件に進む。**「その通りなのだが、先に注意点から話しておこう」**[00:01:39]、**「その通り…と言いたいところだが、Discordには『1ファイル25MBまで』という誓約がある」**[00:06:32] というテンプレで、視聴者の理解を一度肯定してから捻る。

### 会話の比率は明確に偏らせる

魔理沙の発話量は霊夢の 4〜5 倍。霊夢が長く喋るのは [00:01:25] と [00:06:24] の 2 箇所のみで、いずれも「魔理沙の説明をユーザー目線の事例に落とし直す」役割。霊夢を解説に参加させず、**視聴者の頭の中の声**に徹させている。

## 題材の運び方

### 専門用語は「日常物のアナロジー → 用語投入」の順

mp4 構造の説明で典型例が見られる。

> [00:03:17] つまりどういうこっちゃ？
> [00:03:18] 例えば映画フィルムがあって、それを一旦バラバラにして無造作にセロテープで繋げる。その改変フィルムを一昔前の映画館スタッフに渡して「いい感じに再生しろ」と命令する。しかしスタッフはどこから映像が始まり、どこで映像が終わるのか分からず、内容もちぐはぐなため映画として再生する価値は無いと匙を投げるだろう。
> [00:03:35] 同様に「mp4」のような動画ファイルであっても、再生に必要な情報などが揃っていない限り、システムはそれを動画として認めず、再生ができない。

「映画フィルム＋スタッフ」という具象モデルを作ってから「mp4」「再生に必要な情報」と用語で締める。**比喩を先に置いて用語で蓋をする**順番。

### 数字は「率」より「身近な量」で出す

[00:04:25] 「このカラフルなモザイク一枚に格納できる容量はおよそ31.6KB。スネークゲームのような原始的なゲームだったらソースコード丸々格納できるはず。書籍でいうと15ページ分ってとこだな。」

31.6KB という抽象量を「スネークゲームのソースコード」「書籍15ページ」と**用途換算**で言い直す。[00:09:51] 転送速度 6.8MB/s も直後に [00:09:58] 「HDDの一般的な転送速度は約50～100MB/s。SSDだと400～550MB/s。」と比較対象を置く。**裸の数字を一度も出していない**。

### 固有名詞は削らずちゃんと出す

- BK Binary、Dev Detour（人物名）
- mdat、moov（mp4 内部構造名）
- bukkit、jar、プラグイン（実装名）
- ペルソナ・ノン・グラータ（外交用語）

特に [00:13:55] 「スパイの身元が判明してペルソナ・ノン・グラータとなってしまう。」のように、難語をあえてそのまま使い、文脈で意味が分かる作りにしている。**「視聴者が知らないかも」で削る**ことをしない。

### 抽象度の往復が細かい

「mp4 内部に書き込み」という低レイヤの話 [00:02:09-02:48] と「映画フィルム」という日常比喩 [00:03:18] が 30 秒ごとに切り替わる。一段抽象→一段具体→一段抽象を 1 分以内で繰り返し、視聴者を一階層に留めない。

## フレーバー

### 「脱法」というラベルで全体に色を付ける

タイトルの「脱法クラウドストレージ」がすべての笑いどころのトーンを決めている。技術的にはグレー、規約的にアウト、というニュアンスをコールバックとして散らす：

- [00:01:50] 「保管するデータがゲームや音楽といった著作物の場合、著作権侵害として処される可能性もある」
- [00:06:16] 「もちろん、規約的にアカウントがBANされる可能性もあるが」
- [00:10:13] 「規約違反でアカウントがBANされて保存してたデータが全部おしゃかになる未来が待っているだろうから、多分普通にHDDを買ったほうが安上がりだと思う」
- [00:14:42] 「何時間もかけてデータを変換して投稿して、その挙句アカBANなんかされちゃったら失神しちゃうだろうね」

技術解説を最後まで聞かせた上で**現実的なツッコミで地面に戻す**ペースが何度も入り、ロマンと冷静さのギャップが笑いになる。

### ボケのパターン：規模の暴走 → ツッコミ → 小声訂正

[00:13:09] 「削った場所に部屋を作り、村人を良い感じに住ませれば、家と村人が格納された動画を作ることができるのだ！」
[00:13:12] 「ナンテコッタイ・・・」
[00:13:12] 「まあ実際には村人は読み込まれないのだがな。」

提案を一段大きく盛って → 霊夢がオーバーリアクション → 魔理沙が小声で訂正、の三段。

[00:16:04] 「人間そのものをデータストレージとして活用できるかもな」
[00:16:09] 「首の後ろにUSB用のコネクタ付ければ最強の脱法ストレージの完成だね！」

着地でもう一度同じパターンを踏んで終わる。

### 脱線がそのまま伏線になる

[00:02:27] で出した「mdat は容量の 90% 以上を占める」という設定が、[00:12:22] で爆破実験の結果説明として再利用される（「moov は損傷に弱い、mdat は損傷に強い」）。一見独立した実験回が、序盤の知識でちゃんと回収される。視聴者は「あれが効いてくるのか」と気付ける構造。

### スパイ・諜報の長尺アナロジー

[00:13:49-14:23] でマインクラフト隠匿術を**国家機密の送信**に喩える 30 秒弱の長いたとえ話を入れる。

> 某国の大使館にスパイが潜んでいたとする。そのスパイは大使館で得た極秘情報を母国に送信したい。しかし平文や一般的な暗号化で母国に送ろうとすると、某国の諜報機関が何らかの技術で暗号化を解読し、スパイの身元が判明してペルソナ・ノン・グラータとなってしまう。そこで先ほどのような「データをマイクラ内のブロックに変換する」というイレギュラーなデータ変換を噛ますことで、諜報機関にバレずにデータを送信できる可能性が大いに高まる。

技術仕様を技術仕様のまま説明せず、「マイクラ＝最強の暗号化ツール」というキャッチコピーに変換する。**比喩で技術の意味を一段ジャンプさせる**最大ヤマ場。

## 独自の工夫

### 「失敗→改良」を構造の単位にする

サービス章を「成功事例の紹介」で構成せず、必ず**最初の素朴な発想を一度殺してから改良案に進む**。BK Binary 氏 mp4 直書きの失敗、80GB 動画の再エンコード問題、Discord 画像のリンク 1 日切れ問題、マインクラフト案がそもそも非クラウドストレージである件など、毎章で「ダメじゃん」を一度通る。これにより視聴者が**自分で同じ素朴な発想を思いついて即座に却下される**過程をなぞれ、解説者と視聴者の理解が同期する。

### 「目的そのもの」をリフレーミングして章を畳む

[00:11:39] 「じゃあクラウドストレージじゃないじゃん！」というツッコミに [00:14:28] 「クラウドストレージとして用いることは難しいが、情報を隠すのには最適だろう」と答える。**章の前提（クラウドストレージ）を否定する形で章を閉じ、別目的（情報隠匿）に開いて締める**。同じ素材を別レンズで見せ直す手法。

### 結論を「列挙して開く」

結論で 3 つの応用例（オンライン将棋・Cities: Skylines・DNA ストレージ）を投げっぱなしで提示する。**詳細解説を与えない**ことで「視聴者が他の例も思いつけそう」という余韻を残す。チャンネル運営的にもコメント欄のネタになる構成。

### タイトル一語の繰り返し

「脱法クラウドストレージ」というキャッチーな造語を [00:00:28] [00:14:35] [00:16:09] の冒頭・締め・最終ボケで 3 回明示的に呼ぶ。視聴者の脳内に**動画固有のラベル**を刻む。

## 自分の台本へ移植したい要素

- **冒頭 30 秒は「結論3例 + 命名」テンプレ**：説明ゼロで「結果」だけを 3 つ提示し、霊夢の短い驚き反応で挟み、最後に動画のテーマ名を一語で命名する（例：「これは〜」「え？」を 3 ループ → 「今回は〜と呼ばれる〜を紹介していくぜ！」）。
- **章は必ず「素朴な発想 → 致命的欠陥 → 改良案」の螺旋にする**：いきなり完成形を見せず、視聴者が思いつきそうな素朴案を一度提示してから「残念なことに〜」「その通り…と言いたいところだが〜」で否定し、改良に進む。各章 3〜4 分。
- **専門用語は「日常物の比喩 → 用語投入」の順で固定**：「つまりどういうこっちゃ？」と霊夢に聞かせ、魔理沙が日常物（映画フィルム、書籍ページ数、バーコードなど）で比喩を作り、最後の 1 文で「同様に〜のような〜」と専門語で蓋をする。比喩を後付け説明にしない。
- **裸の数字を出さない**：容量・速度・サイズはすべて「書籍 X ページ分」「HDD は 50-100MB/s」「スネークゲームのソースコード丸々」など**用途換算 or 既知のもの比較**とセットで提示。
- **霊夢の発話は 4 種に限定**：(a) 短い驚き「え？」「は？」、(b) 自分の生活への置き換え「容量パンパンになったとするじゃん〜って事！？」、(c) 次の話題を引き出す質問「そんでどうなんの？」、(d) 一行ツッコミ「ダメじゃん！」「いや何で？」。霊夢に長い解説を絶対させない。
- **章末のリフレーミング**：章の前提を最後にひっくり返して別目的に開く（例：「ストレージとしては使えないが、情報隠匿には最適」）。失敗章を否定で終わらせず、別レンズで価値を残す。
- **結論は列挙開放型**：本編の総まとめではなく、本編で扱わなかった応用例を 2〜3 個投げっぱなしで提示し、最後に飛んだボケ（人間 USB コネクタなど）で締めて挨拶 1 行で終わる。長いまとめを書かない。
