# お手本台本に共通する設計パターン

霊夢／魔理沙系「ゆっくり解説」の人気動画 4 本（[cloud_storage](analysis_cloud_storage.md) ／ [library_of_babel](analysis_library_of_babel.md) ／ [philosophical_razors](analysis_philosophical_razors.md) ／ [mosaic_destruction](analysis_mosaic_destruction.md)）に共通する設計パターンを軸ごとに整理し、最後にワークフローの台本生成へ反映すべき具体的な操作を punch list にする。

個別の引用とより詳細な観察は各 `analysis_*.md` に分離してある。本書は横断する規則性だけを抽出した蒸留版。

## 1. 序論（フック）の作り方

**共通則：原理・概論より先に「現象」「結果」「断定」を投げる。** 説明から入る台本は 4 本中ゼロ。

| 動画 | フックの型 |
|---|---|
| library_of_babel | **「これは X である」断定** → 同型例 3 連 → 主題の二項対立（「ある／探せない」）→ タイトル |
| cloud_storage | **「結果 3 例 + 短い驚き反応」をループ** → 動画固有のキャッチ造語で命名 |
| mosaic_destruction | メタネタで掴む → 「常識 → 反証」→ **実証映像を先出し** → 主題宣言 |
| philosophical_razors | 聞き手の **素朴な誤読**（「床屋でも開くの？」）→ 事後的に定義を回収 |

共通の時間設計：**30 〜 60 秒以内でフックを完成させ、その末尾でタイトル／主題語を 1 度宣言する**。フック中の聞き手の発話はすべて短い（1 行）。

「主題の二項対立」は library_of_babel と mosaic_destruction が顕著（ある／探せない、暴く／隠す）。一見矛盾する 2 つの命題を導入で並べて、動画全体の縦串にする。

## 2. ボディの構造

セクションを単純な「項目 N 個の羅列」にしない。各セクションが**内側に小さな起承転結**を持ち、かつセクション同士が**横の連結**を持つ。

### セクション内部の型

| 型 | 出典 | 内容 |
|---|---|---|
| 素朴な発想 → 致命的欠陥 → 改良案 の螺旋 | cloud_storage | 視聴者が思いつきそうな案を一度提示→「残念なことに〜」で否定→次の手法に進む |
| 具体例 → 一般化 → 具体例 のサンドイッチ | mosaic_destruction | 抽象度を単調増加させず、命題を具体例で何度も着地させる |
| 定義 → 日常例 → 歴史 → 思想的含意 の昇降 | philosophical_razors | 同上、抽象度の上下を意識的に交互に |
| 結果先出し → 原理 → 限界 → 応用 | mosaic_destruction | 章全体でこの順序。「マジで？」を先に作る |

### セクション間の横の連結

- N-1 の説明を N の素材として**再利用する**（philosophical_razors：アルダー→ポパーの説明を直後のグライスで「これは駄文」とメタに使う）。
- 章末を**次章のフック**にする（cloud_storage：前章の致命的欠陥が次章の動機）。
- 章末で**前提そのものをリフレーミング**して別目的に開く（cloud_storage：「クラウドストレージじゃない」→「情報隠匿に最適」）。

### 中盤の「全体マップ提示」セクション

4 本中 2 本（mosaic_destruction、cloud_storage）が、最初のテーマを解説した直後に**全分類のメタマップ**を見せる。残りの章を「マップ巡り」に変える構造転換。

mosaic_destruction：ブロックモザイク解説 → 「モザイク 4 分類」のマップ → ガウスぼかし編へ。
cloud_storage：YouTube 章 → ストレージサービス全体への展開。

## 3. 結論の畳み方

**4 本とも総括・要約を一切しない。** 「今日の話をまとめると〜」型のクロージングはゼロ。

代わりに、以下のいずれか（または複合）で締める：

- **主題のスケール反転 → 世界観への昇格**：library_of_babel「実世界も無限に近い有限の選択の一つ」、mosaic_destruction「『見えないものを見たい』っていう人類の本能なのかもしれない」
- **列挙開放型**：本編で扱わなかった応用例を 2〜3 個投げっぱなしで提示し、最後に飛んだボケで畳む（cloud_storage：将棋 → Cities: Skylines → DNA ストレージ → 人間 USB コネクタ）
- **メタファーの再ひねり**：冒頭の比喩を最後にもう一段展開して畳む（philosophical_razors：剃刀 → 十徳ナイフ → 論理のスキンヘッド）
- **アイロニーで主題を再定義**：mosaic_destruction「同じ技術が暴くことにも隠すことにも使われる」

挨拶（「じゃ、さらだばー！」）は 1 行で済ませて余韻を引きずらない。

## 4. 掛け合いの設計

ロール分担は 4 本すべてで固定：**解説役（魔理沙）が情報を出す／聞き手役（霊夢）が反応する**。逆転や役割の揺れは一切ない。

### 発話量の比率

- 解説役：聞き手 ≈ 4〜5 : 1
- 聞き手の発話は基本 1〜2 行。例外は「視聴者目線で言い換える」場面のみ（cloud_storage の「容量パンパンになったとするじゃん〜って事！？」型）。

### 聞き手の発話の機能分類

4 本に共通して観測される発話タイプは 5 つ：

| タイプ | 例 |
|---|---|
| 短い情動・驚き | 「え？」「は？」「強そう」「なにそれ怖い」 |
| 視聴者目線への言い換え | 「つまり、〜って事？」「あの時習った〜は〜のためだったんだね！」 |
| 次の話題を引き出す質問 | 「そんでどうなんの？」「で、その技術が回り回って〜」 |
| **誤った仮説の提示**（典型的勘違い） | 「掛け算と冪乗の混同」「これ編集じゃないの？」 |
| 短いツッコミ | 「ダメじゃん！」「いや何で？」「じゃあクラウドストレージじゃないじゃん！」 |

**「誤った仮説の提示」は 4 本すべてで観測**：聞き手が視聴者代表として典型的勘違いを先回りして発話する。解説役が訂正することで、視聴者が頭の中で立てた仮説が回収される。

### 解説役が聞き手に問いかける

聞き手が一方的に反応するだけでなく、**解説役の側から二択質問を投げる**場面がボディ中盤に少なくとも 1 回ある（philosophical_razors：「『宇宙人に会った』と『パン屋で友達に会った』、どっちにより確かな証拠が必要だと思う？」）。視聴者の判断を参加させる節目。

## 5. 題材の運び方

### 専門用語・固有名詞

**4 本とも、固有名詞・年号・論文誌名・人物名は削らずに出す**。「ある人物」「ある技術」型の抽象化は禁じられている。観測された具体名の例：

- philosophical_razors：1980 年ロバート・ハンロン、Phys. Rev. B、Sr2RuO4、Google Scholar 引用 10 回
- cloud_storage：BK Binary、Dev Detour、mdat / moov、ペルソナ・ノン・グラータ
- mosaic_destruction：Level2Jeff、KoKuToru、Thunderbolt5、銀河 M87
- library_of_babel：ホルヘ・ルイス・ボルヘス、itch.io、10^961755、400 ペタバイト

### 専門用語の導入順

ほぼ全ケースで **「現象の説明 → 専門用語のラベリング」** の順。視聴者に身構えさせない後付け命名：

- mosaic_destruction：「線の境界では白から黒に一気に変わるだろ？ これを『高周波成分』っていうんだが」
- philosophical_razors：「電気抵抗ゼロで電流が流れる特殊な現象がある」→「電気抵抗ゼロ？」→「これが超伝導だ」（流れの中で命名）
- cloud_storage：「映画フィルム＋スタッフ」比喩で具象モデルを作ってから「mp4」「再生に必要な情報」と用語で蓋

聞き手が**オウム返し質問**（「○○？」）で語を反復し、その語だけに絞った 1〜2 文の補足を入れる型も頻出。

### 数値の体感化

**裸の数字を出さない**。同じ量を複数の単位系で 2〜3 回叩く：

- library_of_babel：漢数字（1僧掲邏摩）→ 指数表記（10^961755）→ 物理行為（0 を 11 日間唱える長さ）
- mosaic_destruction：2.2 ミクロン → 髪の毛の 50 分の 1、秒速 500km → 音速の 1500 倍
- cloud_storage：31.6KB → スネークゲームのソースコード → 書籍 15 ページ分、6.8MB/s → HDD 50-100MB/s と並列比較

「率」より「身近な量・既知の物との比」で。

### アナロジーの多層化

同じ概念に複数の比喩を当てるパターンが目立つ：

- mosaic_destruction：ガウスぼかしの逆算困難性 →「コップに垂らしたインク」（解説役）→「角砂糖を水に溶かす」（聞き手が別比喩で言い直す）
- philosophical_razors：ヒュームの剃刀 →「雨だから室内に入る」→「鳥は雨の時こそ餌探し」「魚に『濡れたくない』は意味不明」の反例 2 段重ね

聞き手が**別の比喩で言い直す**ことが理解の二重化として機能する。単なるオウム返しではない。

### 抽象度のミクロな往復

「一段抽象 → 一段具体 → 一段抽象」を 30 秒〜1 分単位で交互に。視聴者を一階層に留めない（cloud_storage の mp4 内部構造 ↔ 映画フィルム比喩の往復が典型）。

## 6. フレーバー

### 笑いと本筋の関係

**4 本とも、笑いが技術内容と地続き**。ボケのために本筋を止めない。共通の笑いの型：

- **規模の暴走 → ツッコミ → 小声訂正** の三段（cloud_storage：「家と村人が格納された動画」→「ナンテコッタイ・・・」→「まあ実際には村人は読み込まれないのだがな」）
- **常識 → 反証** のテンプレ（mosaic_destruction：「平成に流行った都市伝説」→「GitHubに公開されてな」）
- **メタな自己言及**（philosophical_razors：直前の自分の長い説明を「これは駄文だった」と自分で削り直す／library_of_babel：「この動画の 3 分後のスクショもバベル内に存在する」）

### コールバック・伏線

序盤に張った設定は必ず後半で機能する：

- cloud_storage：序盤の「mdat は容量の 90% を占める」が、後半の爆破実験の結論「moov は損傷に弱い／mdat は強い」で回収
- philosophical_razors：ヒュームを 1 度目の剃刀の主として出した後、アルダーの剃刀の説明でヒュームが論理実証主義を崩す役で再使用
- library_of_babel：マイクラ版バベルの BGV を冒頭で宣言し、後段「数時間彷徨って見つかったのは『いす』『農協』だけ」でボゴソート比喩に接続

### トーン

**4 本とも終始ドライ・フラット**。誇張・絶叫・ハイテンションのリアクションはない。煽り表現や攻撃的なラベリングもない。知的好奇心と軽い皮肉の温度で押し切る。

## 7. 横断する独自テクニック

各動画固有の工夫の中で、移植可能性が高いもの：

- **動画の構造そのもので概念を実演する**（philosophical_razors：グライスの剃刀の説明として、直前の自分の長文を駄文と認めて削る／逆オッカムの説明として、シンプルに済む話を陰謀論で 2 分半語る）。視聴者は定義文ではなく動画の動きから概念を体得する。
- **題材の中に動画を組み込むメタ構造**（library_of_babel：寸劇は「ゆっくりバベルの歩き方」というバベル内蔵書からの引用、動画自身のスクショもバベル内に存在）。動画の外側まで主題で覆う。
- **タイトル一語の繰り返し**（cloud_storage：「脱法クラウドストレージ」を冒頭・締め・最終ボケで 3 回明示的に呼ぶ）。視聴者の脳内に動画固有のラベルを刻む。
- **「使えない技術」の救済としての歴史パート**（mosaic_destruction：ガウスぼかしを「使えない」と一度落とした後、ハッブル望遠鏡のサクセスストーリーで「使える」に覆す）。負 → 正の感情曲線で歴史パートを正当化。

## 8. ワークフローへの反映（punch list）

`object_centric_video/01_script.md` の台本作成段で実行すべき具体的な操作。

### 序論

- [ ] フックは **原理ゼロ・現象先出し** で構築する。30〜60 秒以内に主題語／タイトルを 1 度宣言。
- [ ] フックに **主題の二項対立**（ある／届かない、強い／弱い、暴く／隠す）を 1 組含める。動画全体の縦串になる。
- [ ] 序論末でずんだもん（解説役）が **動画タイトル・主題語を口に出す**（書き文字に頼らない）。

### ボディ

- [ ] 各セクション内部を **3 段以上の起承転結**（素朴な発想 → 欠陥 → 改良／具体 → 一般 → 具体／結果 → 原理 → 限界 など）にする。フラットな羅列禁止。
- [ ] セクション同士に **横の連結**を最低 1 か所仕込む（前章の素材を次章で再利用、前章末を次章のフックにする）。
- [ ] ボディ中盤に **「全体マップ提示」セクション** を 1 つ入れて、後半を別の旅に変える（複数のトピックを跨ぐ動画で特に有効）。

### 結論

- [ ] **総括・要約を書かない**。今日の話をまとめると〜型のクロージング禁止。
- [ ] 以下のいずれかで畳む：
  - 主題のスケールを一段上げて世界観・人間観に反転
  - 本編で扱わなかった応用例 2〜3 個の列挙開放
  - 冒頭で張ったメタファーをラストにもう一段展開
- [ ] 締めの挨拶は 1 行で済ませる。余韻を引きずらない。

### 掛け合い

- [ ] **発話量比 解説役 : 聞き手 = 4〜5 : 1** を守る。
- [ ] 聞き手（めたん）の発話は **5 タイプに限定**：短い情動・視聴者目線の言い換え・次話題を引き出す質問・誤った仮説の提示・短いツッコミ。長い解説をさせない。
- [ ] **「誤った仮説」を聞き手に最低 1 回提示させる**。視聴者が頭の中で立てそうな典型的勘違いを先回りして発話 → 解説役が訂正。
- [ ] 解説役から聞き手への **二択質問** をボディ中盤に最低 1 回入れる。聞き手に答えさせてから本論へ。

### 題材の運び方

- [ ] 専門用語は **「現象の説明 → 用語ラベリング」の順** に固定。「これを○○といって〜」のような用語先出しを禁止。
- [ ] 固有名詞・年号・人物名・論文誌名・引用回数・金額は **削らずそのまま出す**。「ある研究者」「ある技術」型の抽象化禁止。
- [ ] 重要な数値は **複数の単位系で 2〜3 回叩く**（式表現 → 桁感の言い換え → 人間スケールの行動）。裸の数字を出さない。
- [ ] 重要なアナロジーは **聞き手に別の比喩で言い直させる**ターンを作る（オウム返しではなく別比喩）。
- [ ] 抽象度を **30 秒〜1 分単位で上下に振る**。一階層に留まらない。

### フレーバー

- [ ] 笑いは **技術内容と地続き** にする。ボケのために本筋を止めない。
- [ ] ボケパターンの定型「規模の暴走 → ツッコミ → 小声訂正」を最低 1 回。
- [ ] 序盤に張った要素を **後半で必ず回収**する伏線設計を最低 1 つ。
- [ ] トーンは **終始ドライ・フラット**。誇張・絶叫・煽り禁止（既存 [[feedback_theme_tone]] と整合）。

### 攻めの拡張（高難度）

- [ ] **動画の構造そのもので概念を実演する**瞬間を 1 か所作る（直前の説明を自分で削る／主題の通りに動画自身が動く）。
- [ ] 動画固有の **キャッチ造語** を作り、冒頭・中盤・締めで 3 回呼ぶ。視聴者の脳内に動画固有ラベルを刻む。

## 既存ルールとの照合

これらのパターンは、既存の作業ルールと矛盾しない：

- **[[feedback_beginner_script_design]]**（抽象化しすぎNG、日常アナロジー → 技術用語 → 深掘り、固有名詞削らない）→ §5 で全 4 本が一致する観察として裏付けられる
- **[[feedback_character_roles]]**（ずんだもん解説／めたん聞き手、逆転禁止）→ §4 のロール固定と完全に整合
- **[[feedback_topic_entry_breadth]]**（入口は広く・中身は専門）→ §1 のフック設計と §5 の専門語温存が両立を担保
- **[[feedback_theme_tone]]**（共感×知的好奇心、煽りNG）→ §6 のトーン観察と整合
- **[[feedback_sequel_self_contained]]**（前作参照は概要欄止まり）→ 参照 4 本ともシリーズ前提のコールバックはなく単体で完結

新規追加として強い行動指示になるのは、§3 の「総括禁止」、§2 の「全体マップ提示」、§4 の「誤った仮説の提示」、§5 の「現象 → 用語ラベリング順」、§7 の「構造での実演」と「キャッチ造語の 3 回呼び」。
