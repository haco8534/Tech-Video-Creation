# オブジェクト中心動画 — マスターワークフロー

題材を1本の解説動画にするための4段階パイプライン。**仕組み**（オブジェクト中心ステージ、台本駆動、Track 補間）は段階間で固定する。**世界観**（形・動き・アクセント色・トーン）は題材ごとに決める——ただし**白テーマと文字の大きさは固定ベース**として全題材で共通（→ `02_design_overall.md`「固定ベース」）。全4段に共通する**デザイン思想**は `00_philosophy.md` にある。

## パイプライン全体図

```
入力（タイトル + 補足説明）
  │
  ▼
[1] script.md ──────── 台本
  │   ガイド: 01_script.md
  │
  ▼
[2] design_overall.md ─ 全体デザイン方針
  │   ガイド: 02_design_overall.md
  │
  ▼
[3] design_spec.md ──── セリフ単位の挙動スペック
  │   ガイド: 03_design_spec.md
  │
  ▼
[4] Composition.tsx ─── Remotion 実装
      ガイド: 04_remotion.md
```

各段は前段の成果物を受け取り、1種類の情報だけを足す。**script = 何を言うか / overall = どう見せるか / spec = いつ起こすか / tsx = 実装**。

## 各段の責務 — 受け渡し契約

| 段 | 成果物 | 足す情報（持つもの） | 持たないもの |
|---|---|---|---|
| [1] | `script.md` | 対話（誰が何を喋るか）と論理区切り（序論／ボディ／ボディ内セクション／結論）。**セクション区切り = 画面の切り替わり単位** | デザイン、タイミング、event |
| [2] | `design_overall.md` | 各セクション（= 1画面）の自然言語デザイン。**画面の見え方はここで決め切る**——図の見た目（カタログ・ディテール必須）／文字の節制ルール／最も混む瞬間の止め絵／時間順の状態遷移（3〜5セリフに1ステップ）／演出ノート。貫通する世界観もここ。粒度は**セクション単位** | フレーム数、event 名、セリフ単位の正確なタイミング。**ピクセル座標も書かない（テキスト内包箱の文字基準サイズだけ例外）** |
| [3] | `design_spec.md` | event アンカーの定義と各セリフ行への割り当て、セリフ × オブジェクト挙動の対応表（**どのセリフで何が起きるか**）。内容 event は全セリフの 20〜30% 目安 | 座標値 |
| [4] | `Composition.tsx` | Track 群と SVG プリミティブによる実装、台本駆動の**対話字幕**。座標は直書きで現物合わせ、**ただしテキスト内包箱の幅はテキストから派生計算（固定リテラル禁止）** | — |

### 段をまたぐときの原則

- **創造的判断は [1] script と [2] overall に集約する**。何を語り、どう見せるかはこの2段で決め切る。[3] spec と [4] tsx は、決まった設計をズレなく形にする段——新しい創造的判断を持ち込まない。
- **下流でズレが見つかったら、まず前段の散文を直す**。spec がおかしいなら overall を、tsx がおかしいなら spec を疑う。コードは散文の翻訳結果であって、原本ではない。
- **座標は紙の上で決めきれない**。design_spec は「いつ・何が起きるか」までを定義し、ピクセル座標は tsx でプレビューを見ながら現物合わせする。

## 台本のセリフはどこへ行くか

[1] で書いた対話は、最終的な動画で視聴者に届かなければならない。経路は2つ：

- **対話字幕（必須）** — 画面下部に、話者名つきでセリフ本文を尺に同期して表示する。`Composition.tsx` が `script.md` の全セリフから自動描画する（→ `04_remotion.md`）。**これが無いと「台本が動画のどこにも出てこない」状態**になる。台本駆動の `lineDurations`（各セリフの表示フレーム数）は、字幕・アニメ event の共通のタイミング源。
- **ボイスオーバー音声（任意レイヤー）** — VOICEVOX 等でセリフを読み上げた音声。生成手段がある場合に足す。音声を足したら `lineDurations` を文字数推定から実音声の長さに差し替える（Track 群のロジックは不変）。

絵（オブジェクトの動き）が概念を語り、対話字幕（と音声）がそれに言葉を添える。design_overall は画面下部に字幕帯のぶんの余白を確保すること。

## 画面の単位 — セクション = 1画面

台本の **ボディ内セクション**（と序論・結論）の区切りが、そのまま**画面の切り替わり単位**になる。台本を書く時点で「ここで画面が変わる」前提でセクションを割る。

動きは2階層：

- **セクション内**: オブジェクトの集合が状態を連続変化させる。カット切り替えはしない（オブジェクト中心ステージ）
- **セクション境界**: 画面が切り替わる。前セクションの全オブジェクトがフェードアウト → 新セクションのオブジェクトがフェードイン（標準 30f クロスフェード）

## サブガイド

各段の詳細は同フォルダ内のサブガイドを参照：

- **[00_philosophy.md](00_philosophy.md)** — デザイン思想。「絵が説明そのもの」を軸に、図のつくり方（3原則）と動かし方（7原則）。全段の土台（パイプラインの「段」ではない）
- **[01_script.md](01_script.md)** — 台本作成。良い台本の2本柱（本質を射抜く解説／見続けさせる掛け合い）、リサーチと切り口、序論／ボディ／結論の3部構成
- **[02_design_overall.md](02_design_overall.md)** — 全体デザイン方針。貫通する世界観（色・形・動きの語彙）、各セクションの登場オブジェクトと動き方を自然言語で
- **[03_design_spec.md](03_design_spec.md)** — セリフ単位のスペック。event アンカーの定義、各セリフへの割り当て、オブジェクト挙動の対応表、ライフサイクル契約
- **[04_remotion.md](04_remotion.md)** — Composition.tsx 実装。Track/resolve/blendNumeric、State 数値のみルール、SVG プリミティブ直接組み、セクション可視性ゲート、対話字幕、音声差し替え、Root.tsx 登録

## 全段階に通底する原則

1. **デザイン思想に従う**（→ `00_philosophy.md`）。絵が説明そのもの——図は仕組みを動いて見せ、概念にラベルを貼る飾りにしない。下流の spec・tsx も例外ではない
2. **続編でも単体で完結する**。前作参照は概要欄止まり、本編冒頭で前作コールバックしない（→ `feedback_sequel_self_contained`）

## 進め方の標準パターン

入力（タイトル + 補足）が来たら、4段を順に進める。**各段の成果物をユーザーに見せ、合意を取ってから次段へ進む**。前段が固まる前に次段を書かない（→ `feedback_design_spec_first`）。

1. **[1] script.md** リサーチ＋切り口 → 見せて合意 → 台本ドラフト → 見せて指摘反映
2. **[2] design_overall.md** ドラフト → 見せる → 世界観・オブジェクトの指摘を反映
3. **[3] design_spec.md** ドラフト → 見せる → 挙動・タイミングの指摘を反映
4. **[4] Composition.tsx** 実装 → preview/snapshot で通し見 → 微調整

レンダ（mp4）は重いので、確認は preview/snapshot を優先（→ `feedback_render_preview_preference`）。

## 参考実装

- `sandbox/experiments/public_key_crypto/` — 4段パイプラインで作った完成例。白テーマ、5画面、公開鍵暗号の図解、台本駆動の対話字幕。`research.md` / `script.md` / `design_overall.md` / `design_spec.md` / `Composition.tsx` が各段の成果物の見本

見た目の語彙（形・動き・アクセント色・トーン）は題材ごとに別物（白テーマと文字サイズ下限は固定ベースとして共通）。コピーするのは仕組み——4段の進め方、Track 補間、画面切替、対話字幕の出し方——であって、題材固有の世界観ではない。
