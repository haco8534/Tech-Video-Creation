## セリフは字幕と音声になる

ここで書いた対話は、完成動画で**画面下部の対話字幕**として全行表示される（音声を付ける場合はボイスオーバーの読み上げ原稿にもなる）。だからセリフは、聞いて自然で・読んで分かる長さに書く。1行が長すぎると字幕が収まらない——**目安は1行40字程度まで**、超えるなら文を分ける。

## キャラクター役割（固定）

- **ずんだもん = 解説役**（調べてきた人、〜なのだ口調）
- **めたん = 聞き手役**（素朴な疑問、ですわ口調）

逆転禁止（→ `feedback_character_roles`）。役割を入れ替えると「めたんが説明、ずんだもんが質問」になり違和感が出る。

## 文書構造

```markdown
# {題材タイトル}

# ▶ 序論

めたん：{素朴な疑問・日常的な入口}
ずんだもん：{解説の口火}
めたん：{受け}
ずんだもん：{今日のテーマ提示}

# ▶ ボディ

## {セクション1の見出し}

ずんだもん：……
めたん：……

## {セクション2の見出し}

ずんだもん：……
めたん：……

# ▶ 結論

ずんだもん：{まとめ}
めたん：{腑に落ちる受け}
```

- `# ▶ 序論 / ▶ ボディ / ▶ 結論` … 3部構成の大区分
- `# ▶ ボディ` 配下の `## {見出し}` … ボディ内セクション。見出しは「何の話か」を表す内容ラベル（デザイン名ではない）
- セリフは `話者：本文` の1行。ト書き・効果音・注記は書かない

## セクション = 1画面

**`▶ 序論` 全体、`▶ ボディ` の各 `##` セクション、`▶ 結論` 全体が、それぞれ1画面**になる。画面 = ひとつの連続ステージで、その中ではオブジェクトが状態を連続変化させる。画面が切り替わる瞬間だけ、前画面が一掃され新しい世界に入れ替わる。

台本を書く時点で「ここで画面が変わる」前提でセクションを割る。これが下流の design_overall.md の設計単位を決める。

### セクションの割り方

- **1セクション = 視覚的な世界が一貫しているひと続き**。同じ題材・同じ比喩・同じレイアウトで語れる範囲は、会話が長くても1セクションに収める（オブジェクト中心ステージは長い会話を1画面で支えられる）
- **画面を切り替えるのは、扱う対象や比喩が本質的に変わる節目だけ**。例：「旧来の方式」→「新しい方式」のように世界観ごと入れ替わるとき
- **迷ったらセクションを割らない**。連続が基本、画面切替は例外。割りすぎると 3Blue1Brown 的な連続性が壊れ、ただのスライド送りになる
- **序論と結論は各1画面**。序論が2画面要ると感じたら、入口が長すぎる
- 目安：1本の動画でボディ 2〜4セクション、総画面数 4〜6。これを大きく超えるなら切りすぎ

## 会話設計の原則

### 1. 入口は日常・中身は技術（→ `feedback_beginner_script_design`）

- 抽象化しすぎない。流れは「日常アナロジー → 技術用語の紹介 → 深掘り」
- 固有名詞は削らない（「URL」「DNS」「Self-Attention」を「アドレス」「住所帳」「仕組み」に逃がさない）

### 2. テーマの入口と専門性の対称（→ `feedback_topic_entry_breadth`）

- 日常的な広い入口には、門外漢が付いてこられる中身を揃える
- 入口だけ広くて中身がマニアック／入口は専門なのに中身が浅い、はどちらも NG

### 3. 続編でも単体で完結（→ `feedback_sequel_self_contained`）

- 前作の内容やコメント欄に依存する導入は NG。前作参照は概要欄止まり、本編冒頭で前作コールバックしない

### 4. 攻撃的・煽り的トーン禁止（→ `feedback_theme_tone`, `feedback_slide_critique_tone`）

- 共感 × 知的好奇心ベース。問題指摘テーマでも中立観察に徹する。煽りラベル・支配メタファー禁止

## 口調の参考

**めたん（ですわ口調）**
- 「〜ですわ」「〜ですの」「〜ですわよ」「あら」「まぁ」
- お嬢様キャラだが知識マウントはしない（あくまで聞き手）

**ずんだもん（〜のだ口調）**
- 「〜なのだ」「〜のだ」「〜だぞ」「実はこれ、〜なのだ」「ここからが面白いのだ」
- 解説役だが説教臭くしない、軽快に

## 採否判断

台本ドラフト後にチェック：

- [ ] 序論／ボディ／結論の3部構成になっているか
- [ ] 各セクションが「画面が変わる」節目として妥当か（割りすぎ・割らなすぎがないか）
- [ ] 序論が単体で成立しているか（前作・コメント欄に依存していないか）
- [ ] キャラ口調が一貫しているか（ずんだもん=解説／めたん=聞き手、逆転なし）
- [ ] 固有名詞を一般語に逃がしていないか
- [ ] デザイン・タイミング・event を書き込んでいないか（純粋に対話と区切りだけか）
- [ ] 推定総尺が想定範囲内か（目安：全セリフ文字数 ÷ 8〜10 文字/秒）
