# [1] 台本作成

パイプラインの最初の段。題材をリサーチし、切り口を決め、`script.md` を書く。script.md には**対話と論理構造だけ**を入れ、デザイン・タイミング・event は持たせない。次段 `design_overall.md` の入力になる。

script.md にデザインの語や「ここでアニメ」といった注記を書かない。それは下流の段の仕事。

## セリフは字幕と音声になる

ここで書いた対話は、完成動画で**画面下部の対話字幕**として全行表示される（音声を付ける場合はボイスオーバーの読み上げ原稿にもなる）。だからセリフは、聞いて自然で・読んで分かる長さに書く。1行が長すぎると字幕が収まらない——**目安は1行40字程度まで**、超えるなら文を分ける。

## リサーチ

切り口を決め、台本を書く前に、題材を調べる。リサーチは2つを支える——**正確さ**（間違ったことを書かない）と**切り口**（独自の角度は、調べた中の引っ掛かりから見つかる）。

3つの視点で調べる：

- **主流** — 基本の定義、仕組み、代表的な研究やデータ、なぜ広く使われているか
- **異議** — 通説への反証、例外、限界。「実はみんなが思っているのと違う」点。切り口はここに眠っていることが多い
- **背景** — 歴史的経緯、設計思想、専門家の見解、日常生活との接点

調べた事実は、出典と信頼度（一次資料／専門家の発言／未確認）を添えて `research.md` に控える。**未確認の情報は台本に使わない。** 同じ事象に複数の説があれば主因と副因を切り分け、確定できなければ「諸説ある」として扱う。

## 切り口を決める

リサーチを踏まえ、台本を書き出す前に、この動画の**切り口**を決める。切り口とは、題材に対する一つの角度——どの問い・どの矛盾・どの視点からその題材に入るか。

切り口がないと、台本は「題材をひと通り解説する」だけになる。分かりやすくても、見続ける理由が弱い。

- **切り口はテーマそのものではない。**「{題材}とは」「{題材}を解説」は切り口ではない。「{題材}を、〇〇という角度から見る」が切り口
- **切り口は、引っ掛かりのある場所に宿る。** 通説とのズレ、直感に反する事実、素朴な疑問の奥にある深い構造——リサーチの「異議」で見つけた引っ掛かりが、そのまま切り口になることが多い
- **テスト**：この動画が「何に切り込むか」を一言で言えるか。言えないなら、まだ切り口がない

切り口が決まると、序論でどんな疑問から入り、結論で何に答えるかが自然に定まる。

## キャラクターと掛け合い

この動画の面白さは、構成の仕掛けではなく**二人の掛け合いそのもの**から生まれる。掛け合いは解説の飾りではなく、台本の主役のひとつとして設計する。

### 役割（固定）

- **ずんだもん = 解説役**（調べてきた人、〜なのだ口調）
- **めたん = 聞き手役**（素朴な疑問、友達と話すような普通の口調）

逆転禁止（→ `feedback_character_roles`）。役割を入れ替えると「めたんが説明、ずんだもんが質問」になり違和感が出る。

### 二人のスタンス

| | ずんだもん（解説役） | めたん（聞き手役） |
|---|---|---|
| 立ち位置 | 解説に没頭している | 解説の「外側」にいる |
| トーン | 淡々・真面目・軽快 | 気取らない素の口調、無邪気で子供っぽい |

ずんだもんは調べてきたことを順に語り続ける。めたんは視聴者の代弁者であり、画面に彩りを添えるおちゃらけ役。**淡々としたずんだもんと、おちゃらけるめたんの落差**そのものが、この動画の可笑しさの源になる。

### めたんの3つのモード

めたんの発言は、つねに次のどれか：

- **A 素朴な問い** — 「なぜなの？」「つまり？」。解説を前に進める土台。視聴者がいま抱く疑問と重なる
- **B ユーモアの返し** — 解説を受けた、少しズレた感想や例え
- **C 子供のふざけ** — 内容に関連した所で本筋を離れ、無邪気に遊ぶ

A が背骨。B・C は味付けで、二つの制約を守る：

1. **内容に接続する** — いま解説していることから出たボケだけにする。無関係なギャグ・内輪ネタは入れない
2. **その一行で完結する** — ずんだもんは拾わないので、相手の落としを前提にした一発ギャグは宙に浮く。「解説された概念を自分に当ててみる」式の、それ単体で可笑しい発言にする

### ずんだもんの応答ルール

ずんだもんは、めたんの発言を「実のあるもの」と「お笑い」に分けて扱う：

- **実のあるもの**（モードA の問い、反論、先走り）には応じる。問いには答え、良い問いは褒める（「いい質問なのだ」）。めたんが先走ったら「まだ早いのだ、順番に行くのだ」とたしなめて先へ進む
- **お笑い**（モードB・C のボケ）には反応しない。間も置かず、解説を続ける。ツッコミは書かない

ボケが宙に浮いたまま解説が進む——その宙ぶらりんを、視聴者が心の中でツッコむ。これがこの掛け合いの笑いの構造。

### リズム

ひと続きの長い解説を、めたんの短い発言（相槌・問い・先走り・ボケ）で区切る。めたんの行は短く、ずんだもんの解説行は長め——この長短の交替が、聞き続けられるテンポを作る。

### 口調

**めたん（友達と話すような普通の口調）** — 「〜なの？」「〜なのよ」「〜でしょ」「〜じゃない？」「えっ」「へえ」「ねえ」。気取らない、素のままの喋り方。聞き手であって、知識マウントはしない。

**ずんだもん（〜のだ口調）** — 「〜なのだ」「〜のだ」「〜だぞ」「実はこれ、〜なのだ」「ここからが面白いのだ」。解説役だが説教臭くせず、軽快に。

## 文書構造

```markdown
# {題材タイトル}

# ▶ 序論

めたん：{素朴な疑問・日常的な入口}
ずんだもん：{解説の口火}
めたん：{受け}
ずんだもん：{今日のテーマ提示}

# ▶ ボディ

## {セクション1の見出し}

ずんだもん：……
めたん：……

## {セクション2の見出し}

ずんだもん：……
めたん：……

# ▶ 結論

ずんだもん：{まとめ}
めたん：{腑に落ちる受け}
```

- `# ▶ 序論 / ▶ ボディ / ▶ 結論` … 3部構成の大区分
- `# ▶ ボディ` 配下の `## {見出し}` … ボディ内セクション。見出しは「何の話か」を表す内容ラベル（デザイン名ではない）
- セリフは `話者：本文` の1行。ト書き・効果音・注記は書かない

## セクション = 1画面

**`▶ 序論` 全体、`▶ ボディ` の各 `##` セクション、`▶ 結論` 全体が、それぞれ1画面**になる。画面 = ひとつの連続ステージで、その中ではオブジェクトが状態を連続変化させる。画面が切り替わる瞬間だけ、前画面が一掃され新しい世界に入れ替わる。

台本を書く時点で「ここで画面が変わる」前提でセクションを割る。これが下流の design_overall.md の設計単位を決める。

### セクションの割り方

- **1セクション = 視覚的な世界が一貫しているひと続き**。同じ題材・同じ比喩・同じレイアウトで語れる範囲は、会話が長くても1セクションに収める（オブジェクト中心ステージは長い会話を1画面で支えられる）
- **画面を切り替えるのは、扱う対象や比喩が本質的に変わる節目だけ**。例：「旧来の方式」→「新しい方式」のように世界観ごと入れ替わるとき
- **迷ったらセクションを割らない**。連続が基本、画面切替は例外。割りすぎると 3Blue1Brown 的な連続性が壊れ、ただのスライド送りになる
- **序論と結論は各1画面**。序論が2画面要ると感じたら、入口が長すぎる
- 目安：1本の動画でボディ 2〜4セクション、総画面数 4〜6。これを大きく超えるなら切りすぎ

## 会話設計の原則

### 1. 入口は日常・中身は技術（→ `feedback_beginner_script_design`）

- 抽象化しすぎない。流れは「日常アナロジー → 技術用語の紹介 → 深掘り」
- 固有名詞は削らない（「URL」「DNS」「Self-Attention」を「アドレス」「住所帳」「仕組み」に逃がさない）

### 2. テーマの入口と専門性の対称（→ `feedback_topic_entry_breadth`）

- 日常的な広い入口には、門外漢が付いてこられる中身を揃える
- 入口だけ広くて中身がマニアック／入口は専門なのに中身が浅い、はどちらも NG

### 3. 続編でも単体で完結（→ `feedback_sequel_self_contained`）

- 前作の内容やコメント欄に依存する導入は NG。前作参照は概要欄止まり、本編冒頭で前作コールバックしない

### 4. 攻撃的・煽り的トーン禁止（→ `feedback_theme_tone`, `feedback_slide_critique_tone`）

- 共感 × 知的好奇心ベース。問題指摘テーマでも中立観察に徹する。煽りラベル・支配メタファー禁止

### 5. 分かりやすさの先に、見続けたくなることを置く

台本のゴールは「分かりやすい」だけではない。視聴者が次の一言を聞きたいと思い続けること——それも同じだけ狙う。

ただし、これを構成の型で作らない。フック・引き・回収を枠として決め打ちし、そこを埋めるように書くと、台本は形式的になる。見続けたくなる流れは、めたんが心から知りたくて問い、その問いが視聴者の問いと重なるところ——掛け合いそのものから生まれる。

見続けたくなる流れを、いくつかの規律が下支えする。冒頭で長い前置きや挨拶をしない——視聴者が見に来たものをすぐ出す。同じ調子を続けない——問いの質・テンポ・話題の角度を変えていく。各セクションは、次のセクションへの興味（めたんが抱く次の疑問）で閉じる。

### 6. 解説の正確さと中立性

ずんだもんは「調べてきた人」であって権威ではない。専門家・現場の人・その分野に強い思い入れのある人が見ても、引っかからない解説にする。

- **断定にはスコープを添える** — 時代・地域・集団・文脈。「いつ・どこで・誰にとって」が抜けた言い切りをしない
- **簡略化したら、簡略化したと明かす** — 噛み砕くのはよいが、「正確にはもっと複雑なのだ」と一言残す
- **複数の立場が共存する話題を、勝手に閉じない** — 思想・定義・評価・予測のように正解が一つでない題材で、単一の結論を押し付けない。「これは一つの見方」を残す
- **ポジショントーク・大きすぎる主語を避ける** — 「開発者は」「みんな」式の過大な一般化をしない。評価を述べるときは主語を明示する（誰がそう言っているのか）
- **異論のある主張は、補足で相対化する** — そのまま流さず、めたんの問いやずんだもんの一言で「ただし〜」を添える

## 採否判断

台本ドラフト後にチェック：

- [ ] リサーチを済ませ、切り口が一言で言えるか（「題材の解説」で終わっていないか）
- [ ] 序論／ボディ／結論の3部構成になっているか
- [ ] 各セクションが「画面が変わる」節目として妥当か（割りすぎ・割らなすぎがないか）
- [ ] 序論が単体で成立しているか（前作・コメント欄に依存していないか）
- [ ] キャラ役割・口調が一貫しているか（ずんだもん=解説／めたん=聞き手、逆転なし）
- [ ] めたんのふざけ（B・C）が内容に接続し、その一行で完結しているか
- [ ] ずんだもんがボケに反応していないか（ツッコミを書いていないか）
- [ ] 解説の正確さと中立性（原則6）を満たすか——裏付け・スコープ・議論を閉じていないか
- [ ] 固有名詞を一般語に逃がしていないか
- [ ] デザイン・タイミング・event を書き込んでいないか（純粋に対話と区切りだけか）
