# [1] 台本作成

パイプライン最初の段。題材をリサーチし、切り口を決め、`script.md` に**対話と論理構造**を書く。デザイン・タイミング・event は持たせない——それは下流の段の仕事。`script.md` が次段 `design_overall.md` の入力になる。

## 良い台本がすること

台本は同時に2つを成立させる。多くの台本は、片方を弱いまま放置して失敗する。

1. **解説が、本質を射抜く** — 題材をなぞるのではなく、「そうなるしかない」と視聴者に感じさせる。
2. **掛け合いが、見続けさせる** — 二人のやりとりそのものが、動画を進めるエンジン。

この2つが台本の魂。以下のすべては、この2つに奉仕する。

### 解説は、本質を射抜く

分かりやすいだけの解説は、表層をなぞって終わる。「Xは A・B・C でできていて、Aがこれをする」——正確で、明快で、何も残らない。視聴者は部品を見分けられても、なぜそうなるのかを自分で再構成できない。少しでもその題材を知っている人にしか、芯が届かない。

狙うのは、視聴者が**「そうなるしかない」と腑に落ちる**こと。一拍ごとの解説に、次の4つを効かせる。

- **答えより先に、問題を見せる。** 解決策をいきなり説明しない。まず「何が困るのか」を、視聴者がむずがゆくなるまで描く。答えは、情報としてではなく、解放として届く。
- **「何をするか」でなく「なぜそうするしかないか」。** 各ステップが、前のステップから強いられて出てくるように繋ぐ。「だからこうなる」の連鎖にする。手順の列挙にしない。
- **例えに、説明を背負わせる。** 例えは、その中で考えれば本物の結論に辿りつくものを選ぶ。例えを出したそばから「本当はこれは〜ではなく〜なのだ」と外側から補正し続けているなら、その例えは飾り——親しみは買えても、理解は運んでいない。背負う例えは補正が要らない。例えの論理が、そのまま仕組みの論理になっている。
- **本当に知らない人を、置き去りにしない。** その題材を初めて聞く人が、どのステップでも付いてこられるか。固有名詞は削らない（「URL」「DNS」「Self-Attention」を「アドレス」「住所帳」「仕組み」に逃がさない → `feedback_beginner_script_design`）。固有名詞は名指したまま、背負う例えで支える——それが、易しさと正確さを両立させる唯一のやり方（→ `feedback_topic_entry_breadth`）。

### 掛け合いが、見続けさせる

この動画の面白さは、構成の仕掛けからではなく、**二人の掛け合いそのもの**から生まれる。フック・引き・回収を型として決め打ちし、枠を埋めるように書くと、台本は形式的になる。見続けたくなる流れは、めたんが心から知りたくて問い、その問いが視聴者の問いと重なるところから生まれる。

#### 役割（固定）

- **ずんだもん＝解説役。** 調べてきた人。〜のだ口調。解説に没頭し、淡々・真面目・軽快に語り続ける。
- **めたん＝聞き手役。** 解説の「外側」にいる、視聴者の代弁者。やわらかな女性口調、無邪気で子供っぽい。

#### めたんがすること

めたんの行は、つねに次のどちらか。両方を、動画を通して切らさない。

- **問う。** 視聴者がいま抱く疑問を、めたんが代わりに口にする。解説を前に進める背骨。
- **ふざける。** 解説されたばかりの概念で、子供っぽく遊ぶ。これが、見続けさせる。

**各画面で、めたんは最低一度は本気でふざける。** そして、長いずんだもんの解説を、めたんの短い行（相槌・問い・先走り・ボケ）で区切る。長短の交替が、聞き続けられるテンポを作る。

#### ふざけ方

効くふざけの作り方：

- **解説されたばかりの概念を、自分の生活に持ち帰る。** いま聞いた仕組みを、自分の暮らしに本気で当てはめ、その間抜けな結論に、めたん自身が大喜びする。感想はZ世代的でよい。
- **言い切る。** 思いつきを名案だと信じて言い切る。「〜かしら？」と問いの形に逃げた瞬間、ふざけは死ぬ（本物の疑問なら問いの形でよい——ボケを問いに化けさせない、という意味）。
- **一行で完結する。** ずんだもんは拾わない。落としを相手に預けず、それ単体で可笑しい一行にする。
- **内容に接続する。** いま解説していることから出たボケだけ。無関係なギャグ・内輪ネタは入れない。

> 弱い例：「わたしのお部屋の鍵も、配って回れば無くさず済むのかしら。」——問いの形で、しかも理屈が正しそうで、ふざけになっていない。
> 良い例：「わたしも玄関に付けるわ。ご近所さんに毎日パチンと閉めてもらうの。」——言い切りで、仕組みの価値を取り違えていて、可笑しい。

#### ずんだもんの応じ方

ずんだもんは、めたんの発言を二つに分けて扱う。

- **問い・反論・先走り**には応じる。問いに答え、良い問いは褒め（「いい質問なのだ」）、先走りはたしなめて先へ進む（「まだ早いのだ、順番に行くのだ」）。
- **ふざけ**には反応しない。間も置かず、解説を続ける。ツッコミは書かない。

ボケが宙に浮いたまま解説が進む——その宙ぶらりんを、視聴者が心の中でツッコむ。これが掛け合いの笑いの構造。

#### 口調

- **めたん** — 一人称は「わたし」。語尾は「〜ね」「〜かしら」「〜わ」「〜のね」「〜じゃない？」「〜でしょう」、相づちは「あら」「まあ」「ふうん」「ねえ」。上品でやわらかいが気取らない。聞き手であって、知識マウントはしない。ふざけるときも、この口調のまま。
- **ずんだもん** — 「〜なのだ」「〜のだ」「〜なんだよ」「実はこれ、〜なのだ」。解説役だが説教臭くせず、軽快に。

## 作る手順

### リサーチ

切り口と正確さは、リサーチから出る。3つの視点で調べる。

- **主流** — 基本の定義、仕組み、代表的な研究やデータ、なぜ広く使われているか
- **異議** — 通説への反証、例外、限界。「実はみんなが思っているのと違う」点
- **背景** — 歴史的経緯、設計思想、専門家の見解、日常との接点

### 切り口を決める

台本を書き出す前に、この動画の**切り口**——題材に入る一つの角度——を決める。

切り口がないと、台本は「題材をひと通り解説する」だけになる。分かりやすくても、見続ける理由が弱い。

- 「{題材}とは」「{題材}を解説」は切り口ではない。「{題材}を、〇〇という角度から見る」が切り口
- 切り口は引っ掛かりに宿る——通説とのズレ、直感に反する事実、素朴な疑問の奥の構造。リサーチの「異議」が、そのまま切り口になることが多い

切り口が決まると、序論でどんな疑問から入り、結論で何に答えるかが定まる。引っ掛かりから入る切り口は、それ自体が解説を表層の外へ連れ出す。

## script.md の書き方

`script.md` には**対話と論理構造だけ**を書く。デザインの語・「ここでアニメ」などの注記・タイミング・event は書かない——下流の段の仕事。

### 文書構造

```markdown
# {題材タイトル}

# ▶ 序論
めたん：{素朴な疑問・日常的な入口}
ずんだもん：{解説の口火}
めたん：{受け}
ずんだもん：{今日のテーマ提示}

# ▶ ボディ

## {セクション1の見出し}
ずんだもん：……
めたん：……

## {セクション2の見出し}
ずんだもん：……
めたん：……

# ▶ 結論
ずんだもん：{まとめ}
めたん：{腑に落ちる受け}
```

- `# ▶ 序論 / ▶ ボディ / ▶ 結論` … 3部構成の大区分
- `# ▶ ボディ` 配下の `## {見出し}` … ボディ内セクション。見出しは「何の話か」を表す内容ラベル（デザイン名ではない）
- セリフは `話者：本文` の1行。ト書き・効果音・注記は書かない
- セリフは完成動画で**画面下部の対話字幕**として全行表示される（音声を付ける場合は読み上げ原稿にもなる）。1行は読んで分かる長さに——**目安40字、超えたら文を分ける**

### セクション = 1画面

`▶ 序論` 全体、`▶ ボディ` の各 `##`、`▶ 結論` 全体が、それぞれ1画面になる。画面の中ではオブジェクトが状態を連続変化させ、画面の切り替わりだけが、世界をリセットする唯一の点。台本を書く時点で「ここで画面が変わる」前提でセクションを割る。

- 1セクション＝視覚的な世界が一貫しているひと続き。同じ題材・同じ比喩・同じレイアウトで語れる範囲は、会話が長くても1画面に収める
- 画面を切り替えるのは、扱う対象や比喩が本質的に変わる節目だけ（例：「旧来の方式」→「新しい方式」）。迷ったら割らない——割りすぎると、ただのスライド送りになる
- 序論と結論は各1画面。目安：ボディ2〜4セクション、総画面数4〜6

## 守ること

- **正確さと中立性。** ずんだもんは先生のような立場かつ「調べてきた人」であって権威ではない。専門家・現場の人が見ても引っかからない解説にする。
  - 断定にはスコープを添える——いつ・どこで・誰にとって
  - 簡略化したら「正確にはもっと複雑なのだ」と一言残す
  - 正解が一つでない題材（思想・定義・評価・予測）で、単一の結論を押し付けない
  - 「開発者は」「みんな」式の過大な一般化を避け、評価を述べるときは主語を明示する
  - 異論のある主張は、めたんの問いやずんだもんの一言で「ただし〜」を添えて相対化する
- **冒頭で間を取らない。** 長い前置き・挨拶をしない。視聴者が見に来たものを、すぐ出す

## 採否判断

台本ドラフト後にチェックする。

**本質**
- [ ] 切り口が一言で言えるか（「題材の解説」で終わっていないか）
- [ ] 各解説が「なぜそうなるしかないか」を見せているか（表層をなぞって、手順を列挙していないか）
- [ ] 例えが説明を背負っているか（出したそばから外側で補正し続けていないか）
- [ ] その題材を初めて聞く人が、どのステップでも付いてこられるか。固有名詞を一般語に逃がしていないか

**掛け合い**
- [ ] 各画面で、めたんが最低一度は本気でふざけているか
- [ ] ふざけが言い切りの形か（問いの形に逃げていないか）。内容に接続し、その一行で完結しているか
- [ ] ずんだもんがボケに反応していないか（ツッコミを書いていないか）
- [ ] キャラ役割・口調が一貫しているか（解説／聞き手の逆転なし）

**構造と正確さ**
- [ ] 序論／ボディ／結論の3部構成で、各セクションが「画面が変わる」節目として妥当か（割りすぎ・割らなすぎがないか）
- [ ] 正確さと中立性（スコープ・簡略化の明示・議論を閉じない）を満たすか
- [ ] デザイン・タイミング・event を書き込んでいないか（純粋に対話と区切りだけか）
- [ ] 推定総尺が想定範囲内か（目安：全セリフ文字数 ÷ 8〜10 文字/秒）
