# [2] 全体デザイン方針（design_overall.md）

パイプライン2段目。`script.md` を入力に、**動画の見え方をほぼ全部**決める。デザイン思想は `00_philosophy.md`。下流の spec は時間軸への割り付け、tsx は数値への翻訳——**画面に何が見えるかは、ここで完全に決め切る**。読み手は、このドキュメントだけで各画面の止め絵と動きの段取りが頭に描けなければならない。

## 出力の骨格

```markdown
# {番号}_{題材} — design overall

## 1. 世界観
（色・形・動きの語彙）

## 2. 文字の節制
（画面に出す文字を最小化する判断基準）

## 3. 図カタログ
（各図の静的仕様。見た目はここで決め切る。各図に必須のディテール）

## 4. セクション別デザイン
### {script の画面1}
- 眼目 / 舞台 / 時間順の状態遷移 / 演出ノート
### {script の画面2}
…

## 5. 採否判断
```

**数値は書かない。** 座標・サイズはピクセル値ではなく相対で（「主役の1/3」「画面幅の8割」「字幕帯と同じ厚み」）。フレーム数も書かない（タイミングは特定のセリフ抜粋で）。

**ただし「文字を内包する箱」は例外。** 箱のサイズは「中の文字の長さ × フォント基準サイズ × 1.05 ＋ 左右余白」以上、と**文字基準で書く**（→ 3. 図カタログ）。相対値だけで決めると、tsx でテキストが箱からはみ出す事故が必ず起きる。

## 1. 世界観

### 固定ベース（全題材共通・変えない）

- 背景 `#f5f7fa` ／ パネル面 `#ffffff` ／ 主文字 `#243044`
- 読ませる文字は 42px 以上（具体値は `04_remotion.md`）

### 題材ごとに決める3軸

| 軸 | 決めること |
|---|---|
| 形の語彙 | 「同じ意味は常に同じ形」のマッピング |
| 色の語彙 | アクセント色 2〜4 色。各色が動画を通して同じ意味を運び続ける（途中で別用途に流用しない） |
| 動きの語彙 | 使う動き／封印する動き。**ループ動は全題材で禁止**（浮遊・振動・明滅・脈動・常駐背景モーション） |

## 2. 文字の節制

**文字は最後の手段。** まず絵だけで通せないかを問う。絵だけで意味が立つなら、ラベル・見出し・タイトルを足さない（→ `00_philosophy.md`「絵が説明そのもの」）。

### 削るもの

1. **セクションタイトル（画面上部の大見出し「拡散モデル」「自己回帰モデル」など）は原則なし。** 画面の主題はその画面の絵そのものが語る。テロップで「この画面は◯◯の話」と書くのは、絵が説明を背負えていないサイン。例外を出すなら、その画面で「タイトル無しだと絵が成立しない」理由を1行で書く
2. **軸ラベル・色凡例は、色だけで意味が伝わるなら省く。** 世界観で `AUTOREG=暖黄` を「自己回帰側」と決めていて、絵がその色で識別できるなら、わざわざ「自己回帰モデル」と書かない。1度どこかで「黄＝自己回帰」と定着させたら、以降は色だけで通す
3. **箇条書きリスト（「① 〜 ② 〜 ③ 〜」）は禁止。** 共通点や手順の列挙は対話字幕（声）の役割。絵に「①②③」を書くと、声と画面が同じことを二重に言う冗長になる

### 出してよいもの

- **数式・変数名・添字**（`E=mc²` `q·kᵀ` `xₜ` など）—— 読ませる文字ではなく式の構成要素なので、図に含めて良い。フォントサイズ下限の例外
- **対話字幕**（画面下部・全画面共通）—— 台本を視聴者に届ける必須経路（→ `04_remotion.md` §7）
- **どうしても絵で表せない固有名詞**（製品名・人名・年号など）—— ただし1画面に1〜2個まで

### 量の上限

**1画面に出す独立テキスト ≤ 3**（数式構成要素・対話字幕は除く）。3を超えそうなら、絵で背負えるものが残っていないかを疑う。

## 3. 図カタログ — 各図の静的仕様

**各図の「見た目」はここで完全に決め切る。** 各画面の記述で見た目を書き直さない。図カタログを参照すれば、その図の絵が頭に描ける状態にする。

各図について、以下を書く：

- **役** — 何を表すか（1文）
- **形** — 図形の種類（角丸長方形／円／三角／線画／グリッド…）と細部（角の丸み、線の太さ、塗り or 中抜き）
- **サイズ感** — 画面に対する大きさ（主役なら画面高の○割、補助なら主役の○分の1、字幕帯と同じ厚み、など。相対で）。**ただし文字を内包する箱は、文字基準で書く**（次項）
- **色** — 世界観の語彙のどれを、どこに使うか（輪郭は SUBJECT、塗りは SUBJECT_SOFT、影は…）
- **構成要素（必須）** — サブパーツの内訳と位置関係を箇条書きで。**プリミティブ1〜2個の組合せ（「角丸長方形＋ラベル」など）で済ませない**——次項のディテール必須要件を満たすこと
- **ディテール（次項を参照、必須2つ以上）** — 質感／内部構造／スケール階層 から2つ以上を明記
- **登場画面** — どの画面に出るか

### 文字を内包する箱のサイズ

ラベル付きカード、軸ラベル帯、トークン箱、注記など、内部にテキストを持つ箱は**テキスト基準でサイズを書く**：

> 「フォント基準サイズ Npx、最大文字数 M 字、左右余白 padX。箱の最小幅 = M × N × 1.05 + 2·padX。最小高 = N × 1.6」

「Canvas の 1/3」のような相対値だけで決めない（テキスト幅の保証にならず、tsx で必ずはみ出す）。テキストの内容と長さが画面ごとに変わるなら、画面ごとに最大長を計算する。

### ディテール — 質感・内部構造・スケール階層（必須2つ以上）

各図は、見栄えを成立させるために以下の3要素のうち**最低2つ**を持たせる。プリミティブ1〜2個の組合せは「箱に矢印」と同レベルでパワポ的——必ず作り直す。

- **質感** — 単色塗りで済ませない。グラデーション／薄い影／枠線の太細／薄いハッチング・ドット・罫線・紙の粒立ち、のどれかで質感を持たせる
- **内部構造** — 角丸長方形ひとつで済まさず、サブパーツの階層として設計する（外枠 + 内枠 + 中身 + 装置部品 のように）。各サブパーツは数えると 3〜6 個を目安
- **スケール階層** — 同じ図の中で要素のサイズに大小がある（主従が一目で分かる）。全パーツが同サイズで並ぶと表組み的になる

ディテールは「無意味な装飾」ではない——意味を運ぶ要素のみ足す。「絵が説明そのもの」の原則は崩さず、ディテールも何かを語ること。

## 4. 図の設計の原則

- **背骨を1本通す。** 動画全体で姿を変えていく中心の図を1つ決める。ほかの図はその加工工程として連なる
- **概念1つに図1つ。** 「この図は何を見せるか」が一言で言えなければ、その図は削る

## 5. セクション別デザイン

`script.md` の1画面 = 出力の1サブセクション。各画面で次の4項目を埋める：

### 眼目

絵だけで伝える中心概念（1文）。この画面が無かったら動画から失われるもの。

### 舞台 — 最も混む瞬間の止め絵

その画面が会話のあいだに通る複数状態のうち、**最も混む瞬間の画面の絵**を、読み手が頭の中で描けるレベルで言葉にする。書くこと：

- 主役（背骨の図）はどこにいて、どの大きさで、どの状態か
- 補助の図たちはどこに置かれ、それぞれが何を見せているか
- 画面の上下左右に意味を割り当てるなら、その対応（左＝入力／右＝出力 など）
- 余白の位置（字幕帯のぶんの下端余白は常に確保）

この記述で、その画面の止め絵が一枚、迷わず描けること。

### 時間順の状態遷移

最も混む瞬間に至るまでの動きを、**ステップ列**で書く。各ステップで「何が」「どう変わる」を動詞で書き、紐づくセリフを特定できる抜粋で添える（spec 段でこれを event に翻訳する）。

#### テンポ目標 — 3〜5セリフに1ステップ

**画面のセリフ N 行に対して、状態遷移ステップは概ね N/4 〜 N/3 個。** 1画面が 12 セリフなら、ステップ 4〜5 個では少ない（画面が長く無音になる）。最低 4〜5 個、できれば 6〜8 個を狙う。

**ミクロな動きも積極的に足す。** 大きなオブジェクトの登退場・モーフだけが「動き」ではない。次のような小さな反応もステップとして数えてよい：

- 既存図の一部分の色が一段濃く／薄くなる
- 補助線がふっと現れ、数秒で消える
- 図の輪郭が一拍だけ強調される（枠が一瞬太くなって戻る）
- 既存図の中の一要素が脈動を伴わず一度きり震える／指される
- 数値・添字がカウントアップする

ただし**ループ動は禁止のまま**（→ `00_philosophy.md` 7）——ミクロな動きも一度きりで終わる。

#### 書式

```
(a) ずんだもん「軸を引くのだ」近辺：軸が左から右へ描き起こされる
(b) めたん「あら、こうなるのね」近辺：軸の左端に完全砂嵐パッチが、ふわっと浮かび上がる
(c) ずんだもん「一手目」近辺：軸の上に一手アイコン（DIFFUSION 色の小さな三角）が現れる
(d) ずんだもん「次の絵が」近辺：(c) の右隣に、ノイズが少し薄い絵が地続きに現れる
(e) ずんだもん「もう少し薄く」近辺：同上、もう一段薄い絵が現れる
(f) ずんだもん「猫が立つ」近辺：右端に猫の線画がはっきり立つ。直前にひと呼吸の間
```

#### 書くときに守ること

- **「{セリフ抜粋} 近辺」は必須**——「{セリフ抜粋}」だけで script.md の特定の1行に一意特定できる程度に書く。「次のセリフで」「しばらくして」のような曖昧表現は禁止（spec が紐付け先を決められない）
- **登場・退場の出生元／退場先を1つに決める。** 「画面外から湧く」「フェードして消える」だけで済ませない。「主役の右肩から枝分かれする」「補助の枠の中に吸い込まれて消える」のように、既存の図に紐付ける
- **動きそのものを動詞で描写する。** 「現れる」だけで済ませず、「ふわっと浮かび上がる」「左から滑ってきて止まる」「主役の輪郭をなぞって描き起こされる」のように、見えるさまを言う
- **連続変形（モーフ）と新規登場を区別する。** 既存の図を作り直さず、姿を変えるなら「{図} が {状態A} から {状態B} へ連続変形」と明示

### 演出ノート

止め絵と遷移を補強する、表情の指示。書くこと：

- **テンポと強弱** — ゆっくり開く／さっと閉じる／ためらいの一拍を入れる／クライマックスでひと呼吸止める
- **カメラワーク**（あれば） — ズームイン／引き／パン。何を凝視させ、何を俯瞰させるか
- **象徴的なディテール** — 完成した瞬間に外枠だけが光る／失敗の矢が落ちると地面が少し沈む／最後の一手が決まるとき、ほかの要素が一段薄くなる
- **絶対やらないこと**（必要があれば） — 「ループする揺れは入れない」「複数の要素を同時にアニメさせない」のように、誘惑を断つ指示

## 画面の整合性 — 4つの制約

**ここを満たさない画面は、必ず作り直す。**詰め込み・同時並行・対比過多・重なりが、デザイン破綻の四大要因。

1. **同時に居る、意味的に独立した補助は最大3つ。** 字幕帯・背骨の主役を除いた**意味的に独立した補助の図**は、画面内のどの瞬間も3つを超えない。ここで言う「補助の図」は意味の単位——1つの図がサブパーツとして 4〜6 個のプリミティブを抱えるのは「1補助」と数える（→ 3. 図カタログ「内部構造」）。意味の独立した補助が4つ以上なら、本筋に関係ない図が紛れているか、画面を割る合図
2. **同時に動くのは1つ。** 視聴者の視線は一点しか追えない。因果的に一体の動き（視点移動＋登場 など）を除いて、別の図を並行で動かさない（→ `00_philosophy.md` 3）
3. **1画面で扱う対比軸は1つ。** 「2方式を比べる」「得手・不得手を見せる」「ゴールが同じだと示す」を1画面に詰めない。詰めたくなったら画面を割る——足し算で見せ場を増やすほど、視聴者は焦点を失う
4. **「最も混む瞬間」で全図が重ならない。** 各図に占有ゾーンを与え、動いた図の着地点を空けておく。途中の状態は最も混む瞬間の部分集合になる

## 密度 — 余白を残しすぎない

「補助は最大3」「同時に動くのは1」を守ると画面はスカスカに振れがち。**密度の下限**も持つ：

- **主役 + 補助の占有面積が、中央エリア（字幕帯・上余白を除いた高さ約 700px ぶん）の 60% 以上**。これより低いと「描くものが無くて空いている」状態
- 密度を上げる先は**装飾ではなく構造**——図カタログのディテール3要素（質感・内部構造・スケール階層）で持ち上げる。中央エリアに何もない時間を作らない

## レイアウトの約束

- 画面下部は対話字幕の帯（2行）が常駐。主役の図は上寄り〜中央。下端に重要物を置かない
- 上余白も控えめでよい（セクションタイトル原則なし → § 2 文字の節制）。タイトル帯のぶんを当たり前に空けない
- スマホ視聴前提。要素数は絞り、各要素はディテールを持つ（節制と密度の両立）
- 画面切替は標準30fクロスフェード（境界処理は一律でよい）

## 採否判断

**世界観**
- [ ] 固定ベース（白テーマ・文字下限）を守り、アクセント3軸を題材ごとに設計した
- [ ] 各アクセント色が「何を意味するか」が決まっていて、最後まで流用していない

**文字の節制**
- [ ] 各画面の独立テキスト ≤ 3（数式構成要素・対話字幕を除く）
- [ ] セクションタイトル（画面上部の大見出し）を原則出していない。出す画面があれば、無いと絵が成立しない理由を書いた
- [ ] 軸ラベル・色凡例を、色で識別できる場面では省いている
- [ ] 箇条書きリスト（「① 〜 ② 〜」）を絵に書いていない

**図カタログ**
- [ ] 各図の見た目（形・サイズ感・色・構成要素・ディテール）が書かれている
- [ ] 図カタログだけ読めば、その図の絵が頭に描ける
- [ ] 各図が「質感／内部構造／スケール階層」のうち**2つ以上**を持つ（プリミティブ1〜2個の組合せで終わっていない）
- [ ] テキストを内包する箱のサイズが、文字基準（最大文字数 × 基準フォントサイズ × 1.05 ＋ 余白）で書かれている
- [ ] 背骨が1本通っていて、各図は「何を見せるか」が一言で言える

**セクション別デザイン**
- [ ] 各画面に「眼目」「舞台」「時間順の状態遷移」「演出ノート」がそろっている
- [ ] 「舞台」を読めば、最も混む瞬間の止め絵が一枚、迷わず描ける
- [ ] 各画面の状態遷移ステップ数が、そのセクションのセリフ数の **1/4〜1/3** に達している（無音ストレッチが出ない）
- [ ] 各ステップが特定のセリフ抜粋で一意に紐付いている（「次のセリフで」のような曖昧表現なし）
- [ ] 登場・退場の出生元／退場先が、既存の図に紐付いている（「画面外から湧く」で済ませていない）
- [ ] 動きの描写が動詞で具体的（「現れる」だけで済ませていない）
- [ ] 演出ノートに、テンポと象徴的なディテールが書かれている

**整合性と密度**
- [ ] 各画面の意味的に独立した補助 ≤ 3（サブパーツは数えない）
- [ ] 各画面で同時に動くのは1つ
- [ ] 各画面で扱う対比軸は1つ
- [ ] 全状態で図どうしが重ならない
- [ ] 中央エリアの 60% 以上が主役＋補助で埋まる（スカスカの空白画面になっていない）

**数値**
- [ ] ピクセル座標・フレーム数を書いていない（相対サイズと言葉のタイミングに留める）
- [ ] **例外**: テキスト内包箱のサイズだけは文字基準で書いている
