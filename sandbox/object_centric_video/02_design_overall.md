# [2] 全体デザイン方針（design_overall.md）

パイプライン2段目。`script.md` を入力に `design_overall.md` を作る。**各画面にどんな図が居て、どう動くか**と、**動画全体を貫く世界観**を、自然言語で設計する。

デザイン思想は `00_philosophy.md`。この段は、その思想を題材に当てはめる作業。**創造的なデザイン判断はこの段に集約する**——下流の spec と tsx は、ここで決めた設計を形にするだけ。

## 書くこと・書かないこと

| 書く | 書かない |
|---|---|
| 貫通する世界観（色・形・動きの語彙） | event アンカー、セリフ単位のタイミング（→ spec） |
| 各画面に登場する図と、その動き方 | 座標値、フレーム数（→ tsx） |

**すべて質的に書く。** レイアウトは「中央に横並び」、動きは「左から右へ走査」のように言葉で。数値は一切書かない——下流の仕事。

## 世界観を6軸で決める

下の6軸で世界観を決める。**ただし固定ベースの2点（白テーマ・文字の大きさ）は題材で変えない。** 固定ベースに従い、残りを題材ごとにゼロから組む。

### 固定ベース（全題材共通・変えない）

- **白テーマ** — 背景とパネル面は白系で固定する（背景 `#f5f7fa` 系／面 `#ffffff`／主文字 `#243044` 系）。明るい作業台の上で図が動く画作りに統一する。アクセント色（強調・各装置の色）と図の形の語彙は、これまでどおり題材ごとに設計してよい——白い土台の上で題材固有の語彙が映える。
- **文字は大きく** — スマホ視聴前提。画面に出る文字は原則「読ませる文字」として大きく描く。小さくしてよいのは、大量に並んだ図形の連番・添字のような「記号としての文字」だけ。具体値は `04_remotion.md`「白テーマと文字サイズ（固定ベース）」に従う（→ `feedback_slide_font_size`）。

| 軸 | 決めること |
|---|---|
| 貫通する比喩 | 動画全体の手触り（白い土台の上で何が起きるか） |
| 形の語彙 | 「同じ意味は常に同じ形」のマッピング |
| 色の語彙 | **背景・面は白で固定**。題材ごとに決めるのはアクセント色（強調・各装置の色）とその意味だけ |
| 動きの語彙 | 使う動き／封印する動き（ズーム・回転・パン・変形…）。**ループし続ける動き（浮遊・振動・明滅・脈動・常駐背景モーション）は全題材で封印**——動きは1〜2回で終え、止める（→ `00_philosophy.md` 7） |
| 文字方針 | 対話字幕（話者色・位置）と、図中のラベル。読ませる文字は大きく、小さくするのは記号としての文字だけ（固定ベース） |
| ナビ補助 | セクションタイトル・進捗表示の有無と形式 |

## 図を設計する

`00_philosophy.md` の通り、図はひとつの仕組みを見せる専用品として設計する。手順：

1. **背骨を一本決める** — 動画全体で姿を変えていく中心の図（例：語の「意味」を表す数値の図形）。全画面がこの一つの変化の物語になるようにする
2. **各概念に図を一つ割り当てる** — script の各セクションの中心概念に対し、「それが起きるさまを絵で起こす」図を1つ設計する
3. **図を工程としてつなぐ** — 各図は背骨を順に加工する一連の工程。「この図は何を見せるか」を言えない図は削る

図の数は題材の密度しだい（長尺・多概念なら10以上になることもある）。基準は数ではなく、各図が固有の仕組みに対応しているか。

図カタログには各図の **見せる仕組み** を必ず書く。詳細なライフサイクル（誕生・退場）は次段 spec の仕事。

## 遊びを設計する

図が厳密に正しく動くだけでは、動画は「真面目な人が作ったもの」に見える。`00_philosophy.md`「図は厳密に、しかし動画は硬くしない」のとおり、図が**何を語るか**は厳密に、**どんな表情で語るか**には遊びを入れる。design_overall で、その遊びを世界観の一部として設計する。

- **人物っぽい図に、表情と反応を。** 送り手・受け手・利用者など、図の中に人物的なものが現れたら、話の節目で誇張して反応させる（驚き・喜び・困惑）。動画に体温が入るのはここ
- **遊び方は、この題材のために作る。** 色・形・動きの語彙と同じく、ふざけ方も題材ごとにゼロから設計する。気に入った一つのふざけを、どの動画でも繰り返さない
- **遊びは台本の節目に乗せる。** どの反応も script のあるセリフ・出来事に紐づく一回きりの反応。常時揺れる飾りではないので、ループ禁止の規律と衝突しない

実装上、表情などの切り替えは数値 `mix`（0→1）のクロスフェードで持つ（→ `04_remotion.md` の State 規則）。

## セクション別デザインを書く

**script.md の1セクション（= 1画面）に対し、出力の1サブセクションを書く。** 画面リストと1対1で対応させる。各画面で4項目を埋める：

- **登場する図** — その画面に居る図の顔ぶれ
- **レイアウト** — 空間配置を質的に（「中央に横並び」「上半分に円形」「左に主役・右に補助」）
- **動き** — 会話が進むにつれ図がどう変化するか。変形・登場のしかた・遷移を質的に書く（→ `00_philosophy.md`「図の動かし方」）。例：「左から右へ走査」「元の図から湧き出る」「役目を終えた補助だけ消える」
- **遊び** — この画面で図のどこに遊びを入れるか（人物っぽい図の誇張した反応など → 「遊びを設計する」）
- **絵の眼目** — この画面が絵だけで伝える中心概念。1文で

書いたら検査する：

- **絵で語れるか** — 「絵の眼目」が絵だけで示せるか。絵的な手がかりが思いつかない画面は、概念を絵で語れていない。台本側に差し戻す
- **静止していないか** — 図はその画面の会話の間に必ず状態変化する。「最初から最後まで同じ」の画面は設計し直す
- **湧いていないか** — 新しい図の出生元（どの既存の図から生まれるか）と退場先（どこへ溶けるか）を散文で書く。画面の切り替わりだけがリセットを許す

## レイアウトの約束

- 画面下部は対話字幕の帯（2行）が常駐する。主役の図は上寄り〜中央に置き、下端に重要物を置かない
- スマホ視聴が主。文字を大きく取るぶん、1画面に置けるものは少ない。詰め込まず、余白を多めに残す
- 画面の入れ替わりは標準30fクロスフェード。この段では各画面の中身に集中し、境界処理は一律でよい

### 重なりを設計で防ぐ

1画面は、会話が進むあいだに複数の状態を通る（図が増え、動き、姿を変える）。**レイアウトは、その画面が通るすべての状態で破綻しないように設計する。** ある一瞬だけ成立する配置は不可。

- **いちばん混む状態を基準にする。** その画面で図が最も多く同時に出ている瞬間を想定し、その全部が重ならず収まる配置を組む。途中の状態は、その配置の部分集合になる
- **各図に占有ゾーンを与える。** 図ごとに「ここはこの図の場所」という領域を決め、ほかの図をそこへ侵入させない
- **新しい図は、空いたゾーンへ移動して生まれる。** 既存の図から派生する図は、出生元に重なったまま留まらない。湧き出たら、自分の空きゾーンへ移動しきる
- **動いた先を空けておく。** 図が移動・変形して着く位置に、別の図を残さない。そこに図が居るなら、その図は先に退場・移動を終えている（→ `03_design_spec.md` がこの前後関係を時間軸に落とす）

## 採否判断

ドラフト後にチェック：

- [ ] script.md の全画面に対応するデザインがあるか
- [ ] 固定ベース（白テーマ・文字の大きさ）を守り、残る世界観を題材ごとに決めたか
- [ ] 背骨（姿を変えていく中心の図）が一本通っているか
- [ ] 図カタログの各図が「見せる仕組み」を一言で言えるか（ラベルを貼るだけの箱になっていないか）
- [ ] 各画面に「絵の眼目」があり、画面内で図が状態変化するか
- [ ] 図の動き・登場・遷移が `00_philosophy.md`「図の動かし方」に沿っているか（意味のない動き・一斉登場・全消し遷移・ループし続ける動きがないか）
- [ ] 湧く／消える図がないか（出生元・退場先が既存の図か、画面の入退場か）
- [ ] 画面が通るどの状態でも図どうしが意図せず重ならないか（占有ゾーンが分かれ、新しい図・動いた図の着地点が空いているか）
- [ ] 動画が硬くなっていないか（図の表情・遊びが、題材固有の形で設計されているか）
- [ ] 座標値・フレーム数・セリフ単位タイミングを書いていないか（質的記述に留まっているか）
