# 暗号化って具体的に何をやっているのか — 全体デザイン方針

## 背骨

舞台は一本の**透明なビット列レーン**。左に平文、中央に鍵から作られるマスク生成機、右に暗号文があり、動画全体を「**平文との関係がどこで切れるか／どこに残ってしまうか**」の一つの図で貫く。暗号化を箱や南京錠として見せず、ビット列・マスク・nonce・タグを、同じレーン上で出たり消えたりする部品として扱う。各画面は、ビット列に残る関係をひとつずつ断ち切っていく工程として連なり、最後に「暗号文はぐちゃぐちゃな文字ではなく、関係を断たれたビット列」という構図へ戻る。

## 画面リスト

| # | 画面 | 絵の眼目（一文） |
|---|---|---|
| 1 | 導入／ビットの関係 | 文字が 0/1 の列にほどけ、鍵で決まる巨大な変換に通ることで、元のパターンとの関係が断たれる |
| 2 | nonce と鍵ストリーム | 鍵と nonce から毎回ちがうマスクが生成され、同じ平文でも暗号文の指紋が変わる |
| 3 | nonce 再利用の破綻 | 同じマスクを二度使うと、二つの暗号文を重ねた瞬間にマスクが消え、平文同士の関係が露出する |
| 4 | モードの役割 | AES の 128 ビット歯車だけでは長いデータを扱えず、モードがブロック列を安全な流れに組み替える |
| 5 | 認証タグ | 暗号文に触られても、タグ照合で改ざんを検出し、読めないことといじられていないことを分けて見せる |
| 6 | 鍵の作り分け | 共有秘密から用途別の交通鍵が枝分かれし、同じ秘密をあちこちで使い回さない構造を見せる |
| 7 | 漏れる外形 | 本文は隠れても、宛先・長さ・タイミング・端点の平文はレーン外に残り、暗号化の射程を示す |
| 8 | 結論 | 平文・鍵・nonce・モード・タグ・外形を一列に並べ直し、暗号化の仕事を「関係を切る技術」として回収する |
