# 画像生成AI

# ▶ 序論

めたん：ねえずんだもん。最近よく見る画像生成AI、あれって不思議じゃない？
ずんだもん：何が不思議なのだ？
めたん：プロンプトに「猫の写真」って打つと、本当に猫が出てくるのよ。
ずんだもん：うんうん、出てくるのだ。
めたん：あれって、AIが頭の中で完成図を思い浮かべて、ぱっと描いてる感じかしら？
ずんだもん：実はね、そこが多くの人が誤解してるところなのだ。
めたん：あら、違うの？
ずんだもん：画像生成AIはね、一発で絵を当てることを、最初から諦めてるのだ。
めたん：諦めてる？それで絵が出てくるの？
ずんだもん：むしろ、諦めたからこそ出せるのだ。
めたん：諦めて、出せるようになる。逆みたいで気になるわ。
ずんだもん：今日はそこを解きほぐすのだ。
ずんだもん：主役は二つ。拡散モデルと、自己回帰モデル。
めたん：聞いたことはあるわ。Stable DiffusionとかGPT-4oとか。
ずんだもん：そのへんで使われてる仕組みなのだ。
ずんだもん：見た目はぜんぜん違うんだけど、骨組みはほとんど同じ。
めたん：同じなの？それぞれ別物の発明だと思ってたわ。
ずんだもん：同じ作戦の、軸違いなのだ。
めたん：軸違い。なんだかパズルみたいね。
ずんだもん：順番に解いていくのだ。
ずんだもん：まずは「一発で当てるのが、なぜ無謀なのか」から。

# ▶ ボディ

## 一発で当てるという無謀

ずんだもん：画像って、コンピューターから見ると、ただの数字のマス目なのだ。
めたん：マス目ね。
ずんだもん：よく見るSNS向けの正方形の絵は、だいたい1024×1024のマス。
ずんだもん：マスの数は、約100万個。
めたん：100万のマスね。
ずんだもん：1マスは、赤・緑・青の三色を混ぜて表現する。
ずんだもん：それぞれ256段階あって、1マスで約1600万通りの色が選べるのだ。
めたん：100万のマスに、それぞれ1600万通り。
ずんだもん：可能な絵の総数は、もう数字で書く意味がないくらい巨大なのだ。
めたん：想像つかないわね。
ずんだもん：その膨大な可能性のほとんどは、人間が見ても、ただのザーザーの砂嵐に見える絵。
めたん：あら、絵じゃなくて？
ずんだもん：絵じゃない。完全にランダムな色のマスが並んでるだけ。
めたん：そっか、全部の組み合わせの中身は、ほとんど無意味な絵なのね。
ずんだもん：その通り。人間が「猫」と認めるパターンは、その大海の中の、ごく小さな点。
めたん：ふうん。じゃあ、絵に見える組み合わせを引き当てるのが、難しいってこと？
ずんだもん：難しい、というか、無謀なのだ。
ずんだもん：100万マスの色を、全部いっぺんに当てる問題、なのだ。
めたん：いっぺんに、全部。たしかに目眩がするわね。
ずんだもん：そして、当たったかどうかは、絵が全部完成したあとにしか分からない。
めたん：途中で「あ、ここ違うな」って気づけないってこと？
ずんだもん：そう。1マスずつ確かめる手段がない。最後の絵を見て、初めて当たり外れが分かる。
めたん：それは、無謀どころか、最初から勝負になってないわね。
ずんだもん：だから、画像生成AIは、この戦い方をまず捨てたのだ。
めたん：捨てて、何をしたの？
ずんだもん：そこが次の話なのだ。
めたん：聞かせて。

## バラし方、二つ

ずんだもん：捨てた代わりに、こう考え直したのだ。
ずんだもん：「一発で当てる」を、「易しい一手のくり返し」にバラそう、と。
めたん：易しい一手。
ずんだもん：たとえばこう。完成済みの絵に、ほんの少しだけ手を加える、なら易しいのだ。
めたん：ふむふむ。
ずんだもん：「ほとんど完成」を「もう一歩だけ完成に寄せる」のは、全部当てるよりずっと簡単。
めたん：たしかに、一気にゼロから描くより、ちょっとずつ直す方が楽そうね。
ずんだもん：そう。じゃあ、最初は「ほとんど完成」じゃなくて、もっと荒れた状態だったらどうする？
めたん：それでも、ちょっとずつ完成に近づければ良いってこと？
ずんだもん：そう。少しだけ寄せる、を何度も何度もくり返せば、最後には完成にたどり着く。
めたん：大きな問いを、小さな問いの列に分けたのね。
ずんだもん：その通り。一発当ての無謀を、易しい一手の連鎖に置き換えた。
めたん：それなら、各ステップは学べそうな気がするわね。
ずんだもん：そこが鍵。一発当ては学べないけど、易しい一手なら学べる。
めたん：で、その「バラし方」に二通りある、って話だったわよね。
ずんだもん：そう。同じ作戦を、別の軸でバラした、二つの仕組みがあるのだ。
めたん：その二つ、ちょっと先に教えてもらえる？
ずんだもん：まず一つ目。最初の状態は、画面ぜんぶがザーザーの砂嵐から始まる。
めたん：完全な砂嵐ね。
ずんだもん：そこから、画面全体のザラつきを、少しだけ薄める。
ずんだもん：もう少し薄める。さらに薄める。
めたん：画面全体が、だんだんはっきりしていくのね。
ずんだもん：最後にはザラつきが消えて、絵が現れる。これが拡散モデル。
めたん：もう一つは？
ずんだもん：もう一つは、最初の状態が、何も描かれていない、空っぽのキャンバスから始まる。
めたん：真っ白ね。
ずんだもん：そこに、左上のマスから順に、一マスずつ色を置いていく。
めたん：マスを一個ずつ、順に埋めていくのね。
ずんだもん：全部埋まれば、絵が完成する。これが自己回帰モデル。
めたん：あら、ぜんぜん違うわね。一方は「画面全体を少しずつ綺麗に」で、もう一方は「マスを順に埋める」。
ずんだもん：見た目は違うのだ。でも、両方とも、易しい一手の連鎖、という意味では同じ。
めたん：軸が違うだけ、って言ってたのは、こういうことね。
ずんだもん：では、それぞれをじっくり見せていくのだ。
めたん：お願い。

## 拡散モデル — ノイズを少しずつ引く

ずんだもん：まずは拡散モデル。
めたん：お願い。
ずんだもん：絵を作るときは、画面全体が完全な砂嵐から始まる。
めたん：色のついた砂粒が、画面いっぱいに敷き詰められてる感じね。
ずんだもん：そこから、ノイズをほんの少しだけ引く。
めたん：「ノイズを引く」って、具体的にはどういうこと？
ずんだもん：各マスの色を、ほんの少しだけ「絵に近そうな色」に寄せる、ということ。
めたん：絵に近そうな色。
ずんだもん：完全にランダムだった砂粒が、わずかに偏りを持ち始める。
めたん：まだほとんど砂嵐だけど、なんとなく構図のヒントが見えてきた感じ？
ずんだもん：そんな感じ。次のステップで、もう少しノイズを引く。
ずんだもん：また次のステップで、もう少し引く。
めたん：だんだん、絵がはっきりしてくるのね。
ずんだもん：それを20回とか50回くり返すと、絵が完成する。
めたん：待って。各ステップで「ノイズを少し引く」って、AIはどうやって判断してるの？
ずんだもん：いい質問なのだ。秘密は学習のときの仕掛けにある。
めたん：仕掛け？
ずんだもん：学習のときは、逆をやるのだ。完成済みの絵に、ノイズを少しずつ足していく。
めたん：絵を、わざと汚していくのね。
ずんだもん：そう。完成した絵から始まって、ちょっとずつ砂嵐に近づけていく。
めたん：手順を、生成と反対方向に回すのね。
ずんだもん：そう。すると、各段階で、ノイズが少ない絵と、もう少し多い絵の、ペアができる。
めたん：ふむふむ。
ずんだもん：そのペアを大量に集めて、AIに「多い側を見て、少ない側を当てよ」と訓練する。
めたん：あら、それなら答えがちゃんとあるわね。
ずんだもん：一発当ては答えがないけど、こっちは答えがある。
めたん：教師ありの問題に化けるのね。
ずんだもん：そう、これが鍵。AIは「答えのある問題」しか学べない。
めたん：絵を汚す手順を自分で決めたから、汚す前の答えを知ってる、と。
ずんだもん：その通り。だから絵を作るときは、足した手順を逆にたどるだけで良い。
めたん：純粋な砂嵐から、ちょっとずつ綺麗にしていけば、いつか完成にたどり着くのね。
ずんだもん：これが拡散モデルの正体なのだ。
めたん：ところで、なんで「拡散」っていう名前なの？
ずんだもん：絵にノイズを足していく方の手順を「拡散」と呼ぶのだ。
ずんだもん：絵が砂嵐に向かって、染料が水の中に広がるみたいに、ぼやけていく様子に似てるから。
めたん：あら、ちゃんと意味のある名前ね。
ずんだもん：生成側の手順は、その逆だから「逆拡散」とも呼ばれるのだ。
めたん：絵を作る側が「逆」って呼ばれるの、ちょっと面白いわね。
ずんだもん：学習の都合上、汚す方向が「正方向」と決まってるのだ。
めたん：これを使ってるのが、Stable DiffusionとかMidjourneyとかDALL-Eね。
ずんだもん：そう。今の世のお絵かきAIの主流は、ほとんどこの仕組みなのだ。

## 自己回帰モデル — マスを左上から埋める

ずんだもん：次は、自己回帰モデル。
めたん：お願い。
ずんだもん：絵を作るときは、何も描かれていない、空っぽのキャンバスから始まる。
めたん：真っ白ね。
ずんだもん：そこで、画像を「パッチ」という小さなマスに区切る。
めたん：マスの大きさは？
ずんだもん：仕組みによるけど、例えば16×16ピクセルが1パッチ。
めたん：1024の絵だと、縦横64個ずつ、合わせて4096個のパッチね。
ずんだもん：そう。そのパッチを、左上から順番に、一つずつ埋めていくのだ。
めたん：1パッチずつ、順に色を決めていくのね。
ずんだもん：そう。各ステップで「次のパッチに何が来るか」を予測する。
めたん：文章の続きを書くみたいに？
ずんだもん：まさにそれなのだ。GPTが文章を作るときと、まったく同じ仕組み。
めたん：あら、画像なのに文章と同じ機械でいけるの？
ずんだもん：画像をパッチの「列」に並べ直せば、文字の列とほとんど同じものになる。
めたん：列にして並べるんだ。
ずんだもん：行ごとに左から右、それを上から下へつなげれば、ただの「パッチの並び」になる。
めたん：そうすれば、GPTの仕組みがそのまま使えるってこと？
ずんだもん：そう。だから、自己回帰モデルでの画像生成は、テキスト生成と全く同じ機械で動かせるのだ。
めたん：各ステップは、何を見て、何を予測してるの？
ずんだもん：これまで埋めたパッチを全部見て、次の1パッチの中身を当てる。
めたん：これも、各ステップは易しい問題ね。
ずんだもん：そう。完成全部を一発で当てる問題が、1パッチ当てる問題の列に化ける。
めたん：これも、学習はどうやってるの？
ずんだもん：完成された画像を持ってきて、パッチの列に並べる。
ずんだもん：そして、「途中までを入力に、その次のパッチを答えに」というペアを大量に作る。
めたん：これも教師あり問題に化けるのね。
ずんだもん：そう。答えのある問題に変身させてから学ばせる、というところは拡散モデルと同じ。
めたん：途中までを見せて、続きを当てさせる、っていう作り方なのね。
ずんだもん：そう。だから絵を作るときは、ペアの作り方をそのまま再現すれば良い。
めたん：1パッチ出したら、それを入力に加えて、次の1パッチを出す。
ずんだもん：それを最後まで繰り返せば、絵が完成する。
めたん：ところで、なんで「自己回帰」っていう名前なの？
ずんだもん：「自分の出力を、次の入力に回し続ける」からなのだ。
ずんだもん：1パッチ出力したら、それを次の入力に回す。出力が次の入力になる。
めたん：ぐるぐる回しながら、絵が立ち上がっていくのね。
ずんだもん：これを使ってるのが、GPT-4oや、最近のGoogleのGeminiの画像生成。
めたん：あら、ChatGPTのあれも？
ずんだもん：そう、ChatGPTで絵を作れる機能の中身が、この仕組み。
めたん：テキストと画像が、同じ機械の上で動いてるってことね。
ずんだもん：そう、それが自己回帰モデルの大きな強み。
めたん：拡散モデルとは、ぜんぜん作り方が違うのね。

## 並べてみると、軸が違うだけ

ずんだもん：ここまでで、二つの仕組みを見たのだ。
めたん：うん、ぜんぜん違って見えたわ。
ずんだもん：見え方は違うのだ。でも、骨組みはほとんど同じ。
めたん：骨組み？
ずんだもん：三つの共通点があるのだ。一つずつ並べていく。
めたん：はい。
ずんだもん：一つ目。両方とも、一発当ての無謀を、易しい一手の連鎖にバラした。
めたん：これは共通ね。
ずんだもん：二つ目。両方とも、その連鎖を「前の状態を見て、次を当てる」という形に揃えた。
めたん：拡散モデルは「ノイズが多い絵を見て、少ない絵を当てる」。
めたん：自己回帰モデルは「埋まったパッチを見て、次のパッチを当てる」。
ずんだもん：どっちも、すぐ手前を見て、その次を出す形。
めたん：たしかに、構造は同じね。
ずんだもん：三つ目。両方とも、学習のときに「逆方向」を仕込むことで、答えのある問題に化けさせた。
めたん：あら、これも共通ね。拡散はノイズを足す側、自己回帰はパッチを並べる側。
ずんだもん：自分で答えを作ったから、自分で学べる。
めたん：三つも揃ってると、ほとんど同じ仕組みって言いたくなるわね。
ずんだもん：そう、骨組みは本当にそっくりなのだ。
めたん：じゃあ、違うのはどこ？
ずんだもん：違うのは、バラし方の軸、ただ一つだけ。
めたん：軸。つまり、何にそって分割したか、ってこと？
ずんだもん：その通り。
ずんだもん：拡散モデルは「ノイズの量」を軸にバラした。画面全体のザラつきを段階的に薄める。
ずんだもん：自己回帰モデルは「空間の位置」を軸にバラした。マスを順番に埋める。
めたん：同じ作戦を、別の方向で切ってるだけね。
ずんだもん：そう。同じ作戦を、別の方向で切ったから、出てくる絵の作られ方が違って見えるだけ。
めたん：見た目が違うから、別物に見えてただけ、ということね。
ずんだもん：そう。中身の発想は、ほとんど同じ。
めたん：軸が違うと、得意なことも変わってくる？
ずんだもん：少しだけ違ってくる。脇道として触れておくのだ。
ずんだもん：拡散モデルは、最初から画面全体の構図がぼんやり決まっていく。
めたん：全体のバランスを、最初から見ながら作れるのね。
ずんだもん：そう。だから「全体構図」が要る絵に強い。
ずんだもん：自己回帰モデルは、左上のマスを置いたら、以降の全部がそれに引きずられる。
めたん：最初の一手が、ずっと効いてくるわね。
ずんだもん：でも、テキストや音声と全く同じ機械で扱える。これが大きい。
めたん：「文章を作って、絵を作って、続きの文章を作る」を、一つの機械で？
ずんだもん：それが一本のモデルでできる。
めたん：どちらが上というより、軸が違うから得手が違うのね。
ずんだもん：そう。優劣じゃなくて、得手の方向が違う。
めたん：軸の選び方は、用途で決まる感じ？
ずんだもん：お絵かき特化なら拡散、テキストと混ぜたいなら自己回帰、と棲み分けが進んでる。

# ▶ 結論

ずんだもん：ここまで来たら、最初の話に戻れるのだ。
めたん：最初は「画像生成AIは一発で当てることを諦めてる」って話だったわね。
ずんだもん：諦めたうえで、易しい一手の連鎖にバラした。
ずんだもん：これが、画像生成AIの本当の発明なのだ。
めたん：賢いから絵が描けるんじゃなくて、諦め方が賢いのね。
ずんだもん：そういうこと。一発当ては不可能でも、易しい一手なら学べる。
ずんだもん：学べるから、くり返せる。くり返せるから、絵になる。
めたん：拡散モデルと自己回帰モデルは、その「バラし方」の二つの軸なのね。
ずんだもん：そう。違う仕組みに見えるけど、骨は同じ。違うのは軸だけ。
めたん：ノイズの軸か、空間の軸か。
ずんだもん：そう。これからAIが描いた絵を見るときは、軸を思い浮かべてみるのだ。
めたん：「これは画面全体が砂嵐から立ち上がってきた絵かしら」「これはマス目を順に埋めて作った絵かしら」って。
ずんだもん：そう。中で何が起きてるか、想像が湧くようになる。
めたん：表面の魔法じゃなくて、中の骨組みが見えてくるのね。
ずんだもん：それが今日のお土産なのだ。
めたん：ありがとう、ずんだもん。
ずんだもん：またなのだ。
