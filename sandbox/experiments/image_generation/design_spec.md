# image_generation — design spec

パイプライン3段目。`design_overall.md` の質的な動きを、`script.md` の各セリフに event アンカーで時間割り付けする。座標値・フレーム数は持たない。

## 1. event 一覧と割り当て

発生順。各 event は紐付くセリフの開始フレームで発火する（フレーム計算は tsx）。画面切替 event `scene.X.in` は各画面の先頭セリフに付く。

| event | 紐付くセリフ（画面・話者・抜粋） | 発火する挙動 |
|---|---|---|
| `scene.intro.in` | 序論・めたん「絵を描いてくれるアプリ」 | 序論の画面がフェードイン |
| `intro.prompt` | 序論・ずんだもん「と打てば、宇宙服を着た猫が出てくる」 | プロンプトタグが現れ、矢印が伸び、生成画がぼやけ→くっきり結像 |
| `intro.variant` | 序論・めたん「別の猫が出てきた」 | 生成画が別バージョンの一枚へクロスフェード |
| `intro.notcollage` | 序論・ずんだもん「どこかからの検索でも」 | 写真の切り貼り小片が寄って却下で消え、虫めがねの印が却下で消える |
| `intro.newborn` | 序論・ずんだもん「新しく生まれた一枚」 | 生成画が強調され、中の猫がきょとんと一度反応 |
| `intro.title` | 序論・ずんだもん「土台の話から」 | タイトルが結像 |
| `scene.body1.in` | ボディ1・ずんだもん「絵そのものの正体」 | ボディ1の画面がフェードイン |
| `b1.pixels` | ボディ1・ずんだもん「ピクセルと呼ぶ」 | キャンバスが拡大し、色つきピクセルの格子が見える |
| `b1.random` | ボディ1・めたん「これは……砂嵐だわ」 | グリッドがでたらめな色で埋まり、砂嵐へ連続変形 |
| `b1.space` | ボディ1・ずんだもん「ぜんぶの絵が住む」 | キャンバスが縮小して空間の一点になり、絵の空間フィールドが展開 |
| `b1.thread` | ボディ1・ずんだもん「細い糸のように」 | 意味のある絵の糸が描き起こされ、糸上に絵サムネイルが点々と乗る |
| `b1.walk` | ボディ1・ずんだもん「となりへ一歩ずれる」 | 糸上の点が一歩進み、隣のサムネイルへなめらかに変わる |
| `b1.walkoff` | ボディ1・ずんだもん「意味がこわれていく」 | 点が糸を外れ、数歩で砂嵐サムネイルへ崩れる |
| `b1.oneshot` | ボディ1・ずんだもん「「猫を描け」と命じて」 | 大きなキャンバスが戻り、矢印一本でもやもや猫が出る。猫が困り顔で一度反応 |
| `b1.chain` | ボディ1・ずんだもん「画像生成AIの、背骨」 | もやもや猫が一手ずつくっきりし、糸へ向かう一手の鎖の軌跡が描かれる。猫が得意顔へ |
| `scene.body2.in` | ボディ2・ずんだもん「ディフュージョンと言う」 | ボディ2の画面がフェードイン |
| `b2.axis` | ボディ2・ずんだもん「「ノイズの濃さ」にとる」 | ノイズ濃さの軸（水平の目盛り）が描き起こされる |
| `b2.train` | ボディ2・ずんだもん「完全な砂嵐にしてしまう」 | 写真キャンバスが一段ずつ砂嵐へ連続変形し、訓練の列が左へ伸びる |
| `b2.predict` | ボディ2・ずんだもん「それを見ぬく。それだけ」 | AI装置が現れ、砂嵐入り画像からノイズを抜き出し、足したノイズ＝正解の対応を示す |
| `b2.generate` | ボディ2・ずんだもん「乗っているノイズを見ぬいて、引く」 | 軸左端の砂嵐が一手ごとに右へ進み、グリッドが砂嵐→絵へ多段変形 |
| `b2.emerge` | ボディ2・めたん「霧が晴れていくみたいに」 | 絵が浮かび、猫が結像。猫が安堵の表情で一度反応 |
| `b2.prompt` | ボディ2・ずんだもん「プロンプトを、いっしょに、AIに見せる」 | プロンプトタグが装置に差さり、各一手の軌跡が猫の点へ舵を切る |
| `b2.vary` | ボディ2・ずんだもん「出発の砂嵐が、毎回ちがう」 | 出発の砂嵐を替えると、糸上の別の猫の点に着くことを短く示す |
| `scene.body3.in` | ボディ3・ずんだもん「ChatGPT が文章を書くやり方を、絵に持ちこむ」 | ボディ3の画面がフェードイン |
| `b3.text` | ボディ3・ずんだもん「ひたすら繰り返す」 | 文章のトークン帯が、装置の一手ごとに一語ずつ右へ伸びる |
| `b3.patches` | ボディ3・ずんだもん「小さなマス目のかけらに、刻む」 | 絵キャンバスがグリッド線でかけら（パッチ）に刻まれる |
| `b3.codebook` | ボディ3・ずんだもん「「絵の単語帳」を、用意する」 | 絵の単語帳パレットが現れる |
| `b3.tokenize` | ボディ3・ずんだもん「長い列に、化けるのだ」 | 各かけらが単語帳の最近タイルへ結ばれ置換、2Dグリッドが1Dの列へほどける |
| `b3.merge` | ボディ3・ずんだもん「ひとつながりの列にする」 | プロンプトタグが列の先頭に付き、言葉トークンと絵トークンが一続きになる |
| `b3.write` | ボディ3・ずんだもん「左上の角から、順番に」 | 装置が順にトークンを当てて列を埋め、キャンバスに上から猫が現れる。半分の猫が一度反応、書き上がりで猫が満足げに反応 |
| `scene.body4.in` | ボディ4・ずんだもん「隣に並べて、見くらべる」 | ボディ4の画面がフェードイン、左右に二つの鎖が並ぶ |
| `b4.common` | ボディ4・ずんだもん「やさしい一手の鎖に、置きかえた」 | 両鎖の手前で一発の矢印が×になり、一手の鎖へ置きかわる |
| `b4.learn` | ボディ4・ずんだもん「糸の地形のほう」 | 学習データのペアが両鎖へ流れこみ、中央の糸の地形が強調される |
| `b4.diff` | ボディ4・ずんだもん「ぼんやり全体から、くっきり全体へ」 | 左の拡散は全体が一度に磨かれ、右の自己回帰は左上から一部分ずつ確定する対比再生 |
| `b4.meritD` | ボディ4・ずんだもん「こまかな質感に、強い」 | 左鎖の終点に拡散産の猫と「質感・まとまり」バッジ |
| `b4.meritA` | ボディ4・ずんだもん「こみ入った注文を、よく理解できる」 | 右鎖の終点に自己回帰産の猫と「指示の理解・会話で修正・文字」バッジ |
| `b4.weak` | ボディ4・ずんだもん「まちがいが、あとへ響く」 | 右鎖はある一手の赤が以降へ響き、左鎖は赤が次の段で直る対比 |
| `b4.merge` | ボディ4・ずんだもん「芯の二択は、変わらない」 | 左右の鎖の間にハイブリッドの帯が薄く現れ、二匹の猫が顔を見合わせて並ぶ。芯の二択軸は残る |
| `scene.outro.in` | 結論・ずんだもん「はじめの不思議に、戻る」 | 結論の画面がフェードイン |
| `outro.recap` | 結論・ずんだもん「順に、たどり直す」 | 振り返りアイコンの帯が順に灯る |
| `outro.converge` | 結論・ずんだもん「まだ誰も立っていない一点にも、立てる」 | 二つの鎖が別々の入口から糸上の猫の一点へ歩き、軌跡が収束する |
| `outro.callback` | 結論・ずんだもん「とほうもない広さから、一点を、当てていた」 | 猫の一点が光り、序論の生成画が回帰。猫が満足げに一度反応 |
| `outro.end` | 結論・めたん「まだまだ眠っているのね」 | 糸上にまだ灯らない暗い点々が無数に残り、締めの一文が結像 |

内容 event は 37、画面切替 event は 6。内容 event は全250セリフの約 15%。

## 3. オブジェクト別 挙動メモ

複数 event をまたいで動くオブジェクトの、event 間の連続的な状態変化。

- **canvas（ボディ1）。** 画面入場では序論の生成画の続きとして猫の絵。`b1.pixels` で拡大しピクセルの格子があらわになる。`b1.random` で各セルがでたらめな色へ変わり砂嵐になる。`b1.space` で縮小し、spaceField の中の一点（小サムネイル）になる。以降は spaceField 上の基準点として静止。
- **blurCat（ボディ1）。** `b1.oneshot` で大キャンバス上に、輪郭のぼやけた猫として誕生（ぼけ量・大）。`b1.chain` で、一手の鎖が進むのに合わせてぼけ量が段階的に下がり、くっきりした猫へ。表情は `b1.oneshot` で困り顔、`b1.chain` 完了で得意顔（数値 mix のクロスフェード）。
- **genCanvas2（ボディ2）。** `b2.generate` で noiseAxis 左端に砂嵐（ノイズ量 1）として誕生。一手ごとに軸を右へ移動しつつノイズ量が 1→0 へ多段で下がる。`b2.emerge` でノイズ量が小さくなり猫が結像。`b2.prompt` 以降は、ノイズ量の低下に加え、糸上の「猫の点」へ寄る方向の舵が軌跡に乗る。`b2.vary` では出発ノイズ違いの軌跡を短く重ねて見せる。
- **patchGrid → imageTokenRow（ボディ3）。** `b3.patches` で canvas3 にグリッド線が入りかけらに分かれる。`b3.codebook` 後、`b3.tokenize` で各かけらが codebook の最近タイルへ線で結ばれ、色がタイルの色に置換。続いて 2D 配置が崩れ、左上から順に 1D の長い列（imageTokenRow）へほどける。`b3.merge` で先頭に promptTag（言葉トークン）が連結。
- **imageTokenRow / canvas3 の充填（ボディ3）。** `b3.write` で nextWordEngine が列を左上ぶんから順に確定（カウント駆動 count を 0→N へ）。列の確定数に同期して、canvas3 側にも対応するかけらが上から順に現れる。半分まで埋まった時点で猫は上半身だけ＝半分の猫の反応、N 到達で猫が満足げな反応（表情 mix）。
- **diffRail / autoRail（ボディ4）。** 画面入場で左右に並ぶ。`b4.common` で各鎖手前の一発矢印が×へ、鎖本体が強調。`b4.diff` で diffRail は全体一括変形のループ的でない一回再生、autoRail は左上から確定の一回再生。`b4.weak` で autoRail の一手が赤に変わりその色が後続コマへ伝播、diffRail は赤コマが次段の全体見直しで通常色へ戻る。`b4.merge` で両鎖の間に hybridBand が薄く出る。
- **catD / catA（ボディ4）。** `b4.meritD`／`b4.meritA` で各鎖終点に誕生（最初は互いに張り合う向き）。`b4.merge` で向きが変わり、顔を見合わせて並ぶ（向き・表情の数値変化）。

## 4. 採否判断

- [ ] design_overall の各画面の「動き」がすべて event に分解されているか — 6画面すべて分解済み
- [ ] 各 event が `script.md` の実在するセリフに紐付いているか — 抜粋はいずれも実在の一意な部分文字列
- [ ] 画面切替 event（`scene.X.in`）が各画面の先頭セリフに付いているか — 6画面すべてに付与
- [ ] 内容 event の量が適切か — 37／250 ≈ 15%、目安内
- [ ] 全オブジェクトにライフサイクル契約があるか — 主要オブジェクトすべてに誕生・死・出生元・退場先
- [ ] 出生元・退場先がすべて既存オブジェクトか画面入退場か — 何もない所からの湧きなし。派生はすべて既存オブジェクトから
- [ ] 座標値・フレーム数値を書いていないか — 質的記述に留めた

## 次段への引き継ぎ

`Composition.tsx` 実装（`04_remotion.md`）に渡すもの：

- **event 一覧と割り当て** → `scriptData.ts` の `SCRIPT[]` に event を付与。各 event のセリフ開始フレームが Track のキーフレーム位置
- **ライフサイクル契約** → 各オブジェクトの opacity Track と画面可視性ゲート
- **オブジェクト別 挙動メモ** → 各オブジェクトの状態 Track の組み立て方針
