# セリフ単位スペック — AI画像生成は、なぜ文章から絵を作れるのか

`design_overall.md` の背骨（言葉の目的地へ、ノイズの雲を少しずつ畳んでいく作業台）を event に分解して `script.md` のセリフへ紐付ける。

共通装置：**プロンプト札**（入力文。画面ごとに短文へ差し替わる）、**目的地マーカー**（言葉が指す座標。ミント）、**潜在作業台**（圧縮された画像作業場。白い面）、**ノイズ雲**（グレーの粒群）、**生成画像タイル**（完成側の絵。猫・靴・背景を簡略化した低ディテールの絵）。

## 画面 1: intro

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| `scene.intro.in` | め「最近の画像生成AIって、雑に『白い猫、宇宙服、ラーメン屋』って打つだけで」 | 中央にプロンプト札、右に空の画像枠、奥に薄いノイズ雲が現れる |
| `intro.text_image.gap` | ず「文章は文字の並びで、画像は色の点の並びなのだ」 | プロンプト札と画像枠の間に断絶線が入り、文字列とピクセル格子が対比される |
| `intro.artist.cross` | ず「文章を読んでから頭の中で構図を考えて、筆で描いているわけではない」 | 小さな絵師アイコンの幻が出て、すぐ薄いバツで消える |
| `intro.destination.noise` | ず「言葉を『絵がありそうな場所』へ変換して」 | プロンプト札から目的地マーカーが立ち、ノイズ雲へ細い誘導線が伸びる |
| `intro.question` | ず「文字しか渡していないのに、AIはどうやって」 | 目的地マーカーとノイズ雲が拡大し、画面中央に問いの状態で静止する |

## 画面 2: prompt_map

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| `scene.prompt_map.in` | ず「まず、プロンプトを命令書だと思うと少し外すのだ」 | 命令書の紙が現れ、丸められて地図面へ変わる |
| `map.pairs.in` | ず「ネットには画像と説明文の組が大量にあるのだ」 | 小さな画像カードと説明文カードのペアが複数並ぶ |
| `map.align` | ず「画像と文章を同じ地図の上に置く練習をするのだ」 | ペアカードが点に縮み、同じ地図面に配置される |
| `map.near_far` | ず「対応する画像と文章は近くへ、関係ない組は遠くへ置く」 | 「黒猫」と猫画像が近づき、「消防車」が遠くへ離れる |
| `map.destination` | ず「プロンプトは、絵の描き方を一行ずつ指示する紙ではなく」 | プロンプト札が目的地マーカーへ変形し、地図上の一点を照らす |
| `map.bias` | ず「モデルが知っている『かわいい』は、世界全体の真理ではなく」 | 目的地の周囲に「学習データの傾向」の薄い偏り雲が出る |
| `map.to_workspace` | ず「次は、その地図を使って、どこで絵を作るのかを見るのだ」 | 地図面が縮み、下から潜在作業台がせり上がる |

## 画面 3: latent

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| `scene.latent.in` | め「画像って、結局はピクセルの集まりでしょ？」 | 巨大なピクセル格子が画面いっぱいに広がる |
| `latent.too_big` | ず「1024かける1024の画像なら、点が100万個以上あるのだ」 | ピクセル格子が増殖して過密になり、扱いにくさを示す |
| `latent.compress` | ず「画像を一度ぎゅっと圧縮した作業場なのだ」 | 巨大格子が折り畳まれ、中央の小さな潜在作業台へ圧縮される |
| `latent.encode` | ず「まず画像をエンコーダで潜在表現に押し込む」 | 画像タイルがエンコーダ箱を通って潜在作業台上の低密度な形になる |
| `latent.decode` | ず「最後にデコーダで、人間が見えるピクセル画像へ戻すのだ」 | 潜在作業台の形がデコーダ箱を通り、右の画像タイルへ戻る |
| `latent.tradeoff` | ず「軽くしすぎると、文字や指や細い模様が崩れやすい」 | 作業台を小さくしすぎると細部ラベルが崩れ、元の大きさへ戻される |
| `latent.no_coloring` | ず「この『作業場』があるから、画像生成はただの巨大な塗り絵ではなくなる」 | ピクセル格子が退場し、潜在作業台とノイズ雲だけが残る |

## 画面 4: denoise

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| `scene.denoise.in` | ず「今の画像生成で中心にある考え方の一つが、拡散モデルなのだ」 | 潜在作業台の上に砂嵐状態のノイズ雲が大きく立つ |
| `denoise.forward` | ず「きれいな画像に少しずつノイズを足して壊していくのだ」 | 完成画像タイルに粒が降り、段階カードとして砂嵐へ崩れる |
| `denoise.answer` | ず「自分で足したノイズなら、答えがわかるからなのだ」 | 足した粒だけがアンバーに光り、答え札として分離する |
| `denoise.step_power` | ず「学習が終わると、モデルは『少しだけノイズを減らす』ことができる」 | モデル箱がノイズ雲から一層だけ粒を取り、少し像が出る |
| `denoise.no_big_jump` | ず「一発で完成絵を決めようとすると、可能性が多すぎる」 | 大きな一発矢印がぼやけた平均画像へ伸び、すぐ消える |
| `denoise.chain` | ず「今の状態から、ほんの少しだけマシな状態へ進む」 | 小さな矢印が連なり、ノイズ雲が段々と猫の画像タイルへ近づく |
| `denoise.prompt_guidance` | ず「ただ絵らしいほうではなく、『赤い靴を履いた猫』らしいほうへ」 | プロンプト札と目的地マーカーが戻り、各小矢印の向きを少し曲げる |
| `denoise.next_problem` | ず「なぜAIはたまに『赤い立方体と青い球』を」 | 完成しかけの猫画像が退き、赤青の属性札だけが前面に残る |

## 画面 5: binding

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| `scene.binding.in` | め「『右に犬、左に猫』って頼んだのに、逆になったり」 | 左右に犬・猫の簡略シルエットが出て、位置札が一瞬入れ替わる |
| `binding.tags` | ず「『赤い』『靴』『猫』を知っていても」 | 「赤い」「靴」「猫」のタグが浮かび、猫の足元へ束ねる線が伸びる |
| `binding.cross_attention` | ず「cross-attentionという仕組みが重要になるのだ」 | 画像の場所ごとに、参照する単語タグへの細い線が引かれる |
| `binding.local_focus` | ず「耳のあたりは『cat』を強く見る」 | 耳、足元、背景の各領域が別々の単語タグを強く光らせる |
| `binding.swap_fail` | ず「三つの対象、三つの色、三つの帽子を、正しく束ねないと」 | 色札が誤った対象へ飛び、赤い球・青い立方体の失敗例に変わる |
| `binding.text_fingers` | ず「文字は『なんとなく看板っぽい』ではダメで」 | 看板文字と手のアイコンが現れ、細部が崩れる箇所だけ赤く揺れる |
| `binding.answer` | ず「かなり多くの場合は後者なのだ」 | 「知らない」側の札が薄まり、「結びつけ損ねる」側へチェックが付く |
| `binding.prompt_tip` | ず「対象ごとに属性を近くに置く、数を増やしすぎない」 | タグが対象ごとに近く整理され、線の絡まりがほどける |

## 画面 6: editing

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| `scene.editing.in` | め「昔は一回出して、違ったらプロンプトを変えて、また祈る感じだった」 | 単発生成のサイコロと画像カードが出て、外れカードが横へ流れる |
| `editing.chat` | ず「今は、画像を見せて、ここだけ直して」 | 画像カードの横に会話吹き出しが現れ、修正指示が入る |
| `editing.mask_area` | ず「前のキャラは保って、背景だけ昼にして」 | 画像カードの背景だけがマスクされ、キャラは固定ピンで留まる |
| `editing.context` | ず「前の会話、アップロード画像、修正指示をまとめて扱いやすくなる」 | 会話履歴・参照画像・修正札が一つの文脈束へまとまる |
| `editing.control` | ず「『目的に合う絵を制御して作る』へ寄っている」 | サイコロが消え、目的地マーカーと調整つまみが画像カードにつながる |
| `editing.limits` | ず「細かい制約、厳密な図面、数の一致、ブランドや人物の扱い」 | 画像カードの上に制約チェック項目が出て、いくつかが警告色になる |
| `editing.verify` | ず「正確さが必要な図、医療、法律、証拠、ニュース画像では」 | 画像カードの横に出典・確認印が付き、見た目とは別レイヤーで照合される |

## 画面 7: outro

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| `scene.outro.in` | ず「最初の違和感に戻るのだ」 | 導入のプロンプト札、目的地マーカー、ノイズ雲が戻る |
| `outro.recap_map` | ず「プロンプトは、絵の手順書ではなく、画像と言葉の地図上の目的地」 | プロンプト札が地図上の目的地マーカーへ変わる |
| `outro.recap_workspace` | ず「潜在空間という圧縮された作業場で」 | 地図の下に潜在作業台が開き、ノイズ雲がそこへ載る |
| `outro.recap_steps` | ず「ノイズを一歩ずつ絵らしいほうへ押していく」 | 小矢印の列が走り、ノイズ雲が完成画像タイルへ畳まれる |
| `outro.binding` | め「赤、靴、猫を正しく束ねられた時にうまく出るのね」 | 赤・靴・猫のタグが完成画像内の正しい部位に結ばれる |
| `outro.truth_split` | ず「見た目が自然だからといって、現実に正しいとは限らない」 | 完成画像タイルが「見た目」面と「正しさ」面の二層に分かれる |
| `outro.final` | ず「文章が指す方向へ、無数の色の可能性を少しずつ畳んでいるのだ」 | 目的地マーカー、潜在作業台、完成画像が一直線に重なり、締めの状態で静止する |

## 各オブジェクトの誕生と死

| オブジェクト | 誕生 | 死 |
|---|---|---|
| プロンプト札 | `scene.intro.in` | 画面間で差し替えながら全体を貫通。`editing.control` では調整つまみに変身し、`scene.outro.in` で復帰 |
| 目的地マーカー | `intro.destination.noise` | `map.to_workspace` で一旦退場 → `denoise.prompt_guidance` で復帰 → `outro.final` まで生存 |
| ノイズ雲 | `scene.intro.in` | `intro.question` で退場 → `scene.denoise.in` で主役として復帰 → `outro.recap_steps` で完成画像へ変身 |
| 絵師アイコンの幻 | `intro.artist.cross` | `intro.artist.cross` 内でバツと同時に死 |
| 地図面 | `scene.prompt_map.in` | `map.to_workspace` で潜在作業台へ席を譲る → `outro.recap_map` で復帰 |
| 画像・説明文ペアカード | `map.pairs.in` | `map.align` で地図上の点へ変身 |
| 偏り雲 | `map.bias` | `map.to_workspace` |
| 巨大ピクセル格子 | `scene.latent.in` | `latent.compress` で潜在作業台へ変身 |
| 潜在作業台 | `latent.compress` | 以降の主作業場として全体を貫通し、`outro.final` まで生存 |
| エンコーダ箱・デコーダ箱 | `latent.encode` / `latent.decode` | `latent.no_coloring` |
| モデル箱 | `denoise.step_power` | `denoise.chain` 後に退場 |
| アンバーの答え粒 | `denoise.answer` | `denoise.step_power` |
| 大きな一発矢印＋平均画像 | `denoise.no_big_jump` | `denoise.chain` |
| 小矢印の列 | `denoise.chain` | `denoise.next_problem` → `outro.recap_steps` で復帰 |
| 犬・猫・図形シルエット | `scene.binding.in` | `binding.prompt_tip` |
| 単語タグと結び線 | `binding.tags` | `binding.prompt_tip` で整理後、`scene.editing.in` で退場 → `outro.binding` で復帰 |
| 看板文字・手の細部アイコン | `binding.text_fingers` | `binding.answer` |
| 単発生成サイコロ | `scene.editing.in` | `editing.control` |
| 編集画像カード | `scene.editing.in` | `outro.truth_split` まで形を変えながら生存 |
| 会話吹き出し・文脈束 | `editing.chat` / `editing.context` | `editing.control` で調整つまみへ畳む |
| 制約チェック項目・確認印 | `editing.limits` / `editing.verify` | `scene.outro.in` で退場。確認印は `outro.truth_split` で再登場 |
| 見た目面・正しさ面 | `outro.truth_split` | `outro.final` |

同時表示の最大は、画面4の「潜在作業台＋ノイズ雲＋目的地マーカー＋小矢印」、画面5の「対象シルエット＋単語タグ＋結び線＋失敗例」、画面6の「画像カード＋会話吹き出し＋文脈束＋確認印」で、いずれも 4 個以内に収める。役目を終えた装置は次の装置が立つ前に短く退場させる。
