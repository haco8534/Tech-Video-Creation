# image_generation — design spec

`design_overall.md` の質的な動きを event アンカーに分解し、`script.md` の各セリフに割り当てる。座標値・フレーム数は持たない。

対話字幕は event ではない——全セリフが自動で画面下部の字幕になる（tsx の仕事）。ここで定義する event は、絵の装置の挙動だけを指す。

## 1. event 一覧と割り当て

発生順。画面切替 event（`scene.*.in`）は各画面の先頭セリフに必須。

| `intro.q` | 序論・めたん「あれって、AIが頭の中で完成図を思い浮かべて、ぱっと描いてる感じかしら？」 | 盤面の右上空に巨大な「？」（VOID 色）が現れる |
| `intro.giveup` | 序論・ずんだもん「画像生成AIはね、一発で絵を当てることを、最初から諦めてるのだ。」 | 盤面の上に大きな「✗」（VOID 色）が一拍かぶさり、すぐ消える。猫の線画も薄れて盤面が空白に戻る |

| `scene.body1.in` | ボディ1・ずんだもん「画像って、コンピューターから見ると、ただの数字のマス目なのだ。」 | 画面切替。序論の盤面が中央に残り、罫線が一段濃くなり、マス目構造が強調される（連続変形） |

| `b1.sea` | ボディ1・ずんだもん「その膨大な可能性のほとんどは…ザーザーの砂嵐に見える絵。」 | 拡大マスと色棒が畳まれる。盤面の周囲に縮小盤面（砂嵐パッチ）が画面いっぱいに散らばる（可能性の海の登場）。中央の盤面も砂嵐になる |
| `b1.point` | ボディ1・ずんだもん「人間が「猫」と認めるパターンは、その大海の中の、ごく小さな点。」 | 中央の盤面が砂嵐から猫に変わり、輪郭が SUBJECT 色で強調される。周囲の縮小盤面はすべて砂嵐のまま |

| `b3.more` | ボディ3・ずんだもん「また次のステップで、もう少し引く。」 | 3個目（0.66）と4個目（0.5）の小盤面が左から右へ順に現れる |
| `b3.count` | ボディ3・ずんだもん「それを20回とか50回くり返すと、絵が完成する。」 | 5個目（0.33）と6個目（猫が立つ、0）が現れる。下端に時間軸線（DIFFUSION 太線）が描き起こされ、矢印の先に「20〜50ステップ」の数値帯が灯る |
| `b3.howq` | ボディ3・めたん「待って。各ステップで「ノイズを少し引く」って、AIはどうやって判断してるの？」 | 時間軸列の中央の1ステップだけが拡大ハイライトされる（DIFFUSION 色の枠が一拍太くなる） |
| `b3.learn` | ボディ3・ずんだもん「学習のときは、逆をやるのだ。完成済みの絵に、ノイズを少しずつ足していく。」 | 時間軸列の右端（猫）から上に学習ペアが立ち上がる。上段＝完成猫、下段＝少し汚した盤面。間に HOPE 色の双方向矢印 |

| `scene.body4.in` | ボディ4・ずんだもん「次は、自己回帰モデル。」 | 画面切替。方式名カード「自己回帰モデル」が中央に大きく現れる（副題：「空間軸 — マスを左上から埋める」）。すぐ盤面の下の小帯に縮む |
| `b4.empty` | ボディ4・ずんだもん「絵を作るときは、何も描かれていない、空っぽのキャンバスから始まる。」 | 中央に空っぽの大盤面（16×16 パッチに区切られている）が現れる |

| `b4.unroll` | ボディ4・ずんだもん「行ごとに左から右、それを上から下へつなげれば、ただの「パッチの並び」になる。」 | 展開図の下に「= 文章の列」の比喩矢印が灯り、列の先頭にカーソル（AUTOREG 色）が点く |
| `b4.fromAll` | ボディ4・ずんだもん「これまで埋めたパッチを全部見て、次の1パッチの中身を当てる。」 | 展開図の途中まで（10パッチほど）が一段濃く、次の1パッチに吸い込み矢が向く。盤面側でも対応する位置がハイライト |
| `b4.learn` | ボディ4・ずんだもん「完成された画像を持ってきて、パッチの列に並べる。」 | 盤面の上に学習ペア（上段＝完成猫、下段＝同じものをパッチ列にほどいた姿）が現れる。間に HOPE 色の矢印 |
| `b4.fullCat` | ボディ4・ずんだもん「それを最後まで繰り返せば、絵が完成する。」 | 盤面のパッチが残り全部、左上から右下へ一気に塗り埋められる。盤面に猫が立つ |
| `b4.products` | ボディ4・めたん「あら、ChatGPTのあれも？」 | 盤面の下に「GPT-4o / Gemini 画像生成」の固有名詞が一行で灯る |
| `scene.body5.in` | ボディ5・ずんだもん「ここまでで、二つの仕組みを見たのだ。」 | 画面切替。中央に主盤面（猫）が現れる |

| `b5.horz` | ボディ5・ずんだもん「自己回帰モデルは「空間の位置」を軸にバラした。マスを順番に埋める。」 | 横軸が原点から左右へ立ち上がる（AUTOREG エメラルド）。左端に空白パッチの小盤面、右端にパッチ全埋の小盤面が灯る |

| `outro.end` | 結論・ずんだもん「それが今日のお土産なのだ。」 | 盤面の下に締めの一文「諦め方が、賢い」と副題が灯る。動画が静かに終わる |

## 2. ライフサイクル契約

「画面入場／画面退場」はクロスフェードと同時に opacity 0→1／1→0。盤面（主役）は全画面で背骨だが、画面リセットを挟むため各画面で別個体として描き、視聴者には「同じ盤面」と感じさせる。

## 3. オブジェクト別 挙動メモ

**盤面（背骨）** — 全画面に登場。持つ数値は「砂嵐度」（0＝整った絵／1＝完全砂嵐）、「猫描き率」（0＝空白／1＝猫の線画が立つ）、「パッチ埋め率」（0＝空白／1＝全パッチ埋）、「サイズ」「位置」「opacity」。各画面で別個体だが視覚的には同じ盤面と感じさせる。
- 序論：空白 → `intro.prompt` で猫描き率0→1 一拍 → `intro.giveup` で猫描き率1→0
- ボディ1：序論からモーフで継続（マス目強調）→ `b1.sea` で砂嵐度0→1 → `b1.point` で猫描き率0→1（砂嵐度を1のまま視覚的に猫が浮く）→ `b1.allatonce` で大矢印に攻撃される → `b1.abandon` で✗
- ボディ2：左に縮んで残る
- ボディ3：時間軸列の右端の盤面が完成像。左端の盤面と段階的な中間盤面を順に生成
- ボディ4：パッチ列の主盤面が「パッチ埋め率」を 0→1 に上げる
- ボディ5：原点に主盤面（猫）。サイズは中央付近に占有
- 結論：ボディ5から持ち越し

**時間軸列（ボディ3）** — `b3.start` で1個目（砂嵐）→ `b3.subtract` `b3.more` `b3.count` で順次2〜6個目を追加（各 0.83、0.66、0.5、0.33、0 の砂嵐度）。`b3.howq` で中央1ステップに DIFFUSION 色の枠が一拍太くなる。`b3.identity` で列ぜんたいが一拍光る（schedule: 直前の学習ペアが上に畳まれた後）。

**パッチ列の主盤面（ボディ4）** — `b4.empty` で全パッチ空白の状態で誕生。`b4.fillStart` で「パッチ埋め率」0→0.05（左上3パッチ程度）。`b4.predict` で吸い込み矢が次のパッチへ。`b4.gpt` で右側に展開図がほどけて並ぶ（主盤面はそのまま）。`b4.fromAll` で「埋め率」0.05→0.4 程度（10パッチほど）。`b4.fullCat` で「埋め率」0.4→1。猫が立つ。

**二軸の十字（ボディ5・結論）** — `b5.vert` で縦軸（DIFFUSION）が原点から上下へ立ち上がる（線の長さが 0→full）。`b5.horz` で横軸（AUTOREG）が左右へ立ち上がる。`b5.midpoints` で各軸の中点に小盤面が灯る。`b5.diffStrength` で縦軸全体が一拍光る。結論では `outro.axes` で両軸が一拍光る、`outro.q` で「？」が背景に置き換わる動き。

**数珠（ボディ2）** — `b2.split` で大矢印の右から伸びる（左から右へ順に小矢印が描き起こされる）。`b2.cat` で右端に猫の小盤面が灯る。`b2.learnable` で各小矢印に✓。`b2.fork` で中央から二分岐（上下に分かれる）。`b2.diffSide`/`b2.autoSide` で各分岐の下に小盤面列が並ぶ。

**結論の数珠のこだま** — `outro.chain` で原点の猫盤面の周りに一拍だけ薄く現れて消える（半透明・短時間）。
