# image_generation — design overall

`script.md` を入力に、動画全体を貫通する世界観と、7画面それぞれのデザインを言語で決め切る。座標値・フレーム数は持たない（design_spec と Composition.tsx の仕事）。例外は「文字を内包する箱」のサイズだけ、文字基準で書く。

図解は概念にラベルを貼らない。仕組みが起きるさまを、画面の上で実際に起こす。「ノイズ」と書いた箱は出さない。1024×1024のマス目が画面いっぱいに広がり、その一つひとつのマスが砂粒のように粗れたり、整ったりする——その変化そのものを見せる。

## 背骨

貫通する背骨は **1024×1024 のマス目**。100万個のマスが画面いっぱいに敷き詰められている、その一枚の盤面が動画ぜんたいの主役。動画は、この盤面が「ザーザーの砂嵐」と「猫の絵」のあいだを、二つの異なる軸でどう行き来するかの一本の物語になる。

- 序論：盤面に「猫」が魔法のように立つ瞬間と、その実は不思議という問い
- ボディ1：盤面を全マスいっぺんに当てる困難（可能性の海・ほぼ全部が砂嵐）
- ボディ2：「一発当て」を「易しい一手の連鎖」にバラす発想転換
- ボディ3：拡散モデル — 盤面ぜんたいのザラつきを**時間方向**に段階的に薄める
- ボディ4：自己回帰モデル — 盤面の**空間方向**に左上から1パッチずつ埋める
- ボディ5：二つを並べる。十字の二軸 — 縦＝ノイズ／横＝空間
- 結論：二軸を重ねて締める

中心装置：**盤面**（マス目）／**砂嵐の粒**（ザラつき）／**猫の線画**（完成）／**可能性の海**（縮小盤面群）／**時間軸の列**（盤面が並ぶ）／**パッチ列**（盤面が一列にほどける）／**学習ペア**（汚す前 / 汚した後）／**二軸の十字**。

## 1. 世界観

### 固定ベース

- 背景 `#f5f7fa` ／ パネル面 `#ffffff` ／ 主文字 `#243044`
- 読ませる文字は 42px 以上

### 題材ごとの3軸

| 軸 | 設定 |
|---|---|
| 形の語彙 | 専用の図解装置の集まり（一覧は §3 図カタログ）。背骨は **盤面**（マス目グリッド・薄い罫線つき）と **マスの粒**（個々のセル）。装置：砂嵐パッチ／猫の線画／可能性の海（縮小盤面の散布）／時間軸の盤面列／パッチ列／吸い込みの矢印／学習ペア／二軸の十字。汎用の「ラベル付き箱」「テキストカード」は使わない |
| 色の語彙 | 白テーマで固定（背景 `#f5f7fa`／面 `#ffffff`／枠 `#c4cedd`／主文字 `#243044`／補助文字 `#5d6b82`）。アクセント：**SUBJECT（盤面・絵そのもの・「答え」）＝** ヴァイオレット `#6a4fb6`（AI絵の象徴的な色）。**DIFFUSION（拡散モデルの軸・ノイズ方向）＝** ローズ `#c8458b`（時間で減るもの）。**AUTOREG（自己回帰モデルの軸・空間方向）＝** エメラルド `#1e9c7f`（順に埋まるもの）。**VOID（砂嵐・無謀・困難）＝** ダークグレー `#4a5366`（光を吸う色）。**HOPE（易しい・学べる・OK）＝** アンバー `#d99a2b` |
| 動きの語彙 | 使う＝フェード、軸に沿った直線移動、マス目の色が段階で変わる、砂嵐の濃度が薄くなる、パッチが順に埋まる、矢印が一息で通る／壁の前で詰まる、学習ペアが生まれる、軸が立ち上がる、共通点が順に点灯。封印＝3D回転、カメラのズーム、バウンド、急な明滅、**ループし続ける動き**（浮遊・振動・明滅・脈動・常駐背景モーション）。装置はイベントごとに一度きり動き、終わったら静止 |

レイアウトの基準：viewBox 1920×1080 の中心原点。**画面下部 y ≈ 264〜520 は対話字幕の帯**として確保し、主装置は y < 240 の上半分〜中段に置く。盤面（主役）は中央。時間軸の列は左から右、パッチ列は左上から右下、二軸の十字は中央で交差。

## 2. 文字の節制

絵で意味が立つなら文字は足さない。

### 削るもの

- **セクションタイトル（画面上部の大見出し）は原則なし。** 画面の主題は装置と動きが語る。例外は序論末の動画タイトル「画像生成AI」だけ——題材を最初に同定する必要があるため
- **「拡散モデル」「自己回帰モデル」のラベルは、色で識別できるなら省く。** 一度どこかで「ローズ＝拡散」「エメラルド＝自己回帰」を視聴者に対応づけたら、以降は色だけで通す。各方式の名前は **拡散モデル本編の冒頭** と **自己回帰モデル本編の冒頭** で1度ずつ大きく出して、それ以降の画面（軸の比較・結論）では文字を出さず色帯だけで識別させる
- **箇条書き（「① 〜 ② 〜」）は禁止。** 共通点の3つ・工程の列は対話字幕（声）が運ぶ

### 出してよいもの

- **数式・数値・添字**：`1024×1024`、`≈ 100万マス`、`≈ 1600万色`、`16×16 px`、`4096 パッチ`、`20回 / 50回`。読ませる文字ではなく式の構成要素
- **対話字幕**（画面下部・全画面共通）
- **方式名**（「拡散モデル」「自己回帰モデル」）は各本編冒頭で1度ずつだけ
- **動画タイトル**（序論末「画像生成AI」と副題）

### 量の上限

1画面に出す独立テキスト ≤ 3（数式構成要素・対話字幕は除く）。

## 3. 図カタログ — 各図の静的仕様

各図の「見た目」はここで完全に決め切る。各画面の記述で見た目を書き直さない。

各図は **質感・内部構造・スケール階層** のうち最低2つを持つ。

### 文字を内包する箱のサイズ

ラベル付きカード／補助の説明帯／工程アイコン名／方式名カードなど、内部にテキストを持つ箱は **テキスト基準でサイズを書く**：

> 「フォント基準サイズ N px、最大文字数 M 字、左右余白 padX。箱の最小幅 = M × N × 1.05 + 2·padX。最小高 = N × 1.6」

本作で使う基準：

- **方式名カード**（「拡散モデル」「自己回帰モデル」など、最大7字）：N = 56、padX = 32。最小幅 = 7 × 56 × 1.05 + 64 = 476 px
- **補足ラベル帯**（数値とその意味、最大14字）：N = 38、padX = 24。最小幅 = 14 × 38 × 1.05 + 48 = 606 px
- **工程アイコン名**（共通点・軸名、最大10字）：N = 36、padX = 20。最小幅 = 10 × 36 × 1.05 + 40 = 418 px

## 4. 図の設計の原則

- **背骨は1本**：盤面（マス目）。各画面は盤面が別の状態を取る／別の見立てになる工程として連なる
- **概念1つに図1つ**：「この図は何を見せるか」が一言で言えなければ削る

## 5. セクション別デザイン

### 序論（画面1）：マス目に魔法のように猫が立つ

#### 眼目

「プロンプト → 絵」のあいだに何が起きているかは見ていない。盤面に猫が立つ瞬間が魔法に見える、その不思議。

#### 舞台

- **主役**：中央に盤面（16×16、画面高の 0.55）。最初は空白、最後は猫が立っている
- **補助**：盤面の左にプロンプト枠（小さなふきだし「猫の写真」）／盤面の右上空に巨大な「？」（VOID 色）／盤面の下に動画タイトル「画像生成AI」＋副題「実は、一発で当てるのは諦めている」
- **下端**：字幕帯のぶんの余白

#### 時間順の状態遷移（テンポ目標 21セリフ / 6ステップ）

```
(a) めたん「ねえずんだもん。最近よく見る画像生成AI」近辺：背景の薄い格子が現れる。中央に盤面（空白）がフェードイン。左にプロンプト枠が現れる
(b) めたん「プロンプトに「猫の写真」って打つと、本当に猫が出てくる」近辺：プロンプト枠に「猫の写真」の文字が打ち込まれ、枠から盤面へ細い矢印が伸び、盤面に猫の線画が一拍で立つ
(c) めたん「あれって、AIが頭の中で完成図を思い浮かべて、ぱっと描いてる感じかしら」近辺：盤面の右上空に巨大な「？」が現れる
(d) ずんだもん「画像生成AIはね、一発で絵を当てることを、最初から諦めてるのだ」近辺：盤面の上に大きな「✗」が一度かぶさり、すぐ消える。猫の線画も薄れて盤面が空白に戻る
(e) ずんだもん「主役は二つ。拡散モデルと、自己回帰モデル」近辺：盤面の下に二つの小さな色帯が現れる。左帯＝ローズ（DIFFUSION）、右帯＝エメラルド（AUTOREG）。色だけで何も書かない
(f) ずんだもん「まずは「一発で当てるのが、なぜ無謀なのか」から」近辺：動画タイトル「画像生成AI」と副題「実は、一発で当てるのは諦めている」が盤面の下に灯る。色帯はそのまま小さく残る
```

#### 演出ノート

- (b) の猫が立つ瞬間は **一拍で出す**（魔法らしく）。フェードではなく一瞬で
- (c) の「？」は VOID 色で大きく。視聴者の疑問を画面に外在化する
- (d) の「✗」は赤系（DANGER）ではなく VOID 色で。否定ではなく「魔法じゃない」を示す
- 色帯（ローズ／エメラルド）は (e) で初登場。以降の画面で各方式の識別に使う
- 絶対やらない：盤面が浮遊する／背景の格子が揺れる

### ボディ1（画面2）：一発で当てるという無謀

#### 眼目

100万マスの色を全部いっぺんに当てる困難。可能性の海の中で、猫はごく一点。

#### 舞台

- **主役**：中央に盤面（16×16、画面高の 0.42）。画面の中盤に
- **補助**：盤面の上に数値帯（「1024 × 1024 ≈ 100万 マス」）／盤面の右に1マス拡大＋RGB の三色棒（「256³ ≈ 1600万 色」）／その後、盤面の周囲に **可能性の海**（縮小盤面 24個）が散らばり、中央1点だけが猫
- **下端**：字幕帯のぶんの余白

#### 時間順の状態遷移（テンポ目標 26セリフ / 7ステップ）

```
(a) ずんだもん「画像って、コンピューターから見ると、ただの数字のマス目なのだ」近辺：序論の盤面が中央に残る（モーフで継続）。罫線が一段濃くなり、マス目構造が強調される
(b) ずんだもん「マスの数は、約100万個」近辺：盤面の上に数値帯「1024 × 1024 ≈ 100万 マス」がフェードイン
(c) ずんだもん「それぞれ256段階あって、1マスで約1600万通りの色が選べる」近辺：盤面の右側に1マスが拡大表示で抜き出され、その横に R/G/B の3本の色棒が伸び、下に「256³ ≈ 1600万 色」が灯る
(d) ずんだもん「その膨大な可能性のほとんどは、人間が見ても、ただのザーザーの砂嵐に見える絵」近辺：拡大マスと色棒が畳まれる。盤面の周囲に縮小盤面（砂嵐パッチ）が画面いっぱいに散らばる（可能性の海の登場）。中央の盤面も砂嵐になる
(e) ずんだもん「人間が「猫」と認めるパターンは、その大海の中の、ごく小さな点」近辺：中央の盤面が砂嵐から猫に変わり、輪郭が SUBJECT 色で強調される。周囲の縮小盤面はすべて砂嵐のまま
(f) ずんだもん「100万マスの色を、全部いっぺんに当てる問題」近辺：画面の外（左上）から大きな矢印が中央の猫の盤面めがけて飛び、ヒットする手前で **VOID の壁** にぶつかって停まる
```

## 画面の整合性 — 4つの制約

1. **同時に居る、意味的に独立した補助は最大3つ** — 各画面の補助の数は、序論3／ボディ1 3／ボディ2 3／ボディ3 3（時間軸列・学習ペア・方式名カード）／ボディ4 3（パッチ列盤面・展開図・学習ペア）／ボディ5 3（二軸・共通点ドット・補助ラベル）／結論 2（数珠・締めの一文）
2. **同時に動くのは1つ** — 各ステップは1つの装置の1つの動きに絞る
3. **1画面で扱う対比軸は1つ** — ボディ5だけが「拡散 vs 自己回帰」を扱う。他画面は単一方向の物語
4. **「最も混む瞬間」で全図が重ならない** — 各画面の主役（中央）・補助（上部または右側）・装置の追加（必要なら左側）に占有ゾーンを割り当て

## 密度 — 余白を残しすぎない

中央エリア（字幕帯・上余白を除いた高さ約 700px）の **60% 以上**を主役＋補助が占めるよう、各画面で確認：

- 序論：盤面（中央約 600×600）＋プロンプト枠＋タイトル → 中央エリアの約 65%
- ボディ1：盤面＋数値帯＋可能性の海 → 80%（可能性の海が画面いっぱい）
- ボディ2：数珠（横長）＋二分岐 → 70%
- ボディ3：時間軸列（横長6個）＋学習ペア → 75%
- ボディ4：主盤面＋展開図（横並び）＋学習ペア → 75%
- ボディ5：二軸の十字＋共通点ドット＋ラベル → 70%
- 結論：二軸＋盤面＋締めの一文 → 65%

密度を上げる先は装飾ではなく **構造**——図カタログのディテール3要素（質感・内部構造・スケール階層）で持ち上げる。

## レイアウトの約束

- 画面下部 y ≈ 264〜520 は対話字幕の帯として常駐
- 上余白も控えめ（セクションタイトル原則なし）
- 背景に薄い格子（12px 間隔、`#eaeef4` の 40%）を全画面常駐——マス目世界の通底質感
- 画面切替は標準30fクロスフェード
- スマホ視聴前提。要素数は絞り、各要素はディテールを持つ
