# 02_画像生成AI — design overall

## 1. 世界観

### 固定ベース（全題材共通）

- 背景 `#f5f7fa` ／ パネル面 `#ffffff` ／ 主文字 `#243044`
- 読ませる文字は 42px 以上

### 形の語彙

| 形 | 意味（動画を通して固定） |
|---|---|
| 角丸正方形フレーム（白塗り、細い枠線） | 「絵が出来上がる場所」＝キャンバス。背骨。動画を通して同じフレームが姿を変える |
| 細線で描かれた猫（コーラル色のリネアート） | 「完成絵」の象徴。各画面の終局でキャンバスの中に現れる |
| 砂粒（極小の点群、明度・彩度バラバラ） | 「ノイズ／砂嵐」。拡散モデル軸の語彙 |
| マス目（薄いグレーの細罫） | 「空間の位置」。自己回帰モデル軸の語彙。キャンバス上に重ねる／剥がす |
| 矢印（細く、控えめ） | 「ステップが進む方向」「時間の進行」。装飾には使わない。意味のある時にだけ |
| ステップマーカー（小さな塗りつぶし円、横一列） | 「連鎖の一手一手」。連鎖の概念を可視化するときだけ |

### 色の語彙

| 役 | 色 | 意味（最後まで流用しない） |
|---|---|---|
| SUBJECT | `#d9543c`（暖かいコーラル） | 完成絵そのもの／背骨。猫のリネアート、画像が生まれる場所のアクセント |
| DIFFUSION | `#2f8fb3`（寒色のティール） | 拡散モデル軸の専用色。ノイズ、拡散側のステップ、拡散モデル側の図のラベル |
| AUTOREG | `#d99a2b`（暖かい黄土） | 自己回帰モデル軸の専用色。パッチの埋め、自己回帰側のステップ、自己回帰モデル側の図のラベル |
| VOID | `#7c8aa0`（クールグレー） | 「無謀」「ありえない可能性の海」「諦めた」を示す抑制色。背景的に使う |

### 動きの語彙

- **使う動き** — 描き起こし、ふわっとフェード、横スライド、モーフ（同じ位置で姿が連続変形）、ステップ刻みの順次出現
- **封印する動き** — ループ全般。浮遊、振動、明滅、脈動、常駐背景モーション、回転アニメ、パルス、シマー
- **緩急** — 重要な変化はゆっくり、つなぎは速く。クライマックスでひと呼吸止める

## 2. 図カタログ — 各図の静的仕様

各図の「見た目」はここで完全に決め切る。各画面の記述で見た目を書き直さない。

### Canvas（背骨）

- **役** — 「絵が生まれる場所」。動画を通して姿を変える背骨の図
- **形** — 角丸正方形のフレーム。白塗り、INK 色の細い枠線（2px 相当）、四隅は半径 24px の角丸
- **サイズ感** — 画面高さの 50% 程度（中央 1 枚なら）。並列するときは 35% 程度に縮める
- **色** — 内側は SURFACE、枠線は INK、各画面で内側に入る要素のアクセント色は別途
- **構成要素** — 外フレーム ＋ 内側コンテンツ領域。コンテンツ領域はフレームの内側にだけ収まり、枠を越えない
- **登場画面** — 序論／ボディ1〜5／結論。常に在る

### Cat（完成絵の象徴）

- **役** — 「完成絵」の象徴。各画面の終局で Canvas の中にひとつ立つ
- **形** — コーラル色 1 色の細線リネアート（線幅 4px 相当、塗りなし）。座った猫の輪郭、耳・しっぽ・目だけの最小描写。装飾は無し
- **サイズ感** — Canvas 内側の縦 70% 程度
- **色** — 線は SUBJECT、塗りは無し
- **構成要素** — 輪郭線一筆 ＋ 目（点）2 つ
- **登場画面** — 序論（小さく予告）／ボディ2 終局（拡散側・自己回帰側の各キャンバスに小さく）／ボディ3 終局（拡散モデルの結果として）／ボディ4 終局（自己回帰モデルの結果として）／ボディ5（両並列キャンバスの内側）／結論

### NoiseField（砂嵐パッチ）

- **役** — 「ザーザーの砂嵐」「完全ノイズ」「ノイズの量」を見せる。拡散モデル軸の語彙
- **形** — Canvas 内側に敷き詰めた極小の正方形点（一辺 6〜10px 相当）。座標・色・明度はランダム
- **サイズ感** — Canvas 内側いっぱい
- **色** — VOID を基調にしつつ DIFFUSION のうっすらした寒色も少量混ぜる（純グレーすぎると死んだ画面になる）
- **構成要素** — 数百個の小正方形点の集合。点の密度・色のばらつきを「ノイズの強さ」で制御
- **登場画面** — ボディ1（無謀の海の象徴として）／ボディ2（拡散モデル軸の予告）／ボディ3（出発点から終局まで段階的に薄める）／ボディ5（拡散側キャンバスに薄く）

### PatchGrid（マス目）

- **役** — 「空間の位置」「次のマスを当てる」を見せる。自己回帰モデル軸の語彙
- **形** — 等間隔の薄い罫線で Canvas 内側を縦横に区切る格子。1 マスは Canvas 内側の 1/8 相当の大きさ
- **サイズ感** — Canvas 内側いっぱい
- **色** — 罫線は INK の薄めグレー、埋まったマスの中身は AUTOREG（暖黄）
- **構成要素** — 8×8 の罫線格子 ＋ 各マスの「埋まり状態」（埋まったマスは AUTOREG 色の塗り）
- **登場画面** — ボディ2（自己回帰モデル軸の予告）／ボディ4（出発点から終局まで段階的にマスを埋める）／ボディ5（自己回帰側キャンバスに薄く）

### StepBeads（ステップマーカー）

- **役** — 「易しい一手」「連鎖」「ステップが進む」を可視化
- **形** — 横一列に並んだ塗りつぶし円（直径 18px 相当、間隔 36px 相当）。順に色が点くと「ステップが進んだ」感じになる
- **サイズ感** — Canvas の下、横幅は Canvas の幅程度
- **色** — 未到達はうっすら INK のグレー、到達済みはそのシーンの軸色（拡散側なら DIFFUSION、自己回帰側なら AUTOREG）
- **構成要素** — 8〜10 個の円が一列に並ぶ
- **登場画面** — ボディ2（連鎖の予告で 4 個程度）／ボディ3（拡散モデルの 8 ステップ）／ボディ4（自己回帰モデルの 8 ステップ）

### CombinationsHaze（可能性の海）

- **役** — 「組み合わせ爆発」「ほとんど砂嵐」「正解は薄い点」を抽象的に見せる
- **形** — Canvas の周囲に広く広がる薄い砂粒の雲。中央に小さな SUBJECT 色の点が一つだけ光る
- **サイズ感** — Canvas を超えて画面の左右ぎりぎりまで広がる（背景的に存在）
- **色** — 砂粒は VOID、中央の点だけ SUBJECT
- **構成要素** — 砂粒の雲 ＋ 中央の SUBJECT 点（「人が絵と認めるパターン」）
- **登場画面** — ボディ1 終局でのみ

### AxisLabel（軸ラベル）

- **役** — 各モデルの「軸」を一言で名指す
- **形** — Canvas の下に置く小さなラベル帯（横長の角丸長方形、塗り、白文字）
- **サイズ感** — Canvas の幅の 60% 程度、高さは 56px 相当
- **色** — 拡散モデル側は DIFFUSION 塗り、自己回帰モデル側は AUTOREG 塗り、文字は SURFACE
- **構成要素** — ラベル帯 ＋ 「ノイズの量」または「空間の位置」の白文字
- **登場画面** — ボディ3 終盤／ボディ4 終盤／ボディ5（両キャンバスの下）

### SceneTitle（セクションタイトル）

- **役** — 画面の見出し
- **形** — 画面上部中央の大きな文字。下に短い水平線（INK の細罫）
- **サイズ感** — 文字 76〜92px、水平線は文字幅の 60% 程度
- **色** — 文字 INK、水平線 INK
- **構成要素** — タイトル文字 ＋ 下線
- **登場画面** — ボディ1〜5。序論と結論はタイトル無しで本文の図だけ

### Subtitle（対話字幕）

- **役** — 全セリフを画面下部に表示する話者名つき字幕（必須）
- **形** — 画面下部の帯（高さ 200px 相当）。話者名は左寄せ、本文はその右
- **サイズ感** — 本文 54px、話者名 42px
- **色** — めたん名は `#d6336c`、ずんだもん名は `#2f9e44`、本文は INK
- **構成要素** — 話者名 ＋ 本文（最大 2 行）
- **登場画面** — 全画面

## 3. 図の設計の原則

- **背骨は Canvas。** 動画全体で Canvas が姿を変えていく一本の物語にする。NoiseField／PatchGrid／Cat は Canvas の中に住む。Canvas 外で起きる出来事は CombinationsHaze 以外には無い
- **概念ひとつに図ひとつ。** 「ノイズ＝NoiseField」「空間＝PatchGrid」「完成＝Cat」「一手＝StepBeads」。混ぜない
- **拡散モデル側と自己回帰モデル側は、色と図形で完全に区別する。** 視聴者が「今どっちの話か」を一瞬で見分けられる。拡散モデルは寒色 + NoiseField、自己回帰モデルは暖黄 + PatchGrid。これを徹底する

## 4. セクション別デザイン

### 序論

#### 眼目

「画像生成AIは一発で当てることを諦めている」。Canvas の中で何かが起きる前の、問いの提示。

#### 舞台 — 最も混む瞬間の止め絵

画面中央に Canvas が 1 枚、画面高の 50% で立っている。Canvas の内側は空っぽの SURFACE。Canvas の上に大見出し「画像生成AI」（92px）。Canvas の下、左右に 2 枚の小さなサブカード（それぞれ Canvas の 1/3 サイズ）が横並び。左サブカードに「拡散モデル」のラベルが DIFFUSION 色で、右サブカードに「自己回帰モデル」のラベルが AUTOREG 色で書かれている。画面下端は字幕帯のぶん空けてある。

#### 時間順の状態遷移

```
(a) 「ねえずんだもん」のセリフ近辺：Canvas が中央に、上から描き起こされる。フレームの線が時計回りに引かれる
(b) 「猫の写真」のセリフ近辺：Canvas の内側に Cat が一筆ずつ描き起こされる。線がスーッと現れる
(c) 「画像生成AIはね」のセリフ近辺：大見出し「画像生成AI」が Canvas の上にフェードイン
(d) 「主役は二つ」のセリフ近辺：Canvas の下左右に、空のサブカードが 2 枚、対称に滑り出てくる
(e) 「拡散モデル」のセリフ近辺：左サブカードに DIFFUSION 色のラベル「拡散モデル」がフェードイン
(f) 「自己回帰モデル」のセリフ近辺：右サブカードに AUTOREG 色のラベル「自己回帰モデル」がフェードイン
(g) 「軸違いなのだ」のセリフ近辺：2 枚のサブカードを結ぶ薄い水平の補助線が、INK 色でうっすら引かれる（「同じ作戦」の予告）
```

#### 演出ノート

- **テンポと強弱** — Cat の描き起こしはゆっくり、ためらいの一拍を入れて落ち着いた印象に
- **カメラワーク** — なし。Canvas は中央固定
- **象徴的なディテール** — Cat が完成した瞬間、Canvas の枠線が一瞬だけ太くなって戻る（「絵が成立した」合図）。一度きり、繰り返さない
- **絶対やらないこと** — Cat を点滅させない。サブカードを揺らさない。背景にぼんやりした円や粒を散らさない

### ボディ1 — 一発で当てるという無謀

#### 眼目

「可能な絵の組合せは無限。正解はその中の点。だから一発当ては無謀」を絵だけで腑に落とす。

#### 舞台 — 最も混む瞬間の止め絵

画面中央に Canvas（画面高の 45%）。Canvas の中身は NoiseField で完全に埋まっている（純粋な砂嵐）。Canvas を取り囲むように CombinationsHaze が画面左右ぎりぎりまで広がり、中央に SUBJECT 色の小さな点が一つだけ光る。Canvas の上にセクションタイトル「一発で当てるという無謀」。下端は字幕帯のぶん空け。

#### 時間順の状態遷移

```
(a) 「画像って、コンピューターから見ると」のセリフ近辺：序論の Cat と見出しがフェードアウトし、空の Canvas だけが残る。セクションタイトル「一発で当てるという無謀」が画面上部にフェードイン
(b) 「マスの数は、約100万個」のセリフ近辺：Canvas の内側に PatchGrid 様の薄い格子が、一斉に薄くフェードインする（後で剥がれる仮置き）
(c) 「1マスで約1600万通り」のセリフ近辺：格子のうち、左上の数マスだけが順次違う色で点く。「組み合わせの多さ」を点滅ではなく順次出現で示す
(d) 「巨大なのだ」のセリフ近辺：格子と色付きマスが一気に消え、代わりに Canvas 内側が NoiseField で埋まる（純粋な砂嵐に変身。連続変形のモーフ）
(e) 「ザーザーの砂嵐に見える絵」のセリフ近辺：CombinationsHaze が Canvas の左右から外側へ、薄い砂粒の雲として広がり始める
(f) 「ごく小さな点」のセリフ近辺：CombinationsHaze の中央、Canvas の真上の位置に、SUBJECT 色の小さな点が一つだけスッと現れる（人が絵と認める唯一の点）
(g) 「無謀なのだ」のセリフ近辺：Canvas と CombinationsHaze 全体がほんの少しトーンダウンする（諦めの予感、VOID 寄りに）
```

#### 演出ノート

- **テンポと強弱** — (d) のモーフ（格子→砂嵐）と (f) の SUBJECT 点登場は、視聴者の理解の節目。両方ともゆっくり、たっぷり間を取る
- **カメラワーク** — なし
- **象徴的なディテール** — (f) で SUBJECT 点が現れた瞬間、CombinationsHaze がほんのわずかに薄くなる（点を引き立てる）
- **絶対やらないこと** — NoiseField をループでチカチカさせない。CombinationsHaze の砂粒を浮遊させない。SUBJECT 点を脈動させない

### ボディ2 — バラし方、二つ

#### 眼目

「一発当てを諦めて、易しい一手の連鎖にバラす。バラし方の軸は二つある」。今は中身じゃなくて、二つの軸の予告。

#### 舞台 — 最も混む瞬間の止め絵

画面中央上に StepBeads が 4 個、横一列。下に Canvas が 2 枚、左右に並ぶ（それぞれ画面高の 35%）。左 Canvas の内側は薄い NoiseField で、ステップ進行の途中状態を 1 枚見せている。右 Canvas の内側は PatchGrid で、左上のマス 2 個だけが AUTOREG 色で埋まっている。それぞれの Canvas の下にラベル「拡散モデル」（DIFFUSION 色）「自己回帰モデル」（AUTOREG 色）。Canvas の上にセクションタイトル「バラし方、二つ」。

#### 時間順の状態遷移

```
(a) 「捨てた代わりに」のセリフ近辺：ボディ1の砂嵐 Canvas と CombinationsHaze がフェードアウト。セクションタイトル「バラし方、二つ」がフェードイン
(b) 「易しい一手のくり返し」のセリフ近辺：画面中央に空の StepBeads（4 個）が左から右へ順次フェードインで現れる
(c) 「少しだけ寄せる、を何度も」のセリフ近辺：StepBeads の各円が、左から順番に INK 色から SUBJECT 色へ点灯していく（4 個一気には点けない。一手ずつのリズムで）
(d) 「一発当ての無謀を、易しい一手の連鎖に置き換えた」のセリフ近辺：4 個全部が SUBJECT 色になり、ひと呼吸止まる
(e) 「二つの仕組みがある」のセリフ近辺：StepBeads が画面上方へ縮みながら退き、その下に Canvas が 2 枚（左右）、左右対称に滑り出てくる。両方とも内側は空
(f) 「画面ぜんぶがザーザーの砂嵐から始まる」のセリフ近辺：左 Canvas の内側が NoiseField で埋まる（出発点）
(g) 「ザラつきを、少しだけ薄める」のセリフ近辺：左 Canvas の NoiseField が少しだけ薄まり、Cat の輪郭の気配がうっすら見える状態に連続変形
(h) 「これが拡散モデル」のセリフ近辺：左 Canvas の下に DIFFUSION 色のラベル「拡散モデル」がフェードイン
(i) 「空っぽのキャンバスから始まる」のセリフ近辺：右 Canvas の内側に PatchGrid の罫線が薄くフェードイン
(j) 「左上のマスから順に」のセリフ近辺：右 Canvas の左上のマス 2 個が AUTOREG 色で順に埋まる
(k) 「これが自己回帰モデル」のセリフ近辺：右 Canvas の下に AUTOREG 色のラベル「自己回帰モデル」がフェードイン
```

#### 演出ノート

- **テンポと強弱** — (c) の StepBeads 点灯は一定速度ではなく、「一・二・三・四」とリズムを刻むように。(g) と (j) は両側を同時に動かさない——左 Canvas の演出が落ち着いてから右 Canvas に移る
- **カメラワーク** — なし
- **象徴的なディテール** — StepBeads が点灯する間、Canvas（まだ空）はやや暗めの背景として控える。Canvas が前に出るのは (e) で StepBeads が退いてから
- **絶対やらないこと** — StepBeads を全部一気に光らせない（連鎖の感じが消える）。左右 Canvas を同じタイミングで埋めない（対比軸が散る）

### ボディ3 — 拡散モデル

#### 眼目

「砂嵐から始めて、各ステップで少しずつノイズを引き、最後に絵が立ち上がる。学習は逆方向で教師ありに化ける」を 1 枚の Canvas の連続変形で見せる。

#### 舞台 — 最も混む瞬間の止め絵

画面中央に Canvas が 1 枚、画面高の 55%。Canvas の内側はほぼ綺麗な状態で、Cat が立っており、SUBJECT 色の線が完全に出ている（最後のステップ）。Canvas の下に StepBeads が 8 個、全部 DIFFUSION 色で点灯済み。Canvas の上にセクションタイトル「拡散モデル」。Canvas の下、StepBeads の下に AxisLabel「ノイズの量」（DIFFUSION 色）。

#### 時間順の状態遷移

```
(a) 「まずは拡散モデル」のセリフ近辺：ボディ2の 2 枚 Canvas がフェードアウト。セクションタイトル「拡散モデル」がフェードイン
(b) 「完全な砂嵐から始まる」のセリフ近辺：画面中央に Canvas が単体で現れ、内側が一気に NoiseField で埋まる（純粋な砂嵐の出発点）。Canvas の下に空の StepBeads（8 個、グレー）が横一列に並ぶ
(c) 「ノイズをほんの少しだけ引く」のセリフ近辺：StepBeads の 1 個目が DIFFUSION 色で点灯。同時に Canvas 内の NoiseField が「ほんのわずかに」薄まる（密度が少し下がる）
(d) 「次のステップで、もう少しノイズを引く」のセリフ近辺：StepBeads の 2 個目が点灯。NoiseField がもう少し薄まる
```

## 5. 画面の整合性 — 4つの制約

**ここを満たさない画面は、必ず作り直す。**

1. **同時に居る補助要素は最大3つ。** 序論／結論：Canvas + 大見出し + サブカード2 = 計4要素（サブカードはペアで1つの意味カウント）。ボディ1：Canvas + CombinationsHaze + SUBJECT点 = 3。ボディ2：StepBeads + 2Canvas + 2ラベル（ラベルはCanvasに付帯） = 3。ボディ3／4：Canvas + StepBeads + AxisLabel + 学習小キャンバス群（学習場面のみ、生成場面に戻ったら消える） = 3〜4（学習場面は別フェーズで生成場面と分離するので可）。ボディ5：2Canvas + 中央文字 + 2AxisLabel + 共通点リスト（リストは中央文字と排他、入れ替わり） = 4〜5（左右Canvasはペアで1カウント、AxisLabelはCanvasに付帯）
2. **同時に動くのは1つ。** 全画面で守る。特にボディ2／4／5の左右並列では、左を動かしてから右を動かす。同時に動かさない
3. **1画面で扱う対比軸は1つ。** 序論：諦めの予告（対比なし）。ボディ1：「可能性の海 vs 正解の点」（軸1）。ボディ2：「二つのバラし方の予告」（軸1：軸自体）。ボディ3：「拡散モデルの中身」（対比なし、深掘り）。ボディ4：「自己回帰モデルの中身」（対比なし、深掘り）。ボディ5：「拡散モデル vs 自己回帰モデル」（軸1）。結論：「序論との呼応」（対比なし、回顧）
4. **「最も混む瞬間」で全図が重ならない。** Canvas は内側に住む要素を持つので「重なり」は外周との関係のみ。左右並列の 2 Canvas は中央スペースを空ける。CombinationsHaze は Canvas の外側で広がり、Canvas には侵入しない。StepBeads は Canvas の下に専用ゾーン。AxisLabel は StepBeads の下に専用ゾーン。サブカードは Canvas の下に専用ゾーン

## 6. レイアウトの約束

- 画面下部は対話字幕の帯（高さ 200px 相当）が常駐。Canvas はそれより上で完結する
- セクションタイトルは画面上部、Canvas はその下に中央配置（または左右並列）
- 画面切替は標準 30f クロスフェード
- スマホ視聴前提。1 画面の要素は少なく、余白を多く
