# image_generation — design overall

パイプライン2段目。`script.md` を入力に、各画面の登場図・動き・遊びと、動画全体を貫く世界観を、自然言語で設計する。座標値・フレーム数は持たない。

## 1. 世界観

題材は「画像生成AIの仕組み」。芯のメッセージは——画像生成とは、ありえないほど広い数の空間の中で、意味のある絵がいる極薄の場所（糸）へ、やさしい一手の鎖をつないで歩いて当てること。拡散モデルと自己回帰モデルは、その鎖を張る軸がちがう二流派。世界観は、この「広い空間／細い糸／一手の鎖」を画として成立させるために組む。

| 軸 | この題材での設定 |
|---|---|
| 貫通する比喩 | 白い作業台の上で、一枚の**キャンバス**（マス目の格子）が姿を変えていく。そのキャンバスは、**絵の空間**という広いフィールドの中の一点であり、空間には意味のある絵だけが通る**細い糸**が走っている。生成＝糸の上の一点へ歩いて当てること |
| 形の語彙 | キャンバス＝角丸の正方形グリッド（マス目がはっきり見える）。ピクセル＝色つきの小セル。砂嵐＝乱雑な色セルで埋めたグリッド。絵の空間＝うっすいフィールド面。糸＝空間を横切る、なめらかな帯状の曲線。鎖の一手＝小さなコマ（丸／グリッドの小型版）が連なる。絵の単語帳＝小タイルを並べたパレット。プロンプト＝文字入りの角丸タグ |
| 色の語彙 | **背景・面は白で固定**。アクセントは5系統——拡散モデル＝ティール、自己回帰モデル＝アンバー、意味のある絵・糸・地形＝グリーン、砂嵐・無意味＝グレー、言葉・プロンプト＝バイオレット。装置や軌跡は所属する流派の色をまとう |
| 動きの語彙 | 使う：連続変形（砂嵐⇄絵、ぼやけ⇄くっきり）、セル単位の充填（カウント駆動でトークン・コマが増える）、軌跡を一歩ずつ伸ばす、線・帯の描き起こし、値の連続変化（ノイズ量・充填数・ぼやけ量）。**封印：ループし続ける動き全般**——砂嵐は「ちらつかせない」。乱雑さは静的な状態として持ち、イベントのときだけ一段変化する。浮遊・明滅・脈動・常駐背景モーション・ズーム多用・回転は使わない |
| 文字方針 | 対話字幕は画面下部、話者色つき（固定ベース）。図中の装置名・主要ラベル（「キャンバス」「絵の空間」「拡散モデル」など）は読ませる文字として大きく。プロンプトタグの中の「宇宙服を着た猫」も読ませる文字。グリッドのセルや絵の単語帳のタイルには文字を入れない（色・形だけ）。鎖のコマに振る連番は「記号としての文字」として小さくしてよい |
| ナビ補助 | 左上にセクションタイトル（序論／01 絵は数の格子／02 拡散モデル／03 自己回帰モデル／04 ふたつを並べる／結論）。進捗バーは置かない |

## 2. セクション別デザイン

`script.md` の6セクション（＝6画面）に1対1で対応。

### 序論：見たことのない一枚

- **登場する図**：プロンプトタグ「宇宙服を着た猫」、生成画（宇宙服を着た猫のスタイライズドな一枚）、却下される「切り貼り」「検索」の小片、タイトル。
- **レイアウト**：中央上にプロンプトタグ、そこから下へ矢印、その先に生成画。生成画はやがて画面中央の主役へ。下部は字幕帯の余白。
- **動き**：プロンプトタグが現れる → 矢印が根元から伸びる → 生成画がぼやけた状態から描き起こされ、くっきり結像する。「切り貼り？」で、生成画の脇に写真の小片2枚が寄ってくるが、却下されて消える。「検索？」で虫めがねの印が現れ、却下されて消える。「毎回ちがう」で、生成画が別バージョンの一枚へクロスフェードで切り替わる。最後にタイトルが結像。
- **遊び**：生成画の中の猫は、人物っぽい図。「世界のどこにも無かった絵」と明かされる節目で、猫がきょとんとした顔を一度（自分が今この場で生まれたことに気づいた反応）。この動画の遊びは一貫して「生成画の中の猫が、自分の生成のされ方に反応する」で通す。
- **絵の眼目**：言葉から、世界に無かった一枚が、その場で生まれる。

### ボディ1：絵は数の格子 ── 意味のある絵は、ありえないほど狭い

- **登場する図**：キャンバス（背骨）、ピクセルグリッド、砂嵐グリッド、絵の空間フィールド、意味のある絵の糸、糸の上を歩く点、一発出力のもやもや絵、やさしい一手の鎖。
- **レイアウト**：上半分＝キャンバス／グリッドの実演ゾーン。中〜下半分＝絵の空間フィールドと糸のゾーン。二つのゾーンは占有を分け、重ねない。
- **動き**：序論の生成画がキャンバスとして残り、拡大してマス目（ピクセル）が見える。セルがでたらめな色で埋まり、砂嵐へ連続変形。次にキャンバスが縮小して、絵の空間フィールドの中の一点になる。フィールドに意味のある絵の糸が描き起こされ、糸の上に小さな絵のサムネイルが点々と乗る。糸の上の点が一歩ずつ動くと、隣のサムネイルへなめらかに変わる。点が糸を外れて原っぱへ出ると、数歩で砂嵐サムネイルに崩れる。続いて大きなキャンバスが戻り、「一発出力」＝矢印一本でもやもやした猫が出る。最後に、そのもやもや猫が数コマかけて少しずつくっきりし、「やさしい一手の鎖」が空間の中で糸へ向かう軌跡として描かれる。
- **遊び**：一発出力のもやもや猫は、人物っぽい図。生成されたのに自分の輪郭がぼやけていることに、情けない・困った顔の反応を一度。直後、一手の鎖でくっきり結像したとき、しゃきっと得意顔に変わる反応を一度。
- **絵の眼目**：意味のある絵は、広大な空間の極薄の糸の上だけにあり、生成とは、そこへ一手ずつ歩いて当てること。

### ボディ2：流派その一・拡散モデル ── ノイズを引いて、絵を浮かばせる

- **登場する図**：ノイズの濃さの軸（水平の目盛り）、訓練の列（写真が段階的に砂嵐化する forward の連なり）、ノイズを見ぬくAI装置、生成中のキャンバス、プロンプトタグ、拡散の鎖の軌跡。アクセントはティール。
- **レイアウト**：上部に「ノイズの濃さの軸」を水平に一本（左端＝濃い砂嵐、右端＝くっきりした絵）。中段が訓練・装置・生成の実演ゾーン。プロンプトタグは装置の上に差し込まれる。
- **動き**：拡散モデルの名と、ノイズ濃さの軸が現れる。訓練——一枚の写真キャンバスが、ノイズのふりかけで一段ずつ砂嵐へ連続変形し、軸の左へ列が伸びていく。AI装置が現れ、砂嵐入りの画像から「乗っているノイズ」だけを抜き出して見せ、「足したノイズ＝正解」と答え合わせの対応が示される。生成——軸の左端の砂嵐キャンバスが、装置の一手ごとにノイズを引かれ、軸を右へ一段ずつ進む。グリッドが砂嵐から絵へ連続変形（ノイズ量 1→0）。霧が晴れるように絵が浮かぶ。プロンプトタグが装置に差さると、各一手の進む方向が「猫・宇宙服」へ舵を切り、軌跡が糸の上の猫の点へ寄っていく。最後に、出発の砂嵐を替えると別の猫に着くことを短く見せる。
- **遊び**：生成中のキャンバスの中身は猫になっていく。砂嵐の段階では猫は顔も見えない。霧が晴れて猫が結像した瞬間、猫が「やっと出てこられた」という安堵の反応を一度。
- **絵の眼目**：拡散モデルは、砂嵐から一段ずつノイズを引き、プロンプトで舵を切って、絵を浮かび上がらせる。

### ボディ3：流派その二・自己回帰モデル ── 絵を「言葉の列」にして書く

- **登場する図**：文章のトークン帯、次の一語を当てる装置、絵キャンバス、絵を刻むかけら（パッチ）、絵の単語帳パレット、絵のトークン列、プロンプトタグ。アクセントはアンバー。
- **レイアウト**：上部に文章のトークン帯（ChatGPTの実演）。中段に絵→かけら→単語帳→トークン列の変換ゾーン。下段に、言葉トークンと絵トークンが同居する一本の長い列。占有ゾーンを縦に三段に分ける。
- **動き**：ChatGPTの文章書き——テキストトークンの帯が、装置の一手ごとに一語ずつ右へ伸びる。次に絵キャンバスがグリッド線でかけら（パッチ）に刻まれる。絵の単語帳パレットが現れる。各かけらが、単語帳の最も近いタイルへ線で結ばれ、そのタイル（トークン）へ置きかわる。2Dのグリッドが、一本の長い列へほどける。プロンプトタグが列の先頭に付き、言葉トークンと絵トークンが一続きになる。生成——装置が左上から順に、次のトークンを当てて列を埋めていき（充填はカウント駆動）、列が埋まるぶんだけ、2Dキャンバスにも対応するかけらが上から順に現れる。最後に、次トークンの候補が複数あり、そのつど別を選ぶことを短く見せる。
- **遊び**：キャンバスには上から順に猫が現れる。耳・顔が先に書かれ、下半身がまだ空白の「半分だけの猫」が、続きを待つ／下半身を気にする反応を一度。書き上がった瞬間に、猫が満足げな反応を一度。
- **絵の眼目**：自己回帰モデルは、絵を「絵の単語の列」に変換し、ChatGPTと同じく次の一語を当てて、一語ずつ絵を書く。

### ボディ4：ふたつを並べる ── おなじ芯、ちがう向き

- **登場する図**：中央に絵の空間と糸（共通の的）、左に拡散モデルの鎖（ノイズ濃さの軸・ティール）、右に自己回帰モデルの鎖（描いた量の軸・アンバー）、共通点の図（一発の却下／絵と言葉のペアから学ぶ）、得意・弱みのバッジ、溶ける境目の帯。
- **レイアウト**：左右対称。中央上に共通の地形（空間＋糸）。左ゾーン＝拡散、右ゾーン＝自己回帰。左右の鎖は中央の同じ糸へ向かう。
- **動き**：左右に二つの鎖が並んで現れる。共通点——両方の鎖の手前で「一発の矢印」が×になり、「一手の鎖」へ置きかわる。学習データの図（絵＋説明文のペア）が両方へ流れこみ、覚えるのは個々の絵でなく中央の糸の地形だ、と地形が強調される。ちがい——左の拡散は、グリッド全体が砂嵐からくっきりへ、何度も全体を磨く。右の自己回帰は、グリッドが左上から一部分ずつ確定していく。二つを並べて対比再生。得意——左に「質感・まとまり」、右に「指示の理解・会話で修正・文字」のバッジ。弱み——右の鎖で、ある一手が赤くなると、以降のコマにも赤が響く（直せない）。左の鎖では、ある一手が赤くなっても、次の段で全体を見直して直る。最後に、左右の鎖の間がぼやけてハイブリッドの帯が薄く現れるが、中央の「芯の二択」軸は残る。
- **遊び**：左右の鎖の終点に、拡散産の猫と自己回帰産の猫が一匹ずつ。「どちらが上、ではない」の節目で、張り合っていた二匹が、ふっと顔を見合わせて並ぶ反応を一度。
- **絵の眼目**：二つの流派は、同じ「一手の鎖で地形上の一点を当てる」を、ちがう軸でやっているだけ。

### 結論：狙った一点を当てる

- **登場する図**：振り返りアイコンの帯（数の格子／砂浜の一粒／拡散の鎖／自己回帰の鎖）、絵の空間と糸（全編の舞台が回帰）、二つの鎖の収束軌跡、序論の生成画（回帰）、まだ灯らない一点たち、締めの一文。
- **レイアウト**：上部に振り返りアイコンの帯。中央に絵の空間と糸。糸の上に、序論の猫の一点。
- **動き**：振り返りアイコンが順に灯る。中央に絵の空間と糸が戻る。二つの鎖（拡散・自己回帰）が、空間の別々の入口から、同じ糸の上の「猫の一点」へ向かって歩き、軌跡が一点に収束する。入口を替えると糸の上の別の点に着くことを短く見せる。猫の一点が光り、序論の生成画が浮かび上がる。最後に、糸の上にまだ灯っていない暗い点々が無数に残っているのを見せ、締めの一文が結像する。
- **遊び**：回帰した猫は、序論ではきょとんとしていたのが、結論では視聴者のほうへ、物語を一周し終えた満足げな反応を一度。
- **絵の眼目**：画像生成とは、学んだ地形の上で、狙った一点を当てること。まだ無数の一点が眠っている。

## 4. 採否判断

- [ ] script.md の全6画面に対応するデザインがあるか — ある
- [ ] 固定ベース（白テーマ・文字の大きさ）を守り、残る世界観を題材ごとに決めたか — 白固定、アクセント5系統を題材用に設計
- [ ] 背骨（姿を変えていく中心の図）が一本通っているか — キャンバスが全画面で姿を変える
- [ ] 図カタログの各図が「見せる仕組み」を一言で言えるか — 各図に明記
- [ ] 各画面に「絵の眼目」があり、画面内で図が状態変化するか — 各画面に明記、全画面で連続変化
- [ ] 図の動き・登場・遷移が `00_philosophy.md`「図の動かし方」に沿っているか — モーフ中心、湧き禁止、ループ動作なし。砂嵐は静的状態として持つ
- [ ] 湧く／消える図がないか — 各図の出生元・退場先は次段 spec で固定
- [ ] 画面が通るどの状態でも図どうしが意図せず重ならないか — 各画面で占有ゾーンを分割（ボディ1は上下、ボディ3は縦三段、ボディ4は左右対称）
- [ ] 動画が硬くなっていないか — 遊びを「生成画の中の猫が、自分の生成のされ方に反応する」という題材固有の形で設計。各画面で猫が一度きりの反応をする
- [ ] 座標値・フレーム数・セリフ単位タイミングを書いていないか — 質的記述に留めた

## 次段への引き継ぎ

`design_spec.md` に渡すもの：

- **図の顔ぶれと役割** → spec がライフサイクル契約（誕生・死・出生元・退場先）をセリフに紐付ける
- **各画面での図の質的な動き** → spec がセリフ単位の event に割り付ける
- **世界観の語彙**（色5系統・形・動き・文字方針） → spec と tsx が従う基準
