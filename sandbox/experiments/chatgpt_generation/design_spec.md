# セリフ単位スペック — 【視覚的に理解する】ChatGPTが文章を生成する仕組み

`design_overall.md` の背骨（入力ベルトが確率分布へ変形する生成パイプライン）を、event に分解して `script.md` のセリフに紐付ける。座標値・フレーム数は下流で決める。

## 画面 1: map

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.map.in | め「ChatGPTって、こっちが一文入れると」 | 空の横長パイプラインが現れる |
| input.in | ず「入力は『明日は雨なので、傘を』」 | 左端に入力文カードが置かれる |
| map.steps | ず「文字列がトークン列になり、トークンIDになり」 | パイプライン上に token / id / vector / attention / logits / softmax の駅が順に点灯 |
| map.goal | ず「最後にlogitsという候補点数になる」 | 右端に候補語の確率棒グラフが薄く出る |
| map.loop | ず「一つ選んだトークンを後ろに足して、また同じ流れを回す」 | 右端の「持つ」から入力文末へ戻るループ矢印が描かれる |
| map.question | ず「どうやって『持つ』が高確率になる分布へ変わるのか」 | 全体地図が小さく奥へ下がり、入力文と右端分布だけが強調される |

## 画面 2: loop

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.loop.in | ず「ChatGPTの文章生成は、基本的には次のトークンを一つ選ぶ処理」 | 入力文の右端に空きスロットが現れる |
| dist.preview | ず「今までの文脈から、次に来る部品の確率分布を作る」 | スロットの右に候補「持つ」「差す」「買う」「忘れる」「猫」の棒グラフが出る |
| token.pick | ず「その中から一つを選び、後ろに足す」 | 「持つ」チップがグラフからスロットへ滑り込む |
| loop.reset | ず「足したら、それを含めた新しい文脈で、また次の確率分布」 | 文が「...傘を持つ」へ伸び、新しい空きスロットが開く |
| toy.scope | ず「実際の語彙はこんな5個ではなく」 | 5候補の背後に巨大な候補棚が半透明で広がる |

## 画面 3: tokenize

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.tokenize.in | ず「入力された文字をそのままニューラルネットに入れているわけではない」 | 連続した入力文カードが中央に出る |
| text.cut | ず「『明日』『は』『雨』『なので』『、』『傘』『を』に切る」 | 文カードに切れ目が入り、7つのトークンピースに分かれる |
| ids.in | ず「『明日』が101、『は』が12、『雨』が502」 | 各ピースの下にID番号札が出る |
| embedding.in | ず「IDを意味を持つ数値の並びへ変換する」 | ID札が4本の数値バーへ変形する |
| rain.vector | ず「『雨』の埋め込みを [0.6, -0.1, 0.8, 0.2]」 | 「雨」のバー列に数値ラベルが出る |
| umbrella.vector | ず「『傘』は [0.7, 0.0, 0.6, -0.1]」 | 「傘」のバー列に数値ラベルが出る |
| wo.vector | ず「『を』は [-0.2, 0.1, 0.0, 0.4]」 | 最後の「を」のバー列が拡大される |
| pos.add | ず「位置情報を足す」 | 各トークンの上に位置番号が乗り、バー列に細い位置色が足される |

## 画面 4: attention

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.attention.in | ず「ここでAttentionなのだ」 | tokenized入力が横一列に再登場し、最後の予測地点が光る |
| weights.in | ず「『傘』に0.45、『雨』に0.30、『なので』に0.15」 | 最後の位置から各トークンへ太さの違う注意線が伸びる |
| mix.in | ず「この重みで、前のトークンのベクトルを混ぜる」 | 注意線を通って「傘」「雨」「なので」の色が最後のベクトルへ流れ込む |
| context.vector | ず「[0.78, 0.12, 0.66, -0.08] みたいな文脈ベクトル」 | 最後の「を」のバー列が新しい4本バーへモーフする |
| heads.in | ず「Attentionは一種類ではない」 | 文法・理由・相性の3色の注意線が重なる |
| layer.stack | ず「この処理が何層も積み重なる」 | 同じトークン列が縦に積まれ、上層ほど抽象色に変わる |
| mlp.in | ず「混ざったベクトルを、次の予測に使いやすい形へ曲げ直す」 | 文脈ベクトルが小さな変換ボックスを通る |
| final.vector | ず「[1.10, 0.05, 0.88, -0.12] になる」 | 変換後の最終ベクトルが右へ置かれる |

## 画面 5: logits

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.logits.in | ず「最後の位置には文脈を背負ったベクトルがある」 | 最終ベクトル [1.10, 0.05, 0.88, -0.12] が中央に出る |
| vocab.in | ず「語彙には『持つ』『差す』『買う』『忘れる』『猫』」 | 右側に5つの候補語カードが縦に並ぶ |
| dot.score | ず「候補にも、出力用の重みベクトルがある」 | 最終ベクトルから各候補へ線が伸び、点数計算の印が出る |
| logits.in | ず「『持つ』3.2、『差す』2.5、『買う』1.1」 | 候補カードの横にlogits棒が出る |
| softmax.in | ず「そこでsoftmaxを使うのだ」 | logits棒がsoftmaxゲートを通過する |
| probs.in | ず「『持つ』60パーセント、『差す』30パーセント」 | 棒グラフが確率表示に変わり、合計100%の目盛りが点灯 |
| flow.recap | ず「トークンID、埋め込み、位置つきベクトル」 | 画面下にこれまでの変換列が小さく流れて、現在位置がsoftmaxで光る |

## 画面 6: decode

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.decode.in | ず「確率分布ができたら、次は出力トークンを選ぶ」 | 確率分布だけが大きく中央に出る |
| greedy.mark | ず「一番高いものを必ず選ぶ方法もある」 | 「持つ」棒に王冠状のマークがつく |
| sample.roll | ず「確率に従って少しランダムに選ぶこともある」 | 確率棒の上を選択カーソルが揺れ、今回は「持つ」で止まる |
| append.token | ず「分布から『持つ』が選ばれたとする」 | 「持つ」チップが文末に接続される |
| next.slot | ず「文脈は『明日は雨なので、傘を持つ』に更新される」 | 新しい文の右端に次の空きスロットが開く |
| loop.again | ず「ここからまた同じ処理を回す」 | 文末からパイプライン先頭へ戻るループ矢印が強調される |

## 画面 7: chatgpt

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.chatgpt.in | め「ChatGPTって、ただ続きを書くだけじゃなくて」 | 基本パイプラインの上に「会話として返す」レイヤーが重なる |
| base.model | ず「土台には、次トークンを予測する大きな言語モデル」 | 下段に大きな言語モデル箱が出る |
| align.in | ず「人間の指示に従いやすくする後段の学習や調整」 | モデル箱の出口に調整ダイヤルが付く |
| context.layers | ず「システム側の指示、過去の会話、添付された情報」 | 入力文の前に system / history / files / tools の薄いカードが連結される |
| dist.shift | ず「ChatGPTらしさは、その文脈の作り方と、モデルの調整」 | 同じ候補分布が、丁寧・安全・形式一致の方向へ少し傾く |

## 画面 8: limits

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.limits.in | ず「この仕組みの弱点も見ておくのだ」 | softmax分布が再登場し、横に「現実」レイヤーが空で出る |
| natural.not.true | ず「『自然に続く』と『現実に正しい』は同じではない」 | 確率分布と現実チェックの2枚が少しずれて表示される |
| plausible.error | ず「自然だけど間違った出力になることがある」 | 高確率候補が滑らかに選ばれるが、現実レイヤーでは赤いズレが出る |
| verify.tools | ず「コードなら実行する。事実なら一次情報を見る」 | 実行・一次情報・計算・専門家の照合印が順に押される |
| probability.label | ず「softmaxが出したのは『次に自然な候補の分布』」 | 分布に「自然さ」、現実レイヤーに「検証」とラベルが付く |

## 画面 9: outro

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.outro.in | ず「流れをもう一度通すのだ」 | 入力文が左端に戻る |
| recap.tokens | ず「トークン列になる」 | 切れ目が入る |
| recap.vector | ず「IDは埋め込みベクトルになる」 | IDからバー列へ変わる |
| recap.attention | ず「Attentionで『雨』『傘』『なので』が最後の予測地点へ混ざる」 | 注意線が最後の位置に集まる |
| recap.softmax | ず「logitsはsoftmaxで、『持つ』60パーセント」 | logits棒が確率分布へ変わる |
| final.loop | め「選ばれた『持つ』を後ろに足して、また同じ流れを回す」 | 「持つ」が文末へ接続され、ループ矢印が一周する |
| final.glow | ず「次の一手を選ぶ小さな分布が光っている」 | 文の先端の確率分布だけが残り、静かに光って停止 |

## 各オブジェクトの誕生と死

| オブジェクト | 誕生 | 死 |
|---|---|---|
| 入力文カード「明日は雨なので、傘を」 | input.in | final.glow まで変形しながら継続 |
| パイプライン地図 | scene.map.in | scene.loop.in で小型化、以降は recap / loop として再登場 |
| 空きスロット | scene.loop.in | append.token で一度埋まり、next.slot で再誕生 |
| 候補確率分布 | dist.preview | scene.tokenize.in で退場、probs.in 以降は decode / limits / outro で再登場 |
| トークンピース | text.cut | pos.add 後、scene.attention.in で注意対象として継続 |
| ID番号札 | ids.in | embedding.in でバー列へ変身 |
| 埋め込み・位置バー列 | embedding.in | context.vector で文脈ベクトルへ変身 |
| Attention線 | weights.in | layer.stack で層内表現へ畳まれる |
| 文脈ベクトル | context.vector | mlp.in で最終ベクトルへ変身 |
| 最終ベクトル | final.vector | dot.score でlogitsへ変身 |
| logits棒 | logits.in | softmax.in で確率棒へ変身 |
| 「持つ」出力チップ | token.pick | next.slot で入力文の一部として吸収 |
| ChatGPT調整レイヤー | align.in | scene.limits.in |
| 現実チェックレイヤー | scene.limits.in | final.glow |

同時表示の最大は、画面5の「最終ベクトル＋候補カード＋logits/確率棒」、画面7の「基本パイプライン＋文脈カード＋調整レイヤー」、画面8の「確率分布＋現実レイヤー＋照合印」で、いずれも3〜4個以内に収める。
