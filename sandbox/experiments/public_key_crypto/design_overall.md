# public_key_crypto — design overall

`script.md` を入力に、動画全体を貫通する世界観と、5画面それぞれのデザインを自然言語で設計する。座標値・フレーム数・セリフ単位のタイミングは持たない（design_spec と Composition.tsx の仕事）。

図解は概念に**ラベルを貼る**のではなく、仕組みを**動いて見せる**。「公開鍵」と書いた箱は出さない。錠前が鍵なしでパチンと閉じ、配られた錠前で箱が閉じられ、手元の鍵だけが箱を開ける——その受け渡しそのものを画面の上で起こす。本作の図は、汎用の箱・矢印の寄せ集めではなく、**ひとつの仕組みをひとつ可視化する専用装置**の集まりとして設計する。

## 背骨

貫通する背骨は **錠前と鍵、そして開けた通信路の上を渡る一つの箱**。白い舞台を一本の通信路が横切り、左に送り手、右に受け手。道の上を通るものは、上から見おろす一つの目に、すべて見られている。箱は「施錠度」という一つの数値を持つ——0なら、ふたが開いて中身が見える。1なら、錠前で固く閉じている。

動画全体は、この箱が人目のある通信路をどうやって安全に渡りきるかの一本の物語になる。序論では一本の鍵で開け閉めする素朴な錠前が、鍵を渡す段で破れる。ボディ1で錠前と鍵が「配る錠前（公開鍵）」と「残す鍵（秘密鍵）」に分かれ、ボディ2では錠前の殻が開いて中身が片道の計算だと分かり、ボディ3では同じ錠前と鍵を逆向きに使って送り主を証す。個々の装置——のぞき見る目・一方向の計算・落とし戸・署名の印——は、すべてこの錠前と鍵が箱を「閉じる／開く」工程として連なる。

## 1. 世界観

白テーマ（背景・面が白系）と文字サイズ下限は固定ベース（→ `02_design_overall.md`「固定ベース」）。比喩・図の語彙・アクセント色を、公開鍵暗号固有に設計する。

| 軸 | この題材での設定 |
|---|---|
| 貫通する比喩 | **開けた往来**。白い作業台を一本の通信路が横切り、左端に送り手の台、右端に受け手の台。道の上を通るものは、真上から見おろす一つの目に、すべて見られている。錠前・鍵・箱は、この道の上で受け渡される実体のある道具。視聴者は同じ舞台をずっと眺め、一つの箱が人目の道を安全に渡りきるまでの全工程を追う |
| 形の語彙 | 専用の図解装置の集まり（一覧は §3 図カタログ）。背骨は **箱**（ふた付き、中に小さな手紙）と **錠前**（U字のつるが開閉する南京錠）と **鍵**（持ち手と歯）。装置：のぞき見る目／一方向の計算（数のタイルと行き帰りの二本道）／落とし戸（険しい壁に開く小さな扉）／署名の印（確認のチェック）。汎用の「ラベル付き箱」「意味のない矢印」「テキストだけのカード」は使わない |

| 動きの語彙 | 使う＝フェード、通信路に沿った移動、錠前のつるがパチンと閉じる／鍵でかちゃりと開く、箱のふたの開閉、のぞき見る目の視線、数のタイルが寄って積になる、矢印が行きはすっと一息・帰りは壁の前で詰まる、落とし戸が開く、印が灯る。封印＝3D回転、カメラのズーム／パン、バウンド、急な明滅、**ループし続ける動き（浮遊・振動・明滅・脈動・常駐背景モーション）**。装置は会話の進行にあわせてイベントごとに一度きり動き、終わったら止まる（合間は静止していてよい） |
| 文字方針 | 画面下部に**対話字幕**を常駐（話者名は話者色、本文は濃い藍、スマホ視聴前提で大きめ・2行まで）。図解内のテキストは最小限：公開鍵／秘密鍵のラベル、数のタイルの数字、行き帰りの道の「やさしい／険しい」、落とし戸、署名、工程アイコン名。サイズは固定ベースの下限を守る |
| ナビ補助 | 画面が切り替わるとき、左上に小さくセクション番号と見出し（例「01 錠前と鍵を、別々に」）。進捗ドット等は置かず最小限。字幕帯とは別の位置 |

レイアウトの基準：中心原点。**画面下部 y≈330〜510 は対話字幕の帯**として確保し、主オブジェクトはそれより上に置く。通信路は画面の中ほどに横たえ、送り手を左端・受け手を右端に。のぞき見る目は通信路の上。1画面に装置を全部いっぺんに出さない——会話が進むにつれ装置が順に立ち上がり、役目を終えたら畳まれ、次の装置に場所を譲る。

## 2. セクション別デザイン

### 序論（画面1）：人目のある通信路

- **登場する図**：通信路、送り手の台、受け手の台、のぞき見る目、箱（中に手紙）、素朴な錠前と鍵（一本の鍵で開け閉めするタイプ）、タイトル「公開鍵暗号」
- **レイアウト**：通信路を画面中ほどに横一文字。送り手を左端、受け手を右端の小さな台として。のぞき見る目は通信路の真上。箱は通信路の上を左から右へ動く。タイトルは通信路の下、字幕帯の上の余白に。
- **動き**：通信路と送り手・受け手の台が現れる → 送り手が手紙を箱に入れ、ふたを開けたまま通信路を右へ進める。上の目が箱をとらえ、中の手紙がコーラルに染まる（中身が読まれる）→ 素朴な対策：錠前と鍵が現れ、箱のふたを閉じて錠前をかける。閉じた箱は進んでも目に中身を読まれない → だが受け手は鍵を持たず開けられない。鍵が通信路を進みはじめると、目が鍵をとらえてコーラルに染まり、漏れた鍵で箱が開いてしまう（コーラルの✗）→ 通信路の上に錠前のかかった箱がとどまり、その下にタイトル「公開鍵暗号」が灯る
- **絵の眼目**：人目のある道では、箱に鍵をかけても、その鍵を渡す段で見られてしまう——秘密を送るのに秘密が要る、堂々めぐり

### ボディ1（画面2）：錠前と鍵を、別々のものにする

- **登場する図**：通信路、送り手の台、受け手の台、のぞき見る目、南京錠（つるが開閉）、鍵、箱、配られる公開鍵の群れ、公開鍵・秘密鍵のラベル
- **レイアウト**：通信路は引き続き中ほど。受け手（右）のそばに手元ゾーンを小さく確保し、錠前と鍵の対をそこに置く。配られる錠前は通信路を右から左へ流れる。箱は送り手側で錠前をかけられ、右へ戻る。
- **動き**：一つの南京錠が中央に大きく現れ、つるが開いた状態から押しこむとパチンと閉じる——鍵は使われない（閉めるのに鍵は要らない）→ 受け手が錠前と鍵をひとそろい持ち、鍵は受け手の手元ゾーンへ収まる（秘密鍵）→ 開いたままの錠前が複製され、何個も通信路を左へ流れていく（公開鍵）。上の目もその一つを受け取るが、危険のコーラルではなく公開のティールで描かれる（配ってよい）→ 送り手が流れてきた錠前を一つ取り、箱のふたを閉じてパチンとかけ、箱を右へ送り返す → 目は箱を見るが、ティールの錠前は閉じるだけで開かない → 受け手の手元の鍵（アンバー）が箱の錠前にかちゃりと差さり、ふたが開く（グリーンの✓）。鍵は一度も通信路に乗っていない
- **絵の眼目**：閉める錠前は世界じゅうに配ってよい。開ける鍵だけを手元に残せば、鍵を一度も渡さずに秘密が届く

### ボディ2（画面3）：閉める計算と、開ける計算

- **登場する図**：南京錠（殻が開いて計算盤になる）、二つの素数タイル、積のタイル、行きの道（やさしい）、帰りの道（険しい壁つき）、総当たりの試しタイル群、落とし戸、合言葉の数（秘密の数）、砂時計
- **レイアウト**：中央に錠前。錠前の金属の殻が開くと計算盤になる。盤の左に二つの素数タイル、右に積のタイル。あいだに行き（上の道）と帰り（下の道）。帰りの道のとちゅうに険しい壁、その壁に落とし戸。
- **動き**：ボディ1の錠前が中央で大きくなり、金属の殻が左右に開いて中の計算盤があらわになる → 左の二つの素数タイルが中央へ寄り、掛け合わさって右に大きな積のタイルになる。行きの矢印がすっと一息で通る（掛け算はやさしい）→ 逆向き：積のタイルから帰ろうとする矢印が、険しい壁の前で詰まる。壁の手前に総当たりの試しタイルが次々現れては×になる（戻すのは困難）→ 帰り道の壁に小さな落とし戸が現れ、受け手だけが持つ合言葉の数（アンバー）が差しこまれると落とし戸が開き、険しい壁を迂回するアンバーの近道が通る → 錠前の殻だけを差し出して見せる：殻をいくら調べても、合言葉の数はその中に無い → 帰りの道に砂時計が小さく添えられ、「絶対」ではなく「現実的な時間では」であることを示す
- **絵の眼目**：錠前は片道の計算。閉じる行きはやさしく、開ける帰りは険しい。秘密鍵は、その帰り道に付いた近道（落とし戸）の合言葉

### ボディ3（画面4）：同じ鍵で、送り主を確かめる

- **登場する図**：通信路、受け手の台、受け取る人々（複数）、南京錠、鍵（秘密鍵・アンバー）、配られた公開鍵（ティールの錠前）、箱、署名の印（グリーンのチェック）、鍵と印を結ぶ線
- **レイアウト**：通信路に戻る。今度は受け手（右）が送る側。受け手の手元に秘密鍵。通信路の先（左）に、配られた公開鍵を持つ受け取る人々が並ぶ。
- **動き**：受け手が、箱のふたを手元の秘密鍵（アンバー）で閉じる——錠前ではなく鍵で閉める（さっきと順番が逆）→ 箱が通信路を進み、配られた公開鍵（ティールの錠前）を持つ人なら誰でもその箱を開けられる → 箱がきれいに開く。中身を隠すためではない：開いたという事実そのものに、グリーンの署名の印が灯る → 印から、受け手の手元の秘密鍵へ一本の線が結ばれ、「この箱を閉じられたのは、秘密鍵を持つただ一人」という関係を示す。中身を隠していた同じ装置が、向きを逆にして送り主の印になる
- **絵の眼目**：鍵で閉めて錠前で開ける——逆向きに使うと、開けられたという事実が「ただ一人の持ち主が閉めた」証しになる

### 結論（画面5）：鍵を渡さずに、秘密を送る

- **登場する図**：通信路、箱、工程アイコンの帯（人目の道／錠前と鍵を分ける／片道の計算／署名）、のぞき見る目、錠前、鍵、締めの一文
- **レイアウト**：上に工程アイコンの帯。中央に通信路と箱。
- **動き**：工程アイコンが順に灯る：人目の道 → 錠前と鍵を分ける → 片道の計算 → 署名。アイコンが灯るのにあわせ、中央の通信路の箱が全工程を早送りでなぞる → 序論の不思議に回帰：通信路の上に錠前のかかった箱、その上にのぞき見る目、傍らに「？」→ だが受け手の手元の鍵だけが光り、通信路の上には「鍵が通った跡」が無いことが空白で示される。箱が受け手のもとで開き、締めの一文を残して静かに終わる
- **絵の眼目**：鍵を渡す必要は最初から無かった。閉める錠前は配り、開ける鍵だけを隠す——それで人目の道を越えて秘密が結べる
