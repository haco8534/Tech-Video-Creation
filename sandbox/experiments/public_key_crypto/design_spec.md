# 公開鍵暗号 — セリフ単位スペック

`design_overall.md` の背骨「道を渡るもの／渡らないもの」を、画面ごとの event に分解し、`script.md` のセリフ 1 行に紐付ける。

## 共用の装置と色言語

- **舞台** — 画面を横切る道。左端＝送り手の台、右端＝受け手の台、道の中央上方に**目**。目は視線（線）を対象へ引く
- **箱** — ふた付き。蝶番は本体の上辺の角
- **錠前** — U 字のつるが開閉。蝶番はつるの脚の根元。閉めるのは押すだけ、開けるのは鍵
- **鍵** — 輪＋軸＋歯

色：背景 `#F5F7FA` ／ 主 `#243044` ／ 公開（配るもの）＝ブルー `#3A7CC4` ／ 秘密（隠すもの）＝ゴールド `#C99022` ／ 盗み見＝レッド `#CC4B3C` ／ 証し＝グリーン `#3E9B5F`

## 画面づくりの約束

- **回す装置は蝶番を一つに決める。** 錠前のつるは脚の根元、箱のふたは本体の角を軸に開閉し、軸の位置はぶらさない。
- **密な画面はレーンに分ける。** 画面3 は上帯＝行きの計算／下帯＝帰りの計算に機械的に分離し、矢印やラベルの交差・重なりを設計で消す。画面内テキストは数値・固有名詞・役割ラベル 1 語のみ。
- **構造の遠い図どうしは変身させない。** 錠前→計算盤面（画面3）は連続変形せず、金属面を畳んで輪郭を額縁に残し、中身を描き起こす。

## 画面2 錠前と鍵を、別々のものにする — 錠前だけが道を渡り、鍵は手元から動かない

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.split.in | ず「まず、思いこみをひとつ外すのだ」 | 舞台再構成。中央の見せ場に錠前と鍵のひとそろいが大きく現れる |
| lock.snap | ず「パチンと押しこめば、それだけで掛かる」 | 鍵を脇に置いたまま、つるがパチンと閉まる |
| lock.unlock | ず「開けるときだけ、鍵が要る」 | 鍵が差さってつるが開く |
| pair.home | ず「受け手が、錠前と鍵をひとそろい用意する」 | ひとそろいが受け手の手元へ移動・縮小。鍵は手元枠に収まる |
| locks.spread | ず「いっぽうの錠前は、開いたまま、いくつも」 | 開いた錠前の複製が開始をずらして道を左へ渡る。1 個は道の中央（目の届く所）に残る |
| eye.lock | め「のぞき見している人も、わたしと同じ錠前を」 | 目の視線が中央の錠前へ落ちる（届いていい） |
| eye.lock.close | ず「できるのは『閉める』ことだけなのだ」 | 中央の錠前がパチンと閉じ、それきり何も起きない |
| spare.fold | ず「送り手は、配られた錠前で箱をパチンと」 | 中央の錠前を畳み（時間順畳み）、送り手の箱が湧いて配られた錠前でパチンと閉じる |
| box.cross | ず「のぞいている人にも、その箱は見えるのだ」 | 箱が道を渡る。視線が落ちるが閉じたまま |
| box.unlock | ず「手元の鍵で、かちゃり」 | 鍵が差さり錠前が開き、ふたが開く |
| key.still | め「渡したのは錠前だけ。鍵は、ずっと受け手の」 | 鍵が手元枠で強調される。道の上に渡ったのは錠前と箱だけ |
| names.in | ず「手元に残すこの鍵を、秘密鍵と呼ぶのだ」 | 錠前に「公開鍵」、鍵に「秘密鍵」のラベルが付く |

**誕生と死** — ひとそろい：scene.split.in で生まれ pair.home で受け手の持ち物に転用。複製錠前：locks.spread で生まれ、未使用分は spare.fold で死。箱：spare.fold で生まれ画面退場で死。同時表示の最大は〈舞台・錠前（配り分）・鍵・箱〉。

## 画面3 閉める計算と、開ける計算 — 行きは一瞬、帰りは壁。秘密の数だけが落とし戸を開く

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.calc.in | め「でも、ひとつ引っかかるの。錠前を配るなら」 | 配られた錠前が 1 つ、中央に大きく残る。目が見おろす |
| reverse.try | め「錠前から鍵を逆算されたら、おしまい」 | 錠前から鍵の輪郭（破線）へ逆算の矢印が伸びかけて止まる |
| lock.unveil | ず「そこで、錠前の正体を見るのだ」 | 金属面と鍵の輪郭を畳み、錠前の輪郭を額縁に残して空の盤面を描き起こす |
| lane.go | ず「ある向きには進みやすく、逆向きには」 | 上レーンに行きの矢印が引かれる |
| go.multiply | ず「たとえば、大きな素数をふたつ、掛け合わせる」 | 61 × 53 が上レーンを一瞬で流れ、3233 に届く |
| lane.back | ず「では逆に、掛けた答えだけを渡して」 | 3233 が下レーンへ降り、逆向きの矢印が引かれる |
| wall.up | ず「その総当たりは、とほうもない時間が」 | 下レーンに試し割り（÷2 ÷3 ÷5 …）の壁がせり上がり、3233 が止まる |
| contrast.hold | め「掛けるのは一瞬。もとに戻すのは」 | 行き＝短い一気の矢印、帰り＝壁、の対比で静止 |
| trapdoor.open | ず「受け手だけが知っている数があると」 | 秘密の数（ゴールド）が壁に当たり、落とし戸が開く |
| back.shortcut | め「その隠し扉を通れば、戻る計算が」 | 3233 が落とし戸を抜け、61 × 53 へすうっと戻る |
| door.hidden | ず「錠前を調べても、落とし戸の合言葉は」 | 落とし戸が閉じて壁に紛れ、目の視線は壁の前で止まる |
| digits.grow | ず「使う数の大きさは、時代に合わせて」 | 3233 が桁の長い数字列へ伸びる |

**誕生と死** — 錠前の金属面・鍵の輪郭：lock.unveil で死（輪郭は額縁へ転用）。盤面の数値・壁・戸：各 event で生まれ画面退場で死。同時表示の最大は〈目・額縁・盤面一式〉の 3。

## 画面5 結論 — 道を渡ったものの列に、鍵だけが無い

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.recap.in | ず「通信路ぜんたいを、もう一度たどってみる」 | 道・目・両端の台だけの素の舞台 |
| recap.cycle | ず「はじまりは、人目のある通信路」 | 薄い鍵と循環矢印が浮かぶ（堂々めぐり） |
| cycle.cut | ず「錠前と鍵を、べつべつの手の仕事に」 | 循環矢印が断ち切れ、錠前は道側へ、鍵は受け手の手元へ割れる |
| recap.spread | ず「公開鍵を配り……秘密鍵だけを手元に」 | 錠前が道を渡って並び、鍵は手元枠で灯る |
| recap.calc | め「錠前の中身は、片道の計算」 | 並んだ錠前の 1 つに行き（大）と帰り（壁）の矢印が透ける |
| recap.sign | ず「向きを逆さにすれば、送り主の署名にも」 | グリーンの灯を点した箱が右→左へ渡り、列に加わる |
| parade.align | め「……渡す必要は、はじめから無かったんだ」 | 道を渡ったものが道の上に一列に整列する。列の中に鍵だけが無い |
| roles.in | め「のぞかれてもいい錠前と、たった一つの鍵」 | 列に「配るもの」、鍵に「隠すもの」のラベル |
| title.final | ず「それが、公開鍵暗号のお話だったのだ」 | 題字「公開鍵暗号」が上部に灯り、静止して終わる |

**誕生と死** — 循環矢印：recap.cycle で生まれ cycle.cut で死。薄い鍵：cycle.cut で手元の鍵に転用。錠前列・箱・ラベル・題字：各 event で生まれ動画の終わりまで残る。同時表示の最大は〈舞台・錠前列（1 装置扱い）・箱・鍵〉。
