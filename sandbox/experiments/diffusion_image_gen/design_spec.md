# [3] セリフ単位スペック — diffusion_image_gen

各画面の先頭セリフに `scene.{画面名}.in`。装置の同時表示は各画面 3〜4 個以内。役目を終えた装置は次が立つ前に死なせる。

共通装置：**絵タイル**（格子状の一枚。同じ一枚が「絵→砂嵐→絵」と姿を変える背骨。各セルの色＝画像、グレーの粒＝ノイズ）、**ノイズ粒**（タイルに降る／取れる細かい点。ぼくたちが撒いた答えはアンバー）、**推測する箱**（ノイズを当てる・取るモデル。インディゴ）。

## 画面1：序論「砂嵐の中には、何も無い」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.intro.in` | ぜんぶ砂嵐から作られた | 中央に砂嵐状態の絵タイルが立ち上がる |
| `intro.resolve` | 砂嵐がだんだん猫に化けて | 横にプロンプト「猫」が点り、砂嵐が一枚の絵へ解けていく |
| `intro.carve` | 砂の中から猫を彫り出す | 砂嵐にノミが当たる「彫り出す」絵面（＝勘違い）が重なる |
| `intro.empty` | 砂嵐の中には、何も無い | ノミと隠れ猫の幻が消え、砂嵐がただの空の粒だと示される（勘違いに✗） |
| `intro.nodraw` | 絵の描き方なんて一度も | タイルに「描き方は未習得」の1語ラベル |
| `intro.name` | これが拡散モデル | 下部にタイトル「拡散モデル」 |

- 絵タイル：`scene.intro.in` 誕生、画面終端まで生存（背骨として持ち越す）。プロンプト「猫」：`intro.resolve` 誕生、`intro.empty` で一旦退場。
- ノミ＋隠れ猫の幻：`intro.carve` 誕生、`intro.empty` で死（✗）。同時最大＝タイル＋プロンプト＋ノミで3個。

## 画面2：ボディ1「まず、絵を壊す（答えはこっちが握る）」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.body1.in` | きれいな一枚の絵を用意 | 色のついたきれいな絵タイルが立つ |
| `b1.sprinkle` | ガウスノイズっていう | タイルにアンバーの粒が少し降りかかる |
| `b1.repeat` | これを何百回も | 降りかけが繰り返され、タイルが段々と崩れる |
| `b1.static` | この壊す手順を拡散過程 | 完全な砂嵐に。「拡散過程 / forward」ラベル |
| `b1.ours` | ぼくたち自身なのだ | 降った粒がアンバーで光り「撒いたのはこっち＝答えを握る」と示す |
| `b1.ask` | さっき足した点はどれだ | 中途半端な砂嵐タイルを推測する箱へ入れ、「足した点は？」の問い |
| `b1.nolabel` | ラベルは一切いらない | 「猫」ラベル札が出てすぐ✗で消える（正解ラベル不要） |
| `b1.grade` | その場で丸つけできる | 箱の答えとアンバーの正解を重ね、✓で丸つけ |
| `b1.free` | 練習問題と答えが、タダで | (砂嵐→答え) の対カードが次々に湧く |

- 絵タイル：`scene.body1.in` 誕生 → `b1.static` で砂嵐へ変身（消さずモーフ）→ `b1.free` で対カードの一枚へ縮む。
- アンバーの粒（答え）：`b1.sprinkle` 誕生、`b1.grade` まで生存、`b1.free` で死。推測する箱：`b1.ask` 誕生、`b1.free` で死。
- 同時最大＝タイル＋箱＋アンバー粒＋✓ で4個。`b1.nolabel` のラベル札は一拍で死に、数を増やさない。

## 画面3：ボディ2「一発では、戻せない（刻めば解ける）」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.body2.in` | 一気にきれいな絵へ戻せば | 左に重い砂嵐タイル、右に空の絵枠（戻し先） |
| `b2.guess` | くっきり出ると思うか | 砂嵐から絵枠へ一発の大矢印が引かれる（二択を問う） |
| `b2.blur` | もやっとした塊が出る | 大矢印の先に、ぼやけた塊が出る |
| `b2.many` | ぜんぶの平均を出す | 候補の絵が何枚も重なり、平均されてぼやけると示す |
| `b2.step` | ほんの一歩だけノイズを取る | 大矢印が消え、砂嵐がほんの一歩だけ晴れる小矢印に替わる |
| `b2.fog` | 濃い霧の中で道を探す | 霧が画面を覆い、次の一歩ぶんだけが見える |
| `b2.chain` | 一歩ごとに絵はくっきり | 小矢印が連なり、タイルが段々くっきりしていく |
| `b2.name` | これが逆拡散、サンプリング | 小矢印の連なりに「逆拡散 / sampling」ラベル |
| `b2.steps` | もとは千ステップ | 「1000 → 数十」の数値 |

- 重い砂嵐タイル：`scene.body2.in` 誕生、生存（`b2.step` 以降は晴れていく主役）。
- 大矢印＋ぼやけ塊＋候補の重なり：`b2.guess`〜`b2.many`、`b2.step` で死（小刻みに席を譲る）。
- 小矢印の連なり＋霧：`b2.step`〜`b2.chain` 誕生、生存。同時最大＝タイル＋（大矢印 or 小矢印連）＋ぼやけ／霧で 3〜4 個。

## 画面4：ボディ3「無から、転がす（コピーではなく生成）」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.body3.in` | まっさらな新しい砂嵐を用意 | 序論とは別の、新しい砂嵐タイルが立つ |
| `b3.norand` | 元になった一枚なんて | 「乱数・元は無い」の1語ラベル |
| `b3.roll` | 坂を下るように転がる | タイルが「絵らしさ」の坂を一歩ずつ下り、解けていく |
| `b3.newimg` | 見たことのない新しい絵 | 坂の底で、序論とは別の新しい絵になる |
| `b3.prompt` | プロンプトをささやく | バイオレットのプロンプト「猫」が坂の横に点る |
| `b3.bend` | 坂の傾きを、言葉で曲げる | プロンプトが坂の傾きを「より猫らしいほう」へ曲げる |
| `b3.memo` | そのまま出してるんじゃない | 「保存した写真をコピー？」の勘違い絵面が出る |
| `b3.scale` | 50億枚なのだ | 50億枚の画像の巨大な山と、小さなモデル箱を対比 |
| `b3.byte` | 1バイトも残らない | 山÷モデル＝「1枚あたり 1バイト未満」、コピー不能に✗ |
| `b3.gist` | 猫っぽさ」「毛の流れ | モデル箱が個々の写真でなく「猫っぽさ／毛の流れ」のコツを絞り出す |
| `b3.gen` | コピーじゃなく、生成 | そのコツで新しい猫が組み上がる。「生成 ≠ コピー」 |

- 砂嵐タイル＋坂：`scene.body3.in`〜`b3.bend` 生存、`b3.scale` で死（対比に席を譲る）。プロンプト：`b3.prompt` 誕生、`b3.bend` 後に薄める。
- 画像の山＋モデル箱：`b3.scale` 誕生 → `b3.byte` の✗ → `b3.gist` でコツを出す → `b3.gen` で新しい絵タイル復活。同時最大＝（タイル＋坂＋プロンプト）or（山＋モデル箱＋✗）で3個。

## 画面5：結論「無から、立ち上げる（大理石の反転）」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.outro.in` | 最初の気持ち悪さに | 砂嵐タイルと完成した絵タイルが左右に並ぶ |
| `outro.empty` | 彫り出したんじゃない | 砂嵐側を透かし、中に何も隠れていないと示す |
| `outro.onepower` | ちょっとだけマシにする | 「少しザラついた絵→少しマシな絵」の一歩だけの力を掲げる |
| `outro.rise` | 見たことのない絵が立ち上がる | 砂嵐から小さな一歩が何十も積み、絵が立ち上がる |
| `outro.marble` | 大理石の中に像が眠って | 砂嵐タイルが大理石の塊へモーフし、中に眠る像の幻 |
| `outro.invert` | 像なんて眠ってない | 大理石の中の像の幻が消え、空の塊になる（反転） |
| `outro.choose` | より像らしいほう」を選ぶ | 削る一打ごとに「より像らしいほう」を選ぶ印が積む |
| `outro.end` | 千の小さな選択が | 絵が完成し、隣の砂嵐は空のまま。締めの一文 |

- 砂嵐タイル：`scene.outro.in` 誕生 → `outro.marble` で大理石へモーフ → `outro.choose` で削られ像になる。完成絵タイル：`scene.outro.in` 誕生、生存。
- 眠る像の幻：`outro.marble` 誕生、`outro.invert` で死。同時最大＝砂嵐(/大理石)＋完成絵＋像の幻 で3個。
