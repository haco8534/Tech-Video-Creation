# zip_bomb — design overall

## 1. 世界観

### 形の語彙

- **指示書カード** — 角丸の白カード。zip 爆弾の「軽い側」を一貫して担う。圧縮データ・コピー指示・XML の合言葉など、抽象度を変えずに使い回す
- **実行タイル** — 一辺の揃った小角丸タイルが Grid 状に並ぶ。指示書を律儀に実行した「重い側」。zip 爆弾の 4.5 PB、Billion Laughs の 10 億回展開、ReDoS の計算量を同じ形で運ぶ
- **zip アイコン** — 角丸矩形＋上端の折り目。中央に「ZIP」表記。サイズはそのファイルの圧縮データ量を表す
- **参照矢印** — 細い実線＋三角頭。出生元から退場先へ向かって描き起こされる
- **窓枠** — 破線の角丸矩形。範囲（スライド窓 32 KB、参照範囲など）を示すときだけ使う
- **メーター** — 横長の進行バー。ディスク・メモリ・CPU の枯渇度

### 色の語彙

- 指示書・本体構造：`#1a5490`（落ち着いた紺）
- 実行された中身（膨らんだ側）：`#e07b39`（オレンジ）
- 防御・検査：`#2f9e44`（緑）
- 警告・上限・天井：`#d6336c`（マゼンタ）
- 補助：`#8a96a8`（グレー）

非対称の主軸は **紺（指示書／小）↔ オレンジ（実行／大）** で固定。マゼンタは「天井」「警戒」、緑は「防御」だけに割り当てる。

### 動きの語彙

- **count 駆動の増殖** — タイルが 1→N に増えていく。「実行が膨らむ」を表す中心の動き
- **指さし矢印の伸長** — 参照線が出生元から退場先へ `stroke-dasharray` で描き起こされる
- **横走査** — スライド窓・読み取りヘッドの左→右移動
- **メーターの上昇** — 枯渇の表現。一度上がったら止まる
- **連続変形（morph）** — 「ABCABCABC…」が「ABC + コピー指示」へ姿を変える等、作り直さずに姿を変える

常駐の揺れ・脈動は置かない。背景は静止し、起こる動きが意味を持って届く。

## 2. 図カタログ

### `inst_card`（指示書カード）

- **役** — 圧縮データ・コピー指示・XML の合言葉など「軽い指示書」全般。防御策の項目もこの形で並べる
- **描画区分** — HTML格納器
- **形** — 角丸 16px、白塗、紺の縁取り 3px、内部にラベル
- **サイズ感** — 主役なら横 320–420px、補助なら 220–280px
- **色** — 縁：主役紺（防御策のときは緑、上限のときはマゼンタ）。塗：白
- **構成要素** — 単一テキストを `<foreignObject>` の中央寄せで
- **登場画面** — 全画面

### `exec_grid`（実行タイルグリッド）

- **役** — 指示書を律儀に実行した結果の膨大な中身。zip 爆弾の 4.5 PB、その他攻撃の「実行量」も同じ形
- **描画区分** — SVG幾何要素
- **形** — 一辺 16–28px の小角丸タイルが Grid 配置
- **サイズ感** — 序論で主役として 700×400px、他画面では補助として小さめ
- **色** — タイル：実行オレンジ
- **構成要素** — Grid セル＝`<rect>`。count 駆動で左上から順に出現
- **登場画面** — 序論、ボディ2、結論

### `zip_icon`（zip ファイルアイコン）

- **役** — 圧縮ファイル本体。下に名前と KB ラベルが付くことがある
- **描画区分** — 合成（外枠が SVG、ラベル部が `<foreignObject>`）
- **形** — 角丸矩形、上端に折り目（小三角）、中央に「ZIP」表記
- **サイズ感** — 単独で出るときは 140×180px、グループで並ぶときは 80×100px
- **色** — 主役紺の縁、白塗
- **構成要素** — `<rect>`＋折り目`<path>`＋中央`<text>`、必要なら下に`<foreignObject>`のラベル
- **登場画面** — 序論、ボディ2

### `arrow_ref`（参照矢印）

- **役** — 指示書が「過去の自分」「同じ位置」を指す動作、または「展開する」流れ
- **描画区分** — SVG幾何要素
- **形** — 直線または曲線＋三角頭。線太 4–6px
- **サイズ感** — 端点間の派生計算
- **色** — 主役紺。登場時は `stroke-dasharray` の描き起こし
- **登場画面** — 全画面

### `window_frame`（窓枠）

- **役** — 範囲を示す枠。スライド窓 32 KB、上限の天井など
- **描画区分** — SVG幾何要素
- **形** — 破線の角丸矩形、線太 3px
- **サイズ感** — 内包要素 + 外側 40px
- **色** — マゼンタ（上限・警戒）または主役紺（中立）
- **登場画面** — ボディ1

### `meter_bar`（枯渇メーター）

- **役** — ディスク・メモリ・CPU の使用量
- **描画区分** — 合成（バーが SVG、ラベルが `<foreignObject>`）
- **形** — 横長 220×28px の角丸矩形、内部に塗りバーが左から伸びる
- **サイズ感** — `target_card` 内に 3 本縦並び
- **色** — 枠：グレー、バー：マゼンタ
- **構成要素** — 外枠 `<rect>` + 内部 `<rect>` + 左横にラベル
- **登場画面** — ボディ3

### `target_card`（自動展開する側のカード）

- **役** — アンチウイルス・メールゲートウェイ・アップロード API など「律儀に開ける側」の代表
- **描画区分** — HTML格納器
- **形** — 角丸 16px、白塗、補助グレー縁取り、内部に縦並びの見出し＋3本のメーター
- **サイズ感** — 360×320px
- **色** — 縁・見出し：グレー。中のメーターだけがマゼンタへ寄っていく
- **構成要素** — `<foreignObject>` の中に `<svg>` を再開せず、メーターは外側 SVG として重ねる（カード=HTML、メーター=SVG が同じ位置に重なる）
- **登場画面** — ボディ3

### `family_card`（攻撃家族カード）

- **役** — 結論の「zip 爆弾／Billion Laughs／ReDoS」を並べる比較カード
- **描画区分** — HTML格納器
- **形** — 角丸 16px、白塗、紺縁取り。内部に名前（大）と「指示書サイズ → 実行量」（中）
- **サイズ感** — 400×260px
- **色** — 主役紺の縁、INK 文字
- **構成要素** — `<foreignObject>` 内で VStack：見出し + 説明テキスト
- **登場画面** — 結論

## 3. 図の設計の原則

**背骨を1本通す。** 動画全体を「指示書 → 実行」という非対称の対比図が姿を変えて貫く一本の物語として組む。LZ77 のコピー指示も、再帰展開型のツリーも、Fifield の単発型も、すべて「軽い指示が重い実行を呼ぶ」の具体例として連なる。

**ひとつの図はひとつの仕組みを見せる。** 「この図は何を見せるか」を一言で言えるかを基準にする。

## 4. セクション別デザイン

### 序論：42 KB → 4.5 PB の正体

#### 眼目

42 KB のファイルが 4.5 PB に化けるのは圧縮率の魔法ではなく、「指示書は軽い、実行は重い」非対称の極限である、ということを絵で示す。

#### 舞台

```
画面 = VStack(gap=大) [
  シーンタイトル,
  ステージ = HStack(gap=極大, align=center) [
    指示書側 = VStack(gap=中) [ zip_icon(主役サイズ), inst_card「42 KB」 ],
    arrow_ref(右向き、太め),
    実行側   = VStack(gap=中) [ exec_grid(20×12), inst_card「4.5 PB」 ]
  ],
  強調 = inst_card「1000 億倍」（マゼンタ縁、大）,
  比較バー = HStack(gap=中) [
    inst_card「指示書: 軽い」,
    inst_card「実行: 重い」（オレンジ縁）
  ],
  字幕帯余白
]
```

最も混む瞬間は (g)。zip_icon（左、小）と exec_grid（右、大）が同じ HStack に並び、矢印が左→右へ。中央に「1000 億倍」、下段に「指示書／実行」比較バー。

#### 時間順の状態遷移

- (a) 「ねえずんだもん、これ見て」あたり：シーンタイトルが上端、ステージ HStack 上で zip_icon が中央付近にふわっと出る
- (b) 「サイズはたったの 42 キロバイト」あたり：zip_icon の下に inst_card「42 KB」が出る
- (c) 「4500 兆バイトなのだ」あたり：zip_icon の右側に arrow_ref が描き起こされ、その右に exec_grid が count 駆動で 0→240 タイルまで膨張、下に inst_card「4.5 PB」
- (d) 「1000 億倍」あたり：ステージの下に強調 inst_card「1000 億倍」がマゼンタ縁で出て、1拍スケール 1.0→1.15→1.0 と弾む
- (e) 「有名な zip 爆弾」あたり：zip_icon の上に「42.zip」名札が付く
- (f) 「指示書は軽い、実行は重い」あたり：「1000 億倍」が一段薄くなり、下に比較バー HStack（inst_card 2 枚）が滑り上がる
- (g) 「同じ文字を 1 億回書け」あたり：比較バーの「指示書」側カード内容が「同じ文字を 1 億回書け」に差し替わる（mix で連続変形）

#### 演出ノート

- グリッドの増殖は 60f ほどで一気に。視聴者に「うわ、増えた」と感じさせる
- 「1000 億倍」のマゼンタは「上限・警戒」の語彙の最初の登場。以降の画面でも同じ意味で再利用
- 避けたい：zip_icon と exec_grid を同時に揺らす。主役の対比を背景に薄める

### ボディ1：圧縮の正体——「コピー指示」を埋めこむ

#### 眼目

LZ77 は「過去の自分を指さす」だけで巨大な繰り返しを数バイトに収める。これが zip の中身（DEFLATE）の本体である、ということを絵で示す。

#### 舞台

```
画面 = VStack(gap=中) [
  シーンタイトル,
  ストリング段 = VStack(gap=小) [
    文字列タイル列 = HStack(gap=小) of 15 個の文字タイル「A B C A B C A B C A B C A B C」,
    inst_card「『3 文字前にもどって 3 文字、4 回コピー』」（コピー指示ラベル）
  ],
  オーバーレイ層: arrow_ref(from=後半12文字タイルの上端, to=先頭ABCの上端, 曲線),
  下段 = HStack(gap=中) [
    inst_card「LZ77 (1977)」,
    inst_card「DEFLATE = LZ77 + ハフマン」,
    inst_card「1段あたり 最大 1032 倍」（マゼンタ縁）
  ],
  字幕帯余白
]
```

最も混む瞬間は (g)。15 文字タイルが上段、その下に曲線矢印で「後半 12 文字 → 先頭 3 文字」を結び、コピー指示カードが矢印の終点近くに。下段に LZ77／DEFLATE／1032 倍上限の 3 カードが横並び。

#### 時間順の状態遷移

- (a) 「圧縮って、何をすることか」あたり：シーンタイトル、ストリング段の容器が薄く出る
- (b) 「ABCABCABCABCABC」あたり：15 文字タイルが左から count 駆動で出現
- (c) 「3 文字前にもどって 3 文字ぶんコピー」あたり：先頭 3 タイルが紺で強調、後半 12 タイルがオレンジ縁、曲線矢印が後半→先頭へ描き起こされ、コピー指示カードがフェードイン
- (d) 「LZ77 という呼び名」あたり：下段に inst_card「LZ77 (1977)」が左から滑り込む
- (e) 「ハフマン符号」あたり：真ん中に inst_card「DEFLATE = LZ77 + ハフマン」が追加
- (f) 「1032 倍」あたり：右端に inst_card「1段あたり 最大 1032 倍」がマゼンタ縁で追加、1拍強調
- (g) 直後：上段の文字列＋矢印が一段薄くなる（次画面で「1段の壁を越える」につなぐ）

#### 演出ノート

- 曲線矢印の描き起こしは 30f
- 上限カードのマゼンタは序論の「1000 億倍」と同じ語彙
- 避けたい：ハフマン木の細かい絵。ラベル合成だけにし、深入りしない

### ボディ2：一段の壁と、ふたつの突破口

#### 眼目

1 段の上限 1000 倍を超えるには「再帰展開型（古典）」か「単発型（Fifield）」しかない。両者の構造的違いを並べて示す。

#### 舞台

```
画面 = VStack(gap=中) [
  シーンタイトル,
  二択 = HStack(gap=大, align=top) [
    左カラム = VStack(gap=中) [
      inst_card「再帰展開型 / 1996」（見出し）,
      ツリー = VStack(gap=小) [
        zip_icon 1 個,
        HStack(gap=極小) of zip_icon × 8（残り 8 個は…で省略）,
        inst_card「× 5 段 = 約 100 万個」,
        exec_grid(8×6, 小)
      ],
      inst_card「→ 4.5 PB」（オレンジ縁、結果ラベル）,
      inst_card「深さ上限で防御済」（緑縁、対策タグ）
    ],
    右カラム = VStack(gap=中) [
      inst_card「単発型 / Fifield 2019」（見出し）,
      構造 = VStack(gap=小) [
        zip_icon「本文」,
        目次 = VStack(gap=極小) of 4 行の inst_card「ファイル A」「B」「C」「D」（細身）
      ],
      オーバーレイ層: 4 本の arrow_ref が「目次の各行 → 本文の同じ一点」へ収束,
      inst_card「10 MB → 281 TB」（オレンジ縁、結果ラベル）,
      inst_card「深さ 1 で対策をすり抜け」（マゼンタ縁）
    ]
  ],
  字幕帯余白
]
```

最も混む瞬間は (k)。左に再帰ツリー＋結果＋対策、右に単発型の本文・目次・収束矢印＋結果＋対策。両カラム末尾に結果ラベル（オレンジ縁）と対策タグ（左=緑・右=マゼンタ）。

#### 時間順の状態遷移

- (a) 「ここで、設計が二つに分かれる」あたり：左カラムの見出しと、右カラムの空っぽの容器が出る
- (b) 「1 個の zip の中にたとえば 16 個」あたり：左カラムに zip_icon 1 個、その下に 8 個の HStack が count 駆動で展開
- (c) 「これを 5 段重ねる」あたり：「× 5 段 = 約 100 万個」のカードが追加
- (d) 「ほぼ 4.5 ペタバイト」あたり：底の exec_grid が膨らみ、結果ラベル「→ 4.5 PB」がフェードイン
- (e) 「再帰的に展開する深さ」あたり：左カラム下端に緑縁「深さ上限で防御済」
- (f) 「David Fifield」あたり：右カラムの見出しが現れる
- (g) 「ぜんぶ同じ位置を指す」あたり：右カラムに本文 zip_icon と目次 4 行が出て、4 本の arrow_ref が同じ点に収束する描き起こし
- (h) 「281 テラバイト」あたり：右カラムに結果ラベル「10 MB → 281 TB」
- (i) 「深さの上限を 1 にしても、無傷で通り抜ける」あたり：マゼンタ縁「深さ 1 で対策をすり抜け」が出て一拍強調

#### 演出ノート

- 再帰ツリーの 5 段は省略カードで済ます。実際に 5 段並べない
- 単発型の 4 本の矢印は時間差で 1 本ずつ描き起こす。最後の 1 本が刺さった瞬間に小さなパルス（マゼンタの光輪）を 1 拍重ねる
- 避けたい：両カラムを同時にアニメさせる。視線が割れる

### ボディ3：律儀さが、引き金になる

#### 眼目

zip 爆弾は「データを壊す」のではなく「サービスを止める」攻撃で、防御の動作そのものが引き金になる。対処は「ちゃんとしすぎないこと」——上限と打ち切り。

#### 舞台

```
画面 = VStack(gap=中) [
  シーンタイトル,
  自動展開する側 = HStack(gap=中) [
    target_card「アンチウイルス」 内部に meter_bar × 3 (DISK / MEM / CPU),
    target_card「メールゲートウェイ」内部に meter_bar × 3,
    target_card「アップロード API」  内部に meter_bar × 3
  ],
  防御策 = VStack(gap=小) [
    inst_card「サイズ上限（例: 100 MB）」（緑縁）,
    inst_card「時間上限（例: 30 秒）」（緑縁）,
    inst_card「圧縮率しきい値（1000 倍超で隔離）」（緑縁）,
    inst_card「ストリーミング検査（展開しながら逐次判定）」（緑縁）
  ],
  字幕帯余白
]
```

最も混む瞬間は (e)。上段に 3 つの target_card（メーターはマゼンタで満タン）、下段に 4 つの防御策カードが縦並び。

#### 時間順の状態遷移

- (a) 「zip 爆弾を受け取って、いちばん困るのは誰」あたり：シーンタイトル
- (b) 「自動的に中身を全部展開する側」あたり：上段に 3 つの target_card が左から順にフェードイン（メーターは空）
- (c) 「CPU も食われる」あたり：全 9 本のメーターが同時に左→右へ伸び、最後はマゼンタで満タンになる
- (d) 「最大展開サイズ」あたり：下段の防御策 VStack に最初の 2 枚（サイズ上限・時間上限）が縦に積み上がる
- (e) 「圧縮率そのものを警戒の合図」あたり：3 枚目「圧縮率しきい値」が追加、1拍強調
- (f) 「展開しながら同時に検査」あたり：4 枚目「ストリーミング検査」が追加

#### 演出ノート

- メーターの伸長は 60f 程度で「ぐっと」上がる。一度満タンになったら止める
- 防御策カードはすべて緑縁。target_card はグレー縁（攻撃に晒される中立側）
- 避けたい：メーターを脈動させる

### 結論：律儀さを武器に変える攻撃の家族

#### 眼目

zip 爆弾は「指示書の軽さ／実行の重さ」を最大化する攻撃の家族の一員。Billion Laughs と ReDoS も同じ家族で、対処も同じ——「指示書を最後まで実行しない」。

#### 舞台

```
画面 = VStack(gap=中) [
  シーンタイトル,
  非対称の核 = HStack(gap=中, align=center) [
    inst_card「指示書: 軽い」,
    arrow_ref(右向き、太め),
    exec_grid(12×6, 小)
  ],
  家族 = HStack(gap=中) [
    family_card「zip 爆弾 / 42 KB → 4.5 PB」,
    family_card「Billion Laughs / 数 KB → 10 億回展開」,
    family_card「ReDoS / 短い正規表現 → 天文学的計算」
  ],
  結語帯 = inst_card「対処: 指示書を最後まで実行しない（上限 / 打ち切り）」（緑縁、太め）,
  字幕帯余白
]
```

最も混む瞬間は (d)。上段に非対称の核（序論の対比の縮小再現）、中段に 3 枚の家族カード、下段に結語帯。

#### 時間順の状態遷移

- (a) 「もう一度、最初の数字に戻る」あたり：上段に非対称の核が再登場（序論の主役を縮小して再現）
- (b) 「Billion Laughs ——」あたり：中段に 1 枚目「zip 爆弾」と 2 枚目「Billion Laughs」family_card がフェードイン
