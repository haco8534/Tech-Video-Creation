# zip_bomb — design overall

`script.md` を入力に、動画全体を貫通する世界観と、6 画面それぞれのデザインを自然言語で設計する。座標値・フレーム数・セリフ単位のタイミングは持たない。

図解は概念にラベルを貼るのではなく、仕組みを画面の上で実際に起こす。「LZ77」「展開」と書いた箱は出さない。**ちっぽけな紙片に書きつけた指示が、律儀に実行されると桁外れの実体に化ける**——その差そのものを画面に起こす。

## 背骨

貫通する背骨は **紙片と、それが律儀に実行されたときの実体の対**。画面の中央には常に二つのものが置かれる：左側に**指示書**（小さな紙片、ちっぽけ、書く側の労力で測ったらゼロに近い）、右側に**実行された結果**（積み上がる立方体、書き出される塊、膨らむ箱）。間には一本の作業の流れがあって、指示が左から右へ「律儀に」読まれて、結果が右で膨らみ続ける。

動画全体は、この対の非対称がどこまで開けるか、そして開けすぎたときに誰が焼かれるかを追う一本の物語になる。序論で「同じ文字を 1 億回書け」という1枚の紙片から積み木が桁外れに積み上がり、ボディ1ではその紙片の中身（コピー指示）がのぞき込まれ、ボディ2では1段ぶんの紙片に乗せられる量に天井があると分かり、紙片を入れ子にして掛け算する道（再帰型）と、紙片の目次を細工して同じ実体を何千ファイルに見せる道（単発型）に分かれる。ボディ3では右側で実体を実際に膨らませる「実行者」が現れ、自分の手で自分を焼く。結論では、左に紙、右に実行という同じ構図が zip 以外の家族（Billion Laughs、ReDoS）にも現れる。

個々の装置——コピー指示、目次、深さの上限、打ち切り線、家族の並び——は、すべてこの紙片と実体の対を加工する工程として連なる。

## 1. 世界観

| 軸 | この題材での設定 |
|---|---|
| 貫通する比喩 | **紙片と、それを律儀に実行した実体の対**。画面に常に左＝指示（小さな紙、薄い、軽そう）と右＝実行された結果（嵩のある立方体・箱・データ塊）が並ぶ。両者の間に「読み手」がいて、紙片を真面目に読み取って右側を膨らませている。視聴者は1本の動画を通じて、この左右の非対称がどう拡大されてゆくかを眺める |
| 形の語彙 | 専用装置の集まり（§3 図カタログ）。背骨は **紙片**（小さな白い四角、上に短い1行のコピー指示）と **立方体スタック**（実行で書き出された量を表す積み木、1個＝小さな単位、たくさん積むと量を表す）。装置：**コピー矢印**（紙片の中で「過去の自分を指す」円弧の矢印）／**スライド窓**（紙片の左に幅32KBのうすい窓枠）／**zip 構造の紙**（ローカルヘッダ・本体・巻末目次の3つに分かれた帯）／**目次の矢印**（巻末目次から本体への複数の矢印）／**実行者**（紙片を読んで右側に書き出す小さな顔のないアイコン、メモリ・CPU・ディスクの3バー付き）／**打ち切り線**（右側のスタックに引かれる赤い水平線）／**家族の並び**（zip／XML／正規表現の同じ「軽い指示・重い実行」構図の小カード3枚）。汎用の「ラベル付き箱」「意味のない矢印」「テキストだけのカード」は使わない |
| 色の語彙 | 背景・面は白テーマで固定（背景 `#f5f7fa`／パネル面 `#ffffff`／枠 `#c4cedd`／主文字 `#243044`）。アクセント：**紙片・指示の側**＝クリーム `#f5e9c4`（軽さ・書きつけた感じ）、**実行・膨らんだ実体・展開の側**＝赤橙 `#d9543c`（嵩・熱・量）、**圧縮の道具感（LZ77・コピー指示）**＝ティール `#2f8fb3`（仕組み側の色）、**防御・打ち切り・OK**＝グリーン `#3f9d57`、**警告・異常・攻撃の手口**＝アンバー `#d99a2b`、退いた要素・補助線＝グレー |
| 動きの語彙 | 使う＝フェード、紙片の登場（左から滑り込む）、紙片に文字が書き起こされる、コピー矢印が円弧でする と引かれる、立方体が下から積み上がる、立方体スタックが伸び続けて画面外へ走る、紙片が複製されて束になる、目次の矢印が同じ位置を指して何本も伸びる、実行者の3本バーが伸びてMAXに達して赤くなる、打ち切り線が水平に引かれてその上の積み木が灰色に落ちる。封印＝3D回転、カメラのズーム／パン、急な明滅、**ループし続ける動き（浮遊・振動・明滅・脈動・常駐背景モーション）**。装置はイベントごとに一度きり動き、終わったら静止する |
| 文字方針 | 画面下部に**対話字幕**を常駐（話者名は話者色、本文は濃い藍、スマホ視聴前提で大きめ・2行まで）。図解内のテキストは最小限：紙片に書かれるコピー指示の1行、サイズ数値（42KB／4.5PB／281TB／1032倍／100万個）、スライド窓の「32KB」、zip 構造の「ローカル／本体／目次」、実行者バーの「RAM／CPU／ディスク」、家族カードの「zip／XML／正規表現」 |
| ナビ補助 | 画面が切り替わるとき、左上に小さくセクション番号と見出し（例「01 紙片と、その実行」「02 圧縮の正体」「03 一段の壁、再帰で掛ける」「04 一段のままで、目次で水増し」「05 律儀さが、引き金になる」「06 同じ家族の攻撃」）。進捗ドット等は置かず最小限 |

レイアウトの基準：中心原点。画面下部 y≈330〜510 は対話字幕の帯として確保し、主オブジェクトはそれより上に置く。**画面ごとに、y 軸を上から「タイトル帯（上端）／主役の図（中央）／補足テキスト帯（中央下）／字幕帯（固定）」に分け、各テキストはその帯にしか置かない**——これで重なりは構図段階で機械的に避ける。テキストの位置と図の位置は名前付きアンカー定数を介して書き、矢印の始点・終点もすべてそのアンカーから derive させる。

## 2. セクション別デザイン

### 序論（画面1）：紙片と、その実行

- **絵の眼目（一言）**：ちっぽけな紙片から、桁外れの実体が湧き出している——スケールが極端に非対称
- **登場する図**：紙片（42KB のラベル付き、上に「同じ文字を 1 億回書け」と書かれる）、巨大な立方体スタック（4.5PB のラベル付き、画面外まで伸びる）、間の作業の流れ、ラベル「42.zip」「1000 億倍」、タイトル
- **レイアウト**：画面左 1/3 に小さな紙片（数値ラベル「42KB」を下に添える）、画面右 2/3 に立方体スタック（数値ラベル「4.5PB」「ハードディスク 4 万 5 千台ぶん」を下に添える）。両者の間に細い作業の流れ。立方体スタックは画面上端を突き抜けて続きを示す。タイトル「zip 爆弾」は画面上部の中央
- **動き**：まず紙片だけが現れる（42KB）→ 紙片の上に「同じ文字を 1 億回書け」と短い1行が書き起こされる → 間の作業の流れが右に伸び、右側に最初の立方体が一つ置かれる → 同じペースで2個、3個と積み上がり、ペースが上がって積み木の塔が画面上端に達し、その先も上に伸び続ける（4.5PB のラベル、ハードディスク 4 万 5 千台ぶんが添えられる）→ ラベル「1000 億倍」が紙片と立方体スタックを結ぶ線の上に灯る → 紙片の下に小さく「42.zip」のラベル → 比喩の差し替え：紙片の上に書かれた1行が「ABCABC...」のような圧縮指示の形に書き換わる → 紙片の脇に小さく「指示書」、立方体スタックの脇に小さく「実行」のラベルが浮かび、両者の比が動画の背骨であることが示される
- **画面分割の判断**：素朴な「圧縮率の魔法」観と「指示書と実行の非対称」観は時間軸で順に出す——魔法的に縮んでいる見え方を最初に提示してから、紙片＋実行の見え方に置き換える。左右に並べて対称配置にはしない

### ボディ1（画面2）：圧縮の正体——コピー指示と、1 段の天井

- **絵の眼目（一言）**：圧縮は「前の自分を指す」こと。1 段でできる膨張には 1032 倍という天井がある
- **登場する図**：紙片（中央の主役）、コピー矢印（紙片内の円弧の矢印）、スライド窓（紙片の左から右にずり動く幅 32KB の窓枠）、ハフマン符号の小タイル（よく出る文字に短いビット列）、ゼロ羅列の長尺紙片、258 文字の打ち止め印、1032 倍のラベル
- **レイアウト**：紙片を画面中央に大きく置く。紙片の中に文字列「ABCABCABCABCABC」が現れ、後半が圧縮された状態に書き換わる。スライド窓は紙片の左、ハフマン符号タイルは紙片の右下、ゼロ羅列の長尺と258打ち止めは紙片の中で連続表示。最終的に右に「1032 倍」のラベル
- **動き**：紙片に「ABCABCABCABCABC」が書き起こされる → 後半の 12 文字がコピー矢印（円弧）でまとめられ、「3 文字前にもどって 3 文字をコピー」を 4 回くりかえす指示に化ける → 紙片の脇に「LZ77（1977）」のラベルが灯る → スライド窓の枠が紙片の上をすっと滑り、窓のサイズ「32KB」のラベルが添えられる → 紙片の右下にハフマン符号タイル（「a」「q」のビット列の長さ差を示す）が現れて溶けこむ → 比較：「同じ文字 100 個」を、ハフマン単独と LZ77 で並べて見せ、LZ77 のほうが 10 倍以上短いことが棒の長さで示される（短時間の比較ビュー） → 紙片の上の指示が「ゼロを 1 個書いて、1 文字前から 258 文字コピー」を何度もくりかえす形に書き換わる → 258 の打ち止め印が連続して並ぶ → 紙片下に「最大 1032 倍」のラベルが灯る → 紙片の上に水平に「1 段の天井」と書かれた赤橙のラインが引かれ、これより上には1段では伸びないことが示される
- **画面分割の判断**：DEFLATE の3要素（LZ77・ハフマン・スライド窓）を縦に並べると対称配置になる。だから紙片の中の指示の書き換わり（時間順）として直列に見せる。比較ビューは短時間の挿入カットで、本流から外れない

### ボディ2a（画面3）：一段の壁を、再帰で掛ける

- **絵の眼目（一言）**：ひと粒の zip の中に同じ小粒が 16 個、5 段重ねで 100 万粒、各粒が 1 段で 4GB に膨らんで、合計 4.5PB
- **登場する図**：素朴な誤解（二重圧縮 → 縮まない）の短い比較、紙片（「再帰型」の本体、薄い色の zip アイコンの束）、入れ子の zip ツリー（1 → 16 → 256 → ... → 100 万）、底の 1 段紙片、立方体スタック（底 4GB が 100 万個ぶん積み上がる）
- **レイアウト**：左上に「素朴な誤解：二重圧縮」のミニ比較（圧縮済みは縮まないことを棒の長さで一瞬見せて消える）。中央に zip 入れ子ツリー（上から下へ枝分かれ、5 段、最終層は小点の群れ）。右に立方体スタック（底 4GB × 100 万）。下に「4.5PB＝1000 倍 × 100 万」の式
- **動き**：まず素朴な誤解の比較（左上）：紙片を二重に包んでも棒の長さが変わらない様子を一瞬出して、灰色に落として畳む → 中央に最初の 1 個の zip が現れる → 1 個から 16 個に枝が分かれる（1 段ほどく動作）→ 16 個それぞれからまた 16 個に分かれる、を 5 段重ねる。最終層は無数の小点に → 数値ラベル「16⁵ ≈ 100 万」が灯る → ツリーの最下層の小点のひとつから、右へ作業の流れが伸び、立方体スタック（4GB ぶんの積み木）が現れる → 立方体スタックが 100 万倍に増殖する動き：1 個の積み木の隣にコピーが現れ、全層に散らばって 100 万個ぶんの量塊になる（同じ実体への参照であることが「枝はあるが実体は1個」の見え方で示される。底の紙片は1枚のままハイライトされる）→ 右下に「100 万 × 4GB ≒ 4.5PB」の式 → 左下に小さく「対策：深さの上限」のラベルと、上限線（赤橙の水平線）が深さ 3 のあたりに引かれ、5 段重ねが線の上にはみ出している様子が示される（再帰型はもう引っかかる）

### ボディ2b（画面4）：一段のままで、目次で水増しする

- **絵の眼目（一言）**：1 つの圧縮データを、巻末の目次が「別ファイル」として何千回も指して、深さ 1 のまま 281TB に化ける
- **登場する図**：zip 構造の帯（ローカルヘッダ／本体（圧縮データ）／巻末目次 の 3 区画）、目次の矢印（巻末から本体への複数本の矢印、すべて同じ位置を指す）、特殊な切れ目の印（本体の中に複数の「ファイル終端」マーク）、展開された立方体スタック、ツールの堅さ表（3 ツール × 反応列の小表）、281TB のラベル、深さ「1」のラベル
- **レイアウト**：画面上部に zip 構造の帯（横長、左から「ローカル／本体／目次」）。本体の中に複数の小さな縦線（特殊な切れ目）。目次から本体への複数の矢印（同じ位置を指す）。下に展開された立方体スタック（281TB）と「深さ＝1」のラベル。右下にツール堅さの小表
- **動き**：zip 構造の帯が現れ、3 区画にラベル付け → 巻末目次から本体への矢印が 1 本伸びる（ファイル A）→ 2 本目、3 本目、4 本目……と矢印が増え、すべて本体の同じ位置を指している様子が見える → 本体の中に縦の切れ目が複数現れ、矢印の入り口が少しずつずれている → 矢印の本数が数千本に増え、目次の項目数も増殖 → 矢印の出口（実行側）から立方体が積み上がる：各矢印が「別ファイル」として独立に展開されるので、同じ実体から何千ファイル分の量が出てくる → ラベル「10MB → 281TB」が灯る → 上方向に「深さ＝1」のラベル、深さ上限の赤橙線（深さ 3 など）がはるか上にあって、その線をひっかけずに突き抜けている様子 → 右下にツール堅さの小表が現れ、各ツールがこの単発型をどう扱うか（エラー／律儀展開／途中停止）の 3 列が並ぶ。エラー＝グリーン、律儀＝アンバー、途中＝中間色

### ボディ3（画面5）：律儀さが、引き金になる

- **絵の眼目（一言）**：自動展開する側が、自分の手で自分を焼く——対策は「最後までやらない」
- **登場する図**：実行者（顔のないアイコン、RAM・CPU・ディスクの 3 本バー付き）、紙片（攻撃側の指示、変わらず小さい）、実行者の前に置かれた展開中の立方体スタック、3 種の自動展開者（アンチウイルス／メールゲートウェイ／ウェブサービス、横並びの小カード）、4 つの打ち切り装置（最大サイズの水平線／砂時計＝タイムアウト／圧縮率メーター／ストリーミングの流し）
- **レイアウト**：画面左に小さな紙片（変わらず）。画面中央に大きな実行者（3 本バー）。実行者の手前に展開中の立方体スタック。実行者の上に3 種の自動展開者カードが横並び（小さく）。実行者の右側に打ち切り装置の集まり（4 つを上下に並べる）
- **動き**：紙片（小）と実行者（大）が並ぶ → 実行者が紙片を読み始め、立方体が前に積み上がる → 実行者の RAM バーが伸びてMAXに達し赤くなる → 続いて CPU、ディスクの順にバーが伸びる → 実行者の顔のないアイコンが赤橙に染まる（自滅）→ 上の 3 種カード（アンチウイルス／メールゲートウェイ／ウェブサービス）が順に現れる：それぞれが「中を律儀に開ける」性質を持ち、紙片を読まされる立場であることが小アイコンで示される → 場面が落ち着き、右側に打ち切り装置が登場：最大サイズの赤橙水平線が立方体スタックの途中に引かれ、その上の積み木が灰色に落ちる → 砂時計が立方体スタックの脇に現れ、30 秒経過で停止 → 圧縮率メーター（針）が現れ、1000 倍を超えた地点でグリーンの隔離マークが灯る → 流しそうめんの樋のように展開された立方体が下流で次々検査されるストリーミングのカット → 4 つの装置はすべてグリーン色で「最後までやらない」ことを表す

### 結論（画面6）：同じ家族の攻撃

- **絵の眼目（一言）**：紙片＋実行という同じ構図が XML（Billion Laughs）にも正規表現（ReDoS）にもある。zip 爆弾はその家族の一員にすぎない
- **登場する図**：序論の対（紙片＋立方体スタック）の再帰画面、3 つの家族カード（zip／XML／正規表現、それぞれ「軽い紙片＋重い実行」の縮図つき）、最後の一文「律儀さを武器に変える攻撃」、共通の防御アイコン帯（上限・時間・深さ）
- **レイアウト**：上半分に序論の対（中央に小さく）。下に 3 つの家族カードを横並び（zip／XML／正規表現）。3 枚それぞれの中に、ミニチュアの「紙片→実行」が描かれる
- **動き**：序論の対が中央に小さく再登場（同じ非対称）→ その下に zip のミニカードがフェードイン（紙片＋立方体スタック）→ 隣に XML の Billion Laughs のミニカード（「lol」の入れ子の紙片 → 「haha」の塊 10 億個）→ 隣に正規表現の ReDoS のミニカード（短い正規表現の紙片 → 砂時計が永遠に回る）→ 3 枚のカードの上に「同じ家族」の括弧が引かれる → カード群の下に共通の防御アイコン帯（最大サイズの水平線／砂時計／深さの上限）が灯る → 序論の紙片＋立方体スタックがゆっくりフェードアウトし、画面中央に「律儀さを武器に変える攻撃」の最後の一文が残って終わる

## 3. 図カタログ

| 図 | 見せる仕組み | 基本の見た目 | 登場画面 |
|---|---|---|---|
