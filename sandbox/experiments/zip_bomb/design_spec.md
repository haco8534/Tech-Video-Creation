# [3] セリフ単位スペック — zip_bomb

各画面の先頭セリフに `scene.{画面名}.in`。装置の同時表示は各画面 3〜4 個以内に収め、役目を終えた装置は次が立つ前に死なせる。

共通装置：**指示カード**（小さな角丸カード。圧縮ファイルそのもの。インディゴ）、**展開バー**（カードから右へ伸びる横帯。展開後のデータ。アンバー→レッドで重さを表す）。この対は全画面に通底する背骨。

## 画面1：序論「軽い指示・重い展開」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.intro.in` | これ、たった42キロバイト | 中央に小さな指示カード（42 KB）が立ち上がる |
| `intro.reveal` | 4.5ペタバイトに化ける | カードから展開バーが右へ一気に伸び、画面右端で切れて外へ溢れる（4.5 PB） |
| `intro.scale` | ギガの百万倍 | 溢れたバーの上に「4.5 PB」「= GB ×100万」の数値が乗る |
| `intro.crush` | ディスクを食いつぶして潰れる | バーの先で開いた側のディスク円盤がひしゃげ、赤に転ぶ |
| `intro.name` | これがzip爆弾 | カードに「zip爆弾」ラベルが付く |
| `intro.weight` | 指一本ぶんも重くない | カードは羽のように軽い印、バー側だけ重い印。軽/重の天秤が傾く |

- カード：`scene.intro.in` 誕生、画面終端まで生存（背骨として次画面へ持ち越す）。
- 展開バー：`intro.reveal` 誕生、生存。ディスク：`intro.crush` 誕生、`intro.weight` で天秤に席を譲り死。天秤：`intro.weight` 誕生、画面退場で死。

## 画面2：ボディ1「圧縮＝指示書」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.body1.in` | ふつうのファイルがどうやって小さくなるか | 「あ」が千個並ぶべた塗りの長いバーが出る |
| `b1.collapse` | あ、を千回」と書けば | 長いバーが、矢印一本＋「×千」の短い指示カードへ畳まれる |
| `b1.instr` | データじゃなく、指示書なのだ | 畳んだカードに「指示書」ラベル。中身でなく命令だと示す |
| `b1.backref` | 直前の並びを、また千回 | ひと並びの種＋自分自身へ戻る後方参照の弧。元データはひと並びだけ |
| `b1.lz77` | LZ77と呼ぶのだ | 後方参照の仕掛けに「LZ77 / DEFLATE」ラベル |
| `b1.monotone` | 中身が単調なほど、爆弾は軽くなる | バーを一色のべた塗りに均すほど、指示カードが縮む |

- 長いバー：`scene.body1.in` 誕生 → `b1.collapse` で畳まれ指示カードへ変身（消さずモーフ）。
- 後方参照の弧：`b1.backref` 誕生、生存。LZ77ラベル：`b1.lz77` 誕生。同時表示はカード＋バー＋弧＋ラベルで4個以内。

## 画面3：ボディ2「一枚では爆弾になれない／入れ子と検知」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.body2.in` | 指示書ひとつには、ちゃんと天井がある | 指示カード1枚＋伸びかけの展開バー |
| `b2.ceiling` | 千三十二倍までしか | バーの上に「×1032」の定規が現れ、バーがそこで頭打ちに止まる |
| `b2.small` | 43メガちょっと | 頭打ちのバー脇に「42 KB → 43 MB」。爆弾にならない小ささ |
| `b2.nest` | 指示書のなかに、また指示書 | カードの中にカード、その中にカード…と五段の入れ子がズームで開く |
| `b2.zip42` | 42.zipに化ける正体 | 入れ子が「×16 を5段 = 約100万倍」を経て「42.zip 4.5 PB」へ |
| `b2.tail` | 開けても開けても、また圧縮ファイル | 入れ子を開く動作が繰り返され、毎回また圧縮ファイルが出る尻尾 |
| `b2.detect` | もう開かない」。検査する側 | 検査のレンズが尻尾を捉え、赤✗で入れ子を止める |

- 展開バー＋定規：`scene.body2.in`〜`b2.small`、`b2.nest` で死（入れ子に席を譲る）。
- 入れ子カード群：`b2.nest` 誕生 → `b2.tail` で尻尾の反復 → `b2.detect` で赤✗。検査レンズ：`b2.detect` 誕生。同時最大＝入れ子＋尻尾印＋レンズ＋✗で4個以内。

## 画面4：ボディ3「重ねずに重ねる（overlapping）」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.body3.in` | 2019年に、フィフィールド | 一段だけのzip：上に目次の項目列、下にデータ領域 |
| `b3.toc` | 場所を指す目次なのだ | 目次の各項目から、データ領域へ伸びる指し矢印 |
| `b3.normal` | 別々の中身を指すのだ | ふつう＝各矢印が別々のデータ塊を指す（散らばる） |
| `b3.overlap` | ぜんぶ同じひとかたまりに向けた | 全項目の矢印が、一つのデータ塊へ束ねて重なる |
| `b3.expand` | そのまま千倍に展開される | 一塊が、項目数ぶん（×千）の展開バーへ膨らむ |
| `b3.evade` | 検査の身構えを、すり抜ける | 層は一枚のまま＝尻尾なし。検査レンズが素通り、緑✓ |
| `b3.power` | 46メガで4.5ペタを生む | 「46 MB → 4.5 PB・一段」の数値。42.zipと同威力 |

- 目次＋データ領域：`scene.body3.in` 誕生、生存。指し矢印束：`b3.toc`〜`b3.overlap` で散→束へモーフ。
- 展開バー：`b3.expand` 誕生（背骨のバー再登場）。検査レンズ＋✓：`b3.evade` 誕生。同時＝目次/データ＋矢印＋バー＋レンズで4個、レンズ登場時に矢印は薄める。

## 画面5：結論「軽さが武器・ウイルスへ」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.outro.in` | もう一度、爆弾の正体 | ほぼ空の指示カードが中央に戻る |
| `outro.recap` | 短い指示だけなのだ | カードの中身は一行の短い指示だけ、と示す |
| `outro.burn` | 自分の資源を焼く | カードから伸びる指示が、開いた側の資源を焼く（赤） |
| `outro.virus` | ウイルスは、自分だけでは増えない | カードがウイルス（カプシド）へモーフ、細胞へ近づく |
| `outro.host` | 細胞が、律儀に読んで増やす | 細胞が指示を読み、ウイルスを律儀に量産する |
| `outro.loopback` | その真面目さが、そのまま弱点 | 「律儀に実行する側」に弱点の印。攻/守の非対称を再掲 |
| `outro.end` | いちばん軽いものが、いちばん重い結果 | 軽いカードと重い展開バーの天秤が極端に傾き、締めの一文 |

- 指示カード：`scene.outro.in` 誕生 → `outro.virus` でカプシドへモーフ。細胞：`outro.virus` 誕生。天秤：`outro.end` 誕生。同時表示は局面ごとに3個以内（recap期＝カードのみ、virus期＝カプシド＋細胞、end期＝カード＋バー＋天秤）。
