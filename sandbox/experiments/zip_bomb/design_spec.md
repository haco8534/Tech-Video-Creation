# zip 爆弾 — セリフ単位スペック

背骨は「指示書（軽い）↔ 実行された体積（重い）」の対比。各画面はこの一枚を加工する。

## 語彙

| 役割 | 色 | 形 |
|---|---|---|
| 指示書・圧縮（軽い／親切） | ティール | 折り角のある小さなカード |
| 実行された体積・脅威（重い） | コーラル赤 | 積み上がるブロックの塊 |
| 数値・倍率・天井 | アンバー | 縦ゲージ／数字タイル |
| 防御・打ち切り | グリーン | 直線の打ち切り線／停止記号 |

展開は左（指示書）→右（体積）の矢印で起こす。同じ「カード→塊」の対が全画面に出る。

---

## 画面1 — 序論

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| `scene.intro.in` | 画面にファイルがひとつ | 小さな指示書カードが 1 枚現れる |
| `intro.size` | たったの 42 キロバイト | カードに「42 KB」が刻まれる |
| `intro.expand` | 中身を全部展開すると | カードから右へ巨大ブロックの塊が膨れ上がる |
| `intro.ratio` | 圧縮率にすると | 塊の脇に「4.5 PB／1000 億倍」が立つ |
| `intro.name` | 「42.zip」と呼ばれる | カードに「42.zip」の名が付く |
| `intro.asym` | 計算の非対称なのだ | カード＝軽い／塊＝重い、として両者に重み記号が付く |
| `intro.paper` | 同じ文字を 1 億回書け | カードが「1 億回書け」の紙に一度だけ言い換わり、塊が再生される |

同時表示：指示書カード／実行ブロックの塊／展開の矢印（＋数値ラベル）= 3 装置。

誕生と死：すべて画面退場（`scene.comp.in`）で死ぬ。

## 画面2 — 圧縮の正体

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| `scene.comp.in` | 正確に説明できる人は案外少ない | 文字列の帯が現れる |
| `comp.string` | ABCABCABC | 帯が「ABCABCABCABCABC」になる |
| `comp.fold` | 3 文字前にもどって | 繰り返し部が畳まれ、コピー指示カード（戻り矢印つき）になる |
| `comp.lz77` | LZ77 という呼び名 | カードに「LZ77」の名が付く |
| `comp.deflate` | 内部で使っている DEFLATE | カードに「DEFLATE」が重なる |
| `comp.zeros` | ゼロが 10 億バイト並んだ | 帯が「0」の列に変わり、1 枚のカードへ畳まれる |
| `comp.ceiling` | おおよそ 1032 倍 | 右に倍率ゲージが立ち、天井「1032×」で頭打ちになる |
| `comp.wall` | 1 段ではせいぜい 1000 倍 | ゲージの天井が「1 段の壁」として強調される |

同時表示：文字列／コピー指示カード／倍率ゲージ = 3 装置。文字列はカードへ畳まれた後は退く。

誕生と死：文字列は `comp.fold` で畳まれて退場。カードとゲージは画面退場（`scene.recur.in`）で死ぬ。

## 画面3 — 再帰型と、その封じ

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| `scene.recur.in` | どうやって作るのか | 中央に 1 段の壁ゲージ（1000×）が残って現れる |
| `recur.double` | 二重圧縮、三重圧縮 | zip の中に zip を入れた殻が重なるが、ほとんど縮まない |
| `recur.random` | 見た目はランダムに近い | 二重殻が「効かない」として薄く沈む |
| `recur.nest` | 16 個の同じ zip | 殻が消え、1 個から 16 個へ枝分かれする入れ子ツリーが生える |
| `recur.stack` | これを 5 段重ねる | ツリーが 5 段に伸び、各段で 16 倍に分岐する |
| `recur.peta` | ほぼ 4.5 ペタ | 葉の総数「約 100 万個 ×1000 倍 ＝ 4.5 PB」が立つ |
| `recur.depth` | 再帰的に展開する深さ | ツリーに展開深さの打ち切り線が引かれる |
| `recur.obsolete` | 現代の主流の対策にはもう引っかからない | 線より下が灰色に沈み、ツリー全体に「封じ済み」が掛かる |

同時表示：壁ゲージ→二重殻→入れ子ツリー＋深さ打ち切り線。二重殻はツリー登場前に沈む。最大 3 装置。

誕生と死：二重殻は `recur.nest` で死ぬ。ツリーは画面退場（`scene.single.in`）で死ぬ。

## 画面4 — 単発型

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| `scene.single.in` | もう一つの設計 | 横長の zip の帯（本文＋巻末）が現れる |
| `single.oneshot` | 1 回展開しただけで | 入れ子なしの 1 段だと示される |
| `single.fifield` | David Fifield | 帯に「Fifield, 2019」が付く |
| `single.struct` | セントラルディレクトリ | 帯の末尾に目次ブロックが分かれて立つ |
| `single.tail` | 巻末の目次のほうを優先 | 読み取りの視線が巻末の目次へ向かう |
| `single.point` | ぜんぶ同じ位置を指す | 目次の複数エントリが、すべて同じ 1 つの実体へ矢印を伸ばす |
| `single.multi` | 何度も何度も書き出して | 1 つの実体が右へ何度も書き出され、ファイルの山になる |
| `single.tera` | 10 メガバイトのファイルから | 山の脇に「10 MB → 281 TB」が立つ |
| `single.depth1` | 深さの上限を 1 にしても | 深さ打ち切り線が引かれるが、1 段なので素通りする |

同時表示：zip 帯／目次エントリ群＋同一実体への矢印束／書き出されたファイルの山 = 最大 3 装置。

誕生と死：zip 帯と目次は画面退場（`scene.trigger.in`）で死ぬ。

## 画面5 — 律儀さが、引き金

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| `scene.trigger.in` | いちばん困るのは誰 | 中央に展開する検査機（歯車）が現れ、指示書カードを呑む |
| `trigger.human` | ディスク容量が足りません | 人の手が「容量不足」で止まる（深刻にならない側）として薄く退く |
| `trigger.auto` | 自動的に中身を全部展開する側 | 検査機が膨らむブロックを律儀に展開し始める |
| `trigger.resource` | どれが先に音を上げるか | 検査機の脇にディスク・メモリ・CPU の 3 本バーが伸び、満杯へ近づく |
| `trigger.dos` | サービスを止める攻撃 | 検査機が固まり「停止」になる |
| `trigger.who` | アンチウイルスソフト | 検査機の入口に検査する側の役割が順に灯る |
| `trigger.irony` | その防ごうとする動作で自滅 | 防ごうとした動作そのものが自滅、と検査機が赤く沈む |
| `trigger.cut` | 展開を最後までやりきらない | 膨張の途中にグリーンの打ち切り線が入る |
| `trigger.limit` | 最大展開サイズ | 打ち切り線に「サイズ・時間・深さの上限」が添う |
| `trigger.ratio` | 圧縮率そのものを警戒の合図 | 入口に圧縮率メーターが付き、振り切れたら隔離になる |

同時表示：検査機／膨張ブロック／資源バー／打ち切り線 = 最大 4 装置。資源バーは打ち切り線が入る頃に役を終える。

誕生と死：資源バーは `trigger.cut` 後に退場。検査機と打ち切り線は画面退場（`scene.outro.in`）で死ぬ。

## 画面6 — 結論

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| `scene.outro.in` | もう一度、最初の数字 | 背骨の対比（小さなカード↔巨大な塊）が中央に戻る |
| `outro.kindness` | 送る側の気遣いの技術 | カードがティールに灯り「やさしさ」として立つ |
| `outro.weapon` | 狙う武器に化けた | カードがそのまま反転し、塊が受け手側を焼く向きになる |
| `outro.family` | Billion Laughs | 下に同じ非対称の家族として小アイコンが 1 つ並ぶ |
| `outro.redos` | ReDoS という攻撃 | 家族にもう 1 つ（ReDoS）が並び、zip 爆弾と横並びになる |
| `outro.common` | 指示書を最後まで実行しない | 家族すべてに共通の防御「最後まで実行しない」が掛かる |
| `outro.end` | 律儀さを武器に変える攻撃 | 締めの一文が立つ |

同時表示：背骨の対比図／家族の小アイコン 3 つ（横並び）= 2 装置群。

誕生と死：最後の画面なので死なない。
