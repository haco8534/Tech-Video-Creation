# セリフ単位スペック — 【視覚的に理解する】チェスAIはどうして世界1位を倒せるのか

`design_overall.md` の背骨（チェス盤から未来の木が伸び、評価の光で枝が刈られ、学習ループで評価の目が育つ）を、event に分解して `script.md` のセリフへ紐付ける。

## 画面 1: intro

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.intro.in | め「チェスAIって、もう人間の世界チャンピオンより強いんでしょ？」 | 中央に8x8のチェス盤、上に王冠アイコン、右に `AI` の黒い分析ボックスが現れる |
| board.same | め「盤面は同じ8かける8で、駒の動きも人間と同じ」 | 人間側とAI側の同じ盤面が左右に並び、駒の動き矢印が一致する |
| brute.false | め「全部読んでるんじゃないの？」 | 盤面から全方向に枝が一気に伸び、すぐ画面外で赤く詰まる |
| three.devices | ず「チェスAIの中には、大きく三つの装置がある」 | AIボックスが開き、`探索` `評価` `学習` の三装置が縦に並ぶ |
| question.pin | ず「読めないほど広い未来から、勝ちにつながる一手を選んでいるのか」 | 盤面の中央に大きな問いピンが立ち、三装置へ線がつながる |

## 画面 2: explosion

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.explosion.in | ず「チェスがどれくらい大変なゲームかを見る」 | 盤面だけが大きく表示され、合法手候補が小さな矢印として周囲に出る |
| moves.count | ず「一つの局面で、合法手はだいたい数十個ある」 | 矢印の横に `候補 数十手` のカウンターが灯る |
| tree.grow | ず「相手にも数十個の返しがある。さらにその次も数十個」 | 一手、二手、三手と木が階層ごとに増殖し、画面奥へ広がる |
| tree.overflow | ず「未来の木は一気に巨大になる」 | 枝が画面端まであふれ、遠くが霧で見えなくなる |
| god.false | め「AIも、全部わかってる神様ではないのね」 | 木の上に出た全知の目アイコンが薄くなり、取り消し線で消える |
| score.need | ず「途中の局面を見て『これは良さそう』と判断する力」 | 木の枝先に小さな評価メーターが付き、良い枝だけが明るく残る |

## 画面 3: search

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.search.in | ず「探索とは、今の局面から候補手を出して、その先を読む」 | 未来の木が整理され、左に自分の手、右に相手の返しとして交互に色分けされる |
| minimax.in | ず「自分は一番得する手を選び、相手は一番嫌な手を選ぶ」 | 自分ノードは上向き矢印、相手ノードは下向き矢印になり、最悪返しの枝が強調される |
| alpha.beta | ず「alpha-beta という枝刈りを使う」 | もう選ばれない枝にハサミ型の光が入り、枝が短く切られる |
| prune.safe | ず「論理的に不要になった場所を切る」 | 切られた枝の先に薄い `読まない` ラベルが出て、残った枝の密度が下がる |
| order.reuse | ず「良さそうな手を先に読む順番づけ、同じ局面を再利用する表」 | 残った枝が優先順位の番号で並び、同一局面ノードが小さな表に吸収される |
| stop.edge | ず「どこかで読むのを止める必要がある」 | 木の深さに水平の停止ラインが引かれ、枝先が評価待ちの丸に変わる |

## 画面 4: evaluation

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.evaluation.in | ず「評価関数は、局面を点数に変える装置」 | 停止ライン上の局面が中央の評価器へ流れ、右から点数バーが出る |
| handcrafted.in | ず「駒の価値、キングの安全、ポーンの形」 | `駒得` `王の安全` `ポーン構造` `駒の働き` の札が評価器に差し込まれる |
| deepblue.badge | ず「Deep Blue も、強力な探索と、人間の知見を入れた評価関数」 | 評価器の横に `Deep Blue型` の古い金属バッジが一瞬点灯する |
| hidden.good | ず「採点表にない良さを見落とす」 | 札では低評価の枝の奥に、後から効く圧力の光が見えて保留される |
| nnue.in | ず「現代の Stockfish は、NNUE という軽いニューラルネット評価」 | 評価器がニューラルネットの小さな層へモーフし、盤面特徴が数字粒として入る |
| fast.update | ず「一手で盤面の一部だけが変わることを利用して、高速に更新」 | 動いた駒の周辺だけが光り、評価器全体ではなく一部のメーターだけが差分更新される |
| score.output | ず「探索で使える点数に変換する目」 | 点数バーが未来の木の枝先へ戻り、良い枝が太くなる |

## 画面 5: alphazero

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.alphazero.in | ず「AlphaZero が面白いのは、人間の棋譜や定跡を中心に覚えたわけではない」 | 定跡本アイコンが横へ退き、空の盤面と `ルールだけ` ラベルが残る |
| selfplay.loop | ず「自己対局を繰り返す」 | AI同士の小さな盤面が円環状に並び、勝敗チップが中央へ集まる |
| update.net | ず「勝った局面、負けた局面、良かった手を材料にして、ニューラルネットを更新」 | 勝敗チップがネットワークへ入り、重みの点が少し配置を変える |
| policy.value | ず「一つは policy、もう一つは value」 | ネットワークから `policy: 有望手の分布` と `value: 勝ちやすさ` の二本の出力が出る |
| mcts.guide | ず「policy が読む枝の優先順位を決め、value が枝先の良し悪しを見積もる」 | 未来の木に戻り、policy が枝を照らし、value が枝先へ点数を付ける |
| visits.gather | ず「有望そうな枝へ探索回数を集める」 | 探索チップが明るい枝へ多く流れ、薄い枝には少しだけ流れる |
| intuition.verify | め「直感で候補を絞って、探索で裏を取る」 | `直感` の光と `検証` の光が同じ枝で重なり、最終候補が一手に収束する |

## 画面 6: human

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.human.in | め「人間のトップ棋士だって、何十年も勉強してる」 | 左に人間の思考雲、右にAIの未来の木が並ぶ |
| human.filter | ず「人間は、経験と直感で候補手をかなり絞る」 | 人間側では候補手が3本程度に整理され、太い直感ラインになる |
| ai.verify | ず「AIはお腹が空かない」 | AI側では多数の候補に小さな探索チップが規則正しく流れ続ける |
| weird.move | ず「気まずい手を避けない」 | 人間側で薄くされた変な手が、AI側では点数上昇で明るく復活する |
| pressure.future | ず「長期的な圧力、キングの安全、逃げ道の少なさ」 | 盤面上に相手キングを囲う圧力線、逃げ道の閉鎖マークが順に出る |
| worlds.split | ず「人間は意味で盤面を見る。AIは数値と探索で見る」 | 人間側の言葉ラベルとAI側の数値ヒートマップが同じ盤面に半分ずつ重なる |

## 画面 7: loop

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.loop.in | ず「本当にすごいのは、改善ループを作ったこと」 | 盤面、探索木、評価器、学習器が円形に配置される |
| loop.steps | ず「候補手を出す。探索する。評価する。対局結果で学習する」 | `候補` `探索` `評価` `対局` `学習` の矢印が順に点灯して一周する |
| stronger.teacher | め「強くなった自分が、次の先生になる」 | 円環の外側に一回り大きいAIが重なり、次の自己対局盤面へ戻る |
| stockfish.test | ず「Stockfish も、分散テストで小さな改善を大量に検証」 | 無数の小さなテスト盤がグリッド状に点灯し、勝率の高い変更だけが評価器へ入る |
| loop.accelerate | ず「探索、評価、学習、テストが回り続ける仕組み」 | 円環が一周ごとに少し太くなり、中央の強さメーターが上がる |
| thesis.lock | め「賢い評価器を持った探索マシンが、無限に近い練習で磨かれる」 | 未来の木、評価器、学習ループが一本の太い構図に合体して静止する |

## 画面 8: outro

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.outro.in | ず「最初の問いは、読めないほど広い未来から勝ちにつながる一手を選ぶのか」 | 導入の盤面と問いピンが戻る |
| recap.search | ず「探索で枝を伸ばし、不要な枝を刈り」 | 未来の木が伸び、枝刈りが一筆で再生される |
| recap.eval | ず「評価関数で途中の局面を点数化する」 | 枝先に点数バーが灯り、良い枝だけが太くなる |
| recap.generations | ず「Deep Blue、AlphaZero、現代の Stockfish」 | `Deep Blue: 探索+手作り評価` `AlphaZero: 自己対局+policy/value` `Stockfish: alpha-beta+NNUE` の三層カードが重なる |
| map.fold | ず「未来の木、局面を点数化する目、勝敗から自分を直すループ」 | 木、評価の目、学習円環が一枚の地図へ畳まれる |
| final.answer | ず「探索と評価と学習のループで、勝ちに近い形へ圧縮できるから」 | 地図から最終候補の一手だけが盤面へ戻り、矢印が静止する |

## 各オブジェクトの誕生と死

| オブジェクト | 誕生 | 死 |
|---|---|---|
| 8x8チェス盤 | scene.intro.in | final.answer まで全体の土台として生存 |
| 王冠アイコン | scene.intro.in | three.devices |
| AI分析ボックス | scene.intro.in | three.devices で三装置へ展開、outro で地図へ統合 |
| 三装置ラベル（探索・評価・学習） | three.devices | scene.explosion.in で一度退場、loop で円環として復帰 |
| 未来の木 | brute.false | final.answer で一手へ収束 |
| 全知の目アイコン | god.false | god.false 内で即退場 |
| 評価メーター／点数バー | score.need | final.answer まで枝先の評価として継続 |
| minimax の自分／相手ノード | minimax.in | stop.edge |
| 枝刈りハサミ | alpha.beta | prune.safe 内で短命に退場 |
| 置換表 | order.reuse | scene.evaluation.in |
| 手作り評価札 | handcrafted.in | nnue.in |
| Deep Blue型バッジ | deepblue.badge | hidden.good |
| NNUE評価器 | nnue.in | loop で評価器として継続、map.fold で地図へ統合 |
| 自己対局円環 | selfplay.loop | update.net で学習器へ吸収、loop で復帰 |
| policy / value 出力 | policy.value | mcts.guide で探索木へ接続、scene.human.in |
| 探索チップ | visits.gather | intuition.verify |
| 人間の思考雲 | scene.human.in | worlds.split |
| 数値ヒートマップ | worlds.split | scene.loop.in |
| 改善ループ円環 | scene.loop.in | map.fold |
| 分散テスト盤グリッド | stockfish.test | loop.accelerate |
| 世代カード（Deep Blue / AlphaZero / Stockfish） | recap.generations | map.fold |

同時表示の最大は画面5の「盤面＋未来の木＋policy/value＋探索チップ」、画面7の「盤面＋探索木＋評価器＋学習器」で4個以内に収める。
