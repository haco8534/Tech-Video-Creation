# セリフ単位スペック — LLMは、なぜ「次の言葉を当てるだけ」で賢く見えるのか

`design_overall.md` の背骨（文の帯＋空きスロットの予測装置／背後に育つ地図）を、event に分解して `script.md` のセリフに紐付ける。

## 画面 1: intro

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.intro.in | め「LLMってさ、よく『次の言葉を予測してるだけ』って説明される」 | 文の帯（数トークン）＋右端の空きスロットが現れる |
| strip.fill | め「でもそれでコードを書いたり、文章を直したり」 | 候補語チップが3つ湧き、最有力がスロットに滑り込む |
| strip.chain | ず「その説明は間違いではないけど、薄すぎるのだ」 | 埋まるたび帯が伸びて新スロットが開き、数語ぶん連鎖する |
| strip.question | ず「なぜ、文章の続きを当てる練習をしているだけで」 | スロットが拡大し「？」が灯って静止（今日の問い） |

## 画面 2: deep

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.deep.in | ず「まず、『次の言葉を当てる』を軽く見ないほうがいい」 | 帯が雨の例文「今日は 雨が 降って いるので 傘を ▢」へモーフ |
| rain.fill | ず「次は『持つ』とか『差す』が自然なのだ」 | 候補「持つ」「差す」が出て、太い方がスロットに収まる |
| rain.links | ず「『雨』『傘』『外出』が関係していることを、ぼくたちは知っている」 | 帯の該当トークンから上へ意味の結び（ノードと線）が湧き出す |
| code.in | ず「『このプログラムはファイルを開いたあと、最後に』と来たら」 | 意味の結びを畳み、帯がコード例文へモーフ、候補「閉じる」 |
| code.rule | ず「開いたものを閉じることが多い、というルールっぽいものが必要」 | 「開いた」と「閉じる」が括弧対応のような弧で結ばれる |
| map.in | ず「文章の背後にあるパターンを内部に作るしかないのだ」 | 帯の背後に多数の文の流れが集まり、一枚の地図（路線網）へ圧縮され始める |
| map.grow | ず「『こういう文脈ならこう続く』という圧縮された地図を作っている」 | 地図の路線が増え、地図からスロットへ細い線が降りて候補を選ぶ |
| map.apart | ず「地図は便利だけど、地図と現地は違うのだ」 | 地図が浮き上がり、元の文章群（現地側）と距離が開いて静止 |

## 画面 3: tokens

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.tokens.in | ず「LLMは、文章をそのまま読んでいるわけではないのだ」 | 切れ目のない一文の帯が現れる |
| strip.cut | ず「最初に、文章を小さな部品へ切るのだ」 | 帯に切れ目が走り、部品がほどけて間隔を開けて並ぶ |
| token.size | ず「よく出るかたまりは大きめに、珍しいかたまりは小さめに」 | よく出る語は1ピースのまま、珍しい語がさらに細かく割れる |
| token.numbers | ず「各トークンは、コンピュータが扱える数字の並びに変換される」 | 1つのトークンの下から数値バーの列が湧き出す |
| pair.near | ず「たとえば『犬』と『猫』は意味が近いのだ」 | バー列が点に縮み、「犬」「猫」の2点が近くに灯る |
| pair.far | ず「一方で『犬』と『消費税』は遠いのだ」 | 「消費税」の点が遠くに灯る |
| space.in | ず「言葉を数字の空間に置くのだ」 | 薄い平面（意味の空間）が開き、他の語の点が散らばる |
| space.cluster | ず「似た意味の点は近くに集まり、違う意味の点は離れていく」 | 点群がゆっくり群れに分かれて整列する |
| akarui.blur | ず「『明るい』という言葉は、部屋にも性格にも未来にも使えるのだ」 | 「明るい」の点がにじみ、3方向へ薄く伸びたまま保留（次画面への引き） |

## 画面 4: attention

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.attention.in | ず「ここで『Attention』という言葉を導入するのだ」 | 例文のトークン列が現れる |
| arc.example | ず「『めたんはケーキを買った。冷蔵庫にそれを入れた』とするのだ」 | トークン列がその例文へ差し替わる |
| arc.it | ず「『それ』はケーキを指しているのだ」 | 「それ」から「ケーキ」へ弧が根元から描き起こされる |
| arc.weights | ず「前の『ケーキ』に強く注意を向ける必要があるのだ」 | 「それ」から他のトークンへも細い弧が伸び、ケーキへの弧だけ太い |
| arcs.kinds | ず「代名詞を見る線、文法を見る線、話題を見る線」 | 色違いの弧が種類ごとに少しずつずれて引かれ、束になる |
| layers.in | ず「Attentionを使って文章を何層も処理する仕組みを、Transformerと呼ぶのだ」 | トークン列が上へ複写され、層が積み上がる（各層で弧が引き直される） |
| layers.rewrite | ず「層を通るたびに、役割や文脈を含んだ表現へ変わっていくのだ」 | 上の層ほどトークンの色が文脈色へ染まっていく |

## 画面 5: icl

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.icl.in | め「LLMって、例をいくつか見せると急に合わせてくることあるわよね」 | 帯が「A 1 B 2 C 3 D ▢」の列へモーフ |
| icl.answer | ず「『Dは？』と聞くと、4と答えやすいのだ」 | スロットに「4」が滑り込む |
| weights.in | ず「ここで面白いのは、モデルの中身を更新していないことなのだ」 | 帯の背後に地図が戻り、錠前が掛かる |
| weights.frozen | ず「モデルの重みは変わっていないのだ」 | 地図は静止のまま錠が一拍強調され、帯側の例だけが順に光る |
| rule.emerge | ず「『今はこういうルールの話だな』と推定して」 | 例ペア（A-1, B-2, C-3）の間に共通の規則線がゴーストで浮かぶ |
| rule.apply | め「文脈を読んで、モードを切り替えてる感じね」 | 規則線がDへ延長され、「4」を指す |
| context.swap | ず「どんな文脈に置くかで、引き出される力が変わるのだ」 | 例が別規則（A 2 B 4 C 6）に差し替わり、地図はそのままスロットが「8」へ変わる |
| steps.in | ず「文脈の中に中間状態を置くと、次に何をすべきかを追いやすくなるのだ」 | 帯の途中に小さな中間チップが挟まり、スロットへの段差が低い連続になる |

## 画面 6: hallucination

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.hallucination.in | め「ここでハルシネーションの話ね」 | 帯＋スロットが上部に現れる |
| strip.smooth | ず「かなり自然な文章で、かなりそれっぽく間違うのだ」 | スロットに候補が滑らかに収まる——正しい時と全く同じ動きで |
| map.front | ず「LLMは、真実だけをきれいに保存した辞書ではないのだ」 | 帯が畳まれ、地図が前面へ出る |
| map.flaws | ず「地図に載っている道が古かったり、そもそも間違っていたり」 | 地図の一部の道が色あせ、一本が誤った向きで描かれているのが見える |
| blank.fill | ず「地図の空白をそれっぽく補うのが得意なのだ」 | 空白（霧の領域）に、周囲のパターンを延長した道がすっと引かれる——既存の道と同じ見た目で |
| check.in | ず「もう一つは、その内容が現実と合っているかを確かめる力」 | 現地の断片（正しい道形）が降りてきて、引いた道に重ねられる |
| check.mismatch | め「文章をうまくする係と、現実チェック係は別に置けってことね」 | 重ねた道のズレが赤く浮く |
| check.tools | ず「コードなら実行する。論文なら引用元を見る。」 | 照合の印が「実行」「引用」「照合」と3連で打たれる |
| draft.value | ず「地図を使って、見取り図を描き、仮説を出し、説明の順番まで組めるのだ」 | 地図上に下書きの経路が複数すっと引かれる |

## 画面 7: outro

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.outro.in | ず「最初の違和感は、『次の言葉を当てるだけで、なぜ賢く見えるのか』だったのだ」 | 導入と同じ姿の帯＋空きスロットが戻ってくる（回収） |
| recap.flow | ず「LLMは文章をトークンに切り、数字の空間に置き、Attentionで文脈内の関係を見て」 | 帯が左から右へ、切れ目→点化→弧→層→充填の全工程を一筆で通しでやり直す |
| map.final | ず「LLMが持っているのは、文章から作った世界の地図なのだ」 | 工程の下に地図が開く |
| map.zones | ず「地図が古い場所や空白の場所では、自然な嘘を作ることがある」 | 地図が明るい領域（頼れる）と霧の領域（空白）に塗り分かれる |
| final.mark | ず「どこまでが地図で、どこから現地確認が必要かを見分けることなのだ」 | 霧の領域に照合の印が灯って静止 |

## 各オブジェクトの誕生と死

| オブジェクト | 誕生 | 死 |
|---|---|---|
| 文の帯＋スロット（背骨） | scene.intro.in | 全画面を貫通（map.front で一時退場、scene.outro.in で復帰） |
| 候補語チップ | strip.fill | 各画面でスロット充填と同時に帯へ吸収（短命の従属物） |
| 「？」の灯 | strip.question | scene.deep.in |
| 意味の結び（雨-傘-外出） | rain.links | code.in（畳む） |
| ルール対応弧（開く-閉じる） | code.rule | map.in（畳む） |
| 地図 | map.in | scene.tokens.in で退場 → weights.in で復帰 → 以降 outro まで生存 |
| 数値バー列 | token.numbers | pair.near（点に縮む＝変身） |
| 意味の空間＋点群 | pair.near | scene.attention.in |
| Attention 弧の束 | arc.it | layers.in（層に引き直し）→ scene.icl.in |
| 層スタック | layers.in | scene.icl.in |
| 錠前 | weights.in | scene.hallucination.in |
| 規則線（ゴースト） | rule.emerge | scene.hallucination.in |
| 中間チップ | steps.in | scene.hallucination.in |
| 霧の領域＋補完された道 | blank.fill | 画面6の主役のまま outro の map.zones へ転用 |
| 現地の断片＋照合印 | check.in | scene.outro.in（照合印は final.mark で再登場） |

同時表示の最大は画面2の「帯＋候補＋意味の結び」「帯＋地図」、画面5の「帯＋地図＋規則線」、画面6の「地図＋現地断片＋照合印」でいずれも 3 個以内。
