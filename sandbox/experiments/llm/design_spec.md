# [3] セリフ単位スペック — LLMの仕組み

各画面の先頭セリフに `scene.{画面名}.in`。装置の同時表示は各画面 3〜4 個以内に収め、役目を終えた装置は次が立つ前に死なせる。

共通装置：**意味の地図**（淡い座標面。語が置かれる舞台）、**語の点**（ラベル付きの点＝トークン。インディゴ。背骨の単位）。文脈の引き＝ティール、研ぎ澄まされた一点＝アンバー、選ばれた語・当たり＝グリーン、外れ・でたらめ・記号操作＝レッド。

## 画面1：序論「次の語だけ・中身は場所」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.intro.in` | なんで賢いの | 中央に短い文「むかしむかし、あるところに」と末尾の点滅する空欄が立つ |
| `intro.story` | って聞かれたら | 空欄の上に候補語が数枚ふわりと並ぶ |
| `intro.pick` | おじいさんと」？ | 候補の一枚「おじいさんと」だけが空欄へ降りて文に足される |
| `intro.repeat` | 一番ありそうな一語 | さらに数語が一つずつ末尾に足され、文が右へ伸びていく |
| `intro.empty` | 意味なんて一個も | 足された語の一つを開くと、「意味」の枠が空っぽ（取り消し線）と分かる |
| `intro.place` | 言葉の置かれた場所 | 空の枠から、地図上の一点だけが残り、中身の正体＝場所だと示す |

- 文＋空欄：`scene.intro.in` 誕生、画面終端まで生存。候補語：`intro.story`〜`intro.pick` で死。
- 「意味」の空枠：`intro.empty` 誕生 → `intro.place` で点へ譲り死。語の点：`intro.place` 誕生（背骨の点として次画面へ持ち越す）。

## 画面2：ボディ1「言葉を地図に置く」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.body1.in` | 数字しか扱え | 一本の数直線に、りんご・みかん・政府が番号順で並ぶ |
| `b1.number` | みかん2番 | 番号1+2=3 の足し算が「政府」に着き、無意味だと×が付く |
| `b1.map` | 点として置く | 数直線が二次元の地図へ開き、語の点が散る（似た語は近く・違う語は遠く） |
| `b1.embed` | 分散表現 | 地図に「埋め込み／分散表現」のラベルが付く |
| `b1.vector` | 「男」のぶんだけ | 「王様」から −男・+女 の二本の矢印が伸びる |
| `b1.queen` | ぴたっと着く | 矢印の先が「女王」の点にぴたりと重なる（向きに意味が乗る） |
| `b1.dims` | 測る物差し | 上帯に「甘さ・大きさ・高級さ…＝物差しの数だけ次元」 |
| `b1.neighbors` | 一緒に出てくる | りんごと政府の各点に、周りの語の衛星が灯る（似た周り＝近い） |
| `b1.firth` | つきあう仲間を見れば | 上帯にファースの一文「言葉は、つきあう仲間で決まる」 |
| `b1.train` | 最後を隠して | 穴埋め札：文末を隠して当て、外れは赤×→点を少し寄せ、当たりは緑。反復で点がクラスタへ締まる |

- 数直線：`scene.body1.in` 誕生 → `b1.map` で地図へモーフ（消さず変身）。地図＋点：`b1.map` 誕生、画面終端まで生存。
- 埋め込みラベル：`b1.embed` 誕生 → `b1.vector` で死。−男/+女の矢印（王族）：`b1.vector` 誕生 → `b1.queen` 後に死。
- 上帯は順送り：`b1.dims`（次元）→ `b1.neighbors`（衛星）→ `b1.firth`（一文）と一つずつ入れ替え。穴埋め札：`b1.train` 誕生、点の締まり（settle）はここで起こす。同時＝地図＋点＋（矢印 or 上帯 or 穴埋め札）で 3 個以内。

## 画面3：ボディ2「文脈で研ぎ澄ます」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.body2.in` | 同じ点なのだ | 地図上、「冷たい」が一点きり。水寄り・人寄りの薄い影が重なってぼやける |
| `b2.pronoun` | 猫が魚を食べた | 文が出て「それ」の点が、魚とも猫ともつかぬ宙ぶらりんで立つ |
| `b2.lookaround` | 関係ある？ | 「それ」から全部の語へ細い線が伸び、関係を問う |
| `b2.move` | 「魚」の側へ | 「魚」への線だけが太くなり、「それ」の点が魚の側へ引かれて動く |
| `b2.attention` | 注意機構 | この見渡し＋注目に「Attention／Transformer」のラベル |
| `b2.quiz` | 性格の明るさ | 「明るい」が、部屋の明るさ極と性格の明るさ極の間に立ち、性格側へ寄る |
| `b2.focus` | 点がぼやけてる | ぼやけた点が、文脈のピントで一点に締まる（重ねるたびに鋭く） |
| `b2.general` | 丸暗記じゃない | 初見の文の点も、既知の点を混ぜて地図のどこかに置かれる |

- 「冷たい」点：`scene.body2.in` 誕生 → `b2.pronoun` で「それ」へ役を譲り死。「それ」点＋近傍語（魚・猫）：`b2.pronoun` 誕生。
- 問いの線束：`b2.lookaround` 誕生 → `b2.move` で一本だけ残し他は薄れる。`b2.quiz` の極・`b2.focus` のピント・`b2.general` の初見点は、前の装置を畳んでから順に立てる。同時＝地図＋活点＋（線束 or 極 or 初見点）で 3〜4 個以内。

## 画面4：ボディ3「次の一語を引いて、また引く」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.body3.in` | 確率をふる | 上に文「私はラーメンを＿」、下に最後の点（アンバー）から確率の山が立ち上がる |
| `b3.dist` | 60パーセント | 候補語が縦バーに並び「食べた60／すすった10／作った5」の数値が乗る |
| `b3.pick` | 一個だけ選ぶ | 山にしたがって一本（食べた）が緑に選ばれる |
| `b3.temp` | つまみで変えられる | つまみで山の形が変わる（低＝尖る・高＝平ら） |
| `b3.halluc` | それっぽい」の区別 | 事実でない語が、自信たっぷりに赤で選ばれる（賢さと同じ仕掛け） |
| `b3.append` | 文のお尻に | 選ばれた語が文末にくっつき「私はラーメンを食べた」になる |
| `b3.loop` | 自己回帰 | 出力語が入力へ戻る輪が回り、山がまた立て直される（くりかえし） |
| `b3.noplan` | 建物が建つ | 設計図なしに、語のブロックが一つずつ積まれて筋の通った建物になる |

- 文：`scene.body3.in` 誕生、画面終端まで生存。確率の山：`scene.body3.in` 誕生 → `b3.append` ごとに引き直し。
- つまみ：`b3.temp` 誕生 → `b3.halluc` で死。輪：`b3.loop` 誕生。建物：`b3.noplan` 誕生。同時＝文＋山＋（つまみ or 輪 or 建物）で 3 個以内。

## 画面5：結論「場所と距離だけ・中国語の部屋」

| event | 紐付くセリフ抜粋 | 何が起きるか |
|---|---|---|
| `scene.outro.in` | LLMの中身を見る | 地図と点だけが戻る。中を覗いても意味は無い |
| `outro.empty` | 場所と距離だけ | 点と点の間に距離だけが引かれ、「意味」は一文字も無いと示す |
| `outro.room` | こんな話をした | 地図が「中国語の部屋」へモーフ：部屋・中の人・分厚いマニュアル |
| `outro.symbol` | 規則どおり記号を返す | 記号が左から入り、規則で別の記号が右へ出る。外からは理解に見える |
| `outro.human` | 周りでどう使われてる | 部屋の隣に人の頭。中には同じ意味の地図がある |
| `outro.mirror` | おどろくほど同じ | 人とLLMの地図が重なり、使う材料が同じだと示す |
| `outro.end` | 問い返してくる | 理解と計算の境目が薄れ、問いがこちらへ向き直って締め |

- 地図＋点：`scene.outro.in` 誕生 → `outro.room` で部屋へモーフ。部屋＋人＋マニュアル：`outro.room` 誕生 → `outro.mirror` で畳む。
- 人の頭（同じ地図）：`outro.human` 誕生。重なり／問い：`outro.mirror`〜`outro.end`。同時＝（部屋 or 地図）＋人＋記号で 3 個以内。局面ごとに前を畳んでから次を立てる。
