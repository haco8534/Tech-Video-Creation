# transformer — design spec

`design_overall.md` の質的な動きを event アンカーに分解し、`script.md` の各セリフへ割り当てる。座標値・フレーム数は持たない。

対話字幕は event ではない——全セリフが自動で画面下部の字幕になる（tsx の仕事）。ここで定義する event は、絵の装置の挙動だけを指す。

## 1. event 一覧と割り当て

発生順。画面切替 event（`scene.*.in`）は各画面の先頭セリフに必須。

| event | 紐付くセリフ（画面・話者・抜粋） | 発火する挙動 |
|---|---|---|
| `scene.intro.in` | 序論・めたん「ねえずんだもん。スマホで文字を打っていると…」 | 序論の画面がフェードイン。語カードが左から並び短い文を作り始める |
| `intro.predict` | 序論・ずんだもん「「おはよう」と打つと「ございます」が候補に出てくる、あれなのだ。」 | 文末に「？」カードが灯り、その下から次語確率バーが立ち上がる。いちばん高い候補がアンバーで灯る |
| `intro.uses` | 序論・ずんだもん「外国語をなめらかに訳す自動翻訳も…ChatGPT も──根っこは全部おなじ…」 | 語カード列を囲んで3つの用途パネルが現れ、各パネルの中で「？→確率バー」のミニ動作が走る |
| `intro.merge` | 序論・ずんだもん「…Transformer というのだ。」 | 3つの用途パネルが中央下へ吸い寄せられ統合コアへ融合。コアに「Transformer」「2017」が灯る |
| `intro.question` | 序論・ずんだもん「…「機械はどうやって文章を読んでいるのか」──それを、絵で一歩ずつ…」 | 統合コアが脈打ちはじめ、その下に問いテキストが現れる |
| `scene.body1.in` | ボディ1・ずんだもん「たとえば、「かける」という言葉を、ひとつだけ取り出してみるのだ。」 | 画面切替。「かける」カードが単独で中央に立ち、右に意味空間パネルが現れる |
| `kakeru.waver` | ボディ1・ずんだもん「「めがねをかける」。「いすに腰をかける」。「友だちに電話をかける」。「保険をかける」。」 | 「かける」の意味指紋が複数パターンの間で明滅。意味空間の点がぼやけて広がり、どの意味領域にも定まらない |
| `kakeru.context` | ボディ1・ずんだもん「…決めているのは「めがね」や「電話」──まわりにいる言葉なのだ。」 | 文脈カード「電話」「を」が「かける」の脇から滑り出て並ぶ |
| `kakeru.resolve` | ボディ1・ずんだもん「…「電話を」と来た時点で、もう「かける」の意味を、電話用にしぼっているのだ。」 | 意味指紋が一つのパターンに落ち着き、意味空間の点がぼやけを失って「電話」領域へ収束し鋭い一点になる |
| `order.flip` | ボディ1・めたん「…「ねこが いぬを 追う」と「いぬが ねこを 追う」では──」 | 「かける」から下方に語順デモ（語カード3枚＋関係の矢印）が分かれ出て、矢印の向きを反転すると関係が逆転する |
| `oldmachine.dict` | ボディ1・ずんだもん「…昔の機械は、ひとつの言葉に、ひとつの決まった意味を、あてがっていたのだ。」 | 語順デモが「かける」へ戻る。辞書の図が現れ、文脈と無関係に灰色の固定指紋を貼る。「めがね」でも「電話」でも同じ灰色指紋 |
| `scene.body2.in` | ボディ2・ずんだもん「ここからが、Transformer の心臓部なのだ。名前は Self-Attention。…」 | 画面切替。一文の語カード列（5枚）が横一列に並ぶ |
| `attn.ask` | ボディ2・ずんだもん「…ほかの全部の言葉に向かって、こう問いかけるのだ。「きみは、ぼくの意味に関係ある？」」 | 焦点カードから、他の全カードへ細い均一の問い線が伸びる |
| `attn.weight` | ボディ2・ずんだもん「「かける」は「電話」に強く注目するのだ。逆に「を」みたいな言葉には、ほとんど注目しないのだ。」 | 問い線の太さに差がつき、その差が語カード列の上の注目配分バー（全長一定の帯）にまとまる |
| `qkv.open` | ボディ2・ずんだもん「…言葉はそれぞれ、三つの顔を持つのだ。Query、Key、Value──質問、鍵、中身、の三役なのだ。」 | 各カードが三つの顔を開く：上辺に Query ソケット、下辺に Key タブ、内部の意味指紋が Value |
| `qk.match` | ボディ2・ずんだもん「ある言葉の Query と、相手の言葉の Key。この二つを照らし合わせて、ぴたっと噛み合うほど──その相手への注目が、強くなるのだ。」 | 焦点カードの Query ソケットを相手の Key タブへ合わせる。噛み合えば施錠して発光、噛み合わなければ収まらない。結果で注目配分バーが更新 |
| `value.sum` | ボディ2・ずんだもん「Value は「中身」なのだ。…相手の Value──持っている中身そのものを、注目の強さのぶんだけ、受け取るのだ。」 | 各相手の Value＝意味指紋を配分の太さぶん取り出し、焦点カード内部で一段ずつ積み上げる（加重和の組み立て） |
| `focus.dressed` | ボディ2・ずんだもん「「かける」は、もう「電話の文脈を着こんだ、かける」になっているのだ。…」 | 積み上がった新しい意味指紋が古い指紋と入れ替わり、焦点カードの色味が変わる。意味空間の小パネルが現れ、点が「電話」寄りへ移って鋭くなる |
| `attn.all` | ボディ2・ずんだもん「…そして文章じゅうの言葉が、いっせいに、これをやるのだ。」 | 全カードが一斉に同じことを行い、行ぜんたいの意味指紋が更新される |
| `qkv.learn` | ボディ2・ずんだもん「…「どんな Query と Key を作れば…」を、すこしずつ調整していくのだ。最初は、まるででたらめなのだ。」 | Query ソケットと Key タブの形が最初はいびつ。文章を読むたびに形が整い、噛み合うようになる |
| `scene.body3.in` | ボディ3・めたん「でも、ずんだもん。「関係がある」と言っても、いろいろな関係が、ありませんこと？」 | 画面切替。一文の語カード列と、ボディ2の注目をまとめた注目グリッド（N×N）が現れる |
| `heads.split` | ボディ3・ずんだもん「…これを Multi-Head Attention──「複数の頭」と呼ぶのだ。」 | 注目グリッドが3〜4色のグリッドに分裂し、色ごとに濃いセルの位置が異なる |
| `heads.merge` | ボディ3・めたん「そして最後に、その線を全部かさね合わせて、ひとつの理解に、まとめる。…」 | 色付きグリッドが重なり合って一枚に統合される |
| `layers.stack` | ボディ3・ずんだもん「…Self-Attention の層を、何段も積み重ねるのだ。下の階で出てきた結果が、そのまま、上の階の入り口…」 | 統合結果の語カード列が複製され上へ段々に積まれる。段を上がるごとに意味指紋が細部から全体へ変化する |
| `pos.shuffle` | ボディ3・ずんだもん「…注目し合うだけだと、順番の情報が、すっぽり抜け落ちるのだ。」 | 注目グリッドだけ残して語カードの並び順をシャッフルすると、グリッドが同じに見えて区別がつかない |
| `pos.wave` | ボディ3・ずんだもん「…順番の目印を、こっそり付け足してやるのだ。これを、位置エンコーディングと呼ぶのだ。」 | 各カード下辺に位置の波（位置ごとに違う波形）が灯り、意味指紋に加わる。以後シャッフルするとグリッドも変わり順番が復元する |
| `scene.body4.in` | ボディ4・めたん「ねえずんだもん。Transformer の前にも、機械は文章を読んでいましたわよね。…」 | 画面切替。画面が上下2段に分かれる（上＝RNN、下＝Transformer） |
| `rnn.relay` | ボディ4・ずんだもん「RNN は、文章を「前から一語ずつ」、順番に読んでいくのだ。」 | 上段で単一状態トークンが先頭カードから1枚ずつ右へ移り、触れたカードだけ点灯する（逐次・遅い） |
| `tf.parallel` | ボディ4・ずんだもん「Transformer の Self-Attention は、全部の言葉を「いっぺんに」見比べるのだ。順番待ちが、そもそも、無いのだ。」 | 下段で注目グリッドが一度に全点灯する（並列・待ちなし） |
| `rnn.fade` | ボディ4・ずんだもん「…文章の最初のほうに出てきた大事な言葉が、後ろへ届くころには、うすれて、消えかけてしまうのだ。」 | 上段で、状態トークンが運ぶ先頭カード由来の意味指紋の成分が、末尾へ着くころには薄れて消えかける |
| `tf.direct` | ボディ4・ずんだもん「でも Transformer は、注目で、いきなり直接つながるのだ。文の一語目と百語目でも、ひとっとびで…」 | 下段で先頭カードと末尾カードが一本の太い直結線で結ばれる（距離ゼロ） |
| `learn.game` | ボディ4・ずんだもん「…文章の途中の言葉をかくして、「次はなんだ」と当てさせる。外れたら、すこし直す。それを、とほうもない回数…」 | 上下の対比が中央へ畳まれ、学習：途中カードを「？」で隠し末尾に次語確率バーを立て、外れたら指紋を直す、を高速で反復する |
| `learn.scale` | ボディ4・ずんだもん「…そして並列に処理できるからこそ、その膨大な練習を、現実的な時間で、終わらせられるのだ。」 | 学習の反復を並列の複数文で同時に走らせ、その量を糧に中央の統合コアが大きく育つ |
| `scene.outro.in` | 結論・ずんだもん「ここまでを、絵といっしょに、振り返るのだ。」 | 画面切替。文脈を着こみ終えた一文の語カード列が中央に現れる |
| `recap.devices` | 結論・ずんだもん「まず──ことばの意味は、まわりが決める。」 | 語カード列の上に装置アイコン帯が現れ、意味空間→注目グリッド→色付きグリッドと層スタック→直結線、の順に再点灯していく |
| `outro.loopback` | 結論・めたん「はじめのお話に、戻ってきますわね。スマホが、次の言葉を予想してくれるとき──」 | 行末に「？」カードが再登場し、次語確率バーが立ち、いちばん高い候補が灯る |
| `outro.end` | 結論・ずんだもん「機械が言葉を「見わたす」しくみ──Transformer のお話、だったのだ。」 | 装置アイコン帯と語カード列が中央の統合コアへ収束し、コアが静かに脈打って終わる |

内容 event は31個（全153セリフの約20%）。「5〜15%目安」を上回るが、15分・6画面の長尺で、各画面が複数の独立した仕組みを順送りで扱い、仕組みごとに専用装置を立ち上げ・畳みする題材のため。装置の登場・退場そのものが説明の本体なので、event は装置の数に比例する。

## 3. オブジェクト別 挙動メモ

複数 event をまたいで連続変化するオブジェクトの、event 間の状態の向き先。

**意味指紋（背骨）** — すべての語カードの内部部品。ボディ1：`kakeru.waver` で「かける」の指紋が複数パターンの間で明滅（不定）→ `kakeru.resolve` で一つのパターンへ収束（定まる）。ボディ2：`value.sum` で焦点カードの指紋が、相手の Value 片を配分の太さぶん加重和して新パターンに組み上がる → `focus.dressed` で旧指紋と入れ替わる → `attn.all` で全カードの指紋が同様に更新。ボディ3：`layers.stack` で段を上がるごとにパターンが細部の凹凸から大きなまとまりへ変化 → `pos.wave` で位置の波形が指紋に加算される。ボディ4：`rnn.fade` で RNN 単一状態トークンが運ぶ先頭由来の成分が減衰する。

**焦点カード（ボディ2）** — 語カード列の右端寄りの1枚（「かける」）。画面入場時はほかと同じ見た目。`attn.ask`〜`qk.match` は明るく持ち上がって「いま処理している主体」を示す。`value.sum` で内部に加重和を組み立て、`focus.dressed` で指紋が入れ替わり面の色味が変わる（文脈を着こんだ姿）。`attn.all` 以降はその姿を保つ。

**注目配分バー（ボディ2）** — 全長一定の帯。`attn.weight` で誕生し、注目線の太さの差を相手ごとの区画として表す（総和は常に一定＝固定された注目量の配分）。`qk.match` で照合の結果に従って各区画の幅が更新される。`value.sum` で各区画の幅が、対応する Value 片の取り出し量を決める。

**注目グリッド（ボディ3）** — `scene.body3.in` で1枚誕生（行＝見る側、列＝見られる側、セルの濃さ＝注目）。`heads.split` で3〜4色のグリッドへ分裂（色付きグリッドが派生）→ `heads.merge` で重なって1枚に戻る。`pos.shuffle` で語カードの並びをシャッフルしてもセル配置が変わらず（順番が消えている）、`pos.wave` 後はシャッフルでセル配置が変わる（順番が効いている）。

**RNN 単一状態トークン（ボディ4）** — `rnn.relay` で RNN 行の先頭カードに生まれ、1枚ずつ右へ移動。触れたカードだけを点灯させる。`rnn.fade` までに、最初のカードから受け取った意味指紋の成分が、移動するほど薄れていく。`learn.game` で RNN 行とともに中央へ畳まれて消える。

**統合コア** — 序論・ボディ4・結論に各1個。いずれも「複数のものが中央に集まって融合し、脈打つ円になる」という同じ振る舞い。序論では用途パネルから、ボディ4では並列の文群から、結論では装置アイコン帯と語カード列から生まれる。画面リセットを挟むので3個は別個体だが、視聴者には「同じコアに戻ってきた」と感じさせる。

**語カード列** — 全画面に登場する主役。各画面で別個体（画面リセット）だが、形・色の語彙と内部の意味指紋は共通。画面内では決して unmount せず、状態を連続変化させる。

## 次段への引き継ぎ
