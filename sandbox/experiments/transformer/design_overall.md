# transformer — design overall

`script.md` を入力に、動画全体を貫通する世界観と、6画面それぞれのデザインを自然言語で設計する。座標値・フレーム数・セリフ単位のタイミングは持たない（design_spec と Composition.tsx の仕事）。

テーマ指定は**ホワイトテーマ**。明るい背景の上で、言葉が「数値の実体」を持ったオブジェクトとして処理されていくさまを描く。

## 設計の原則

図解は概念に**ラベルを貼る**のではなく、仕組みを**動いて見せる**。「Self-Attention」と書いた箱を画面に出すのではなく、注目が配分され中身が混ざる過程そのものを画面上で起こす。だから本作の「形の語彙」は、汎用の箱・矢印の寄せ集めではなく、**ひとつの仕組みをひとつ可視化する専用装置**の集まりとして設計する。視聴者は装置が動くのを見て、解説を聞く前に仕組みを目で理解できる——これが本作の見やすさの核心。

貫通する背骨は **意味指紋**。すべての語カードは、内部に小さなバーの並び＝その語の「意味の数値」を抱えている。動画全体は、この意味指紋が「単独では揺れている」状態から、「まわりから中身を集めて定まる」「層を重ねて深くなる」「順番の目印を得る」へと変化していく一本の物語になる。個々の装置——Q/K/V グリフ・注目配分バー・加重和の組み立て・層スタック・位置の波——は、すべてこの意味指紋を計算し、組み替え、磨くための道具として連なる。装置はバラバラの飾りではなく、ひとつの数値を加工していく一連の工程として互いにつながっている。

## 1. 世界観

| 軸 | この題材での設定 |
|---|---|
| 貫通する比喩 | **明るい解析ノート**。やわらかい白の紙面が一台の作業台で、その上で「言葉」が数値の実体（意味指紋）を持った部品として運ばれ、専用の装置にかけられ、別のものへ組み替わっていく。図解教材のように清潔で見通しがよく、装置の動きを目で追えば仕組みが分かる。視聴者は同じ作業台をずっと眺めながら、ひとつの言葉が処理されていく全工程を追う |
| 形の語彙 | 専用の図解装置の集まり（一覧は §3 装置カタログ）。背骨は **意味指紋**＝語の意味を表す小バーの並び。基本部品は **語カード**＝角丸カプセル＋内部に意味指紋。以降のすべての装置（Q/K/V グリフ・照合・注目配分バー・注目グリッド・加重和・層スタック・位置の波・次語確率バー・RNN 単一状態・統合コア）は、語カードと意味指紋を入力に「何かを計算する道具」として設計する。汎用の「ラベル付き箱」「意味のない矢印」「テキストだけのカード」は使わない |
| 色の語彙 | 背景＝やわらかい白 `#f5f7fa`。語カード＝面 `#ffffff`／枠 `#9fb1c9`／文字 `#243044`、淡い影でわずかに浮かせる。意味指紋のバー＝グリーン系 `#3f9d57`（語の「中身」そのもの＝Value と同色）。Query＝ティール `#0f9b94`。Key＝バイオレット `#7a4fd0`。Value＝グリーン `#3f9d57`。注目・焦点の光・確率バーの当たり＝暖色アンバー `#e08a1e`（紙面でいちばん効く色）。古い・薄れる・固定された要素＝グレー `#aab2bf`。Q/K/V の3色は「三つの役」という意味そのものを担うので必要。対話字幕の話者色＝めたん `#d6336c`／ずんだもん `#2f9e44` |
| 動きの語彙 | 使う＝フェード、滑らかな整列・移動、バーの伸縮、指紋パターンの組み替え、形が噛み合う／収まらない、濃淡の変化、複製（層・頭が増える）、収束・融合、ゆるやかな脈動。封印＝3D回転、カメラのズーム／パン、バウンド、急な点滅。白地なので濃さ・影・色味で奥行きと状態を出し、運動は静かに。装置は1画面の中で順に登場・退場し、常に何かが変化している（静止画面なし） |
| 文字方針 | 画面下部に**対話字幕**を常駐（話者名は話者色、本文は濃紺、スマホ視聴前提で大きめ・2行まで）。図解内のテキストはカード内の単語、Q/K/V の記号、確率バーの候補語、層番号など最小限。専門用語（Self-Attention／Query／Key／Value／Multi-Head／位置エンコーディング／RNN）は装置のラベルとして絵の中に置く |
| ナビ補助 | 画面が切り替わるとき、左上に小さくセクション番号と見出し（例「02 単語どうしが、見つめ合う」）。進捗ドット等は置かず最小限。字幕帯とは別の位置 |

レイアウトの基準：中心原点（実装は viewBox `-960 -540 1920 1080`）。**画面下部 y≈350〜510 は対話字幕の帯**として確保し、主オブジェクトはそれより上（おおむね y≈-420〜300）に置く。下端に重要物を置かない。1画面に装置を全部いっぺんに出さない——会話が進むにつれ装置が順に立ち上がり、役目を終えたら畳まれ、次の装置に場所を譲る。

## 2. セクション別デザイン

### 序論（画面1）

- **登場オブジェクト**：語カード列（「おはよう」などの短文）＋末尾の「？」カード、次語確率バー、3つの用途パネル（予測変換／翻訳／ChatGPT）、統合コア、年号「2017」、問いテキスト
- **レイアウトの考え方**：中央やや上に語カード列を横並び。「？」カードはその右端。次語確率バーは「？」カードの下から立ち上げる。用途パネルは語カード列を三方から囲むように配置。統合コアは語カード列の下、中央。
- **動き**：語カードが左から1枚ずつ並んで短文をつくる → 末尾に「？」カードが灯る → 「？」カードの下から次語確率バーが立ち上がる（候補語が縦に並び、それぞれ高さの違うバー＝確からしさ）。いちばん高い候補がアンバーで灯る——「次の言葉を当てる」を絵にした瞬間 → 語カード列を囲んで3つの用途パネルが現れ、その各パネルの中でも同じ「？→確率バー」のミニ動作が走る（3つとも根は同じ装置）→ 3つのパネルが中央下へ吸い寄せられ統合コアへ融合、コアに「Transformer」、傍らに「2017」→ コアが静かに脈打ち、その下に問いテキストが現れる
- **絵の眼目**：予測変換も翻訳も ChatGPT も、同じ「次語確率バーを立てる」一つの装置で動いている

### ボディ1：ことばの意味は、まわりが決める（画面2）

- **登場オブジェクト**：単独の語カード「かける」（意味指紋つき）、意味空間パネル＋意味の点、文脈カード（「電話」「を」）、関係の矢印と語順の対例カード、辞書（一義を固定する旧式の図）
- **レイアウトの考え方**：左に「かける」カードを大きく、右に意味空間パネル（薄い格子の2D 面、領域がいくつかにゆるく分かれている）。文脈カードは「かける」の左右から滑り込む。語順の対例と辞書は、話が進むにつれ同じ紙面で入れ替わって登場する。
- **動き**：「かける」カードが単独で中央に立つ → 内部の意味指紋のバーが、いくつかのパターンの間で明滅して定まらない → 意味空間パネルの中で、その語の「点」がぼやけて広がり、複数の意味領域（めがね／いす／電話／保険）のどこにも定まらない → 文脈カード「電話」「を」が脇から滑り出て「かける」の隣に並ぶ → 意味指紋のバーが一つのパターンに落ち着き、同時に意味空間の点がぼやけを失って「電話」領域へすっと収束し、鋭い一点になる → 語順の対例：同じ語カード3枚（ねこ／いぬ／追う）のあいだに関係の矢印が引かれ、矢印の向きだけを反転させると「だれがだれを追う」が正反対になる → 昔の機械：辞書の図が現れ、文脈カードと無関係に「かける」へ灰色の固定指紋を一枚あてがう。「めがね」でも「電話」でも貼られる指紋は同じ灰色のまま
- **絵の眼目**：単独の語の意味指紋は揺れていて、まわりの語が来て初めて一点に定まる。昔の機械は揺れを許さず、固定の灰色指紋を貼っていた

### ボディ2：単語どうしが、見つめ合う（画面3）

- **登場オブジェクト**：一文の語カード列（5枚ほど、各自に意味指紋）、Q/K/V グリフ（Query＝ソケット型／Key＝タブ型／Value＝意味指紋そのもの）、照合の動作、注目配分バー、加重和の組み立て、焦点カード、意味空間の小パネル（カムバック）
- **レイアウトの考え方**：一文の語カードを中央に横一列。焦点カードは少し持ち上げて明るくする。Query グリフはカード上辺、Key グリフは下辺に芽吹かせる。照合は焦点カードと相手カードの中間で行う。注目配分バーは語カード列の上に一本通す。加重和は焦点カードの内部で起こす。
- **動き**：一文の語カードが横並びになる → 焦点カード「かける」から、ほかの全カードへ細い問い線が伸びる → 各カードが三つの顔を開く：上辺に Query ソケット、下辺に Key タブ、中身の意味指紋が Value → 焦点カードの Query ソケットを、相手カードの Key タブへ順に合わせていく（照合）。形が噛み合えば錠が掛かって光り、噛み合わなければ収まらない（「電話」の Key は噛み合い、「を」の Key は噛み合わない）→ 照合の結果が、語カード列の上の一本の注目配分バーに溜まっていく。バーは全長が決まっていて、噛み合った相手ほど太い区画を占める——総量は固定で、それを配分する → 配分が決まると、各相手の Value＝意味指紋を、配分の太さのぶんだけ取り出し、焦点カードの内部で一段ずつ積み上げる（加重和の組み立て）→ 積み上がった新しい意味指紋が、焦点カードの古い指紋と入れ替わる。焦点カードの面の色味がわずかに変わる（文脈を着こんだ語）→ 意味空間の小パネルがカムバックし、「かける」の点が「電話」寄りへ移って鋭くなる → 続いて全カードが一斉に同じことを行い、行ぜんたいの意味指紋が更新される → 学習：Query ソケットと Key タブの形は最初いびつで噛み合わない。文章を読むたびに形が少しずつ整い、噛み合うようになっていく
- **絵の眼目**：焦点の語は、Q と K の噛み合いで注目を配分し、その配分どおりに相手の Value を加重和して、新しい意味指紋へ組み替わる

### ボディ3：視点を増やし、積み重ねる（画面4）

- **登場オブジェクト**：一文の語カード列、注目グリッド（N×N）、複数色の注目グリッド（マルチヘッド）、層スタック、位置の波（波形スタンプ）、位置の目盛り
- **レイアウトの考え方**：上部に注目グリッド（行＝見る側の語、列＝見られる側の語、セルの濃さ＝注目の強さ）。語カード列は中央。層スタックは語カード列を上方向へ段々に積み上げて表す。位置の波は各カードの下辺に灯す。
- **動き**：ボディ2の注目配分バー群が畳まれて、一枚の注目グリッドに集約する（全ペアの注目を一望できる）→ その注目グリッドが3〜4色のグリッドに分裂し、色ごとに濃いセルの位置が異なる（頭ごとに見ている関係が違う：文法をたどる頭、意味をたどる頭…）→ 色付きグリッドが重なり合って一枚に統合される → 統合の結果の語カード列が複製され、すぐ上にもう一段積まれる。下の段の出力が、そのまま上の段の入口になる（層スタック）→ 段を上がるごとに、各カードの意味指紋のパターンが、こまかい凹凸から大きなまとまりへと変わっていく（細部→全体）→ 層スタックの注目グリッドだけを残して語カードの並び順をシャッフルすると、グリッドが同じに見えて順番の区別がつかない → 各カード下辺に位置の波（位置ごとに違う波形スタンプ）が灯り、その波形が意味指紋に加わる。以後シャッフルするとグリッドも変わり、順番が復元できる
- **絵の眼目**：ひとつの注目を、色（視点の数）と段（層の高さ）で増やすと意味指紋が深くなる。波形の目印が順番を取り戻す
- 補足：3小トピック（マルチヘッド・層・位置）を順送りで扱う最も密度の高い画面。装置を順に立ち上げ・畳みしながら、語カード列だけは画面を通して連続して残す。

### ボディ4：なぜ Transformer は世界を変えたのか（画面5）

- **登場オブジェクト**：RNN 側＝一列の語カード＋単一状態トークン（移動するひとつの状態）、Transformer 側＝同じ一列の語カード＋注目グリッド、行列の比喩、次語確率バー（カムバック）、並列に並ぶ複数の文、統合コア
- **レイアウトの考え方**：前半は画面を上下2段に分け、上段に RNN、下段に Transformer を置いて対比する。後半は対比を中央へ畳み、学習と並列処理を中央で見せ、統合コアへ収束させる。
- **動き**：上段の RNN ＝ 単一状態トークンが先頭カードから1枚ずつ右へ移り、触れたカードだけが点灯する（逐次）。状態トークンは進むほど、最初のカードから受け取った意味指紋の成分が薄れていく（バケツリレーで最初の水が届かない）→ 下段の Transformer ＝ 注目グリッドが一度に全点灯。先頭カードと末尾カードが一本の線で直結する（距離ゼロ）→ 上下の対比を中央へ畳む → 学習：一文の途中のカードを「？」カードで隠し、末尾に次語確率バーを立てて当てさせる。外れたら指紋を少し直す。これを高速で反復 → その反復を、並列に並んだ複数の文で同時に走らせる → 反復の量を糧に、中央の統合コアが大きく育つ
- **絵の眼目**：逐次の単一状態は遅く・遠くが薄れる。並列の注目は速く・距離ゼロ。並列だから膨大な次語あて練習ができ、巨大な AI が建つ

### 結論（画面6）

- **登場オブジェクト**：文脈を着こみ終えた一文の語カード列、これまでの装置の小アイコン（意味空間／注目グリッド／色付きグリッドと層スタック／直結線）、「？」カード、次語確率バー、統合コア
- **レイアウトの考え方**：中央に完成した語カード列。その上に装置アイコンの帯。最後はすべてが中央の統合コアへ収束する。
- **動き**：これまでの装置が順に小さく再点灯する：意味空間で点が定まる → 注目グリッドが灯る → 色付きグリッドと層スタックが重なる → 先頭と末尾の直結線が引かれる → 行末に「？」カードが再登場し、次語確率バーが立ち、いちばん高い候補がアンバーで灯る → 全アイコンと語カード列が中央の統合コアへ収束し、コア「Transformer」が静かに脈打って終わる
- **絵の眼目**：すべての装置が、最初の「次語確率バーを立てる」という一点に集まる

## 3. 装置カタログ

|---|---|---|---|

| 意味指紋 | 語の意味が「数値の実体」であること。背骨 | カード内の小バーの並び（グリーン系）。高さのパターンが意味 | 全画面 |

| 用途パネル | 予測変換／翻訳／ChatGPT が同じ装置で動くこと | 枠つきパネル、中で確率バーのミニ動作 | 序論 |
| 統合コア | Transformer 本体。脈打つ円 | 静かに脈動する円「Transformer」 | 序論・ボディ4・結論 |
| 意味空間 | 意味の「揺れ」と「定まり」を位置で示す | 薄い格子の2D 面と、その上の点（ぼやけ↔鋭い一点） | ボディ1・ボディ2・結論 |
| 文脈カード | 主役の意味を定める周囲の語 | 語カードと同形、脇から滑り込む | ボディ1・ボディ2 |
| 関係の矢印 | 語順・どの語がどの語にかかるか | カード間の向きつき矢印。向きの反転で関係が逆転 | ボディ1 |
| 辞書 | 昔の機械の、文脈によらない固定の一義 | 旧式の図。灰色の固定指紋を貼り出す | ボディ1 |
| Q/K/V グリフ | 語が持つ三つの顔と、相性で噛み合う仕組み | Query＝ソケット型／Key＝タブ型／Value＝意味指紋 | ボディ2 |
| 照合 | Query と Key の相性が注目の強さを決める | ソケットにタブを合わせ、噛み合えば施錠して発光 | ボディ2 |
| 注目配分バー | 注目の総量は固定で、相手へ配分される | 全長一定の帯を、相手ごとの区画に分割 | ボディ2 |

| 注目グリッド | 全ペアの注目を一望する（一斉処理） | N×N グリッド、セルの濃さ＝注目の強さ | ボディ3・ボディ4・結論 |
| 色付きグリッド | マルチヘッド＝視点ごとに違う関係を見る | 注目グリッドが複数色に分裂・重畳 | ボディ3・結論 |
| 層スタック | 層を積むと意味指紋が細部から全体へ深まる | 語カード列を上方向へ段々に複製、指紋が変化 | ボディ3・結論 |
| 位置の波 | 順番の目印（位置エンコーディング） | 位置ごとに違う波形スタンプ、意味指紋に加算 | ボディ3 |
| RNN 単一状態 | 逐次処理とバケツリレー（遅い・遠くが薄れる） | 一個の状態トークンが左→右へ移動、成分が減衰 | ボディ4 |
