# diffusion — design spec

`design_overall.md` の質的な動きを event アンカーに分解し、`script.md` の各セリフへ割り当てる。座標値・フレーム数は持たない。

対話字幕は event ではない——全セリフが自動で画面下部の字幕になる（tsx の仕事）。ここで定義する event は、絵の装置の挙動だけを指す。

## 1. event 一覧と割り当て

発生順。画面切替 event（`scene.*.in`）は各画面の先頭セリフに必須。

| event | 紐付くセリフ（画面・話者・抜粋） | 発火する挙動 |
|---|---|---|
| `scene.intro.in` | 序論・めたん「言葉を打ちこむだけで、絵を描いてくれるAI…」 | 序論の画面がフェードイン。キャンバスが現れ、まっさらな明るい紙を映す |
| `intro.subtract` | 序論・ずんだもん「足すんじゃなく、引いていくのだ。」 | 紙の上の線が消え、キャンバス全面が砂あらしに変わる（ノイズ量0→1） |
| `intro.reveal` | 序論・ずんだもん「絵になるのだ。霧が晴れて、奥の景色が見えてくる──…」 | 砂あらしのノイズ量がすこしずつ下がり、ねこのドット絵が立ち現れる |
| `intro.mystery` | 序論・めたん「…ただの砂あらしの奥に、絵なんて入ってないでしょ？」 | キャンバスが砂あらしに戻り、虫めがねが一点に寄る。粒しか映らない |
| `intro.name` | 序論・ずんだもん「この仕組みには、名前があるのだ。拡散モデル、というのだ。」 | キャンバスが砂あらしと絵の中間でとどまり、下にタイトル「拡散モデル」が灯る |
| `scene.body1.in` | ボディ1・ずんだもん「ここに、きれいな写真が一枚あるとするのだ。…」 | 画面切替。主役キャンバスが中央上に現れ、鮮明なねこを映す |
| `b1.sprinkle` | ボディ1・ずんだもん「この写真に、ほんの少しだけ、ノイズを振りかけるのだ。…」 | ノイズが振りかけられキャンバスのノイズ量がわずかに上がる。いまの姿が階段の一段目に写し取られる |
| `b1.repeat` | ボディ1・ずんだもん「そこへ、もう一回。さらに、もう一回、振りかけるのだ。」 | 振りかけがくりかえされ、キャンバスがざらつき、階段が一段ずつ右下へ伸びる |
| `b1.full` | ボディ1・めたん「……あっ、もう、ねこの形が残ってない。ただの砂あらし。」 | 階段が最下段（完全な砂あらし）に達し、主役キャンバスも砂あらしになる |
| `b1.small` | ボディ1・ずんだもん「…「ほんの少し」を、何十回も重ねたのだ。」 | 階段の隣り合う二段が並んで強調され、一段の差がごく小さいことを示す |
| `b1.record` | ボディ1・ずんだもん「…一段ずつ控えておくのだ。」 | 階段の各段の上に、その段で足したノイズの記録ノイズ片が一枚ずつ灯る |
| `scene.body2.in` | ボディ2・ずんだもん「機械に出すのは、こういう問題なのだ。…」 | 画面切替。上辺にノイズ階段、中央にノイズ予測器が立つ |
| `b2.pick` | ボディ2・ずんだもん「そう。たとえば、半分くらい壊れた、ねこを取り出すのだ。」 | 階段の途中の一段が出題キャンバスとして中央左に据えられる |
| `b2.guess` | ボディ2・ずんだもん「機械はじっと絵を見て、答えを出すのだ。「足されたノイズは、たぶん、これ」と。」 | ノイズ予測器が出題キャンバスを読み、右上へ予測ノイズ片を一枚はき出す |
| `b2.check` | ボディ2・ずんだもん「機械の答えと、記録された正解。この二つを、ならべて見くらべるのだ。」 | 予測ノイズ片の下に記録ノイズ片（正解）が灯り、二枚の下のずれメーターの針がずれ幅まで振れる |
| `b2.fix` | ボディ2・ずんだもん「ずれていたら、機械の中身を、ほんの少し調整するのだ。…」 | ノイズ予測器の中身がわずかに整い、ずれメーターの針が少し戻る |
| `b2.loop` | ボディ2・ずんだもん「別の段を選び、別の写真を選ぶ。…」 | 出題キャンバスに犬・山・顔の写真が次々と切り替わり、予測→答え合わせ→修正が高速反復。反復カウンタが跳ね上がり、ずれメーターの針が回ごとに小さくなる |
| `b2.skill` | ボディ2・めたん「つまり機械は、たった一つの技を、身につけたってこと？」 | 反復が収束し、ノイズ予測器が安定して光る（技が定着） |
| `scene.body3.in` | ボディ3・めたん「ノイズ当てが、絵を描く力になる。まだ、ぴんと来ないな。」 | 画面切替。砂あらしの主役キャンバスと、右脇に小型のノイズ予測器が現れる |
| `b3.subtract` | ボディ3・ずんだもん「絵から、当てたノイズを引くのだ。すると、絵が、ほんの一段ぶん、きれいになるのだ。」 | ノイズ予測器が予測ノイズ片を出し、引き算記号でキャンバスから引かれ、ノイズ量が一段下がる（階段を一段のぼる） |
| `b3.climb` | ボディ3・めたん「当てて、引く。当てて、引く。一段ずつ、階段をのぼっていく。」 | 予測→引く→のぼるが一段ずつ反復し、キャンバスがすこしずつ鮮明になっていく |
| `b3.top` | ボディ3・ずんだもん「のぼりきれば、ノイズはすっかり消えて、きれいな絵だけが残るのだ。」 | のぼり階段が頂上に達し、キャンバスのノイズ量が0になって元のねこが戻る |
| `b3.fresh` | ボディ3・ずんだもん「…その場で作った、まっさらででたらめな砂あらし。」 | 主役キャンバスが、元の絵を持たない別の砂あらしにすりかわる（のぼり階段はリセット） |
| `b3.emerge` | ボディ3・ずんだもん「何もなかったはずの砂あらしから、見たこともない絵が、ぼうっと浮かび上がってくるのだ。」 | 新しい砂あらしから予測→引くをのぼり、見たことのない新しい絵が立ち現れる |
| `b3.learned` | ボディ3・ずんだもん「…「ノイズの少ない絵は、こういう手ざわり」だと。」 | 背後に学習で見た写真サムネ群が淡くよぎり、ノイズを引くたびキャンバスが「より絵らしいほう」へ寄ることを示す |
| `b3.variety` | ボディ3・ずんだもん「ちがう絵になるのだ。出発の砂あらしが変われば、…」 | 出発の砂あらしを変えた数組（砂あらし→絵）が横に並び、それぞれ違う絵にのぼり着く |
| `scene.body4.in` | ボディ4・めたん「こまったこと、って？」 | 画面切替。砂あらしの主役キャンバスと、のぼり階段が現れる |
| `b4.random` | ボディ4・ずんだもん「…何が出るかは、砂あらしまかせ。」 | キャンバスの先が複数の到達先候補（ねこ／山／花）へふらふら枝分かれする |
| `b4.prompt` | ボディ4・ずんだもん「…打ちこんだ言葉──プロンプトを、使うのだ。」 | プロンプト文字列「宇宙服を着たねこ」が画面上部に現れる |
| `b4.compass` | ボディ4・ずんだもん「…「こっちが宇宙服のねこの方角」と指す、矢印にするのだ。」 | プロンプト文字列が方位磁針へ姿を変え、針が「宇宙服のねこ」の方角を指す |
| `b4.steer` | ボディ4・ずんだもん「一段ごとに、すこしずつ、宇宙服のねこのほうへ舵を切る。…」 | のぼる一段ごとに方位磁針が参照され、枝分かれが一本の道へ束ねられ、キャンバスが宇宙服のねこへ寄っていく |
| `b4.dial` | ボディ4・ずんだもん「…この方位磁針は、効き目を強くも弱くもできるのだ。」 | 強度つまみが現れ、弱（注文から外れる）↔強（注文どおりだが硬い）を行き来し、ほどよい位置に止まる |
| `b4.result` | ボディ4・ずんだもん「そうやって、でたらめな砂あらしから、注文どおりの絵を引き出すのだ。」 | のぼり階段が頂上に達し、キャンバスに宇宙服を着たねこのドット絵が結ぶ |
| `scene.outro.in` | 結論・ずんだもん「ここまでを、絵といっしょに振り返るのだ。」 | 画面切替。中央にキャンバス、その上に空の工程アイコン帯が現れる |
| `outro.recap` | 結論・ずんだもん「はじまりは、一枚のきれいな写真を、すこしずつノイズで壊すこと。」 | 工程アイコン（壊す→練習→のぼる→方位磁針）が順に灯り、キャンバスが全工程を早送りでなぞる |
| `outro.loopback` | 結論・ずんだもん「…「砂あらしの奥に、絵は隠れているのか」という不思議。」 | キャンバスが砂あらしになり、虫めがねが一点に寄る。やはり粒しか映らない |
| `outro.end` | 結論・ずんだもん「機械が絵を「描く」んじゃなく「霧から見つけ出す」しくみ──…」 | のぼる道のりが光の筋として描かれ、その先にキャンバスが宇宙服のねこを映して終わる |

内容 event は31個（全193セリフの約16%）。「5〜15%目安」をわずかに上回るが、15分前後・6画面の長尺で、各画面が複数の独立した工程を順送りで扱い、工程ごとに専用装置を立ち上げ・畳みする題材のため。装置の登場・退場そのものが説明の本体なので、event は装置の数に比例する。

## 2. ライフサイクル契約

## 3. オブジェクト別 挙動メモ

複数 event をまたいで連続変化するオブジェクトの、event 間の状態の向き先。座標・フレーム数は持たない。

**キャンバス（背骨）** — 全画面の主役。持つ数値は「ノイズ量」（0＝鮮明な絵／1＝完全な砂あらし）と、映す絵の種類。
- 序論：画面入場で紙（線を足す表示）→ `intro.subtract` でノイズ量0→1 → `intro.reveal` で1→0付近（ねこが現れる）→ `intro.mystery` で再び1 → `intro.name` で中間値（0.5付近）で保持。
- ボディ1：画面入場でノイズ量0（鮮明なねこ）→ `b1.sprinkle` でわずかに上昇 → `b1.repeat` で段階的に上昇 → `b1.full` で1。以後保持。
- ボディ2：出題キャンバスは `b2.pick` 以降ノイズ量0.5付近で保持。`b2.loop` の反復中だけ、映す絵の種類が犬・山・顔と切り替わる。
- ボディ3：画面入場でノイズ量1 → `b3.subtract`〜`b3.top` で1→0（ねこが戻る）→ `b3.fresh` で別の砂あらしに替わりノイズ量1へ戻す → `b3.emerge` で再び1→0（新しい絵）。映す絵は `b3.fresh` で別物に切り替わる。
- ボディ4：画面入場でノイズ量1 → `b4.steer`〜`b4.result` で1→0、映す絵は宇宙服のねこへ収束。
- 結論：画面入場でノイズ量0（完成した宇宙服のねこ）→ `outro.recap` で全工程を早送り（0→1→0）→ `outro.loopback` で1 → `outro.end` で1→0（宇宙服のねこに戻り保持）。

**ノイズ階段** — ボディ1・2・3・4・結論に登場。崩れ具合の違う小キャンバスの列。ボディ1：`b1.sprinkle` で一段目が生まれ、`b1.repeat`〜`b1.full` で段数が増える（下る方向）。`b1.small` で隣り合う二段が強調され、`b1.record` で各段の上に記録ノイズ片が点灯。ボディ3：`b3.subtract` で一段目、`b3.climb`〜`b3.top` で段が増える（のぼる方向）。`b3.fresh` で階段がリセットされ、`b3.emerge` で新しい砂あらしから再び段が増える。

**ノイズ予測器** — ボディ2で誕生し技を獲得、ボディ3・4へ小型で持ち越す（画面リセットを挟むので別個体だが、視聴者には「同じ機械」と感じさせる）。ボディ2：`scene.body2.in` で出現（中身は未熟）→ `b2.guess` で予測ノイズ片を出す → `b2.fix` で中身がわずかに整う → `b2.loop` の反復で安定へ向かう → `b2.skill` で安定して光る。ボディ3・4：脇に控え、`b3.subtract`／`b4.steer` のたびに予測ノイズ片を出す。

**ずれメーター（ボディ2）** — `b2.check` で誕生し、針が予測と正解のずれ幅まで振れる。`b2.fix` で針が少し戻る。`b2.loop` の反復で、針の振れ幅が回を追うごとに小さくなり、ずれが収束する。

**方位磁針（ボディ4）** — `b4.prompt` でプロンプト文字列が生まれ、`b4.compass` でそれが方位磁針へ姿を変える。針は「宇宙服のねこ」の方角を指して保持。`b4.steer` ののぼり中、針はキャンバスが引かれる向きを補正し続ける。`b4.dial` で強度つまみの位置に応じて補正の強さが弱↔強に変わる。

**虫めがね** — 序論と結論に登場。いずれも「砂あらしの一点に寄り、粒しか映らない（奥に絵は無い）」という同じ振る舞い。序論では不思議の提示、結論ではその不思議への回帰として。

## 4. 採否判断

- [x] design_overall の各画面の「動き」がすべて event に分解されている
- [x] 各 event が `script.md` の実在するセリフに紐付いている
- [x] 画面切替 event（`scene.*.in`）が各画面の先頭セリフに付いている（6画面ぶん）
- [x] 内容 event の量：31個・約16%（理由を §1 末尾に明記）
- [x] 全オブジェクトにライフサイクル契約（誕生・死・出生元・退場先）がある
- [x] 出生元・退場先がすべて既存オブジェクトか画面入退場（何もない所から湧いていない）
- [x] 座標値・フレーム数値を書いていない
- [x] 対話字幕を event 扱いしていない（字幕は SCRIPT 全行から自動描画）

## 次段への引き継ぎ

`Composition.tsx` 実装（`04_remotion.md`）へ：

- **event 一覧** → `scriptData.ts` の該当セリフに `event` を付与。各 event のセリフ開始フレームが Track のキーフレーム位置
- **ライフサイクル契約** → 各オブジェクトの opacity Track（誕生 0→1／死 1→0）と画面可視性ゲート
- **挙動メモ** → キャンバス（ノイズ量）・ノイズ階段・ノイズ予測器・ずれメーター・方位磁針の状態 Track の組み立て方針
- 座標・サイズ・フレーム数値は tsx 内で直書きし、プレビューで現物合わせする
