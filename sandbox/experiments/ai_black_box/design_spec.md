# セリフ単位スペック — AIのブラックボックスを開けてみる

`design_overall.md` の背骨（黒い箱を透明な変換工場へ開いていく）を、event に分解して `script.md` のセリフへ紐付ける。

## 画面 1: intro

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.intro.in | め「AIって、文章も書くし、絵も作るし」 | 中央に黒い箱 `AI`、周囲に文章・画像・コード・音声の小アイコンが浮く |
| icons.orbit | め「あれ、中で何が起きてるのか全然わからない」 | アイコンが箱へ吸い込まれ、箱の表面に「？」が灯る |
| doctor.false | め「小さい博士でも入ってるのかと思うわ」 | 箱の中に博士シルエットが一瞬出て、赤い取り消し線で消える |
| search.false | め「じゃあ巨大な検索エンジン？」 | 検索窓シルエットが出て、これも取り消される |
| box.open | ず「中には、入力を数字に変えて、何層もの変換を通し」 | 箱が観音開きになり、数字粒・層・出力ゲートの簡略工場が見える |
| question.pin | ず「なぜ、ただの計算機から、いまのように高性能なAIが作れるようになったのか」 | 工場の上に大きな問いのピンが立ち、画面が静止する |

## 画面 2: map

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.map.in | ず「最初に、AIという言葉を少しだけ整理する」 | 大きな円 `AI` が現れ、その中に複数の領域が薄く区切られる |
| map.branches | ず「迷路を探索するAI、ルールで判断するAI」 | `探索` `ルール` `確率推定` `機械学習` `深層学習` `生成AI` の領域名が順に点灯 |
| map.chatgpt.apart | め「全部ChatGPTの親戚だと思ってたわ」 | `ChatGPTだけ` という小さなラベルが広いAI円から外れて縮む |
| deep.zoom | ず「今よく話題になる文章生成や画像生成は、深層学習の系統」 | `深層学習` 領域へズームし、その中の `大規模ニューラルネット` が強調される |
| scope.frame | ず「この動画で開けるブラックボックスは、AI全部ではなく」 | 画面に太い枠が出て、今回のスコープだけを囲む |
| modalities.in | ず「文章AIなら大規模言語モデル、画像AIなら拡散モデル」 | 枠の周囲に文章・画像・音声の小窓が並び、中核回路へ線でつながる |

## 画面 3: weights

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.weights.in | ず「AIの中には、人間が書いた巨大なルール表が入っていると思われがち」 | 棚いっぱいのルールカードが箱の中に表示される |
| rules.fade | ず「そういうルールを人間が一つずつ書いているわけではない」 | ルールカードが薄くなり、背後の層回路が見え始める |
| knobs.in | ず「重みと呼ばれる数字のつまみが大量にある」 | 層の間に大量の小さなつまみと線が現れる |
| signal.flow | ず「入力が入ると、その数字のつまみによって信号の強さが変わり」 | 左から数字粒が入り、つまみを通るたび太さや色を変えて右へ流れる |
| knobs.distributed | ず「つまみ一個に『猫の耳』とか『敬語』とか名前が付いているわけではない」 | ひとつの意味ラベルが一つのつまみに貼られかけ、ばらけて多数のつまみに分散する |
| transparent.maze | ず「中身の数字は見られる。でも、その数字の組み合わせが」 | 回路全体が透明化し、数字は見えるが複雑な迷宮線として重なる |

## 画面 4: training

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.training.in | ず「ニューラルネットは、最初から賢いわけではない」 | 乱れたつまみ配置の回路が現れ、出力が的から外れる |
| loss.meter | ず「どれくらい外れたかを測るのだ」 | 出力と正解の距離が赤いメーター `損失` として表示される |
| loop.start | ず「損失が大きいなら、重みのつまみを少し変える」 | 損失メーターから逆向きの赤い信号が層へ戻る |
| knobs.adjust | ず「どのつまみをどっちに回すと損失が減りそうかを、計算でたどる」 | いくつかのつまみが小さく回り、次の出力が的へ近づく |
| tasks.swap | ず「文章AIなら、文章の続きを予測する練習」 | 的が `次の言葉`、次に `ノイズから画像` へ切り替わる |
| pattern.choice | ず「見た例をそのまま保存しているから出るのか、それとも」 | `丸暗記` と `パターン圧縮` の二択が出て、後者だけが重み回路へ接続される |
| terrain.in | ず「間違いが大きい高い場所から、間違いが小さい低い場所へ」 | 回路が高次元地形の簡略図へ変形し、点が谷へ少しずつ降りる |
| train.freeze | ず「チャット中にやっているのは、学習済みの重みを使って答えを出す『推論』」 | 調整ループが止まり、重みに錠前が掛かって次画面へつながる |

## 画面 5: inference

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.inference.in | ず「推論では、まず入力がコンピュータの扱える形に変換される」 | 入力文が左に現れ、切り分けゲートへ入る |
| token.cut | ず「文章なら、まず小さな部品に切られる」 | 文がトークン粒へ分割される |
| vector.in | ず「その部品は数字のベクトルに変えられる」 | トークン粒が縦バーの数字ベクトルへ変形する |
| layers.pass | ず「その数字の列が何層もの変換回路を通る」 | ベクトル列が複数の層を左から右へ通過し、色が変わる |
| attention.arc | ず「Attentionは、他のどの部品をどれくらい見るかを決める」 | トークン同士に太さの違う弧が引かれる |
| attention.example | ず「『それ』を考える時、本に強く注目する」 | `それ` から `本` への弧だけ太く光る |
| transformer.stack | ず「その処理を何層も重ねるのが、Transformer」 | 弧付きの層が縦に積み上がり、表現が上へ行くほど整理される |
| output.dist | ず「次に出すトークンの候補に確率を付ける」 | 右端に候補語チップが確率バー付きで並び、最有力が出力へ出る |

## 画面 6: scale

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.scale.in | ず「今のAIが強くなった理由は、一つではない」 | 変換工場の下に四つの空の支柱スロットが現れる |
| pillar.arch | ず「まず構造。Transformerは、文の中の離れた関係を扱いやすく」 | 1本目の支柱 `構造` が立ち、Attention線が橋のように伸びる |
| pillar.data | ず「次にデータ。インターネット上の文章、画像、音声、コード」 | 2本目の支柱 `データ` が立ち、多様なデータ粒が工場へ流れ込む |
| data.quality | ず「量だけではなく、質や重複の処理も重要」 | データ粒の一部がフィルタで除かれ、整った粒だけが残る |
| pillar.compute | ず「三つ目が計算資源。GPUなどの並列計算機」 | 3本目の支柱 `計算` が立ち、多数の小コアが同時に点灯する |
| balance.scale | ず「モデルサイズ、データ量、計算量を増やすと性能がかなり規則的に伸びる」 | 三つのスライダー `モデル` `データ` `計算` が連動して伸び、性能メーターが上がる |
| pillar.align | ず「四つ目が、人間に合わせる工程」 | 4本目の支柱 `合わせ込み` が立ち、人間評価のチェックが出力ゲートへ入る |
| factory.grow | ず「この積み重ねが、今のAIを作っている」 | 四本の支柱に支えられて工場全体が大きくなり、文章・画像・音声の出口が増える |

## 画面 7: limits

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.limits.in | め「でも結局、ブラックボックスなんでしょ？」 | 透明工場の一部に黒い影が戻る |
| probes.in | ず「中の表現を調べたり、どの入力が出力に効いたかを見たり」 | 虫眼鏡・プローブ線・ヒートマップが回路上に重なる |
| partial.light | ず「部品は見える。動きもある程度測れる」 | 回路の一部が明るく可視化される |
| maze.remains | ず「いつでも人間にわかる説明へ完全に翻訳できるわけではない」 | 可視化されない部分が透明な迷宮として残る |
| split.two | ず「内部でどう計算しているか。もう一つは、その出力を現実の判断に使っていいか」 | 画面が `内部計算` と `現実判断` の二領域に分かれる |
| verify.line | ず「どこから検証が必要なのかを見分けること」 | 出力から現実側へ太い検証ラインが引かれる |
| factory.human | め「数字の巨大工場はあったわけね」 | 工場の外側に人間の操作盤が現れ、次の画面へ畳まれる |

## 画面 8: outro

| event | 紐付くセリフ（抜粋） | 何が起きるか |
|---|---|---|
| scene.outro.in | ず「最初の問いは、『なぜ、ただの計算機から高性能なAIが作れるようになったのか』」 | 導入の黒い箱が戻るが、半透明になっている |
| recap.flow | ず「入力を数字の表現に変え、重みの回路で何層も変換し」 | 箱の中で `入力→数字→層→出力` の流れが一筆で再生される |
| recap.scale | ず「Transformerのような構造、膨大なデータ、強い計算資源」 | 四本支柱が小さく再点灯する |
| box.label | ず「中身は見える。ただ、その意味が人間の言葉にきれいに分解されていない」 | 箱の表面ラベルが `魔法` から `学習された変換回路` へ貼り替わる |
| final.boundary | ず「箱の中で数字がどう変わっていくのかを見ること」 | 地図上に `使える領域` と `検証が必要な領域` の境界線が引かれて静止 |

## 各オブジェクトの誕生と死

| オブジェクト | 誕生 | 死 |
|---|---|---|
| 黒いAI箱 | scene.intro.in | box.open で透明工場へ変身、scene.outro.in で半透明箱として復帰 |
| 文章・画像・コード・音声アイコン | scene.intro.in | box.open |
| 博士シルエット | doctor.false | doctor.false 内で即退場 |
| 検索窓シルエット | search.false | search.false 内で即退場 |
| AI全体地図 | scene.map.in | scene.weights.in |
| スコープ枠 | scope.frame | scene.weights.in |
| ルールカード棚 | scene.weights.in | rules.fade |
| 重みつまみ回路 | knobs.in | training 以降も形を変えて継続、scene.outro.in で箱内へ畳む |
| 信号粒 | signal.flow | 各変換の終端で出力へ吸収 |
| 損失メーター | loss.meter | train.freeze |
| 逆向き学習信号 | loop.start | knobs.adjust |
| 二択ラベル | pattern.choice | terrain.in |
| 高次元地形 | terrain.in | train.freeze |
| 錠前 | train.freeze | scene.inference.in |
| トークン粒 | token.cut | vector.in（ベクトルへ変身） |
| 数字ベクトル | vector.in | layers.pass で層表現へ吸収 |
| Attention弧 | attention.arc | transformer.stack で層内表現へ統合 |
| 候補語チップ | output.dist | scene.scale.in |
| 四本支柱 | scene.scale.in | scene.limits.in で縮小退場、recap.scale で再登場 |
| プローブ・ヒートマップ | probes.in | split.two |
| 透明な迷宮 | maze.remains | final.boundary まで残る |
| 検証ライン | verify.line | final.boundary で境界線へ統合 |

同時表示の最大は画面6の「工場＋支柱＋データ粒＋性能メーター」、画面7の「工場＋プローブ＋迷宮＋検証ライン」で4個以内に収める。
