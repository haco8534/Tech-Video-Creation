# cloud_storage — design overall

`script.md` を入力に、動画全体を貫通する世界観と、7画面それぞれのデザインを自然言語で設計する。座標値・フレーム数・セリフ単位のタイミングは持たない（design_spec と Composition.tsx の仕事）。

図解は概念に**ラベルを貼る**のではなく、仕組みを**動いて見せる**。「データ」「動画」と書いた箱は出さない。一塊の素材が mp4 の部屋に入ろうとして頭の印にぶつかり、色のマス目にほぐれて動画になり、サーバーの圧縮で一マスだけ別の色に化け、画像や文字列に変装し、最後はマイクラのブロックの山に積み変わる——その変装の連鎖そのものを画面の上で起こす。本作の図は、汎用の箱・矢印の寄せ集めではなく、**ひとつの仕組みをひとつ可視化する専用装置**の集まりとして設計する。

## 背骨

貫通する背骨は **一塊の素材タイル**。オレンジ色の小さな四角タイルで、表面にうっすら "0 1 1 0 1 0 0 1..." と薄字でビット列が浮かんでいる。一個一個は離散した粒で、寄り集まると一塊になる。

動画全体は、この一塊の素材タイルが場所を変えるたびに姿を変えていく一本の物語になる。序論では素材の塊が「データそのもの（0と1の並び）」として現れる。ボディ1では mp4 の建物の中に押しこまれようとして弾かれる。ボディ2では色のマス目に展開されて動画フレームに化け、ぱらぱらめくられて YouTube に流れこむ。ボディ3ではサーバーの中で一マスだけ色が揺らされ、列の一文字が化ける。ボディ4では別の建物（Discord）に向かって、こんどは画像一枚、もしくは文字列の山として変装する。ボディ5ではマインクラフトのブロックの山に立体的に積み変わる。結論では、これまで通ってきた変装の連鎖が一筋に並び、最後のフレームで素材タイルが DNA らせんの上に小さく一粒として置かれる。

すべての画面で、素材タイルは消えない・新しく湧かない。常に同じ粒が、置かれる場所と並べかたを変えていく。

## 1. 世界観

白テーマ（背景・面が白系）と文字サイズ下限は固定ベース（→ `02_design_overall.md`「固定ベース」）。比喩・図の語彙・アクセント色を、脱法クラウドストレージ固有に設計する。

| 軸 | この題材での設定 |
|---|---|
| 貫通する比喩 | **置き場所と中身の食い違い**。画面の中央に「いま舞台になっている建物」が一つ立つ（mp4 の建物／YouTube のサーバー塔／Discord の窓／マインクラフトの地面）。建物のかたわらに素材の塊が一塊あり、建物のかたちに合わせて変装しながら、入口にすべりこもうとする。視聴者は建物が切り替わっても、同じ素材の塊がどう変装するかを追えるよう、塊の表面のビット列はうっすら一貫して透けつづける |
| 形の語彙 | 専用の図解装置の集まり（一覧は §3 図カタログ）。背骨は **素材タイル**（オレンジの正方形、表面に薄いビット列）。建物：**mp4 の家**（小さな目印部屋＋おっきな中身部屋の二部屋）、**YouTube サーバー塔**（圧縮の歯車が一つ）、**Discord の窓**（画像枠と文字列の二つの口）、**マインクラフトの地面**（種類ちがいのブロックを並べる立体格子）。変装：**色のマス目**（4色／多色）、**ぱらぱら動画**（マス目を時間方向に重ねた束）、**文字化け文字列**、**ブロックの立体格子**。汎用の「ラベル付き箱」「意味のない矢印」「テキストだけのカード」は使わない |
| 色の語彙 | 背景・面は白テーマで固定（背景 `#f5f7fa`／パネル面 `#ffffff`／枠 `#c4cedd`／主文字 `#243044`）。アクセント（題材ごと）：**素材＝置きたいデータ** ＝アンバー `#d99a2b`（背骨の素材タイルが常にこの色。表面のビット列だけ濃い藍）。**変装の場＝動画・画像・テキスト・ブロック**＝ティール `#2f8fb3`（建物の枠、フレームの輪郭）。**失敗・弾かれる・規約違反・化ける**＝コーラル `#d9543c`（mp4 の目印が崩れる×、ピクセルが化ける、規約 BAN）。**成功・置けた・戻ってきた**＝グリーン `#3f9d57`（動画として受理される✓、デコード成功）。通信路・補助線・退いた要素＝グレー |
| 動きの語彙 | 使う＝フェード、建物への寄せ／退き、素材タイルが解けて色のマス目に並ぶ／集まって元に戻る、マス目の束がぱらぱらめくれる、サーバーの歯車が一回転して一マスだけコーラルに化ける、文字列が下に流れる、ブロックが立体格子の上に降ってくる、変装の連鎖が一筋に並ぶ。封印＝3D回転（マイクラ格子の表示は静止カメラで側面図に留める）、カメラのズーム／パン、バウンド、急な明滅、**ループし続ける動き（浮遊・振動・明滅・脈動・常駐背景モーション）**。建物の歯車もループしない——会話の進行にあわせて一度回って止まる |
| 文字方針 | 画面下部に**対話字幕**を常駐（話者名は話者色、本文は濃い藍、スマホ視聴前提で大きめ・2行まで）。図解内のテキストは最小限：素材タイル表面の薄いビット列、建物の名前（mp4／YouTube／Discord／Minecraft）、変装後のラベル（「色のマス目」「ぱらぱら動画」「文字化け」「ブロックの山」）。「データ」「動画」のような抽象ラベルは置かない——形と色そのもので意味を運ぶ。サイズは固定ベースの下限を守る |
| ナビ補助 | 画面が切り替わるとき、左上に小さくセクション番号と見出し（例「02 ピクセルにほどく」）。進捗ドット等は置かず最小限。字幕帯とは別の位置 |

レイアウトの基準：中心原点。**画面下部 y≈330〜510 は対話字幕の帯**として確保し、主オブジェクトはそれより上に置く。建物は画面中央〜やや右に大きく据え、素材タイルとその変装は建物の入口側（左〜中央）に置く。1画面に装置を全部いっぺんに出さない——会話が進むにつれ装置が順に立ち上がり、役目を終えたら畳まれ、次の装置に場所を譲る。

## 2. セクション別デザイン

### 序論（画面1）：データに、動画のふりをさせる

- **登場する図**：素材タイルの塊（小タイル多数）、変装前の三つの異種ファイル絵（マイクラのブロック山・画像内の音符・動画の中の歯車）、タイトル「脱法クラウドストレージ」
- **レイアウト**：画面中央に素材タイルの塊。その周りの三方向に、変装後の姿として三つの小さな絵がうっすら浮かぶ。タイトルは塊の下、字幕帯の上の余白に。
- **動き**：三つの異種ファイル絵（マイクラ山／画像＋音符／動画＋セーブデータ）が順に画面に立ち上がり、「中身と場所が食い違っている」ことを並べて見せる → それぞれの絵から、共通の素材タイルが一塊ずつ吸い出されて画面中央に集まる。集まった塊の表面に "0 1 1 0..." のビット列がうっすら浮かぶ（どれも同じ素材だった）→ 塊が一度ほぐれて、ばらばらの粒の群れになり、再び塊に戻る——同じ粒は、並べかたを変えれば何にでもなれる → タイトル「脱法クラウドストレージ」が灯る
- **絵の眼目**：場所と中身は決めつけているだけ。同じ0と1の並びが、置き場所に合わせて変装している

### ボディ1（画面2）：mp4 のお部屋に、こっそり詰める

- **登場する図**：素材タイルの塊、mp4 の家（頭の小さな目印部屋＋おっきな中身部屋）、YouTube の受付口（建物の入口）、目印の崩れ印（コーラルの×）、受付の「不受理」印（コーラルの×）
- **レイアウト**：画面中央に mp4 の家。家は左右に二部屋に区切られ、左の小部屋に「目印」、右のおっきな部屋に「中身」。家の右に YouTube の受付口（小さな門）。素材タイルは画面左の手元ゾーン。
- **動き**：mp4 の家が立ち上がり、左の小部屋と右のおっきな部屋に分かれる → 家の中身の九割が右部屋であることを、部屋の大きさそのもので示す → 素材タイルの塊が左から右部屋に流しこまれ、右部屋を埋める。動画としての中身は押しのけられて消える → 同時に、左の小部屋の目印（短い棒状のマーク）が、押しこみの衝撃でぐにゃりと崩れ、コーラルの×が灯る → 家ごと YouTube の受付口に運ばれるが、受付の門で家が止められ、入口の上にコーラルの「不受理」が灯る
- **絵の眼目**：おっきな部屋を塗りつぶすと、頭の小部屋の目印もいっしょに崩れる——動画である印が消えるから、動画として受けつけてもらえない

### ボディ2（画面3）：ピクセルの色に、ほどいて並べる

- **登場する図**：素材タイルの塊、色のマス目（4色：黒・赤・緑・青）、ぱらぱら動画の束（マス目を奥行きに重ねた層）、YouTube の受付口、受理印（グリーンの✓）
- **レイアウト**：画面左に素材タイルの塊、中央に色のマス目、右にぱらぱら動画の束（マス目が手前から奥へ層をなす）、その先に YouTube の受付口。
- **動き**：素材タイルの塊がほぐれて、一粒一粒が中央の格子に並ぶ → 並んだ粒は黒・赤・緑・青の四色に染め分けられ、画面いっぱいの色のマス目になる（一マスに二つぶんのビットが詰まっていることを、色の段階で示す）→ マス目が一枚ぶん完成すると、その後ろにもう一枚、さらにもう一枚と層が重なってぱらぱら動画の束ができる。束は手前から奥に時間方向の厚みを持つ → 束が YouTube の受付口に運ばれ、こんどは受理印のグリーンの✓が灯る
- **絵の眼目**：データを建物の中に隠さず、表面の色として描く。色のマスの集まりが「動画として再生可能なふり」をしながら、中身は完全にデータ

### ボディ3（画面4）：取り出すときに、欠ける

- **登場する図**：ぱらぱら動画の束、YouTube サーバー塔（圧縮の歯車一つ）、化けたマス目（塊の中の一マスだけコーラル）、化けたビット列（"0 1 [1→0] 0..." の薄字に赤線）、画質の粗いマス目（マスが大きくなった版）、データの肥大化を示す目盛り（×20）
- **レイアウト**：画面左にぱらぱら動画の束、中央に YouTube サーバー塔（円筒の中に歯車が一つ）、右に塔から出てきた化けマス目と化けビット列。下に画質を落とした粗マス目を補助として配置。塔の傍らに小さく「×20」目盛り。
- **動き**：左のぱらぱら動画の束が、サーバー塔の入口に流れこむ → 塔の中の歯車が一回転（圧縮）し、出口から動画の束が出てくるが、塊の中の一マスだけがコーラルに染まっている（化けた）→ そのマスを拡大すると、ビット列の真ん中の一字に赤線が引かれる（0 が 1 になった）→ 対策として、入口側のマス目が大きく描き直され、粗マス目になる（化けにくくなる代わりに、運べる量は減る）→ 同時に、塔の傍らで「×20」目盛りが伸び、もとの素材タイルに対して二十倍の動画サイズになることが目盛りそのもので可視化される
- **絵の眼目**：圧縮で一マスが化けただけで、データ全体が読めなくなる。マスを大きくすれば守れるが、運べる量が減る——詰めるか、戻すかの天秤

### ボディ4（画面5）：別のサービスに、別のすがたで置く

- **登場する図**：素材タイルの塊、Discord の窓（建物。中に二つの口：画像枠と文字列枠）、変装1＝画像一枚（多色ピクセルで埋まった大きな枠）、変装2＝文字化け文字列（縦に長く流れる文字の柱）、規約 BAN 印（コーラルの止め札）
- **レイアウト**：画面中央に Discord の窓。建物の中が左右に分かれ、左口に画像枠、右口に文字列の柱。素材タイルは画面左の手元ゾーン。BAN 印は建物の上に小さく予兆として浮かぶ。
- **動き**：mp4 の家から Discord の窓に舞台が切り替わる。建物の入口に「一回ぶんの大きさに上限」を示す小さなゲートが置かれる → 素材タイルが左に集まり、変装ルートを二つに枝分かれする：上ルートは多色ピクセルの画像一枚に変装し、Discord の左口に投函される（サーバーで作りなおされず、四色より多くを使えることを色の豊富さで示す）。下ルートは画像にせず、拡張子だけ付け替えて文字化け文字列の柱になり、右口に貼りつけられる → どちらのルートも入口を通る → 建物の上で BAN 印（コーラルの止め札）が小さく点り、置いた塊と画像と文字列がいっせいに薄く消えかかる（規約で止められるリスクの可視化）
- **絵の眼目**：場所が変われば変装の中身も変わる。画像にも文字列にもなれる——ただし、置けることと、ちゃんと使えることは別

### ボディ5（画面6）：マイクラのブロックの山に、変装する

- **登場する図**：素材タイルの塊、マインクラフトの地面（立体格子）、種類ちがいのブロック群（土・石・木・色違いの石）、立体に組まれたブロックの山、自前サーバーの旗（地面の傍らに立つ）、「クラウドではない」を示すちぎれ線（クラウド絵が薄く消える）、隠匿の眼差し（外から見る目）
- **レイアウト**：画面中央に地面の格子（側面図、奥行きは控えめ）。地面の上に立体に組まれたブロックの山。地面の左に自前サーバーの旗、右上に外から見る一つの目。素材タイルは画面左下。
- **動き**：Discord の窓から、地面の格子に舞台が切り替わる → 素材タイルの塊が、種類ちがいのブロックに一個ずつ変換される（タイルの色とビット列が、ブロックの種類に対応していくのを、変換の線で繋いで見せる）→ 変換されたブロックが地面の格子の上に降ってきて、立体的に積み重なり、不格好な山になる → 地面の左に自前サーバーの旗が立つ。同時に、画面の上にかかっていた「クラウド」の絵が薄くなって消える（自前なのでクラウドではない）→ 右上から外の目が山を見おろすが、目には「ただのブロックの寄せ集め」としてしか映らない（目が動画データだとは思わない、と示す）→ 山の上にうっすら「気づかれない通り道」のラベルが灯る
- **絵の眼目**：ストレージとしては不器用で、クラウドでもない。けれど、見た人が「データだ」と気づかないという一点で、これは別の道具になる——隠す力

### 結論（画面7）：場所が変われば、変装も変わる

- **登場する図**：変装の連鎖（mp4 の家×→色のマス目→ぱらぱら動画→Discord の画像／文字列→マイクラの山）が左から右に一筋に並ぶ、素材タイルの塊（連鎖の上を渡る）、DNA らせん（連鎖の右端）、締めの一文
- **レイアウト**：画面上半に、これまで通ってきた変装の場が小さく一列に並ぶ。下半は通路として空け、字幕帯の上に締めの一文の場所を確保。DNA らせんは連鎖の右端、いちばん新しい場所として小さく立つ。
- **動き**：上半に、これまでの五つの変装の場が左から右へ順に灯る（mp4 の家＝×／色のマス目／ぱらぱら動画／Discord の画像と文字列／マイクラの山）→ 素材タイルの塊が、連鎖の入口（左端）に置かれ、各場の上を通るごとに姿を変える早送りの巡回をする → 連鎖の右端に DNA らせんが現れ、塊の最後の一粒がらせんの上に小さく置かれる（場所が DNA に変わっても、おなじ仕掛けの延長）→ 締めの一文が静かに灯って終わる
- **絵の眼目**：「データを別の場所に置く」というおなじ仕掛けが、変装の中身を変えながらずっと続いている——次の場所が DNA に変わっても、お話は地続き

## 3. 図カタログ

| 図 | 見せる仕組み | 基本の見た目 | 登場セクション |
|---|---|---|---|
| 素材タイルの塊 | 動画全体の背骨。0と1の並びそのもの。場所が変わるたびに姿を変えるが、消えない・湧かない | アンバーの小さな正方形タイル多数。表面にうっすら "0 1 1 0..." の薄字 | 全画面 |
| mp4 の家 | mp4 のファイル構造。頭の小部屋＝目印、右のおっきな部屋＝中身 | 二部屋に区切られた小さな家。左部屋は小さく、右部屋は全体の九割 | ボディ1 |
| 目印の崩れ印 | 中身を塗りつぶすと、頭の目印もいっしょに崩れること | 短い棒状のマーク → ぐにゃりと崩れて上にコーラルの×が灯る | ボディ1 |
| 受付口・受理印・不受理印 | 動画として受けつけられたかどうか | サービス建物の入口の門。上にコーラルの×（不受理）／グリーンの✓（受理） | ボディ1・2 |
| 色のマス目 | データをピクセルの色として描く変装。4色＝一マスに二ビット、多色＝もっと | 画面いっぱいの正方格子。黒・赤・緑・青、もしくは多色 | ボディ2・4 |
| ぱらぱら動画の束 | 色のマス目を時間方向に重ねた、動画への変装 | 色のマス目が奥行きに層をなした、本のような束 | ボディ2・3 |
| YouTube サーバー塔 | 投稿された動画を作りなおす（圧縮する）場所 | 円筒の塔。中央に歯車が一つ。歯車は一度だけ回って止まる | ボディ3 |
| 化けたマス目／化けたビット列 | 圧縮で一マスだけ色が変わり、その下のビットが反転すること | 塊の中の一マスがコーラル。下にビット列が出て真ん中の一字に赤線 | ボディ3 |
| 粗マス目 | 画質を落としてマスを大きくする対策（運べる量は減る） | マス目を大きく描き直した版。マスは数が少なく、各マスがおっきい | ボディ3 |
| ×20 目盛り | 動画にすると素材の二十倍に膨らむこと | サーバー塔の傍らに伸びる細い目盛り。素材タイル一塊と比較して二十倍 | ボディ3 |
| Discord の窓 | 別のサービス。画像と文字列の二つの口を持つ | 縦長の窓型の建物。中が左右に分かれ、左に画像枠、右に文字列枠 | ボディ4 |
| 画像変装 | 色のマス目を一枚の画像にした変装。多色を使える | 多色ピクセルで埋まった一枚の大きな枠 | ボディ4 |
