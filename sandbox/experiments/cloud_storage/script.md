# 脱法クラウドストレージ ── データに、動画のふりをさせる

# ▶ 序論

めたん：ねえずんだもん。さっきね、変なものを見たの。
ずんだもん：どうしたのだ。
めたん：マインクラフトのワールドのなかに、動画ファイルがまるごと入っているって言うのよ。
ずんだもん：入っているのだ。一かたまりのブロックの山が、まるごと一本の動画なのだ。
めたん：ブロックの山が、動画？　どういうこと？
ずんだもん：山のなかのブロック一つ一つが、データの一部を運んでいるのだ。それを集めて読みなおすと、動画になるのだ。
めたん：……戻ってくるの？　ほんとうに？
ずんだもん：戻ってくるのだ。同じやりかたで、画像のなかに音声、動画のなかにマイクラのセーブデータも入れられるのだ。
めたん：……ぜんぶ、入れる場所と中身が、ちぐはぐじゃない。
ずんだもん：ちぐはぐなのだ。けれど、ちゃんと出し入れできているのだ。
めたん：これってつまり、本来データを置く場所じゃないところに、データを置けるってこと？
ずんだもん：そういう変わったストレージの作りかたを、「脱法クラウドストレージ」と呼ぶのだ。
めたん：脱法。響きが物騒ね。
ずんだもん：仕掛けの根っこは、ひとつなのだ。
めたん：根っこ？
ずんだもん：データというものは、ぜんぶ突き詰めると、0と1の並びでしかないのだ。動画も、画像も、ゲームも、もとをたどれば同じ並びなのだ。
めたん：ぜんぶ、おなじ素材でできている。
ずんだもん：だから、別の姿に着せ替えてやれば、別の場所に置いていけるのだ。データに、動画のふりをさせるのだ。
めたん：動画のふりをさせて、動画として置ける場所に、すべりこませる。
ずんだもん：これからほどいていくのは、その「ふりをさせる」やりかたの話なのだ。

# ▶ ボディ

## mp4 のお部屋に、こっそり詰める

ずんだもん：はじめに思いつくのは、いちばんすなおなやりかたなのだ。動画ファイルのなかみだけを、別のデータでこっそり差し替えるのだ。
めたん：動画のふりをしたまま、中身だけ別物にする？
ずんだもん：mp4 というかたちの動画ファイルは、いくつかの部屋に区切られているのだ。
めたん：部屋分けされているのね。
ずんだもん：頭のほうに、再生に必要な目印を置く小さな部屋。そして、映像と音そのものを入れる、おっきな部屋があるのだ。
めたん：おっきな部屋。
ずんだもん：mp4 全体の九割を占める部屋なのだ。十ギガの動画なら、九ギガぶんなのだ。
めたん：そこに、保存したいデータを九ギガぶん、ぎゅっと詰めかえる。
ずんだもん：詰めかえて、動画として YouTube に上げて、必要になったらまた取り出すのだ。
めたん：それ……できそうに聞こえるわ。一本の動画にゲームソフトがまるごと入っちゃうじゃない。
ずんだもん：YouTube は投稿できる本数に上限がないから、動画を増やせば置き場所も増やせるのだ。無限のおさいふみたいに見えるのだ。
めたん：なんでダメなの？
ずんだもん：おっきな部屋を別物で塗りつぶすと、頭の小さな部屋にある「ここから動画が始まる」という目印も、いっしょに崩れてしまうのだ。
めたん：印が崩れると、どうなるの？
ずんだもん：システムが「これは動画」だと、もう認識できなくなるのだ。
めたん：再生できないだけじゃなくて……
ずんだもん：YouTube は、そもそも動画として受けつけないのだ。アップロードの入口で、はじかれるのだ。
めたん：そっか。「動画のふり」をしようとして、動画である目印まで消しちゃうのね。
ずんだもん：中身だけ差し替える、という素直なやりかたは、ここで行きどまるのだ。

## ピクセルの色に、ほどいて並べる

ずんだもん：そこで発想を変えるのだ。中身を差し替えるのではなく、データを表に出すのだ。
めたん：表に出す？
ずんだもん：データはぜんぶ、0と1の並びなのだ。ならば、その0と1を、画面のなかに見える色として描いてしまうのだ。
めたん：見える色、として……
ずんだもん：たとえば、黒のマスを0、白のマスを1とする。画面いっぱいに白黒のマス目を敷きつめるのだ。
めたん：これは、テレビの砂嵐みたい。
ずんだもん：そう見えるのだ。けれど、その砂嵐の一マス一マスが、データの一ビットなのだ。
めたん：マスごとに色を増やしたら、もっと詰めこめる？
ずんだもん：そうなのだ。実際の手法では、黒・赤・緑・青の四色で、一マスに二ビットずつ運ぶのだ。
めたん：マス目一枚で、どれくらい入るのかしら。
ずんだもん：一枚のカラフルな砂嵐に入る量は、小さなゲームのソースコードがまるごと収まるくらいなのだ。書籍にすると十五ページぶんくらいなのだ。
めたん：一枚でそれだけ。それを動画として、ぱらぱら切り替えれば……
ずんだもん：六十分の一秒ごとに、まったく違う砂嵐をめくっていくのだ。一秒で六十枚、別の中身が流れるのだ。
めたん：一秒だけでも、ずいぶんな量が運べるのね。
ずんだもん：そして長い動画ほど、運べる量が積み上がるのだ。何時間もの長さの動画にすれば、一本でゲーム一本ぶんの容量が入ってもおかしくないのだ。
めたん：それなら、動画としてはちゃんと動画。中身は完全にデータ。
ずんだもん：YouTube から見れば、ただの「やけに砂嵐っぽい動画」なのだ。中身が何かは、向こうには分からないのだ。
めたん：これで、無限のおさいふが本当にできあがる？
ずんだもん：そう見えるのだ。ただし、もうひとつ関門があるのだ。

## 取り出すときに、欠ける

めたん：関門？
ずんだもん：YouTube に上げられた動画は、向こうのサーバーで一度作りなおされるのだ。圧縮しなおすのだ。
めたん：圧縮。
ずんだもん：見るぶんには気にならない範囲で、色味やマスのきわが少しずつ揺らされるのだ。視聴者の目には届かないくらい、控えめに。
めたん：あ……砂嵐の一マスが、ほんの少し別の色に化ける？
ずんだもん：化けるのだ。0 だったマスが、1 になることもあるのだ。
めたん：それって、データだとしたら……
ずんだもん：一字化けただけで、ぜんぶ読めなくなることもあるのだ。動画なら一瞬の色のずれで済む話が、データだと致命傷になるのだ。
めたん：とくにダメなのは、どんなとき？
ずんだもん：画面の切り替わりが激しいほど、圧縮の手も荒くなって、揺らぎも大きくなるのだ。砂嵐の動画は、ちょうどそれにあたるのだ。
めたん：なんとかならないの？
ずんだもん：画質を落として、マスを大きくとると、化けにくくなるのだ。
めたん：マスがおっきいほうが、ちょっと色が揺れても踏みとどまれる。
ずんだもん：そういうことなのだ。ただし、その代わりに運べるデータも少なくなるのだ。
めたん：たくさん詰めるか、ちゃんと戻ってくるか。どちらかしか取れないのね。
ずんだもん：もうひとつ厄介なのが、データを動画にするときの大きさなのだ。
めたん：大きさ？
ずんだもん：もとのデータの二十倍くらいに膨らむのだ。一ギガを置きたいだけで、二十ギガぶんの動画を作って上げることになるのだ。
めたん：詰めたいぶんの二十倍を、毎回流しこむ。「無限」と呼ぶには、ずいぶん重たい無限ね。
ずんだもん：そういう天秤を、ずっと抱えている技術なのだ。

## 別のサービスに、別のすがたで置く

めたん：YouTube だけが置き場所じゃないわよね。ほかのサービスでも、おなじやりかたが使えるのかしら。
ずんだもん：いい問いなのだ。Discord というサービスがあるのだ。
めたん：聞いたことあるわ。文字や画像をやりとりするところね。
ずんだもん：そこも、ファイルの置き場所として使えてしまうのだ。ただし、Discord には別の制約があるのだ。
めたん：別の制約。
ずんだもん：投稿できる一つのファイルの大きさに、上限があるのだ。長い動画はそもそも上げられないのだ。
めたん：動画でぱらぱら流す、という作戦が、ここでは使えない？
ずんだもん：そこで、二つの逃げ道が考えられたのだ。一つ目は、動画をやめて、画像一枚一枚で送ることなのだ。
めたん：動画から、画像へ。
ずんだもん：Discord に上げる画像は、YouTube とちがって、向こうで作りなおされないのだ。色味がそのまま残るのだ。
めたん：圧縮で揺らされない。
ずんだもん：だから、四色なんかに絞らず、画像が持てる色をめいっぱい使えるのだ。一マスに、もっと多くのビットを詰められるのだ。
めたん：揺らされないぶん、欲ばっていい。
ずんだもん：もう一つの逃げ道は、もっと思いきっているのだ。画像ですらないのだ。
めたん：画像でも、ない？
ずんだもん：データの拡張子を、ただのテキストに付け替えてしまうのだ。すると、文字化けした文字列の山として、画面に開かれるのだ。
めたん：その文字化けを、そのままチャットに貼りつけるの？
ずんだもん：貼りつけるのだ。戻したいときは、文字列をコピーして、テキストファイルにペーストして、拡張子をもとに戻すのだ。
めたん：もう、画像ですらないのね。ただの文字の並びとして、サービスに置いておく。
ずんだもん：データが、こんどは文字のふりをするのだ。場所が変われば、変装の中身も変わるのだ。
めたん：どこでも、置こうと思えば置ける。
ずんだもん：そう見えるのだ。ただ、置けてしまうことと、ちゃんと使えることは、別の話なのだ。
めたん：どういうこと？
ずんだもん：規約というものがあるのだ。本来の使いかたを大きく外れた使いかたは、見つかればアカウントごと止められるのだ。
めたん：止められると、置いていたデータは。
ずんだもん：いっしょに、消えるのだ。何時間もかけて変換して上げたものが、ある日まるごと消えてもおかしくないのだ。
めたん：無限のおさいふ……どころか、底が抜けることもあるおさいふ、ね。

## マイクラのブロックの山に、変装する

めたん：ねえ、はじめに話していた、マインクラフトの山。あれはどういうやりかたなの？
ずんだもん：マインクラフトの中には、たくさんの種類のブロックがあるのだ。土、石、木、いろんな色のブロックがあるのだ。
めたん：種類があるのね。
ずんだもん：その種類と、置かれた位置の二つを使うのだ。ブロックの種類を、何ビットかぶんの数として読むのだ。
めたん：色のマス目の代わりに、ブロックの種類でビットを表す。
ずんだもん：そして、ブロックを縦横に並べて、平面の代わりに立体のマス目を作るのだ。
めたん：あの山は、立体に組まれた砂嵐だったのね。
ずんだもん：そのとおりなのだ。だから、ぱっと見はただのブロックの寄せ集めなのだ。
めたん：これも、規約で止められるリスクがあるの？
ずんだもん：そこが、これまでとちがうのだ。マインクラフトのサーバーは、自分の手元で立ててもいいのだ。
めたん：自分で立てた家のなかに置くなら、誰かに止められることがない。
ずんだもん：止められないのだ。ただし、これはもう「クラウド」とは呼びにくいのだ。
めたん：クラウドは、よそのサービスを借りるから、無限になるんだものね。自分のサーバーなら、自分の機械の容量どまり。
ずんだもん：そうなのだ。だから「ストレージ」としては、もう向いていないのだ。
めたん：あれ。じゃあ、なんでこのやりかたが話に出てくるのかしら。
ずんだもん：ここで、いちばん面白い性質が見えてくるのだ。
めたん：性質？
ずんだもん：このブロックの山を、いきなり見せられた人を思いうかべるのだ。
めたん：マインクラフトの、ちょっと変な建造物に見える。
ずんだもん：そうなのだ。「これは動画データです」とは、まず思わないのだ。
めたん：「動画をマイクラに入れる」という発想を、そもそも持っていないから。
ずんだもん：暗号というのは、ふつう、計算で破られないことを目指す道具なのだ。でも、別の道もあるのだ。
めたん：そこにある、と気づかせない道。
ずんだもん：脱法クラウドストレージは、ストレージとしては不器用なのだ。けれど、「気づかれない通り道」として見ると、姿が変わるのだ。
めたん：データを動画に変装させたのではなく、「動画のふりをした、別の通信路」を作っていたのね。
ずんだもん：マイクラの山は、そのきわみなのだ。ストレージとしての価値を捨てて、隠す力にぜんぶ振っているのだ。

# ▶ 結論

めたん：ぜんぶ並べてみると、おなじ仕掛けが姿を変えていただけなのね。
ずんだもん：そうなのだ。データは0と1の並びでしかない、というところから始めて、その並びを別の何かに着せ替えていったのだ。
めたん：mp4 の中身に化けようとして、はじかれて。
ずんだもん：砂嵐の色のマスに化けて、動画として置けるようになって。
めたん：圧縮で揺らされるから、画質を落としたり、画像のまま送ったり、ただの文字列に変えたり。
ずんだもん：置き場所が変わるたびに、変装の中身も変わっていったのだ。
めたん：そして最後は、マイクラのブロックの山にまで変装した。
ずんだもん：そこでは、もう「無限の置き場所」ではなくて、「気づかれない置き場所」が主役になっていたのだ。
めたん：ストレージのお話のはずが、いつのまにか、隠しごとのお話になっていたのね。
ずんだもん：どちらも、おなじ仕掛けの別の顔なのだ。
めたん：思えば、はじめのほうから不思議だったのよ。動画にゲームを入れる、画像に音声を入れる、ぜんぶちぐはぐなのに、ちゃんと出し入れできる。
ずんだもん：データが0と1の並びでしかない、という出発点を受け入れると、その「ちぐはぐ」が消えるのだ。
めたん：場所と中身の組みあわせは、わたしたちが決めつけていただけ。
ずんだもん：将棋の棋譜の中にデータを置く、街づくりゲームの区画の並びでデータを表す——そういう変わった置き場所も、おなじ発想で作れるのだ。
めたん：もっと先には、何があるの？
ずんだもん：データを、生きものの DNA の中に書きこむ研究も進んでいるのだ。
めたん：……生きもののなかに、データを？
ずんだもん：ここまでの話の延長なのだ。0と1の並びを、別の何かに着せ替えて、別の場所に置く。場所が DNA に変わるだけなのだ。
めたん：場所が変われば、変装の中身も変わる。やっぱり、おなじお話なのね。
