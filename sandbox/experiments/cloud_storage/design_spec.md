# cloud_storage — design spec

`design_overall.md` の質的な動きを event アンカーに分解し、`script.md` の各セリフへ割り当てる。座標値・フレーム数は持たない。

対話字幕は event ではない——全セリフが自動で画面下部の字幕になる（tsx の仕事）。ここで定義する event は、絵の装置の挙動だけを指す。

## 1. event 一覧と割り当て

発生順。画面切替 event（`scene.*.in`）は各画面の先頭セリフに必須。

| event | 紐付くセリフ（画面・話者・抜粋） | 発火する挙動 |
|---|---|---|
| `scene.intro.in` | 序論・めたん「ねえずんだもん。さっきね、変なものを見たの。」 | 序論の画面がフェードイン。中央の素材ゾーンと、その周りの三方の置き場所が枠だけ立ち上がる |
| `intro.three` | 序論・ずんだもん「…画像のなかに音声、動画のなかにマイクラのセーブデータも入れられるのだ。」 | 三つの異種ファイル絵（マイクラのブロック山／画像と音符／動画とセーブデータ）が画面の三方に順に立ち上がる。中身と場所が食い違っていることが並んで見える |
| `intro.title` | 序論・ずんだもん「そういう変わったストレージの作りかたを、「脱法クラウドストレージ」と呼ぶのだ。」 | 画面下中央にタイトル「脱法クラウドストレージ」が灯る |
| `intro.gather` | 序論・ずんだもん「データというものは、ぜんぶ突き詰めると、0と1の並びでしかないのだ。…」 | 三方の絵から、共通の素材タイルが一塊ずつ吸い出されて中央に集まる。集まった塊の表面に "0 1 1 0..." の薄字ビット列がうっすら浮かぶ |
| `intro.shuffle` | 序論・ずんだもん「だから、別の姿に着せ替えてやれば、別の場所に置いていけるのだ。…」 | 中央の塊が一度ばらばらの粒の群れにほぐれ、再び塊に戻る（同じ粒は並べかたを変えれば何にでもなれる） |
| `scene.body1.in` | ボディ1・ずんだもん「はじめに思いつくのは、いちばんすなおなやりかたなのだ。…」 | 画面切替。中央の素材ゾーンの右側に小さな空き地が現れる |
| `b1.house` | ボディ1・ずんだもん「mp4 というかたちの動画ファイルは、いくつかの部屋に区切られているのだ。」 | 空き地に mp4 の家が立ち上がる。家は左右に分かれ、左の小部屋（目印）と右のおっきな部屋（中身）が大きさの比そのもので九対一に描かれる |
| `b1.fill` | ボディ1・めたん「そこに、保存したいデータを九ギガぶん、ぎゅっと詰めかえる。」 | 素材タイルの塊が左の手元ゾーンから右部屋に流しこまれ、右部屋を埋める。もとの動画の中身は押しのけられて消える |
| `b1.crush` | ボディ1・ずんだもん「おっきな部屋を別物で塗りつぶすと、頭の小さな部屋にある「ここから動画が始まる」という目印も、いっしょに崩れてしまうのだ。」 | 押しこみの衝撃が左の小部屋に伝わり、目印の棒状マークがぐにゃりと崩れる。崩れたマークの上にコーラルの×が灯る |
| `b1.reject` | ボディ1・ずんだもん「YouTube は、そもそも動画として受けつけないのだ。アップロードの入口で、はじかれるのだ。」 | 家ごと YouTube の受付口に運ばれるが、門で家が止められる。入口の上にコーラルの「不受理」印が灯る |
| `scene.body2.in` | ボディ2・ずんだもん「そこで発想を変えるのだ。中身を差し替えるのではなく、データを表に出すのだ。」 | 画面切替。mp4 の家が退き、画面中央が空く。左の手元ゾーンに素材タイルの塊が残る |
| `b2.unravel` | ボディ2・ずんだもん「データはぜんぶ、0と1の並びなのだ。ならば、その0と1を、画面のなかに見える色として描いてしまうのだ。」 | 素材タイルの塊がほぐれ、一粒一粒が中央の格子の各マスに並ぶ。最初は黒と白の二色（0と1の素朴な対応） |
| `b2.color` | ボディ2・ずんだもん「…黒・赤・緑・青の四色で、一マスに二ビットずつ運ぶのだ。」 | 格子の各マスが黒・赤・緑・青の四色に染め分けられる。マス目一枚に詰まる情報量が、色の段階そのもので増える |
| `b2.flip` | ボディ2・ずんだもん「六十分の一秒ごとに、まったく違う砂嵐をめくっていくのだ。一秒で六十枚、別の中身が流れるのだ。」 | 完成したマス目の後ろに、もう一枚、さらにもう一枚と層が重なってぱらぱら動画の束ができる。束は手前から奥に時間方向の厚みを持つ |
| `b2.accept` | ボディ2・ずんだもん「YouTube から見れば、ただの「やけに砂嵐っぽい動画」なのだ。…」 | 束が画面右の YouTube 受付口に運ばれ、入口の上にグリーンの受理印✓が灯る |
| `scene.body3.in` | ボディ3・めたん「関門？」 | 画面切替。ぱらぱら動画の束が左に残り、画面中央に YouTube サーバー塔（円筒に歯車一つ）が立ち上がる |
| `b3.compress` | ボディ3・ずんだもん「化けるのだ。0 だったマスが、1 になることもあるのだ。」 | 左の束がサーバー塔の入口に流れこむ。塔の中の歯車が一回転して止まる。出口から出てきた束の塊の中の一マスだけがコーラルに染まる |
| `b3.flip` | ボディ3・ずんだもん「一字化けただけで、ぜんぶ読めなくなることもあるのだ。…」 | 化けたマスを拡大して画面右に大きく描き、その下にビット列を出す。列の真ん中の一字に赤線が引かれる（0 が 1 になった） |
| `b3.coarse` | ボディ3・ずんだもん「画質を落として、マスを大きくとると、化けにくくなるのだ。」 | 入口側のマス目が大きく描き直され、粗マス目になる。マスの数は少なくなるが、各マスの色がいっそう堅く描かれる |
| `b3.bloat` | ボディ3・ずんだもん「もとのデータの二十倍くらいに膨らむのだ。…」 | 塔の傍らに細い目盛りが伸び、もとの素材タイル一塊に対して二十倍ぶんの動画の長さが、目盛りの伸びそのもので可視化される |
| `scene.body4.in` | ボディ4・めたん「YouTube だけが置き場所じゃないわよね。ほかのサービスでも、おなじやりかたが使えるのかしら。」 | 画面切替。YouTube サーバー塔が退き、画面中央に Discord の窓型建物が立ち上がる |
| `b4.discord` | ボディ4・ずんだもん「そこも、ファイルの置き場所として使えてしまうのだ。ただし、Discord には別の制約があるのだ。」 | 窓の入口に「一回ぶんの大きさに上限」を示す小さなゲートが描かれる。建物の中は左右に分かれ、左口（画像枠）と右口（文字列枠）が現れる |
| `b4.image` | ボディ4・ずんだもん「Discord に上げる画像は、YouTube とちがって、向こうで作りなおされないのだ。色味がそのまま残るのだ。」 | 左の手元ゾーンから素材タイルが上ルートに枝分かれし、多色ピクセルで埋まった画像一枚に変装して、Discord の左口に投函される。マスの色は四色に絞られず多色 |
| `b4.text` | ボディ4・ずんだもん「データの拡張子を、ただのテキストに付け替えてしまうのだ。…」 | 同じ素材タイルが下ルートにも枝分かれし、画像にならず、文字化け文字列の柱として右口に貼りつけられる |
| `b4.ban` | ボディ4・ずんだもん「規約というものがあるのだ。本来の使いかたを大きく外れた使いかたは、見つかればアカウントごと止められるのだ。」 | 建物の上にコーラルの止め札（BAN 印）が小さく点り、左口の画像と右口の文字列がいっせいに薄く消えかかる |
| `scene.body5.in` | ボディ5・めたん「ねえ、はじめに話していた、マインクラフトの山。あれはどういうやりかたなの？」 | 画面切替。Discord の窓が退き、画面中央に立体格子の地面が現れる（側面図、奥行きは控えめ） |
| `b5.ground` | ボディ5・ずんだもん「マインクラフトの中には、たくさんの種類のブロックがあるのだ。…」 | 地面の左端に、種類ちがいのブロック群（土・石・木・色違いの石）のサンプルが小さく並ぶ。地面の傍らに自前サーバーの旗が立つ |
| `b5.blocks` | ボディ5・ずんだもん「そして、ブロックを縦横に並べて、平面の代わりに立体のマス目を作るのだ。」 | 左の素材タイルが、ブロックの種類に一個ずつ変換される（タイルとブロックを結ぶ変換線を一本ずつ通す）。変換されたブロックが地面の格子の上に降ってきて、立体的に積み重なり、不格好な山になる |
| `b5.notcloud` | ボディ5・ずんだもん「そこが、これまでとちがうのだ。マインクラフトのサーバーは、自分の手元で立ててもいいのだ。」 | 画面の上方にうっすら出ていた「クラウド」絵が、ちぎれ線とともに薄く消える。地面の傍らの自前サーバーの旗だけが残る |
| `b5.hidden` | ボディ5・ずんだもん「脱法クラウドストレージは、ストレージとしては不器用なのだ。けれど、「気づかれない通り道」として見ると、姿が変わるのだ。」 | 画面右上から外の目が山を見おろす。目には山が「ただのブロックの寄せ集め」としてしか映らないことを、目の視線が山の表面で止まり中身に届かない描きで示す。山の上に「気づかれない通り道」のラベルが灯る |
| `scene.outro.in` | 結論・めたん「ぜんぶ並べてみると、おなじ仕掛けが姿を変えていただけなのね。」 | 画面切替。これまでの建物がすべて退き、画面上半に空の連鎖レーンが横一文字に現れる |
| `outro.chain` | 結論・めたん「mp4 の中身に化けようとして、はじかれて。」 | 連鎖レーンに、これまでの五つの変装の場が左から右へ順に灯る：mp4 の家（コーラルの×）／色のマス目／ぱらぱら動画／Discord の画像と文字列／マイクラの山 |
| `outro.travel` | 結論・ずんだもん「置き場所が変わるたびに、変装の中身も変わっていったのだ。」 | 素材タイルの塊が連鎖の左端に置かれ、各場の上を通るごとに姿を変える早送りの巡回をする |
| `outro.dna` | 結論・ずんだもん「データを、生きものの DNA の中に書きこむ研究も進んでいるのだ。」 | 連鎖の右端に DNA の二重らせんが現れ、塊の最後の一粒がらせんの上に小さく置かれる |
| `outro.end` | 結論・めたん「場所が変われば、変装の中身も変わる。やっぱり、おなじお話なのね。」 | 連鎖レーンの下、字幕帯の上の余白に締めの一文が静かに灯って終わる |

内容 event は35個（全 130 セリフの約27%）。「5〜15%目安」を上回るが、7画面それぞれが独立した変装工程（家に詰める／色にほどく／圧縮で化ける／画像か文字列に枝分かれ／立体に積む／連鎖に並べる）を順送りで扱い、工程ごとに専用装置を立ち上げ・畳みする題材のため。装置の登場・退場そのものが説明の本体なので、event は工程の数に比例する。

## 2. ライフサイクル契約

## 3. オブジェクト別 挙動メモ

複数 event をまたいで連続変化するオブジェクトの、event 間の状態の向き先。座標・フレーム数は持たない。

**素材タイルの塊（背骨）** — 全画面に登場。持つ数値は「塊度」（0＝ばらばらの粒の群れ／1＝一塊にまとまっている）と、表面の「ビット列濃度」（0＝完全に透ける／1＝薄字としてうっすら浮かぶ）、そして所属画面の中での位置。
- 序論：`intro.three` で三方から吸い出されて塊度0→1（中央に集まる）→ `intro.gather` でビット列濃度0→1（"0 1 1 0..." が浮かぶ）→ `intro.shuffle` で塊度1→0→1（ほぐれて再集合）。
- ボディ1：画面入場で塊度1（左の手元）→ `b1.fill` で mp4 の家の右部屋に流しこまれて溶けこむ（塊度1のまま位置だけ移動）。
- ボディ2：画面入場で塊度1 → `b2.unravel` で塊度1→0（ほぐれて格子の粒になる）→ `b2.color` で粒が四色に染まる（粒のまま色だけ変化）。
- ボディ3：箱／塊自体は登場しない（ぱらぱら動画の束として現れる）。化けたマス目だけが背骨の一部の状態を担う。
- ボディ4：画面入場で塊度1（左の手元）→ `b4.image` で上ルートに枝分かれし画像変装に → `b4.text` で下ルートにも枝分かれし文字列変装に（塊そのものは左に残り、複製が二方向に変装）。
- ボディ5：画面入場で塊度1 → `b5.blocks` でブロックに一個ずつ変換され、塊度1→0（粒に分かれてブロックに化ける）→ ブロックの山として再集合。
- 結論：`outro.travel` で塊が連鎖の上を渡り、各場で姿を変える早送り → `outro.dna` で最後の一粒だけが DNA に置かれる。

**ぱらぱら動画の束** — ボディ2で誕生しボディ3に持ち越される。持つ数値は「層の枚数」（手前から奥に積み重なる枚数）。
- ボディ2：`b2.flip` で層0→多（マス目を時間方向に重ねて積み上がる）→ `b2.accept` で完成して YouTube 受付口に進む。
- ボディ3：画面入場で束が左に残る → `b3.compress` でサーバー塔に流れこみ、出口から一マスだけコーラルに染まった束として出てくる。

**サーバー塔の歯車（ボディ3）** — `b3.compress` で歯車が一回転して止まる（ループしない）。回転と同期して、塔の中で塊の一マスが化ける。

**変装の連鎖レーン（結論）** — `scene.outro.in` で空のレーンとして誕生 → `outro.chain` で五つの場が左から右へ順に灯る（点灯の進み具合を持つ）→ `outro.travel` で素材タイルが上を渡り、各場の上で素材タイルの姿が変わる → `outro.dna` で右端に DNA らせんが追加される。

**化けたマス目／化けたビット列（ボディ3）** — `b3.compress` で塊の中の一マスがコーラルに染まる → `b3.flip` でそのマスが画面右に拡大され、ビット列が出て真ん中の一字に赤線が引かれる。マス自体は連続変形（拡大）で、別物として再実装しない。
