# [1] 台本作成（script.md）

題材をリサーチし、`script.md` に**YouTube解説動画として成立する対話台本**、`references.md` に概要欄用の参考文献を書く段。デザイン・タイミング・event は下流で決める。台本では、正しい情報を並べるだけでなく、視聴者が「なんで？」を追っているうちに核心へ着く流れを作る。

## 目的

台本は次の 3 つを同時に満たす。

- **導入で自然に入る** — 日常の違和感・事件・極端な例から入り、「ということで今回は、〇〇を解説していくのだ」で番組として始める
- **解説が本質を射抜く** — 「何をするか」ではなく「なぜそうするしかないか」を見せる
- **掛け合いが見続けさせる** — めたんの誤解・驚き・ツッコミが、次の説明を引き出す

## 前提

- 尺は15分程度。短くまとめすぎず、問題がほどける過程を残す
- 対象は題材名と流通している一文要約くらいは知っているが、その理屈までは知らない人
- 対象は内容に詳しくはないが、浅い説明では満足しない人。表面をなぞるだけではなく、一般論で終わらせず本質に踏み込む
- 既知の確認は 1〜2 往復で通過する。ゼロから教科書的に始めない
- 題材全体と代表方式を混同しない。「一部がそう」なのか「今よく使われる一系統」なのか必ず分ける
- セクションは画面単位。序論全体・ボディ各 `##`・結論全体で総画面 4〜6 を目安にする

## リサーチ

- 台本前に必ず調査する。最新情報が必要な題材は現在の一次情報を確認し、古い知識だけで書かない
- 優先度は `論文 / 公式資料 / 標準仕様 / 開発元ブログ / 信頼できる解説`。根拠の弱いまとめ記事だけで組まない
- 調べる観点は `体系の地図 / 代表方式 / 例外 / 限界 / 近年の変化 / 動画で使える具体例`
- 引用は権威づけではなく説明の支柱として使う。本文では必要な範囲だけ自然に触れる
- `references.md` を別に作り、概要欄に載せる文献を 5 件前後に絞る。形式は `- タイトル — 著者・組織, 年, URL: 動画で使う根拠`

## 書く前に決める

本文に入る前に、次の 5 点を短く決める。

- **入口の絵** — 最初に見せる日常・事件・数字・失敗例。「〇〇とは何か」から始めない
- **初期誤解** — めたんが自然に言いそうな少しズレた理解
- **道筋と問い** — 題材全体、代表方式、例外を切り分け、「なぜその方法が必要なのか」を置く
- **視聴後の残像** — これがわかると何が見えるようになるか
- **最後の回収** — 導入の違和感・比喩・事件を結論でどう返すか

## 構成

```markdown
# {題材タイトル}

# ■ 導入
めたん: {日常の違和感・事件・極端な例}
ずんだもん: {初期誤解への返し・反転}
ずんだもん: ということで今回は、{題材}について解説していくのだ。
ずんだもん: これがわかると、{視聴後に見えるもの}が見えてくるのだ。

# ■ ボディ
## {セクション見出し}
ずんだもん: ...
めたん: ...

# ■ 結論
ずんだもん: {導入の回収}
```

### 導入

導入は `つかみ → 誤解 → 反転 → テーマ宣言 → 視聴後の残像 → 最初の問い` を 45 秒以内に入れる。定義・歴史・挨拶から入らない。導入は答えでなく問いを作る場所。

### ボディ

各章は情報のまとまりではなく、小さな問題解決にする。最初に体系の地図を置き、その後に代表ルートを深掘る。基本形は `素朴案 → 欠陥 → 改良案`、`結果 → 原因 → 仕組み`、`成功例 → 限界 → 次の避難先` のどれか。章末には次章への引きを置く。前章の素材・比喩・数値を次章で再利用し、寄せ集めにしない。

### 結論

総括で終わらせない。導入の違和感を回収し、最後にもう一段ひねって返す。禁止: 「まとめると」「便利なので今後が楽しみ」「ぜひ調べてみてください」「いかがでしたか」。締めは 1 行で余韻を残す。

## 解説ルール

- 答えより先に問題を見せる。答えは情報ではなく解放として届くようにする
- 体系的な説明を心がける。知識を断片で終わらせず、どのグループのどの位置にいるのかを意識させる
- 手順説明にしない。各ステップが前の困りごとから強いられて出る形にする
- 例えは仕組みの論理を背負うものだけ使う。飾りの比喩は削る
- 専門用語は「現象 → 命名」。ただし既知の語は無理に導入に戻さない
- 固有名詞を逃がさない。名前を出したうえで例えで支える
- 数値は身近な比で 2〜3 回叩く
- 抽象→具体→抽象を短期で往復する
- 断定にはスコープを添え、簡略化したら正確には複雑だと残す

## 掛け合い

- **ずんだもん** — 解説役。主張と情報を持つ。一人称「ぼく」、語尾「〜なのだ」「〜のだ」「〜なんだよ」
- **めたん** — 聞き手役。視聴者の代弁。先読みで正解を言わせない。一人称「わたし」、語尾「〜ね」「〜かしら」「〜わ」「〜じゃない？」
- めたんは「聞いてるけど集中力が少し怪しい」視聴者代表。様子見、ふーん、日常の不満、雑な比喩、話聞いてる？感を混ぜる
- めたんに正解を回り込ませない。ただし自然な誤解・軽いボケ・不安の言い換え・半分だけ理解した反応は積極的に入れる
- めたんの発話は `短い情動 / 初期誤解 / 典型的勘違い / 次の質問 / 視聴者目線の言い換え / ツッコミ / 日常の不満 / 脱線` を混ぜる
- めたんに、ずんだもんの説明を圧縮した断定をさせない。そのセリフをずんだもんが言っても成立するなら、質問かツッコミに開く
- 文字数比は **ずんだもん：めたん ≒ 4〜5：1**。完全な交互応答にしない
- ボディ中盤で二択を投げ、めたんに答えさせるターンを最低 1 回入れる

## 語調

- 書き言葉を避け、喋り言葉で書く。「〜してしまう」は「〜しちゃう」、「〜というのは」は「〜っていうのは」
- 整文用の読点を減らす。セリフ読点は 1〜2 個まで
- 決め所に温度語を 1 発刺す。エグい、ヤバい、化け物、反則、えげつない、凶悪、気持ち悪い、不穏、など
- 温度語は 1 セクション 1〜2 個まで。説明の代わりにしない。題材や視聴者を罵らない

## 出力とチェック

セリフは `話者: 本文` の 1 行で書く。ト書き・効果音・注記は書かない。行の長すぎる字幕表示を考え、聞いて長すぎたら分ける。`references.md` も同時に作る。

ドラフト後、次を確認する: 導入で違和感から入るか / 45秒以内にテーマ宣言しているか / 視聴後の残像があるか / 各章に問題解決と次章への引きがあるか / めたんが要約役でなく聞き手として動いているか / 結論が導入を回収しているか / 参考文献が 5 件前後に絞られているか。
