### 1位：なぜスマホは2年で遅くなるのか（37/40点）

- **推薦理由**: ⑧入口の広さが最高点。スマホを持つ全人類が体験する不満で、USB動画と同じ「日常の不満 × 技術的理由」構造。さらに計画的陳腐化という陰謀論的要素があり、コメント欄で「やっぱりわざとだ」vs「物理的な劣化だ」の議論が必ず起きる。EU修理権（2026年7月施行）というタイムリーな要素もある。
- **動画タイトル案**:
  1. なぜスマホは2年で遅くなるのか
  2. あなたのスマホ、わざと遅くされてるかも
  3. スマホの寿命は「設計」されている？
- **日常接点 → 驚き → 論争の流れ**:
  - 日常接点：2年使ったスマホのもっさり感。誰もが体験済み
  - 驚き：Appleが意図的にCPUを制限していたことを認めた「バッテリーゲート」事件。フランスで2500万ユーロの罰金
  - 論争：「わざとやってる」vs「バッテリーの物理劣化でやむを得ない」vs「OSアップデートが新機種前提」
- **派閥構造**:
  - 陰謀論派「企業が買い替えサイクルを設計している」
  - 技術擁護派「リチウムイオン電池の劣化は物理法則。仕方ない」
  - EU規制派「修理する権利で解決すべき」
  - 自衛派「バッテリー交換すれば3年使える」
- **想定コメント例**:
  - 「iPhoneが2年でもっさりになるのマジでそれ」
  - 「バッテリー替えたら復活したから劣化説が正しい」
  - 「Androidのほうが早く遅くなるんだが」
  - 「Appleの罰金2500万ユーロは安すぎる」
- **構成概要**: 体験の共有 → バッテリーゲート事件 → 遅くなる4つの技術的原因（バッテリー劣化→CPUスロットリング、ストレージの書き込み劣化、OSの肥大化、バックグラウンドプロセス蓄積）→ 計画的陳腐化の歴史（1925年電球カルテル）→ EU修理権という新しい風 → 「あなたはどう思う？」で締め
- **リスク・注意点**: Apple批判に偏るとチャンネルのトーンに合わない。「技術的制約」と「ビジネスインセンティブ」の両面を公平に提示する

### 2位：なぜ日本だけFAXが現役なのか（35/40点）

- **推薦理由**: 日本人なら全員知っている「FAXまだ使ってるの？」という疑問。海外から見ると異常だが、実は合理的な理由がある、という「常識の裏返し」構造。差別化余地が極めて高い（このテーマの深掘り解説動画は少ない）。
- **動画タイトル案**:
  1. なぜ日本だけFAXが現役なのか
  2. FAXを馬鹿にする人が知らない「5つの合理性」
  3. 日本のFAX文化は本当に「遅れている」のか？
- **日常接点 → 驚き → 論争の流れ**:
  - 日常接点：病院・役所・取引先で未だにFAXを使わされる体験
  - 驚き：製造・物流業の50%がFAXを日常使用。セキュリティ面ではインターネットより安全な側面がある
  - 論争：「IT後進国の象徴」vs「実は合理的」vs「下請け構造が変えさせない」
- **派閥構造**:
  - 脱FAX派「河野太郎の廃止方針を支持。いい加減やめろ」
  - 現実派「取引先がFAX指定する限り変えられない」
  - 擁護派「インターネットに繋がってないから安全」
  - 海外驚愕派「日本に来て一番驚いたのがFAX」
- **想定コメント例**:
  - 「うちの会社まだFAXで発注書送ってる…」
  - 「セキュリティ面では理にかなってるんだよな」
  - 「下請けの現場知らないから簡単に廃止って言える」
- **構成概要**: 海外の驚き → 日本の利用実態 → FAXが残る5つの理由（下請け階層構造、手書き文化、セキュリティ、法的証拠能力、高齢化）→ 河野太郎の廃止挑戦とその結果 → デジタル化の本当の障壁は技術ではなく構造
- **リスク・注意点**: 「日本叩き」にならないよう注意。構造的な理由を丁寧に解説し、安易な結論を避ける

### 3位：なぜパスワードのルールはあんなに面倒なのか（35/40点）

- **推薦理由**: 全インターネットユーザーが「大文字・小文字・数字・記号を含む8文字以上」に苛立った経験がある。しかも**NIST（アメリカ国立標準技術研究所）が2017年に自ら「あのルールは間違いだった」と公式に謝罪**しているという衝撃の事実がある。「正しいと思っていたことが実は逆効果だった」という認知ギャップが極めて強い。
- **動画タイトル案**:
  1. なぜパスワードのルールはあんなに面倒なのか
  2. パスワードの「常識」は全部間違っていた
  3. 安全なパスワードに記号は要らない？
- **日常接点 → 驚き → 論争の流れ**:
  - 日常接点：「P@ssw0rd!」を作らされるあの体験
  - 驚き：NIST公式謝罪。「correcthorsebatterystaple」のほうが「P@ss1!」より遥かに安全。パスワードの最大文字数制限は90年代のデータベース設計の遺物
  - 論争：「じゃあなんでまだあのルールなの？」「パスワードマネージャを使えば全部解決」vs「覚えられないから結局メモ帳に書く」
- **派閥構造**:
  - パスワードマネージャ派「全部ランダム生成で解決」
  - パスフレーズ派「覚えやすい長い文が最強」
  - 現実派「会社のルールで3ヶ月ごとに変更強制される」
  - 未来派「パスワード自体がもう時代遅れ。パスキーに移行しろ」
- **想定コメント例**:
  - 「銀行のパスワード4桁数字なんだが」
  - 「3ヶ月ごとに変えさせるやつ本当に意味あるの」
  - 「パスワードマネージャのマスターパスワードを忘れた」
- **構成概要**: 「大文字小文字記号数字」ルールの起源 → NIST 2003年ガイドラインと2017年の撤回 → 長さ＞複雑さの数学的証明 → なぜ銀行は4桁なのか → パスキーという未来 → 今すぐできる改善策
- **リスク・注意点**: セキュリティの具体的な数値（解読時間等）は情報ソースを明確にする

### 4位：プリンタはなぜ2026年でもゴミなのか（34/40点）

- **推薦理由**: NPR、CNN、Redditで繰り返し取り上げられる「全人類の共通の怒り」。プリンタが安い理由（本体赤字のインク商法）、シアンが空だとモノクロ印刷できない理由（DRM）、保証切れで壊れる理由（ページカウンター）と、全てに「なるほど、そういうことか」がある。
- **動画タイトル案**:
  1. プリンタはなぜ2026年でもゴミなのか
  2. なぜプリンタはインクで儲けているのか
  3. プリンタが「ちょうど保証切れ」で壊れる理由
- **日常接点 → 驚き → 論争の流れ**:
  - 日常接点：印刷しようとしたら動かない、インクが高い、謎のエラー
  - 驚き：インクの1mlあたりの価格はシャンパンより高い。プリンタ本体は赤字販売。カラーインクが空だとモノクロ印刷をブロックするのはDRM
  - 論争：「もう全部コンビニ印刷でいい」vs「レーザープリンタが正解」vs「そもそも印刷する時代じゃない」
- **派閥構造**:
  - 怒り派「インク商法は詐欺」
  - 脱プリンタ派「コンビニ印刷で十分」
  - レーザー派「インクジェットを買うやつが悪い」
  - 諦め派「年賀状のときだけ使う」
- **想定コメント例**:
  - 「インク買い替えるたびにプリンタ買い直したほうが安いの本当に腹立つ」
