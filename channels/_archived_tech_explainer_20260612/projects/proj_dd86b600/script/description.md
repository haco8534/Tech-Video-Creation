なぜ「再起動したら直る」が全てを解決してしまうのか

「とりあえず再起動」で直った経験、誰にでもあるはず。でもなぜ直ったのか、ちゃんと説明できますか？
雑な対処に見えるこの一手は、実はHeisenbugの発見から、Crash-Only Software、Erlangの"Let It Crash"、Kubernetesの自己修復まで、数十年のソフトウェア工学が選んできた設計思想そのもの。
直る／直らないを分ける境界、Boeing 787が120日ごとに再起動を義務付けられた逆説、セーフティクリティカル領域で「再起動」が選択肢にならない理由まで、対話でやさしく整理します。
