### 1-2. パフォーマンス分析から抽出したバズ法則

**法則1: リーチの最大決定因子は「非エンジニアでも疑問を持てるか」**
- USB（42,929再生）は全人類が使うモノ → おすすめアルゴリズムに乗った
- Tier D（169〜575再生）は「そもそも疑問に思わない」テーマ

**法則2: コメント爆発の条件は「見る前から体験・意見がある」**
- USBコメント上位は全て体験語り（micro-Bがゆるゆる等）
- Tier D動画（URL、パスワード、Delete）はコメント0〜1件

**法則3: エンジニア向けでもTier B（3,000〜10,000）は達成可能**
- Python、数学、ChatGPTは登録者のファン層育成に寄与

**法則4: いいね率とリーチは独立指標**
- 「言語作り方」はいいね率7.9%（1位）だがリーチはTier B

**法則5: タイトル構造パターン**
- 「なぜ○○なのか」→ 最強（42,929）
- 「○○って○○すぎない？」→ 強い（10,298）
- 「○○は○○か？」哲学型 → 議論は生むがリーチ狭い（2,358）

### 1-3. コメント分析から抽出したテーマの種

USB動画196コメントの分析:
- **体験共有型コメントが主流**: 「micro-Bがゆるゆる」「USB-Aの裏表が許せない」「USB-Cケーブルで充電できなかった」
- **知識マウント型**: 「Type-Cの欠点を理解してるのか」「Thunderbolt4ケーブル使えば解決」
- **派閥対立型**: 「全部Type-Cにしろ」vs「Type-Cにも欠点がある」
- **→ 「USB-Cケーブルの内部の違い」は既にコメント欄で需要が見えている**

プログラミング言語動画66コメントの分析:
- **補足・訂正型が主流**: 「アセンブリ言語入れて」「5:17おかしい」
- **哲学的展開**: 「原理上は全言語を一つに纏められる」
- **→ エンジニア層の知識共有欲を刺激するテーマは安定してTier B**

VSCode動画50コメントの分析:
- **制作手法への言及が多い**: 「Claude codeで動画を作ると起きがちな問題」「AIで動画作ってますか？」
- **→ 「AI制作動画」自体がコンテンツになりうる（ただし今回のテーマ候補には含めない）**

### 1-4. 「成功パターン逆算」による発想

USB動画の成功要素を分解:
```
[日常で全員が使うモノ] × [なぜこうなっている？の疑問] × [技術的に驚く理由] × [全員が体験語りできる]
```

この構造に当てはまるモノをリストアップ:
- スマホ → 遅くなる、バッテリー劣化、権限要求
- プリンタ → 壊れる、インクが高い
- Bluetooth → 切れる、繋がらない
- Wi-Fi → 途切れる、遅い
- パスワード → 面倒、覚えられない
- Cookie同意 → うざい
- ソフトウェアアップデート → なぜ巨大、なぜ勝手に再起動
- 充電ケーブル → すぐ壊れる、速度が違う
- FAX → なぜまだ使ってる（日本限定）
- サブスク → コンテンツが消える
- デジタル所有権 → 買ったのに消える

追加の発想軸（トレンドリサーチから）:
- AI関連 → 学習データ枯渇、AIスロップ、仕事が遅くなる
- Z世代 → ガラケー回帰、アナログ経済
- プライバシー → アプリ権限、データ売買
