## シード分解表

### シード1：「ChatGPTにコードを書かせるのはプログラミングか？」

| 分解軸 | 結果 |
|--------|------|
| 技術レイヤー | LLM、コード生成、プロンプトエンジニアリング、Vibe Coding、AIエージェント |
| 前提・思い込み | 「コードを書ける＝プログラマー」「AIが書いたコードは品質が低い」「プログラミングは手で書くもの」 |
| 関連する技術・概念 | Copilot、Cursor、Vibe Coding、ノーコード/ローコード、ペアプログラミング |
| 歴史・発展 | パンチカード→アセンブリ→高級言語→IDE→AI、抽象度は常に上がってきた |
| 日常接点 | 毎日AIを使ってコード書いてるエンジニアが大量にいる |
| 🔥 派閥・陣営 | AI丸投げ派 vs AIはツール派 vs AI不要派、Vibe Coding肯定派 vs 否定派 |

### シード2：「プログラミング言語多すぎない？」

| 分解軸 | 結果 |
|--------|------|
| 技術レイヤー | 言語設計、型システム、ランタイム、エコシステム、パッケージマネージャ |
| 前提・思い込み | 「1つ覚えれば十分」「最強の言語がある」「新しい言語ほど良い」 |
| 関連する技術・概念 | フレームワーク、ライブラリ、ビルドツール、IDE/エディタ選択 |
| 歴史・発展 | 言語の栄枯盛衰（COBOL→Java→Python→Rust）、なぜ新しい言語が生まれ続けるのか |
| 日常接点 | 「何の言語を学ぶべき？」は全プログラマーの原体験 |
| 🔥 派閥・陣営 | 言語信者（各言語ファン）、静的型 vs 動的型、OOP vs 関数型、「結局○○が最強」論争 |
