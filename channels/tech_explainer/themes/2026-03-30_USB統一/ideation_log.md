# Step 1：シード分解 & 多層連想 & バズ構造スキャン

## シード分解表

| 分解軸 | 結果 |
|--------|------|
| **技術レイヤー** | コネクタ物理形状、電気信号プロトコル、給電規格（USB PD）、命名体系（USB 3.2 Gen 2x2等）、認証・ライセンス、後方互換性設計 |
| **前提・思い込み** | 「USBは統一規格のはずだ」「Type-Cなら全部同じ」「規格が新しいほど良い」「統一すれば消費者に利益」「技術的に統一は簡単なはず」 |
| **関連する技術・概念** | Bluetooth規格乱立、Wi-Fi命名変更、充電規格（Qi/Qi2）、映像出力規格（HDMI/DisplayPort）、通信規格（3G/4G/5G）、文字コード（UTF-8統一への道）、xkcd #927「Standards」 |
| **歴史・発展** | RS-232→USB 1.0(1996)→2.0→3.0→3.1→3.2→4→Type-C物理統一→EU法規制→命名体系刷新。Apple Lightning→Type-C移行。規格名リネーム問題。 |
| **日常接点** | スマホ充電、PCとの接続、データ転送、モニター接続、充電器の使い回し、「このケーブルどれ？」問題 |
| **派閥・陣営** | ⚠️ USB規格自体には派閥がほぼ存在しない（全員「統一しろ」で一致）→ **シードそのままでは論争が生まれない。変換が必要。** |

### シード変換の方針

ワークフローの指示通り、「USBはなぜ統一できないのか」をそのまま動画にするのではなく、以下の方向に発散する：

1. **「統一」という概念そのものへの問い** → 技術の世界で「統一」は本当に良いことなのか？
2. **規格乱立の構造的原因** → なぜ人類は規格を統一できないのか（xkcd #927問題）
3. **USBから派生する「選択」の論争** → 充電器、OS、ツール、言語など「何を選ぶか」の議論
4. **技術の複雑さそのもの** → なぜ技術は複雑になり続けるのか
