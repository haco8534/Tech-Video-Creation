## 2. VPNが実際には守れないもの

| ファクト | 信頼度 | 出典 |
|---|---|---|
| フィッシング攻撃：VPNは通信経路を暗号化するだけで、偽サイトへのログインは防げない。VPN使用中にフィッシングに引っかかれば、データは暗号化されたままハッカーに届く | 🟢高 | vpn-advisors.com, CircleID |
| マルウェア：VPNはファイルをスキャンしない。ダウンロードした悪意あるファイルはVPNで止められない | 🟢高 | 8Bit Security, Surfshark公式ブログ |
| Cookieトラッキング：VPNを使っても既存のトラッキングCookieは消えない。ブラウザ設定や拡張機能との併用が必要 | 🟢高 | Ghostery, WizCase |
| ブラウザフィンガープリンティング：画面解像度・フォント・プラグイン構成などからユニークな「指紋」を生成。VPNでIPを変えても識別される | 🟢高 | Hackaday（2025年11月）, fingerprint.com |
| ログインアカウント：GoogleやFacebookにログインしたままVPNを使っても、サービス側はアカウントで本人を特定できる | 🟢高 | NordVPN公式ブログ, Windscribe |
| Zscalerのセキュリティ専門家は「VPNはネットワークツールであり、セキュリティソリューションではない」と明言 | 🟡中 | Zscaler CXO Revolutionaries（専門家コラム） |

## 3. 「VPNで匿名になれる」という神話

| ファクト | 信頼度 | 出典 |
|---|---|---|
| 完全な匿名VPNは存在しない。VPNはIPアドレスを隠せるが、それ以外の追跡手段は多数残る | 🟢高 | NordVPN公式ブログ（匿名VPN解説） |
| 真に近い匿名性を得るには、VPN＋Tor＋プライバシーブラウザ＋厳格な行動習慣の組み合わせが必要 | 🟡中 | Windscribe, Ghostery |
| メリーランド大学の研究者Ensafi氏：「セキュリティの専門知識が高いユーザーでもVPN使用中に安全と感じる傾向があり、VPNのマーケティングが成功していることを示す」 | 🟡中 | Engadget/TechRadar（UM研究紹介） |
| メリーランド大学のOmer Akgul氏：VPNの広告は「実際に機能するVPNが提供できること以上のものを売り込んでいる」と指摘 | 🟡中 | Engadget（2022年記事） |
| UMのMichelle Mazurek教授：「広告の大半はVPNが何をできるかについて、あいまいまたは誇張したクレームをしている」 | 🟡中 | ACM CHI 2025 Paper（Security Knight in Shining Armor） |

## 4. HTTPS普及率とVPN必要性の低下

| ファクト | 信頼度 | 出典 |
|---|---|---|
| W3Techs 2026年1月データ：上位100,000サイトのうち92.6%がデフォルトでHTTPSを使用 | 🟢高 | W3Techs（2026年4月データ） |
| 米国ではHTTPS普及率が90%超。グローバルでも84%に達している | 🟢高 | SSL Dragon, SSL Insights（2026年） |
| Let's Encryptは無料SSL証明書を提供し、2億2,500万以上のウェブサイトを保護。1日あたり約150万枚の証明書を発行 | 🟢高 | Let's Encrypt公式Stats |
| 2021〜2024年の間にSSL証明書を導入したウェブサイト数は57%増加 | 🟡中 | SSL統計まとめサイト複数 |
| HTTPSが普及した現代では「公衆Wi-Fiで通信が盗まれる」リスクは大幅に低下しており、VPN広告の主要メッセージと現実に乖離がある | 🟡中 | Windscribe Blog, The Register（NordVPN行政指導報道） |

## 5. VPNプロバイダーのログ記録スキャンダル

| ファクト | 信頼度 | 出典 |
|---|---|---|
| **IPVanish事件（2016年）**：「一切ログを取らない」と公言していたにもかかわらず、米国土安全保障省（DHS）に接続ログを提供。ユーザーのComcast IPアドレスと位置情報が特定された | 🟢高 | Restore Privacy / CyberInsider |
| **PureVPN事件（2017年）**：「活動記録を保存しない」と主張していたが、FBIがサイバーハラスメント事件でPureVPNに照会し、詳細なセッションログと実IPアドレスが提供された | 🟢高 | VyprVPN Blog, CyberInsider |
| **NordVPNサーバー侵害（2018年）**：フィンランドのデータセンターにある1台のサーバーが第三者ハッカーに侵害された。発覚〜公表まで約1年かかり（2019年10月公表）、対応の遅さが批判された | 🟢高 | TechRadar, NordVPN公式声明 |
| NordVPNは「ユーザー認証情報は漏洩していない」と主張。該当サーバーのトラフィック監視の可能性は否定できないとも認めた | 🟡中 | NordVPN公式ブログ声明 |
| HideMyAss、UFO VPN、Hotspot Shieldも過去にno-logポリシー違反の疑惑や記録保存が報告されている | 🟡中 | CyberInsider VPNログ調査 |

## 6. 無料VPNのリスク

| ファクト | 信頼度 | 出典 |
|---|---|---|
| 2016年のCSIRO（オーストラリア連邦科学産業研究機構）研究：Androidの無料VPNアプリ283本を調査し、38%にマルウェアが含まれていた | 🟢高 | CSIRO研究論文（IMC 2016発表） |
| 同調査で67%の無料VPNアプリが少なくとも1つのトラッキングライブラリをコードに埋め込んでいた | 🟢高 | CSIRO研究論文 |
| 同調査で84%が何らかの形でユーザートラフィックをリークしていた | 🟢高 | CSIRO研究論文 |
| **Hola VPN事件（2015年）**：最大約5,000万ユーザーを持つ無料VPN「Hola」が、ユーザーのデバイスとネットワーク帯域幅をボットネット用に第三者に販売していたことが判明 | 🟢高 | PCWorld, Tripwire |
| Holaの仕組み：ユーザーがインストールすると他のユーザーの出口ノード（エグジット）になる。帯域幅を有料サービス「Luminati」（現Bright Data）で販売 | 🟢高 | PCWorld（2015年報道） |
| 2021年：HolaのChrome拡張機能がハッカーに侵害され、仮想通貨ユーザーをフィッシングサイトに誘導 | 🟡中 | VPNmentor（Hola VPNレビュー） |
| 無料VPNがデータを収益化する主な手段：広告挿入・ユーザーデータの第三者販売・帯域幅の転売 | 🟡中 | Norton Blog, AdaptNetwork |

## 7. セキュリティ専門家によるVPN広告批判

| ファクト | 信頼度 | 出典 |
|---|---|---|
| Consumer Reports（2021年テスト）：16本のVPNを検証した結果、12本（75%）が製品・技術を不正確に説明するか、誇張・過剰な保護主張をしていた | 🟢高 | Consumer Reports（2021年12月） |
