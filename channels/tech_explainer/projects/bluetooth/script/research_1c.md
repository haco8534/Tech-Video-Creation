# research_1c：歴史的経緯・設計思想・日常接点

## 設計思想の根幹

| # | 内容 | 信頼度 |
|---|------|--------|
| 1 | 1998年設立。当時の目的は「ケーブルを排除する短距離無線」。携帯電話とPCの同期、ヘッドセット、マウス等を想定 | 🟢 |
| 2 | 設計目標: 低コスト・低消費電力・短距離・グローバル共通仕様 | 🟢 |
| 3 | FHSS（周波数ホッピング）の採用理由: 軍用無線由来の技術。妨害耐性とスペクトル共有が目的（CDMA系の選択肢もあったが特許コストが高すぎた） | 🟢 |
| 4 | 1999年Bluetooth 1.0発表時、Wi-Fi（IEEE 802.11b）はまだ普及前。当時2.4GHz帯は「空いている」と認識されていた | 🟢 |
| 5 | 2.4GHz帯選定理由: ITUのISM帯指定がグローバル共通で唯一の現実解。900MHzは米国限定、5GHzは当時技術的にコスト高 | 🟢 |

## 不安定さを生んだ歴史的経緯

| # | 経緯 | 信頼度 |
|---|------|--------|
| 6 | 2000年代前半、Wi-Fi爆発的普及（家庭向け無線LANの普及）→2.4GHz帯混雑問題が顕在化 | 🟢 |
| 7 | 2003年Bluetooth 1.2でAFH（Adaptive Frequency Hopping）追加。Wi-Fi利用中のチャンネルを動的に避ける仕組み（後付け対応） | 🟢 |
| 8 | 2004年Bluetooth 2.0+EDR で速度向上（最大3 Mbps） | 🟢 |
| 9 | 2010年Bluetooth 4.0でBLE導入。これは新規物理層で、Classic（BR/EDR）と互換性なし | 🟢 |
| 10 | 2016年Bluetooth 5.0でLE 2M PHY、Long Range追加。ただし普及は段階的 | 🟢 |
| 11 | 2020年Bluetooth 5.2でLE Audio、LC3コーデック、Auracast導入。ただしClassic Audio (A2DP)は廃止せず併存 | 🟢 |

## 日常接点

| # | 経験 | 視聴者の体感 |
|---|------|-------------|
| 12 | 駅やオフィスなど人混みで音飛び・途切れ | ほぼ全Bluetoothユーザーが経験 |
| 13 | 冷蔵庫・電子レンジ近くで切れる | 多くのユーザー経験 |
| 14 | カバン/ポケットに入れた瞬間に音質低下や切断 | TWSユーザー多数経験 |
| 15 | 通話に切り替えると音質が一気に落ちる | 通話Bluetoothヘッドセット利用者 |
| 16 | 同じ場所でWi-Fiは安定しているのにBluetoothが不安定 | 一般的な体験 |
| 17 | ペアリング登録機器が増えると挙動がおかしくなる | スマホヘビーユーザー |
| 18 | 別デバイスへの切替時に時折つまずく | 複数Bluetooth機器ユーザー |

## アーキテクチャ的負債

| # | 内容 |
|---|------|
| 19 | Core Spec v5.4（2023）は3000ページ超。Bluetoothはもはや単一規格ではなく「物理層・リンク層・プロファイル」の巨大集合体 |
| 20 | Classic（BR/EDR）とLE（BLE）はプロトコルスタックが完全に別物。両対応チップ（Dual-mode）でハードウェア共通化 |
| 21 | プロファイル数は30以上。各プロファイルは独立に進化してきた経緯がある |
| 22 | Bluetooth Mesh（2017）は別規格扱い。BLEを土台にしているが、ペアリングモデルが従来と異なる |
| 23 | LE Audio（2020）の本格普及は2024年以降と見られる。20年以上続いたClassic Audioの置き換えは長期戦 |

## 専門家コンセンサス

| # | 内容 |
|---|------|
| 24 | Bluetooth不安定さの根本原因として、無線環境（2.4GHz共有）、人体吸収、アンテナ物理、プロトコル設計、歴史的負債の5要素が広く指摘される |
| 25 | エンジニアコミュニティでは「USB 3.0 hub の近くにBluetoothドングルを置くな」は定番のトラブルシューティング |
| 26 | Bluetoothの「不安定さ」は技術的失敗というより、25年前の設計が現代の無線環境と利用形態に追いついていないという構造問題 |
